# 裝載支援 API 的原生 WPF 瀏覽器
透過 WPF 主機外部註冊的主動式文件伺服器 \(也稱為 DocObject\)，可簡化在 Web 瀏覽器中裝載 [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] 應用程式的作業。  [!INCLUDE[TLA2#tla_ie](../../../../includes/tla2sharptla-ie-md.md)] 可以直接啟動並且與主動式文件整合。  若要在 Mozilla 瀏覽器中裝載 XBAP 和鬆散的 XAML 文件，[!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] 提供了 NPAPI 外掛程式，它能為 [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] 主動式文件伺服器提供類似 [!INCLUDE[TLA2#tla_ie](../../../../includes/tla2sharptla-ie-md.md)] 所提供的裝載環境。  在其他瀏覽器以及獨立應用程式中裝載 XBAP 和 XAML 文件的最簡單實用的方式，是透過 Internet Explorer 的 Web 瀏覽器控制項。  Web 瀏覽器控制項提供複雜的主動式文件伺服器裝載環境，不過可讓自己的主機自訂並延伸該環境，並且直接與目前的主動式文件物件進行通訊。  
  
 [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] 主動式文件伺服器會實作數個常用的裝載介面，包括 [IOleObject](http://go.microsoft.com/fwlink/?LinkId=162049)、[IOleDocument](http://go.microsoft.com/fwlink/?LinkId=162050)、[IOleInPlaceActiveObject](http://go.microsoft.com/fwlink/?LinkId=162051)、[IPersistMoniker](http://go.microsoft.com/fwlink/?LinkId=162045) 及 [IOleCommandTarget](http://go.microsoft.com/fwlink/?LinkId=162047)。  這些介面裝載於 Web 瀏覽器控制項時，可以從 [IWebBrowser2::Document](http://go.microsoft.com/fwlink/?LinkId=162048) 屬性傳回的物件查詢。  
  
## IOleCommandTarget  
 WPF 主動式文件伺服器的 [IOleCommandTarget](http://go.microsoft.com/fwlink/?LinkId=162047) 實作支援多種標準 OLE 命令群組的巡覽相關以及瀏覽器專屬命令 \(擁有 null 命令群組 GUID\)。  此外，它還能辨識稱為 CGID\_PresentationHost 的自訂命令群組。  目前這個群組中只定義了一個命令。  
  
```  
DEFINE_GUID(CGID_PresentationHost, 0xd0288c55, 0xd6, 0x4f5e, 0xa8, 0x51, 0x79, 0xde, 0xc5, 0x1b, 0x10, 0xec);  
enum PresentationHostCommands {   
   PHCMDID_TABINTO = 1   
};  
```  
  
 PHCMDID\_TABINTO 會根據 Shift 鍵的狀態，指示 PresentationHost 將焦點切換至內容中第一個或最後一個可為焦點的項目。
