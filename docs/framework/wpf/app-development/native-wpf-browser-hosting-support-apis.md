# <a name="native-wpf-browser-hosting-support-apis"></a>裝載支援 API 的原生 WPF 瀏覽器
從 WPF 主機註冊的 Active Document 伺服器（也稱為 DocObject）有助於在 Web 瀏覽器中託管 WPF 應用程式。 Internet Explorer 可以直接啟動活動文檔並與其集成。 對於 Mozilla 瀏覽器中 XBAP 和鬆散 XAML 文檔的託管，WPF 提供了一個 NPAPI 外掛程式，它提供了與 Internet Explorer 類似的 WPF 活動文檔伺服器的託管環境。 但是，在其他瀏覽器和獨立應用程式中託管 XBAPs 和 XAML 文檔的最簡單實用方法是通過 Internet 資源管理器 Web 瀏覽器控制項。 Web 瀏覽器控制項提供複雜的活動文檔伺服器託管環境，但它允許自己的主機自訂和擴展該環境，並直接與當前活動文檔物件通信。  
  
 WPF 活動文檔伺服器實現了幾個常見的託管介面，包括 IOleObject、IOleDocument、IOleInPlaceActiveobject、IPersistMoniker、IOleCommandTarget。 [IOleObject](/windows/win32/api/oleidl/nn-oleidl-ioleobject) [IOleDocument](/windows/win32/api/docobj/nn-docobj-ioledocument) [IOleInPlaceActiveObject](/windows/win32/api/oleidl/nn-oleidl-ioleinplaceactiveobject) [IPersistMoniker](https://docs.microsoft.com/previous-versions/windows/internet-explorer/ie-developer/platform-apis/ms775042(v=vs.85)) [IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget) 在 Web 瀏覽器控制項中託管時，這些介面可以是[IWebBrowser2：:Document](https://docs.microsoft.com/previous-versions/aa752116(v=vs.85))屬性返回的物件中的查詢。  
  
## <a name="iolecommandtarget"></a>IOleCommand目標  
 WPF 活動文檔伺服器對[IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget)的實現支援標準 OLE 命令組（具有空命令組 GUID）的許多與導航相關的瀏覽器特定命令。 此外，它識別名為CGID_PresentationHost的自訂命令組。 目前，此組中僅定義了一個命令。  
  
```cpp  
DEFINE_GUID(CGID_PresentationHost, 0xd0288c55, 0xd6, 0x4f5e, 0xa8, 0x51, 0x79, 0xde, 0xc5, 0x1b, 0x10, 0xec);  
enum PresentationHostCommands {
   PHCMDID_TABINTO = 1
};  
```  
  
 PHCMDID_TABINTO指示演示主機根據 Shift 鍵的狀態將焦點切換到其內容中的第一個或最後一個可聚焦元素。
