 <span data-ttu-id="9d0ef-106">其中一個最重要的因素，判斷[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]效能是可轉譯繫結，您可以呈現，較高成本的效能有更多的像素。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-106">One of the most important factors in determining [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] performance is that it is render bound—the more pixels you have to render, the greater the performance cost.</span></span> <span data-ttu-id="9d0ef-107">不過，轉譯會更可以卸載到[!INCLUDE[TLA#tla_gpu](../../../../includes/tlasharptla-gpu-md.md)]，您可以取得更多的效能優勢。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-107">However, the more rendering that can be offloaded to the [!INCLUDE[TLA#tla_gpu](../../../../includes/tlasharptla-gpu-md.md)], the more performance benefits you can gain.</span></span> <span data-ttu-id="9d0ef-108">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]應用程式的硬體轉譯管線可充分[!INCLUDE[TLA#tla_dx](../../../../includes/tlasharptla-dx-md.md)]支援的最少的硬體上的功能[!INCLUDE[TLA#tla_dx](../../../../includes/tlasharptla-dx-md.md)]7.0 版。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-108">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] application hardware rendering pipeline takes full advantage of [!INCLUDE[TLA#tla_dx](../../../../includes/tlasharptla-dx-md.md)] features on hardware that supports a minimum of [!INCLUDE[TLA#tla_dx](../../../../includes/tlasharptla-dx-md.md)] version 7.0.</span></span> <span data-ttu-id="9d0ef-109">進一步最佳化後，可以支援硬體的[!INCLUDE[TLA#tla_dx](../../../../includes/tlasharptla-dx-md.md)]7.0 版和 PixelShader 2.0 + 功能。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-109">Further optimizations can be gained by hardware that supports [!INCLUDE[TLA#tla_dx](../../../../includes/tlasharptla-dx-md.md)] version 7.0 and PixelShader 2.0+ features.</span></span>  
  
## <a name="software-rendering-pipeline"></a><span data-ttu-id="9d0ef-110">軟體呈現管線</span><span class="sxs-lookup"><span data-stu-id="9d0ef-110">Software Rendering Pipeline</span></span>  
 <span data-ttu-id="9d0ef-111">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]軟體呈現管線完全是 CPU 繫結。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-111">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] software rendering pipeline is entirely CPU bound.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] <span data-ttu-id="9d0ef-112">SSE 及 SSE2 指令的利用設定 CPU 來實作最佳化的完整功能軟體模擬轉譯器。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-112">takes advantage of the SSE and SSE2 instruction sets in the CPU to implement an optimized, fully-featured software rasterizer.</span></span> <span data-ttu-id="9d0ef-113">軟體後援是的每當應用程式的功能無法使用硬體呈現管線呈現順暢。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-113">Fallback to software is seamless any time application functionality cannot be rendered using the hardware rendering pipeline.</span></span>  
  
 <span data-ttu-id="9d0ef-114">最大的效能問題就會發生時軟體模式中的呈現與填滿率，定義為您要轉譯的像素數目。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-114">The biggest performance issue you will encounter when rendering in software mode is related to fill rate, which is defined as the number of pixels that you are rendering.</span></span> <span data-ttu-id="9d0ef-115">如果您擔心在軟體轉譯模式中的效能，請嘗試重新繪製的像素的次數降到最低。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-115">If you are concerned about performance in software rendering mode, try to minimize the number of times a pixel is redrawn.</span></span> <span data-ttu-id="9d0ef-116">例如，如果您的應用程式，以藍色背景，然後呈現稍微透明的影像上方時，會呈現所有兩次的應用程式中的像素。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-116">For example, if you have an application with a blue background, which then renders a slightly transparent image over it, you will render all of the pixels in the application twice.</span></span> <span data-ttu-id="9d0ef-117">如此一來，它會花上兩次來呈現應用程式比您藍色的背景影像。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-117">As a result, it will take twice as long to render the application with the image than if you had only the blue background.</span></span>  
  
### <a name="graphics-rendering-tiers"></a><span data-ttu-id="9d0ef-118">圖形轉譯層</span><span class="sxs-lookup"><span data-stu-id="9d0ef-118">Graphics Rendering Tiers</span></span>  
 <span data-ttu-id="9d0ef-119">它可能很難預測的硬體組態，將會在執行您的應用程式。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-119">It may be very difficult to predict the hardware configuration that your application will be running on.</span></span> <span data-ttu-id="9d0ef-120">不過，您可能要考慮的設計，可讓您順暢地切換功能在不同的硬體上執行時，讓它可以充分利用每個不同的硬體組態的應用程式。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-120">However, you might want to consider a design that allows your application to seamlessly switch features when running on different hardware, so that it can take full advantage of each different hardware configuration.</span></span>  
  
 <span data-ttu-id="9d0ef-121">若要達到這個目的，[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]提供功能，以判斷系統在執行階段的圖形功能。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-121">To achieve this, [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] provides functionality to determine the graphics capability of a system at runtime.</span></span> <span data-ttu-id="9d0ef-122">圖形功能取決於分類做為其中的三個轉譯功能層級的視訊卡。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-122">Graphics capability is determined by categorizing the video card as one of three rendering capability tiers.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] <span data-ttu-id="9d0ef-123">會公開[!INCLUDE[TLA#tla_api](../../../../includes/tlasharptla-api-md.md)]，可讓應用程式查詢轉譯功能層。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-123">exposes an [!INCLUDE[TLA#tla_api](../../../../includes/tlasharptla-api-md.md)] that allows an application to query the rendering capability tier.</span></span> <span data-ttu-id="9d0ef-124">您的應用程式可以在執行階段取決於硬體支援的轉譯層採取不同的程式碼路徑。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-124">Your application can then take different code paths at run time depending on the rendering tier supported by the hardware.</span></span>

-   <span data-ttu-id="9d0ef-130">**多紋理支援**：多紋理支援指的是可以在混色作業期間於 3D 圖形物件上套用兩個以上的不同紋理。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-130">**Multitexture Support** Multitexture support refers to the ability to apply two or more distinct textures during a blending operation on a 3D graphics object.</span></span> <span data-ttu-id="9d0ef-131">多紋理支援的程度取決於圖形硬體上的多紋理單位數目。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-131">The degree of multitexture support is determined by the number of multitexture units on the graphics hardware.</span></span>  
  
 <span data-ttu-id="9d0ef-132">像素著色器、 頂點著色器，以及多紋理功能用來定義特定[!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)]版本層級，它反而用來定義中的不同的轉譯層[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-132">The pixel shader, vertex shader, and multitexture features are used to define specific [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version levels, which, in turn, are used to define the different rendering tiers in [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span></span>

-   <span data-ttu-id="9d0ef-135">**轉譯層 0**：沒有圖形硬體加速。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-135">**Rendering Tier 0** No graphics hardware acceleration.</span></span> <span data-ttu-id="9d0ef-136">[!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)]版本層級低於 7.0 版。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-136">The [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version level is less than version 7.0.</span></span>  
  
-   <span data-ttu-id="9d0ef-137">**轉譯層 1**部分的圖形硬體加速。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-137">**Rendering Tier 1** Partial graphics hardware acceleration.</span></span> <span data-ttu-id="9d0ef-138">[!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)]版本層級是大於或等於 7.0 版中，並**較小者**比 9.0 版。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-138">The [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version level is greater than or equal to version 7.0, and **lesser** than version 9.0.</span></span>  
  
-   <span data-ttu-id="9d0ef-139">**轉譯層 2**：大部分圖形功能都使用圖形硬體加速。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-139">**Rendering Tier 2** Most graphics features use graphics hardware acceleration.</span></span> <span data-ttu-id="9d0ef-140">[!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] 版本層級大於或等於 9.0 版。</span><span class="sxs-lookup"><span data-stu-id="9d0ef-140">The [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version level is greater than or equal to version 9.0.</span></span>
