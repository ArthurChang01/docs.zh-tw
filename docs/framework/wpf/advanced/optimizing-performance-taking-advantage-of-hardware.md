<span data-ttu-id="44ee0-103">的內部架構[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]有兩個轉譯管線: 硬體和軟體。</span><span class="sxs-lookup"><span data-stu-id="44ee0-103">The internal architecture of [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] has two rendering pipelines, hardware and software.</span></span> <span data-ttu-id="44ee0-104">本主題提供這些轉譯管線的相關資訊, 可協助您決定應用程式的效能優化。</span><span class="sxs-lookup"><span data-stu-id="44ee0-104">This topic provides information about these rendering pipelines to help you make decisions about performance optimizations of your applications.</span></span>  
  
## <a name="hardware-rendering-pipeline"></a><span data-ttu-id="44ee0-105">硬體呈現管線</span><span class="sxs-lookup"><span data-stu-id="44ee0-105">Hardware Rendering Pipeline</span></span>  
 <span data-ttu-id="44ee0-106">判斷[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]效能最重要的因素之一就是它會呈現系結, 您必須轉譯的圖元越多, 效能成本就愈高。</span><span class="sxs-lookup"><span data-stu-id="44ee0-106">One of the most important factors in determining [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] performance is that it is render bound—the more pixels you have to render, the greater the performance cost.</span></span> <span data-ttu-id="44ee0-107">不過, 可以卸載至圖形處理器 (GPU) 的更多轉譯, 您可以獲得更多的效能優勢。</span><span class="sxs-lookup"><span data-stu-id="44ee0-107">However, the more rendering that can be offloaded to the graphics processing unit (GPU), the more performance benefits you can gain.</span></span> <span data-ttu-id="44ee0-108">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]應用程式硬體轉譯管線在支援最低 microsoft directx 7.0 版的硬體上, 可享有 microsoft directx 功能的完整優勢。</span><span class="sxs-lookup"><span data-stu-id="44ee0-108">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] application hardware rendering pipeline takes full advantage of Microsoft DirectX features on hardware that supports a minimum of Microsoft DirectX version 7.0.</span></span> <span data-ttu-id="44ee0-109">支援 Microsoft DirectX 7.0 版和無效 2.0 + 功能的硬體可以取得進一步的優化。</span><span class="sxs-lookup"><span data-stu-id="44ee0-109">Further optimizations can be gained by hardware that supports Microsoft DirectX version 7.0 and PixelShader 2.0+ features.</span></span>  
  
## <a name="software-rendering-pipeline"></a><span data-ttu-id="44ee0-110">軟體呈現管線</span><span class="sxs-lookup"><span data-stu-id="44ee0-110">Software Rendering Pipeline</span></span>  
 <span data-ttu-id="44ee0-111">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]軟體轉譯管線完全受限於 CPU。</span><span class="sxs-lookup"><span data-stu-id="44ee0-111">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] software rendering pipeline is entirely CPU bound.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]<span data-ttu-id="44ee0-112">利用 CPU 中的 SSE 和 SSE2 指令集來執行優化、功能完整的軟體轉譯器。</span><span class="sxs-lookup"><span data-stu-id="44ee0-112">takes advantage of the SSE and SSE2 instruction sets in the CPU to implement an optimized, fully-featured software rasterizer.</span></span> <span data-ttu-id="44ee0-113">只要應用程式功能無法使用硬體轉譯管線轉譯, 就可以順暢地回溯到軟體。</span><span class="sxs-lookup"><span data-stu-id="44ee0-113">Fallback to software is seamless any time application functionality cannot be rendered using the hardware rendering pipeline.</span></span>  
  
 <span data-ttu-id="44ee0-114">以軟體模式轉譯時, 您會遇到的最大效能問題, 與填滿速率相關, 其定義為您要轉譯的圖元數目。</span><span class="sxs-lookup"><span data-stu-id="44ee0-114">The biggest performance issue you will encounter when rendering in software mode is related to fill rate, which is defined as the number of pixels that you are rendering.</span></span> <span data-ttu-id="44ee0-115">如果您擔心軟件轉譯模式中的效能, 請嘗試將圖元重繪的次數減至最少。</span><span class="sxs-lookup"><span data-stu-id="44ee0-115">If you are concerned about performance in software rendering mode, try to minimize the number of times a pixel is redrawn.</span></span> <span data-ttu-id="44ee0-116">例如, 如果您的應用程式具有藍色背景, 然後在其上呈現稍微透明的影像, 您會將應用程式中的所有圖元呈現兩次。</span><span class="sxs-lookup"><span data-stu-id="44ee0-116">For example, if you have an application with a blue background, which then renders a slightly transparent image over it, you will render all of the pixels in the application twice.</span></span> <span data-ttu-id="44ee0-117">因此, 使用影像轉譯應用程式所需的時間會比只有藍色背景來得多兩倍。</span><span class="sxs-lookup"><span data-stu-id="44ee0-117">As a result, it will take twice as long to render the application with the image than if you had only the blue background.</span></span>  
  
### <a name="graphics-rendering-tiers"></a><span data-ttu-id="44ee0-118">圖形轉譯層</span><span class="sxs-lookup"><span data-stu-id="44ee0-118">Graphics Rendering Tiers</span></span>  
 <span data-ttu-id="44ee0-119">預測您的應用程式將在其上執行的硬體設定可能非常棘手。</span><span class="sxs-lookup"><span data-stu-id="44ee0-119">It may be very difficult to predict the hardware configuration that your application will be running on.</span></span> <span data-ttu-id="44ee0-120">不過, 您可能會想要考慮允許應用程式在不同硬體上執行時順暢地切換功能的設計, 讓它可以充分利用每個不同的硬體設定。</span><span class="sxs-lookup"><span data-stu-id="44ee0-120">However, you might want to consider a design that allows your application to seamlessly switch features when running on different hardware, so that it can take full advantage of each different hardware configuration.</span></span>  
  
 <span data-ttu-id="44ee0-121">為了達到這個目的[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] , 提供了在執行時間決定系統圖形功能的功能。</span><span class="sxs-lookup"><span data-stu-id="44ee0-121">To achieve this, [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] provides functionality to determine the graphics capability of a system at runtime.</span></span> <span data-ttu-id="44ee0-122">圖形功能的決定方式是將視訊卡分類為三種轉譯功能層的其中一個。</span><span class="sxs-lookup"><span data-stu-id="44ee0-122">Graphics capability is determined by categorizing the video card as one of three rendering capability tiers.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]<span data-ttu-id="44ee0-123">公開可讓應用程式查詢轉譯功能層的 API。</span><span class="sxs-lookup"><span data-stu-id="44ee0-123">exposes an API that allows an application to query the rendering capability tier.</span></span> <span data-ttu-id="44ee0-124">然後, 您的應用程式可以在執行時間根據硬體所支援的轉譯層來採取不同的程式碼路徑。</span><span class="sxs-lookup"><span data-stu-id="44ee0-124">Your application can then take different code paths at run time depending on the rendering tier supported by the hardware.</span></span>  
  
 <span data-ttu-id="44ee0-125">對轉譯層級的影響最大的圖形硬體功能如下︰</span><span class="sxs-lookup"><span data-stu-id="44ee0-125">The features of the graphics hardware that most impact the rendering tier levels are:</span></span>  
  
- <span data-ttu-id="44ee0-126">**視訊 RAM**：圖形硬體上的視訊記憶體數量決定可用於組合圖形的緩衝區大小和數目。</span><span class="sxs-lookup"><span data-stu-id="44ee0-126">**Video RAM** The amount of video memory on the graphics hardware determines the size and number of buffers that can be used for compositing graphics.</span></span>  
  
- <span data-ttu-id="44ee0-127">**像素著色器**：像素著色器是根據像素來計算效果的圖形處理函式。</span><span class="sxs-lookup"><span data-stu-id="44ee0-127">**Pixel Shader** A pixel shader is a graphics processing function that calculates effects on a per-pixel basis.</span></span> <span data-ttu-id="44ee0-128">根據所顯示圖形的解析度，每個顯示框架都可能需要處理數百萬個像素。</span><span class="sxs-lookup"><span data-stu-id="44ee0-128">Depending on the resolution of the displayed graphics, there could be several million pixels that need to be processed for each display frame.</span></span>  
  
- <span data-ttu-id="44ee0-129">**頂點著色器**：頂點著色器是在物件頂點資料上執行數學運算的圖形處理函式。</span><span class="sxs-lookup"><span data-stu-id="44ee0-129">**Vertex Shader** A vertex shader is a graphics processing function that performs mathematical operations on the vertex data of the object.</span></span>  
  
- <span data-ttu-id="44ee0-130">**多紋理支援**：多紋理支援指的是可以在混色作業期間於 3D 圖形物件上套用兩個以上的不同紋理。</span><span class="sxs-lookup"><span data-stu-id="44ee0-130">**Multitexture Support** Multitexture support refers to the ability to apply two or more distinct textures during a blending operation on a 3D graphics object.</span></span> <span data-ttu-id="44ee0-131">多紋理支援的程度取決於圖形硬體上的多紋理單位數目。</span><span class="sxs-lookup"><span data-stu-id="44ee0-131">The degree of multitexture support is determined by the number of multitexture units on the graphics hardware.</span></span>  
  
 <span data-ttu-id="44ee0-132">[圖元著色器]、[頂點著色器] 和 [多紋理] 功能是用來定義特定的 DirectX 版本層級, 而後者則是用來[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]在中定義不同的呈現層。</span><span class="sxs-lookup"><span data-stu-id="44ee0-132">The pixel shader, vertex shader, and multitexture features are used to define specific DirectX version levels, which, in turn, are used to define the different rendering tiers in [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span></span>  
  
 <span data-ttu-id="44ee0-133">圖形硬體的功能決定 [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] 應用程式的轉譯功能。</span><span class="sxs-lookup"><span data-stu-id="44ee0-133">The features of the graphics hardware determine the rendering capability of a [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] application.</span></span> <span data-ttu-id="44ee0-134">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] 系統定義三個轉譯層︰</span><span class="sxs-lookup"><span data-stu-id="44ee0-134">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] system defines three rendering tiers:</span></span>  
  
- <span data-ttu-id="44ee0-135">**轉譯層 0**：沒有圖形硬體加速。</span><span class="sxs-lookup"><span data-stu-id="44ee0-135">**Rendering Tier 0** No graphics hardware acceleration.</span></span> <span data-ttu-id="44ee0-136">DirectX 版本層級小於7.0 版。</span><span class="sxs-lookup"><span data-stu-id="44ee0-136">The DirectX version level is less than version 7.0.</span></span>  
  
- <span data-ttu-id="44ee0-137">轉譯**層 1**部分圖形硬體加速。</span><span class="sxs-lookup"><span data-stu-id="44ee0-137">**Rendering Tier 1** Partial graphics hardware acceleration.</span></span> <span data-ttu-id="44ee0-138">DirectX 版本層級大於或等於7.0 版, 且小於9.0 版 。</span><span class="sxs-lookup"><span data-stu-id="44ee0-138">The DirectX version level is greater than or equal to version 7.0, and **lesser** than version 9.0.</span></span>  
  
- <span data-ttu-id="44ee0-139">**轉譯層 2**：大部分圖形功能都使用圖形硬體加速。</span><span class="sxs-lookup"><span data-stu-id="44ee0-139">**Rendering Tier 2** Most graphics features use graphics hardware acceleration.</span></span> <span data-ttu-id="44ee0-140">DirectX 版本層級大於或等於9.0 版。</span><span class="sxs-lookup"><span data-stu-id="44ee0-140">The DirectX version level is greater than or equal to version 9.0.</span></span>  
  
 <span data-ttu-id="44ee0-141">如需[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]呈現層的詳細資訊, 請參閱[圖形轉譯層](graphics-rendering-tiers.md)。</span><span class="sxs-lookup"><span data-stu-id="44ee0-141">For more information on [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] rendering tiers, see [Graphics Rendering Tiers](graphics-rendering-tiers.md).</span></span>
