# 項目樹狀結構和序列化
WPF 程式設計項目通常是以彼此間某種樹狀結構關係的形式存在。  例如，在 XAML 中建立的應用程式 UI 可以概念化為物件樹狀結構。  此項目樹狀結構可再分成兩個不同但有時平行的樹狀結構：[邏輯樹狀結構](GTMT)和[視覺化樹狀結構](GTMT)。  WPF 中的序列化包括儲存這兩個樹狀結構的狀態以及應用程式狀態，並將其寫入檔案 \(可能採用 XAML 格式\)。
