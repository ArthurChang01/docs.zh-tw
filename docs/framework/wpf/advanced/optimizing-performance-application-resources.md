## <a name="use-static-resources-when-possible"></a><span data-ttu-id="937de-121">盡可能使用靜態資源</span><span class="sxs-lookup"><span data-stu-id="937de-121">Use Static Resources when Possible</span></span>  
 <span data-ttu-id="937de-122">靜態資源會藉由查閱已定義資源的參考，提供任何 XAML 屬性屬性的值。</span><span class="sxs-lookup"><span data-stu-id="937de-122">A static resource provides a value for any XAML property attribute by looking up a reference to an already defined resource.</span></span> <span data-ttu-id="937de-123">該資源的查閱行為類似于編譯時間查閱。</span><span class="sxs-lookup"><span data-stu-id="937de-123">Lookup behavior for that resource is analogous to compile-time lookup.</span></span>  
  
 <span data-ttu-id="937de-124">另一方面，動態資源會在初始編譯期間建立暫存運算式，因此會延遲資源的查閱，直到實際需要要求的資源值才能建立物件為止。</span><span class="sxs-lookup"><span data-stu-id="937de-124">A dynamic resource, on the other hand, will create a temporary expression during the initial compilation and thus defer lookup for resources until the requested resource value is actually required in order to construct an object.</span></span> <span data-ttu-id="937de-125">該資源的查閱行為類似于執行時間查詢，這會影響效能。</span><span class="sxs-lookup"><span data-stu-id="937de-125">Lookup behavior for that resource is analogous to run-time lookup, which imposes a performance impact.</span></span> <span data-ttu-id="937de-126">請盡可能在您的應用程式中使用靜態資源，只有在必要時才使用動態資源。</span><span class="sxs-lookup"><span data-stu-id="937de-126">Use static resources whenever possible in your application, using dynamic resources only when necessary.</span></span>
