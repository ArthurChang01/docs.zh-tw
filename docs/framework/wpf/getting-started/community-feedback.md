# <a name="wpf-community-feedback"></a>WPF 社群意見回應
[!INCLUDE[TLA#tla_ms](../../../../includes/tlasharptla-ms-md.md)] 公開各種社群資源供您學習、討論及提供有關 [!INCLUDE[TLA#tla_wpf](../../../../includes/tlasharptla-wpf-md.md)] 的意見反應，包括 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組成員和社群領導人建立的部落格、論壇及 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 產品意見反應中心。 每一種社群資源分別提供不同的優點。 本文中會說明這些優點，以及利用各項優點的最佳做法，以確保所有社群和 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 都能提供最佳的回應。  
  
> [!NOTE]
>  請勿使用各頁面下方的 [註解] 連結來傳送產品的意見反應。 這些連結僅適用於對文件的意見反應。  
  
 
  
<a name="Blogs"></a>   
## <a name="blogs"></a>部落格  
 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組成員和社群領導人撰寫了針對 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 整體或特定 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 子集的各種[部落格](http://go.microsoft.com/fwlink/?LinkId=98852)。 部落格的內容大致涵蓋了產品開發、動機和原理，乃至於特定案例、問題和 [!INCLUDE[TLA#tla_api](../../../../includes/tlasharptla-api-md.md)] 的詳細討論。 此外，您也可以與最深入了解平台的人員討論或詢問自己有興趣的文章。  
  
<a name="Forums"></a>   
## <a name="forums"></a>論壇  
 [WPF 論壇](http://go.microsoft.com/fwlink/?LinkID=187440) (以及 [!INCLUDE[TLA#tla_winvista](../../../../includes/tlasharptla-winvista-md.md)] 64 的[論壇 (英文)](http://go.microsoft.com/fwlink/?LinkId=98855)) 是討論與解決問題的主要社群資源。 論壇提供了一組完整的支援功能以進行討論及解決問題，包括：  
  
-   搜尋。  
  
-   討論追蹤。  
  
-   豐富的文字和程式碼格式。  
  
-   [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)] 整合。  
  
-   最有價值專家 (Most Valued Professional，MVP) 和社群的參與。  
  
-   透過監視確定文章能夠在最快的時間內獲得回應。  
  
 請注意，雖然您的問題可能與 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 相關，但 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 論壇可能不是最適合提出該問題的地方。 例如，您的問題可能與特定的 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 工具有關，例如 [Expression Blend](http://go.microsoft.com/fwlink/?LinkId=98858)、[Expression Web](http://go.microsoft.com/fwlink/?LinkId=98859) 和 [Expression Design](http://go.microsoft.com/fwlink/?LinkId=98860)。 在這些情況下，您應該使用適當的 [Expression 社群論壇](http://go.microsoft.com/fwlink/?LinkId=98861)。  
  
### <a name="forum-best-practices"></a>論壇最佳做法  
 使用下列最佳做法有助於在最快的時間內解決張貼到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 論壇的問題。 這些作法適用於所有的論壇。  
  
#### <a name="search-existing-posts"></a>搜尋現有文章  
 某些問題經常發生，其他人也曾經遇到相同的問題。 因此，您可以快速解決問題，或是在現有的討論加上您的意見。  
  
#### <a name="use-meaningful-titles"></a>使用有意義的標題  
 簡潔又有意義的標題不只能提升您文章的可探索性，也能讓其他 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 論壇社群成員更容易判斷他們是否能夠解決您的問題。  
  
#### <a name="include-appropriate-content"></a>加入適當的內容  
 您應該描述問題及您嘗試處理該問題的方式。 可能的話，請加入支援程式碼片段，或是可示範問題的最簡短範例。 這些詳細資料將有助於增加問題迅速獲得解答的機會。  
  
 您可在[線上](http://www.microsoft.com/wn3/locales/help/help_en-us.htm)找到完整的論壇最佳做法。  
  
<a name="Product_Feedback_Center"></a>   
## <a name="product-feedback-center"></a>產品回報中心  
 某些問題可能不容易解決或根本無法解決。 發生這種情況的原因包括技術上的 Bug、將技術套用至特定案例時發生困難，或是缺乏特定案例的支援等。 這項資訊對於 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 而言非常重要，而且可以透過[產品意見反應中心](http://go.microsoft.com/fwlink/?LinkId=99381)提供。  
  
 張貼在 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 產品意見反應中心的項目會傳送到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組的內部 Bug 資料庫。 因此，這是將您的意見回報給 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 功能負責人最可靠方式。 此外，您還可以驗證和追蹤建議與 Bug，也可以進行投票，以協助 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組決定問題的優先順序。  
  
### <a name="wpf-product-feedback-center-best-practices"></a>WPF 產品意見反應中心最佳做法  
 當張貼文章至 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 產品意見反應中心時，搜尋現有文章、提供有意義的標題和適當的內容都是重要的最佳做法，就如同張貼文章到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 一樣。 以下是您應該採用的其他最佳做法。  
  
#### <a name="search-existing-posts"></a>搜尋現有文章  
 某些問題經常發生，其他人也曾經遇到相同的問題。 因此，您可以快速解決問題，或是在現有的討論加上您的意見。  
  
#### <a name="use-meaningful-titles"></a>使用有意義的標題  
 簡潔又有意義的標題，可增加問題在最短時間內轉給最適當 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組的機會。 這對於 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 這類包含許多相互關聯功能的技術來說特別重要。  
  
#### <a name="describe-how-to-reproduce-your-bug"></a>說明如何重現您的 Bug  
 當您張貼有關 Bug 的文章時，請務必加入下列各項相關資訊：  
  
-   提供的 Bug 詳細描述。  
  
-   使用程式碼片段輔助說明 Bug。  
  
-   提供示範如何重現 Bug 的步驟清單。  
  
-   加入可重現 Bug 的最簡單程式碼範例。  
  
-   說明 Bug 是否能以相同的方式重現。  
  
-   加入相關的例外狀況資訊。  
  
 如果 Bug 與安裝或設定相關，請附上相關的安裝記錄和快照 (位於您 %temp% 資料夾中開頭有 "dd_" 的檔案)。  
  
 若為編譯或組建問題，請附上組建記錄檔。 您可以設定 MSBuild 支援使用各種詳細等級進行記錄，方法是從命令列使用 /v: 參數，或是從 [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)] 之類的 [!INCLUDE[TLA#tla_ide](../../../../includes/tlasharptla-ide-md.md)] 設定適當的層級。  
  
#### <a name="provide-environment-information"></a>提供環境資訊  
 背景資訊通常有助於增加文章的內容。 特別是您可以說明作業系統平台、處理器系列和架構，例如「Windows XP SP2，Pentium III，x86」。  
  
 如果張貼的問題與轉譯有關，在可能的情況下，您也應該加入圖形卡和驅動程式的詳細資料。 這項資訊非常重要，因為 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 是一種展示架構。  
  
#### <a name="provide-solution-or-project-information"></a>提供方案或專案資訊  
 Bug 可能與您用來開發和建置應用程式的工具，以及您所建置的應用程式類型有關。 因此，列出下列資訊可能會有幫助：  
  
-   您所建置的應用程式類型，例如：  
  
    -   應用程式 (.exe) 或程式庫 (.dll)。  
  
    -   [!INCLUDE[TLA#tla_xaml](../../../../includes/tlasharptla-xaml-md.md)] 瀏覽器應用程式 (XBAP)。  
  
    -   鬆散的 [!INCLUDE[TLA2#tla_xaml](../../../../includes/tla2sharptla-xaml-md.md)] 應用程式。  
  
    -   獨立安裝的應用程式。  
  
    -   獨立 ClickOnce 部署的應用程式。  
  
-   開發工具，例如：  
  
    -   MSBuild。  
  
    -   Expression Graphic Designer。  
  
    -   Expression Interactive Designer。  
  
    -   [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)].  
  
-   方案設定，例如：  
  
    -   方案。  
  
    -   單一專案。  
  
    -   具有多個相依專案的方案。  
  
-   您的應用程式是否有特定語言或非語言相關的資源。 例如，您是否指定了 `Application`、`Page` 和 `Resource` 等類型的 `UICulture` 專案屬性或可當地語系化的中繼資料？  
  
-   您是否在 AssemblyInfo.cs 或 AssemblyInfo.vb 檔案中使用了非語言相關的設定。  
  
#### <a name="provide-scenario-and-impact-information"></a>提供案例和影響資訊  
 當 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組決定是否、何時及如何修正問題，以及是否可以改用可接受的因應措施時，出現 Bug 的案例及造成的影響等相關資訊非常重要。  
  
 一般來說，當機和資料遺失案例的影響較大，因此也最容易設定優先權。 不過，某些 Bug 僅會在罕見的案例中出現，而這些案例可能是某些情況的主線案例。 提供案例和影響的相關內容有助於 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組做出正確的決策。

<a name="MSDN_Community_Content"></a>   
## <a name="msdn-community-content"></a>MSDN 社群內容  
 [Windows Presentation Foundation](../../../../docs/framework/wpf/index.md) 文件提供一組完整的技術內容，以 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 能實現的核心案例為目標。 當然，真實世界中的開發人員會遇到各式各樣有趣且不常見的案例，並因此學習到對 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 開發社群整體可能有所助益的事物。  
  
 為了讓開發人員能與 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 社群分享他們的知識和經驗，文件現在已經加入了 MSDN 社群內容，可讓開發人員在每個概念、參考、操作說明、教學課程和範例主題中新增及更新意見。  
  
 將資訊新增到 MSDN 社群內容的優點是可提供 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 集中、最新且共用的存放庫，而且社群知識和經驗可擴展線上文件的優勢。 此外，內容會經過仲裁，以確保提供的資訊清楚、正確，適當且實用。  
  
### <a name="what-type-of-content-should-be-added-to-msdn-community-content"></a>應該將何種內容類型新增到 MSDN 社群內容？  
 MSDN 社群內容設計用意在於擷取相關且有用的適當資訊，以擴充現有的 MSDN 內容。 張貼 Bug、意見反應，往來討論和問題時，應使用先前所述的適當、更有效的工具。  
  
### <a name="msdn-community-content-moderation"></a>MSDN 社群內容仲裁  
 MSDN 社群內容經過仲裁以確保內容的適當性。 如 MSDN [內容仲裁方針](http://go.microsoft.com/fwlink/?LinkId=98864)中所述，以下的內容類型會被視為不適當的內容：  
  
-   違反 MSDN 社群[管理辦法](http://go.microsoft.com/fwlink/?LinkId=98865)的內容  
  
-   不相關、難以理解和測試的內容  
  
-   重複的內容  
  
-   垃圾郵件  
  
-   文件意見反應  
  
-   錯誤報告  
  
-   問題  
  
 仲裁者會根據 MSDN 社群內容仲裁方針處理這些類型的內容。  
  
### <a name="msdn-community-content-language-support"></a>MSDN 社群內容語言支援  
 MSDN Online 的英文 (美國) 和巴西葡萄牙文版本都提供 MSDN 社群內容。 未來考慮推出日文、法文、德文、義大利文、西班牙文、韓文，以及簡體中文和繁體中文。
