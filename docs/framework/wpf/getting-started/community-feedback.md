# WPF 社群意見回應
[!INCLUDE[TLA#tla_ms](../../../../includes/tlasharptla-ms-md.md)] 公開各種社群資源供您學習、討論及提供有關 [!INCLUDE[TLA#tla_wpf](../../../../includes/tlasharptla-wpf-md.md)] 的意見回應，包括 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組成員及社群領導者創作的網誌、論壇和 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 產品回報中心。  每一種社群資源分別提供不同的優點。  本文說明這些優點，以及使用各項優點的最佳做法，以確保所有社群和 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 都能提供最佳的回應。  
  
> [!NOTE]
>  請勿使用各頁面下方的 \[**註解**\] 連結來傳送產品的意見回應。  這些連結僅適用於文件意見回應。  
  
   
  
<a name="Blogs"></a>   
## 網誌  
 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組成員和社群領導者撰寫了針對 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 整體或特定 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 子集的各種[網誌](http://go.microsoft.com/fwlink/?LinkId=98852)。  網誌的內容大致涵蓋了產品開發、動機和原理，乃至於特定案例、問題及 [!INCLUDE[TLA#tla_api](../../../../includes/tlasharptla-api-md.md)] 的詳細討論。  此外，您也可以與深入了解平台的網友進一步討論或詢問自己有興趣的文章。  
  
<a name="Forums"></a>   
## 論壇  
 [WPF 論壇](http://go.microsoft.com/fwlink/?LinkID=187440) \(以及 [!INCLUDE[TLA#tla_winvista](../../../../includes/tlasharptla-winvista-md.md)] 64 的[論壇](http://go.microsoft.com/fwlink/?LinkId=98855)\) 是討論及解決問題的主要社群資源。  為了協助討論及解決問題，論壇提供了一組多樣化的支援功能，包括：  
  
-   搜尋。  
  
-   討論追蹤。  
  
-   豐富的文字和程式碼格式。  
  
-   [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)] 整合。  
  
-   最有價值專家 \(Most Valued Professional，MVP\) 和社群的參與。  
  
-   透過監視確定文章能夠在最快的時間內獲得回應。  
  
 請注意，雖然您的問題可能與 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 有關，但是 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 論壇可能不是提出問題的最佳位置。  例如，您的問題可能與特定 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 工具有關，例如 [Expression Blend](http://go.microsoft.com/fwlink/?LinkId=98858)、[Expression Web](http://go.microsoft.com/fwlink/?LinkId=98859) 和 [Expression Design](http://go.microsoft.com/fwlink/?LinkId=98860)。  在這種情況下，您應該使用適當的 [Expression 社群論壇](http://go.microsoft.com/fwlink/?LinkId=98861)。  
  
### 論壇最佳做法  
 使用下列最佳做法有助於在最快的時間內處理張貼到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 論壇的問題。  這些做法適用於所有的論壇。  
  
#### 搜尋現有文章  
 某些問題經常發生，其他人也曾經遇到相同的問題。  因此，您可以快速解決問題，或是在現有的討論區加上您的意見。  
  
#### 使用有意義的標題  
 簡潔又有意義的標題不僅能提升文章的可探索性，也讓其他 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 論壇社群成員更容易判斷他們是否能夠解決您的問題。  
  
#### 加入適當的內容  
 您應該描述問題以及嘗試處理該問題的方式。  如果可能，請加入支援程式碼片段，或是可示範問題的最簡單範例。  這些詳細資料將有助於增加問題迅速獲得解答的機會。  
  
 您可在[線上](http://www.microsoft.com/wn3/locales/help/help_en-us.htm)找到完整的論壇最佳做法。  
  
<a name="Product_Feedback_Center"></a>   
## 產品回報中心  
 某些問題可能不容易解決或根本無法解決。  發生這種情況的原因包括技術上的 Bug、將技術套用至特定案例時發生困難，或是缺乏特定案例的支援等。  這項資訊對於 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 而言非常重要，而且可以透過[產品回報中心](http://go.microsoft.com/fwlink/?LinkId=99381)提供。  
  
 張貼在 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 產品回報中心的項目將會傳送到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組的內部 Bug 資料庫。  因此，這是將您的意見回報給 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 功能負責人最可靠的一種方式。  此外，您還可以驗證及追蹤建議和 Bug，也可以對其進行投票，以幫助 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組決定問題的優先順序。  
  
### WPF 產品回報中心最佳做法  
 張貼文章到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 產品回報中心時，搜尋現有文章、提供有意義的標題和適當的內容都是重要的最佳做法，就如同張貼文章到 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 一樣。  以下說明您還應該採用的其他最佳做法。  
  
#### 搜尋現有文章  
 某些問題經常發生，其他人也曾經遇到相同的問題。  因此，您可以快速解決問題，或是在現有的討論區加上您的意見。  
  
#### 使用有意義的標題  
 簡潔又有意義的標題，可增加問題在最短時間內轉給最適當 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組的機會。  這對於 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 這類包含許多相互關聯功能的技術來說特別重要。  
  
#### 說明如何重現您的 Bug  
 當您張貼有關 Bug 的文章時，請務必在適當處加入下列內容：  
  
-   提供 Bug 的詳細描述。  
  
-   使用程式碼片段以輔助 Bug 的說明。  
  
-   提供示範如何重現 Bug 的步驟清單。  
  
-   加入可重現 Bug 的最簡單程式碼範例。  
  
-   說明是否可以相同的方式重現 Bug。  
  
-   加入相關的例外資訊。  
  
 如果 Bug 與安裝或設定有關，請附上相關的安裝記錄和快照 \(Snapshot，也就是 %temp% 資料夾中具有前置字元 "dd\_" 的檔案\)。  
  
 若為編譯或建置 \(Build\) 的問題，請附上建置記錄。  您可以從命令列使用 \/v: 參數，或是從 [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)] 之類的 [!INCLUDE[TLA#tla_ide](../../../../includes/tlasharptla-ide-md.md)] 設定適當的層級，以便將 MSBuild 系統設定為支援使用不同詳細等級進行記錄的功能。  
  
#### 提供環境資訊  
 背景資訊通常有助於增加張貼文章的內容。  請具體列出作業系統平台、處理器系列和架構，例如 "Windows XP SP2, Pentium III, x86"。  
  
 如果張貼的問題與轉譯有關，可能的情況下，您也應該加入圖形卡和驅動程式的詳細資訊。  這項資訊非常重要，因為 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 是一種展示架構。  
  
#### 提供方案或專案資訊  
 Bug 可能與您用來開發及建置應用程式的工具，以及您所建置的應用程式類型有關。  因此，列出下列資訊可能很有幫助：  
  
-   您所建置的應用程式類型，例如：  
  
    -   應用程式 \(.exe\) 或程式庫 \(.dll\)。  
  
    -   [!INCLUDE[TLA#tla_xaml](../../../../includes/tlasharptla-xaml-md.md)] 瀏覽器應用程式 \(XBAP\)。  
  
    -   鬆散 [!INCLUDE[TLA2#tla_xaml](../../../../includes/tla2sharptla-xaml-md.md)] 應用程式。  
  
    -   獨立安裝應用程式。  
  
    -   獨立部署 ClickOnce 的應用程式。  
  
-   開發工具，例如：  
  
    -   MSBuild。  
  
    -   Expression Graphic Designer。  
  
    -   Expression Interactive Designer。  
  
    -   [!INCLUDE[TLA#tla_visualstu](../../../../includes/tlasharptla-visualstu-md.md)].  
  
-   方案組態，例如：  
  
    -   方案。  
  
    -   單一專案。  
  
    -   內含多個相依專案的方案。  
  
-   您的應用程式是否具有語言相關或非語言相關的資源。  例如，您是否指定了 `Application`、`Page` 和 `Resource` 等型別的 `UICulture` 專案屬性或可當地語系化的中繼資料 \(Metadata\)？  
  
-   您是否在 AssemblyInfo.cs 或 AssemblyInfo.vb 檔案中使用了中性語言設定？  
  
#### 提供情節和影響資訊  
 當 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組決定是否、何時及如何修正問題，以及是否可以改用可接受的替代解決辦法時，出現 Bug 的案例及造成的影響等相關資訊對他們來說非常重要。  
  
 一般來說，沒有回應和資料流失等情節的影響較大，因此也最容易成為優先處理的狀況。  不過，某些 Bug 只會在較不常見的情節中出現，而這些情節可能也是部分案例的主線情節。  提供相關情節和影響的內容將可幫助 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 小組做出正確的決定。

<a name="MSDN_Community_Content"></a>   
## MSDN 社群內容  
 [Windows Presentation Foundation](../../../../docs/framework/wpf/index.md) 文件提供一組完整的技術內容，其以 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 所能實現的核心情節做為主要探討對象。  在真實世界中，開發人員當然會遇到各式各樣有趣且較不常見的情節，也因此可以學到對 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 開發社群整體可能有所助益的事物。  
  
 為了讓開發人員能夠與 [!INCLUDE[TLA2#tla_wpf](../../../../includes/tla2sharptla-wpf-md.md)] 社群分享他們的知識和經驗，文件現在已經加入了 MSDN 社群內容，供開發人員在每個概念、參考、使用說明和範例等主題中加入及更新意見。  
  
 在 MSDN 社群內容中加入資訊的優點是可以提供最新的集中共用儲存機制，用以儲存可擴充線上文件之優勢的 [!INCLUDE[TLA2#tla_ms](../../../../includes/tla2sharptla-ms-md.md)] 與社群知識和經驗。  此外，內容也會經過控管，以確保提供的資訊清楚、正確且有幫助。  
  
### 應該在 MSDN 社群內容中加入何種內容？  
 MSDN 社群內容的設計用意是為了取得相關、有用且適當的資訊，藉以擴充現有的 MSDN 內容。  張貼 Bug、意見回應、討論串和問題時，都應該使用前面所述適當的、更有效的工具。  
  
### MSDN 社群內容控管  
 MSDN 社群內容已透過控管的方式來確保內容的適當性。  如 MSDN [內容控管方針](http://go.microsoft.com/fwlink/?LinkId=98864) \(英文\) 所述，控管人員會將下列類型的內容視為不適當的內容：  
  
-   違反 MSDN 社群[行為規範](http://go.microsoft.com/fwlink/?LinkId=98865)的內容  
  
-   不相關、難以理解和測試用的內容  
  
-   重複的內容  
  
-   垃圾郵件  
  
-   文件意見反應  
  
-   Bug 報告  
  
-   問題  
  
 控管人員將根據 MSDN 社群內容控管方針來處理這些類型的內容。  
  
### MSDN 社群內容語言支援  
 英文 \(美國\) 和巴西葡萄牙文兩種版本的 MSDN Online 上都有 MSDN 社群內容。  未來考慮推出的版本包括日文、法文、德文、義大利文、西班牙文、韓文以及簡體和繁體中文。
