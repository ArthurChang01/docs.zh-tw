# WIF 程式碼範例索引
Windows 識別基礎 4.5 的程式碼範例如下：  
  
-   [ClaimsAwareWebApp](完整.嗎？LinkID%20=%20248405) \-這個範例會使用基本驗證 externalization \(至本機測試安全性語彙基元服務身分識別與存取工具從 Visual Studio 11\) 示範在傳統的 ASP。NET 應用程式中 \(相對於該網站\)。  
  
-   [ClaimsAwareWebService](完整.嗎？LinkID%20=%20248406) \-這個範例會示範典型的 WCF 服務的驗證 externalization 基本用途。  
  
-   [ClaimsAwareMvcApplication](完整.嗎？LinkID%20=%20248407) \-這個範例會示範如何將 WIF 與 MVC，其中包括非資料組保護與程式碼一樣，會優先處理表單驗證重新導向至登入控制站用完。  
  
-   [ClaimsAwareWebFarm](完整.嗎？LinkID%20=%20248408) \-這個範例會示範伺服陣列可以開始的工作階段快取區 \(相對於 tokenreplycache\)，讓您可以使用參考，而不是交換大 cookie 的工作階段。  它還示範保全 cookie 在伺服陣列中的更簡單的方法。  
  
-   [ClaimsAwareFormsAuthentication](完整.嗎？LinkID%20=%20248409) \-此非常簡單的範例會示範如何知道這。NET 4.5 宣告進去在您的原則，不論您如何驗證您的使用者。  
  
-   [ClaimsBasedAuthorization](完整.嗎？LinkID%20=%20248410)\-這個範例會示範如何使用您的 CLaimsAuthorizationManager 類別和 ClaimsAuthorizationModule 來套用您自己的授權原則。  
  
-   [FederationMetadata](完整.嗎？LinkID%20=%20248411) – 這個範例會示範同時 \(在自訂的 STS\) 動態地產生和消耗動態率 \(信賴憑證者的合作對象應用程式\) 的中繼資料文件。  
  
-   [CustomToken](完整.嗎？LinkID%20=%20248412) – 這個範例會示範如何建立自訂的簡單 Web 語彙基元 \(SWT\) 語彙基元型別。
