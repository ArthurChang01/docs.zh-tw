# <a name="ihostmanualeventwait-method"></a>IHostManualEvent::Wait 方法
導致目前的[IHostManualEvent](../../../../docs/framework/unmanaged-api/hosting/ihostmanualevent-interface.md)實例等到其擁有，或指定的時間量過後才等候。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT Wait (  
    [in] DWORD dwMilliseconds,  
    [in] DWORD option  
);  
```  
  
## <a name="parameters"></a>參數  
 `dwMilliseconds`  
 在如果目前的 `IHostManualEvent` 實例不是擁有的，則傳回之前要等候的毫秒數。  
  
 `option`  
 在其中一個[WAIT_OPTION](../../../../docs/framework/unmanaged-api/hosting/wait-option-enumeration.md)值，表示當此作業封鎖時，主機應採取的動作。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|已成功傳回 `Wait`。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime （CLR）尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 就無法在進程內使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
|HOST_E_DEADLOCK|主機在等候間隔期間偵測到鎖死，並選擇目前的 `IHostManualEvent` 實例作為鎖死犧牲者。|
