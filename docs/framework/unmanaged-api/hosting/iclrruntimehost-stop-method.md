# <a name="iclrruntimehoststop-method"></a>ICLRRuntimeHost::Stop 方法
由 common language runtime (CLR) 停止執行程式碼。  
  
> [!IMPORTANT]
> 這個方法不會將資源釋放到主機、卸載應用程式域或終結執行緒。 您必須終止進程以釋放這些資源。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT Stop();  
```  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`Stop`已成功傳回。|  
|HOST_E_CLRNOTAVAILABLE|CLR 尚未載入進程中, 或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 如果方法傳回 E_FAIL, 則 CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|
