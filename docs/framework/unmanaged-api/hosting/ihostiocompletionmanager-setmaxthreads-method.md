# <a name="ihostiocompletionmanagersetmaxthreads-method"></a>IHostIoCompletionManager::SetMaxThreads 方法
設定主機 allots 至服務 i/o 要求的執行緒數目上限。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT SetMaxThreads (  
    [in] DWORD dwMaxIoCompletionThreads  
);  
```  
  
## <a name="parameters"></a>參數  
 `dwMaxIoCompletionThreads`  
 在要為 i/o 要求所分配的最大執行緒數目。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|已成功傳回 `SetMaxThreads`。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime （CLR）尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 就無法在進程內使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
|E_NOTIMPL|主機未提供 `SetMaxThreads`的執行。|  
  
## <a name="remarks"></a>備註  
 `SetMaxThreads` 提供 CLR 一個機會來設定 i/o 埠上服務要求可用的執行緒數目上限。 主機可能需要獨佔控制執行緒集區的大小，原因包括執行、效能或擴充性。 基於這個理由，不需要主機來執行 `SetMaxThreads`。 在此情況下，主機應該會從這個方法傳回 E_NOTIMPL。
