# <a name="iclrtaskneedspriorityscheduling-method"></a>ICLRTask::NeedsPriorityScheduling 方法
取得值，指出目前正在切換的工作是否需要標示為高優先順序以進行重新排定。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT NeedsPriorityScheduling (  
    [out] BOOL *pbNeedsPriorityScheduling  
);  
```  
  
## <a name="parameters"></a>參數  
 `pbNeedsPriorityRescheduling`  
 [out] `true`，如果主機應該儘快嘗試重新排定目前的工作實例;否則，`false`。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|已成功傳回 `NeedsPriorityRescheduling`。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime （CLR）尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 就無法在進程內使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  
 在工作接近垃圾收集行程收集的情況下，CLR 會將 `pbNeedsPriorityScheduling` 的值設定為 `true`，表示高優先順序的重新排定。 這可讓主機快速重新排程工作，藉此將垃圾收集延遲的可能性降到最低，並讓主機和執行時間能夠共同作業以節省記憶體資源。
