# <a name="icorruntimehoststart-method"></a>ICorRuntimeHost::Start 方法
啟動 common language runtime (CLR)。  
  
## <a name="syntax"></a>語法  
  
```  
HRESULT Start ();  
```  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|此作業成功。|  
|S_FALSE|無法完成操作。|  
|E_FAIL|發生未知的嚴重失敗。 若方法會傳回 E_FAIL，CLR 就不會再處理序中。 任何裝載的應用程式開發介面的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
|HOST_E_CLRNOTAVAILABLE|CLR 尚未載入到處理程序，或 CLR 正在中它無法執行 managed 程式碼，或成功地處理呼叫的狀態。|  
  
## <a name="remarks"></a>備註  
 它通常是不需要呼叫`Start`方法，因為 CLR 執行 managed 程式碼的第一個要求時自動啟動。
