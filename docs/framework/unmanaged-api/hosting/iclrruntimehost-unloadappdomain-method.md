# <a name="iclrruntimehostunloadappdomain-method"></a>ICLRRuntimeHost::UnloadAppDomain 方法
卸載對應至指定之數位識別碼的 managed <xref:System.AppDomain>。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT UnloadAppDomain(  
    [in] DWORD dwAppDomainId  
    [in] BOOL  fWaitUntilDone  
);  
```  
  
## <a name="parameters"></a>參數  
 `dwAppDomainId`  
 在要卸載之應用程式域的數值識別碼。  
  
 `fWaitUntilDone`  
 [in] `true`，表示 common language runtime （CLR）必須等到它完成執行應用程式的目前線程，然後再嘗試卸載應用程式域。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|已成功傳回 `UnloadAppDomain`。|  
|HOST_E_CLRNOTAVAILABLE|CLR 尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 如果方法傳回 E_FAIL，則 CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  
 您可以藉由呼叫[GetCurrentAppDomainId](../../../../docs/framework/unmanaged-api/hosting/iclrruntimehost-getcurrentappdomainid-method.md)，取得目前線程執行所在之應用程式域的數值識別碼。 這個識別碼會對應至 managed <xref:System.AppDomain> 類型的 <xref:System.AppDomain.Id%2A> 屬性。
