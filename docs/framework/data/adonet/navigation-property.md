# <a name="navigation-property"></a><span data-ttu-id="25fc8-102">導覽屬性</span><span class="sxs-lookup"><span data-stu-id="25fc8-102">navigation property</span></span>
<span data-ttu-id="25fc8-103">A*導覽屬性*上是選用屬性[實體類型](../../../../docs/framework/data/adonet/entity-type.md)，可允許從一個導覽[結束](../../../../docs/framework/data/adonet/association-end.md)的[關聯](../../../../docs/framework/data/adonet/association-type.md)至另一端。</span><span class="sxs-lookup"><span data-stu-id="25fc8-103">A *navigation property* is an optional property on an [entity type](../../../../docs/framework/data/adonet/entity-type.md) that allows for navigation from one [end](../../../../docs/framework/data/adonet/association-end.md) of an [association](../../../../docs/framework/data/adonet/association-type.md) to the other end.</span></span> <span data-ttu-id="25fc8-104">不同於其他[屬性](../../../../docs/framework/data/adonet/property.md)，導覽屬性不會包含資料。</span><span class="sxs-lookup"><span data-stu-id="25fc8-104">Unlike other [properties](../../../../docs/framework/data/adonet/property.md), navigation properties do not carry data.</span></span>  
  
 <span data-ttu-id="25fc8-105">導覽屬性包含下列定義：</span><span class="sxs-lookup"><span data-stu-id="25fc8-105">A navigation property definition includes the following:</span></span>  
  
-   <span data-ttu-id="25fc8-106">名稱。</span><span class="sxs-lookup"><span data-stu-id="25fc8-106">A name.</span></span> <span data-ttu-id="25fc8-107">(必要項)</span><span class="sxs-lookup"><span data-stu-id="25fc8-107">(Required)</span></span>  
  
-   <span data-ttu-id="25fc8-108">巡覽的關聯。</span><span class="sxs-lookup"><span data-stu-id="25fc8-108">The association that it navigates.</span></span> <span data-ttu-id="25fc8-109">(必要項)</span><span class="sxs-lookup"><span data-stu-id="25fc8-109">(Required)</span></span>  
  
-   <span data-ttu-id="25fc8-110">巡覽的關聯 End。</span><span class="sxs-lookup"><span data-stu-id="25fc8-110">The ends of the association that it navigates.</span></span> <span data-ttu-id="25fc8-111">(必要項)</span><span class="sxs-lookup"><span data-stu-id="25fc8-111">(Required)</span></span>  
  
 <span data-ttu-id="25fc8-112">請注意，在關聯各端點的實體類型上，導覽屬性是選擇性的。</span><span class="sxs-lookup"><span data-stu-id="25fc8-112">Note that navigation properties are optional on both entity types at the ends of an association.</span></span> <span data-ttu-id="25fc8-113">如果您在關聯其中一個端點的實體類型上定義導覽屬性，就不必在關聯另一個端點的實體類型上定義導覽屬性。</span><span class="sxs-lookup"><span data-stu-id="25fc8-113">If you define a navigation property on one entity type at the end of an association, you do not have to define a navigation property on the entity type at the other end of the association.</span></span>  
  
 <span data-ttu-id="25fc8-114">導覽屬性的資料類型由[多重性](../../../../docs/framework/data/adonet/association-end-multiplicity.md)其遠端的[關聯 end](../../../../docs/framework/data/adonet/association-end.md)。</span><span class="sxs-lookup"><span data-stu-id="25fc8-114">The data type of a navigation property is determined by the [multiplicity](../../../../docs/framework/data/adonet/association-end-multiplicity.md) of its remote [association end](../../../../docs/framework/data/adonet/association-end.md).</span></span> <span data-ttu-id="25fc8-115">例如，假設 `OrdersNavProp` 實體類型上有一個導覽屬性 `Customer`，則該導覽屬性可巡覽 `Customer` 和 `Order` 之間一對多的關聯。</span><span class="sxs-lookup"><span data-stu-id="25fc8-115">For example, suppose a navigation property, `OrdersNavProp`, exists on a `Customer` entity type and navigates a one-to-many association between `Customer` and `Order`.</span></span> <span data-ttu-id="25fc8-116">因為導覽屬性的遠端關聯 End 具有「多 (\*)」多重性，因此其資料型別是 (`Order` 的) 集合。</span><span class="sxs-lookup"><span data-stu-id="25fc8-116">Because the remote association end for the navigation property has multiplicity of many (\*), its data type is a collection (of `Order`).</span></span> <span data-ttu-id="25fc8-117">同樣地，如果 `CustomerNavProp` 實體類型上有一個導覽屬性 `Order`，其資料類型應為 `Customer`，因為遠端 End 的多重性是「一 (1)」。</span><span class="sxs-lookup"><span data-stu-id="25fc8-117">Similarly, if a navigation property, `CustomerNavProp`, exists on the `Order` entity type, its data type would be `Customer`, because the multiplicity of the remote end is one (1).</span></span>
