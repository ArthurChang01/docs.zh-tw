# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="91ca4-102">在 SQL Server 中簽署預存程序</span><span class="sxs-lookup"><span data-stu-id="91ca4-102">Signing Stored Procedures in SQL Server</span></span>
 <span data-ttu-id="91ca4-103">數位簽章是指使用簽署者之私密金鑰 (Private Key) 加密的資料摘要。</span><span class="sxs-lookup"><span data-stu-id="91ca4-103">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="91ca4-104">私密金鑰可確保數位簽章對於其 Bearer 或擁有者而言是唯一的。</span><span class="sxs-lookup"><span data-stu-id="91ca4-104">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="91ca4-105">您可以簽署預存程序、 函式 （除了內嵌資料表值函式）、 觸發程序，以及組件。</span><span class="sxs-lookup"><span data-stu-id="91ca4-105">You can sign stored procedures, functions (except for inline table-valued functions), triggers, and assemblies.</span></span>  
  
 <span data-ttu-id="91ca4-106">您可以使用憑證或非對稱金鑰來簽署預存程序。</span><span class="sxs-lookup"><span data-stu-id="91ca4-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="91ca4-107">這是針對無法透過擁有權鏈結繼承權限或擁有權鏈結中斷 (例如動態 SQL) 的案例所設計。</span><span class="sxs-lookup"><span data-stu-id="91ca4-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="91ca4-108">然後您可以建立使用者對應到憑證，授與憑證的使用者權限的預存程序需要存取的物件。</span><span class="sxs-lookup"><span data-stu-id="91ca4-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  

 <span data-ttu-id="91ca4-109">您可以也建立登入對應到相同的憑證，然後授與任何必要的伺服器層級權限給該登入，或加入至一或多個固定的伺服器角色的登入。</span><span class="sxs-lookup"><span data-stu-id="91ca4-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="91ca4-110">這為了避免在啟用`TRUSTWORTHY`資料庫的需要較高的層級權限中的設定。</span><span class="sxs-lookup"><span data-stu-id="91ca4-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="91ca4-114">建立憑證</span><span class="sxs-lookup"><span data-stu-id="91ca4-114">Creating Certificates</span></span>  
 <span data-ttu-id="91ca4-115">當您註冊使用憑證或非對稱金鑰，其中包含預存程序程式碼，以及執行加密雜湊的資料摘要的預存程序-做為使用者，會建立使用私密金鑰。</span><span class="sxs-lookup"><span data-stu-id="91ca4-115">When you sign a stored procedure with a certificate or asymmetric key, a data digest consisting of the encrypted hash of the stored procedure code, along with the execute-as user, is created using the private key.</span></span> <span data-ttu-id="91ca4-116">在執行階段中，系統會使用公開金鑰 (Public Key) 來解密資料摘要，並與預存程序的雜湊值 (Hash Value) 進行比較。</span><span class="sxs-lookup"><span data-stu-id="91ca4-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="91ca4-117">變更 execute-當使用者以便數位簽章不再相符，則雜湊值。</span><span class="sxs-lookup"><span data-stu-id="91ca4-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="91ca4-118">修改預存程序會卸除簽章完全，這可防止沒有私密金鑰存取權變更預存程序程式碼的人。</span><span class="sxs-lookup"><span data-stu-id="91ca4-118">Modifying the stored procedure drops the signature entirely, which prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="91ca4-119">在任一情況下，您必須重新簽署程序每次您變更程式碼或 execute 位使用者身分。</span><span class="sxs-lookup"><span data-stu-id="91ca4-119">In either case, you must re-sign the procedure each time you change the code or the execute-as user.</span></span>  
  
 <span data-ttu-id="91ca4-120">有兩個所需的步驟簽署模組：</span><span class="sxs-lookup"><span data-stu-id="91ca4-120">There are two required steps involved in signing a module:</span></span>

1.  <span data-ttu-id="91ca4-124">使用 Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` 陳述式並搭配憑證來簽署程序。</span><span class="sxs-lookup"><span data-stu-id="91ca4-124">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>  

<span data-ttu-id="91ca4-125">一旦已經簽署模組，一個或多個主體，就需要建立為了保存應與憑證相關聯的其他權限。</span><span class="sxs-lookup"><span data-stu-id="91ca4-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>  

<span data-ttu-id="91ca4-126">如果此模組需要其他的資料庫層級權限：</span><span class="sxs-lookup"><span data-stu-id="91ca4-126">If the module needs additional database-level permissions:</span></span>  
  
1.  <span data-ttu-id="91ca4-127">使用 Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` 陳述式來建立與憑證相關聯的資料庫使用者。</span><span class="sxs-lookup"><span data-stu-id="91ca4-127">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="91ca4-128">這個使用者只會在資料庫中存在，而且不與登入相關聯，除非也已經從相同的憑證建立登入。</span><span class="sxs-lookup"><span data-stu-id="91ca4-128">This user exists in the database only and is not associated with a login unless a login has also been created from that same certificate.</span></span>  
  
1.  <span data-ttu-id="91ca4-129">授與憑證使用者的必要的資料庫層級權限。</span><span class="sxs-lookup"><span data-stu-id="91ca4-129">Grant the certificate user the required database-level permissions.</span></span>  
  
<span data-ttu-id="91ca4-130">如果此模組需要額外的伺服器層級權限：</span><span class="sxs-lookup"><span data-stu-id="91ca4-130">If the module needs additional server-level permissions:</span></span>  
  
1.  <span data-ttu-id="91ca4-131">若要將憑證複製`master`資料庫。</span><span class="sxs-lookup"><span data-stu-id="91ca4-131">Copy the certificate to the `master` database.</span></span>  
 
1.  <span data-ttu-id="91ca4-132">建立與使用 Transact SQL 該憑證關聯的登入`CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]`陳述式。</span><span class="sxs-lookup"><span data-stu-id="91ca4-132">Create a login associated with that certificate using the Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` statement.</span></span>

> [!NOTE]
