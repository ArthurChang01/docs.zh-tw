## <a name="network-access"></a><span data-ttu-id="afde7-109">網路存取</span><span class="sxs-lookup"><span data-stu-id="afde7-109">Network Access</span></span>  
 <span data-ttu-id="afde7-110">基於安全性考量，SQL Server Express 中的網路通訊協定預設是停用的。</span><span class="sxs-lookup"><span data-stu-id="afde7-110">For security reasons, networking protocols are disabled by default in SQL Server Express.</span></span> <span data-ttu-id="afde7-111">這樣做可防止外部使用者的攻擊，避免危害裝載 SQL Server Express 執行個體的電腦。</span><span class="sxs-lookup"><span data-stu-id="afde7-111">This prevents attacks from outside users that might compromise the computer that hosts the instance of SQL Server Express.</span></span> <span data-ttu-id="afde7-112">您必須明確啟用網路連接並啟動 SQL Server Browser 服務，才能從另一部電腦連接至 SQL Server Express 執行個體。</span><span class="sxs-lookup"><span data-stu-id="afde7-112">You must explicitly enable network connectivity and start the SQL Server Browser service to connect to a SQL Server Express instance from another computer.</span></span>  
  
 <span data-ttu-id="afde7-113">一旦網路連接啟用之後，SQL Server Express 執行個體與其他 SQL Server 版本便具有相同的安全性需求。</span><span class="sxs-lookup"><span data-stu-id="afde7-113">Once network connectivity is enabled, a SQL Server Express instance has the same security requirements as the other editions of SQL Server.</span></span>  
  
## <a name="user-instances"></a><span data-ttu-id="afde7-114">使用者執行個體</span><span class="sxs-lookup"><span data-stu-id="afde7-114">User Instances</span></span>  
 <span data-ttu-id="afde7-115">使用者執行個體是 SQL Server Express Database Engine 的獨立執行個體，由 SQL Server Express 父執行個體所產生。</span><span class="sxs-lookup"><span data-stu-id="afde7-115">A user instance is a separate instance of the SQL Server Express database engine that is generated by a parent instance of SQL Server Express.</span></span> <span data-ttu-id="afde7-116">使用者執行個體的主要目標是允許在最小權限使用者帳戶底下執行 Windows 的使用者，針對其本機電腦上的 SQL Server Express 執行個體擁有系統管理員 (`sysadmin`) 權限。</span><span class="sxs-lookup"><span data-stu-id="afde7-116">The primary goal of a user instance is to allow users who are running Windows under a least-privilege user account to have system administrator (`sysadmin`) privileges on the SQL Server Express instance on their local computer.</span></span> <span data-ttu-id="afde7-117">使用者執行個體不適用於在其電腦上具有系統管理員身分的使用者。</span><span class="sxs-lookup"><span data-stu-id="afde7-117">User instances are not intended for users who are system administrators on their own computers.</span></span>  
  
 <span data-ttu-id="afde7-118">使用者執行個體是從代表使用者的 SQL Server 或 SQL Server Express 主要執行個體產生。</span><span class="sxs-lookup"><span data-stu-id="afde7-118">A user instance is generated from a primary instance of SQL Server or SQL Server Express on behalf of a user.</span></span> <span data-ttu-id="afde7-119">它會在使用者的 Windows 安全性內容底下當做使用者處理序執行，而非當做服務執行。</span><span class="sxs-lookup"><span data-stu-id="afde7-119">It runs as a user process under the Windows security context of the user, not as a service.</span></span> <span data-ttu-id="afde7-120">此時，系統不允許 SQL Server 登入，僅支援 Windows 登入。</span><span class="sxs-lookup"><span data-stu-id="afde7-120">SQL Server logins are disallowed; only Windows logins are supported.</span></span> <span data-ttu-id="afde7-121">這樣做可防止使用者執行個體上執行的軟體進行該使用者無權執行的整個系統範圍變更。</span><span class="sxs-lookup"><span data-stu-id="afde7-121">This prevents software executing on a user instance from making system-wide changes that the user would not have permissions to make.</span></span> <span data-ttu-id="afde7-122">此外，使用者執行個體也稱為子或用戶端執行個體，而且有時候會使用 RANU 縮寫 (「以一般使用者的身分執行」) 表示。</span><span class="sxs-lookup"><span data-stu-id="afde7-122">A user instance is also known as a child or client instance, and is sometimes referred to by using the RANU acronym ("run as normal user").</span></span>  
  
 <span data-ttu-id="afde7-123">每個使用者執行個體都與其父執行個體以及在相同電腦上執行的其他使用者執行個體隔離。</span><span class="sxs-lookup"><span data-stu-id="afde7-123">Each user instance is isolated from its parent instance and from other user instances running on the same computer.</span></span> <span data-ttu-id="afde7-124">安裝在使用者執行個體上的資料庫只會以單一使用者模式開啟，因此多位使用者無法連接至這些資料庫。</span><span class="sxs-lookup"><span data-stu-id="afde7-124">Databases installed on user instances are opened in single-user mode only; multiple users cannot connect to them.</span></span> <span data-ttu-id="afde7-125">使用者執行個體的複寫、分散式查詢和遠端連接都是停用的。</span><span class="sxs-lookup"><span data-stu-id="afde7-125">Replication, distributed queries and remote connections are disabled for user instances.</span></span> <span data-ttu-id="afde7-126">當使用者連接至使用者執行個體時，他們對於 SQL Server Express 父執行個體並沒有任何特殊權限。</span><span class="sxs-lookup"><span data-stu-id="afde7-126">When connected to a user instance, users do not have any special privileges on the parent SQL Server Express instance.</span></span>
