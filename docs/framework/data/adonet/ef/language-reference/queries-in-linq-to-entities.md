# <a name="queries-in-linq-to-entities"></a><span data-ttu-id="07de3-102">LINQ to Entities 中的查詢</span><span class="sxs-lookup"><span data-stu-id="07de3-102">Queries in LINQ to Entities</span></span>
<span data-ttu-id="07de3-103">查詢是指從資料來源中擷取資料的運算式。</span><span class="sxs-lookup"><span data-stu-id="07de3-103">A query is an expression that retrieves data from a data source.</span></span> <span data-ttu-id="07de3-104">查詢通常會以特定的查詢語言來表示，例如 SQL 用於關聯式資料庫，而 XQuery 用於 XML。</span><span class="sxs-lookup"><span data-stu-id="07de3-104">Queries are usually expressed in a specialized query language, such as SQL for relational databases and XQuery for XML.</span></span> <span data-ttu-id="07de3-105">因此，開發人員必須針對他們所查詢的每種資料來源或資料格式，學習新的查詢語言。</span><span class="sxs-lookup"><span data-stu-id="07de3-105">Therefore, developers have had to learn a new query language for each type of data source or data format that they query.</span></span> <span data-ttu-id="07de3-106">Language-Integrated Query (LINQ) 提供了一種較簡單且一致的模型，可處理各種資料來源和格式的資料。</span><span class="sxs-lookup"><span data-stu-id="07de3-106">Language-Integrated Query (LINQ) offers a simpler, consistent model for working with data across various kinds of data sources and formats.</span></span> <span data-ttu-id="07de3-107">在 LINQ 查詢中，您一定會使用程式設計物件。</span><span class="sxs-lookup"><span data-stu-id="07de3-107">In a LINQ query, you always work with programming objects.</span></span>  
  
 <span data-ttu-id="07de3-108">LINQ 查詢作業由三個動作構成：取得資料來源、建立查詢和執行查詢。</span><span class="sxs-lookup"><span data-stu-id="07de3-108">A LINQ query operation consists of three actions: obtain the data source or sources, create the query, and execute the query.</span></span>  
  
 <span data-ttu-id="07de3-109">實作 <xref:System.Collections.Generic.IEnumerable%601> 泛型介面或 <xref:System.Linq.IQueryable%601> 泛型介面的資料來源可以透過 LINQ 進行查詢。</span><span class="sxs-lookup"><span data-stu-id="07de3-109">Data sources that implement the <xref:System.Collections.Generic.IEnumerable%601> generic interface or the <xref:System.Linq.IQueryable%601> generic interface can be queried through LINQ.</span></span> <span data-ttu-id="07de3-110">執行個體的泛型<xref:System.Data.Objects.ObjectQuery%601>類別，實作泛型<xref:System.Linq.IQueryable%601>介面中，做為資料來源[!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)]查詢。</span><span class="sxs-lookup"><span data-stu-id="07de3-110">Instances of the generic <xref:System.Data.Objects.ObjectQuery%601> class, which implements the generic <xref:System.Linq.IQueryable%601> interface, serve as the data source for [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] queries.</span></span> <span data-ttu-id="07de3-111"><xref:System.Data.Objects.ObjectQuery%601> 泛型類別表示傳回零個或多個具型別物件之集合的查詢。</span><span class="sxs-lookup"><span data-stu-id="07de3-111">The <xref:System.Data.Objects.ObjectQuery%601> generic class represents a query that returns a collection of zero or more typed objects.</span></span> <span data-ttu-id="07de3-112">您也可以讓編譯器推斷實體的型別使用 C# 關鍵字`var`(Dim 在 Visual Basic 中)。</span><span class="sxs-lookup"><span data-stu-id="07de3-112">You can also let the compiler infer the type of an entity by using the C# keyword `var` (Dim in Visual Basic).</span></span>  
  
 <span data-ttu-id="07de3-113">在此查詢中，您可以精確地指定想要從資料來源中擷取的資訊。</span><span class="sxs-lookup"><span data-stu-id="07de3-113">In the query, you specify exactly the information that you want to retrieve from the data source.</span></span> <span data-ttu-id="07de3-114">此外，查詢也可以指定該項資訊傳回之前應該如何排序、分組和成形。</span><span class="sxs-lookup"><span data-stu-id="07de3-114">A query can also specify how that information should be sorted, grouped, and shaped before it is returned.</span></span> <span data-ttu-id="07de3-115">在 LINQ 中，查詢會儲存在變數內。</span><span class="sxs-lookup"><span data-stu-id="07de3-115">In LINQ, a query is stored in a variable.</span></span> <span data-ttu-id="07de3-116">如果查詢傳回一連串值，查詢變數本身必須是可查詢型別。</span><span class="sxs-lookup"><span data-stu-id="07de3-116">If the query returns a sequence of values, the query variable itself must be a queryable type.</span></span> <span data-ttu-id="07de3-117">這個查詢變數不會採取任何動作，也不會傳回任何資料。它只會儲存查詢資訊。</span><span class="sxs-lookup"><span data-stu-id="07de3-117">This query variable takes no action and returns no data; it only stores the query information.</span></span> <span data-ttu-id="07de3-118">在您建立查詢之後，必須執行該查詢以便擷取任何資料。</span><span class="sxs-lookup"><span data-stu-id="07de3-118">After you create a query you must execute that query to retrieve any data.</span></span>  
  
## <a name="query-syntax"></a><span data-ttu-id="07de3-119">查詢語法</span><span class="sxs-lookup"><span data-stu-id="07de3-119">Query Syntax</span></span>  
 [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] <span data-ttu-id="07de3-120">查詢可以在兩個不同的語法來撰寫： 查詢運算式語法和以方法為基礎的查詢語法。</span><span class="sxs-lookup"><span data-ttu-id="07de3-122">不過，[!INCLUDE[dnprdnshort](../../../../../../includes/dnprdnshort-md.md)] Common Language Runtime (CLR) 無法讀取查詢運算式語法本身。</span><span class="sxs-lookup"><span the option of calling them directly by using method syntax, instead of using query syntax.</span></span> <span data-ttu-id="07de3-126">如需詳細資訊，請參閱 [LINQ
