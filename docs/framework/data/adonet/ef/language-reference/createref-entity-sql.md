## <a name="remarks"></a><span data-ttu-id="c3f80-108">備註</span><span class="sxs-lookup"><span data-stu-id="c3f80-108">Remarks</span></span>

 <span data-ttu-id="c3f80-111">在下面的範例中，Orders 和 BadOrders 兩者都是 Order 型別的實體集，而 Id 則相當於 Order 的單一索引鍵屬性。</span><span class="sxs-lookup"><span data-stu-id="c3f80-111">In the example below, Orders and BadOrders are both entitysets of type Order, and Id is assumed to be the single key property of Order.</span></span> <span data-ttu-id="c3f80-112">此範例說明如何產生 BadOrders 中實體的產考。</span><span class="sxs-lookup"><span data-stu-id="c3f80-112">The example illustrates how we may produce a reference to an entity in BadOrders.</span></span> <span data-ttu-id="c3f80-113">請留意，此參考可能會懸空。</span><span class="sxs-lookup"><span data-stu-id="c3f80-113">Note that the reference may be dangling.</span></span>  <span data-ttu-id="c3f80-114">換言之，此參考可能無法實際識別特定的實體。</span><span class="sxs-lookup"><span data-stu-id="c3f80-114">That is, the reference may not actually identify a specific entity.</span></span> <span data-ttu-id="c3f80-115">在這種情況下，對該參考的 `DEREF` 作業將會傳回 null。</span><span class="sxs-lookup"><span data-stu-id="c3f80-115">In those cases, a `DEREF` operation on that reference returns a null.</span></span>

FROM LOB.Orders AS o
