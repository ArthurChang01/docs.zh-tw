<span data-ttu-id="6b77d-179">以集合為基礎的彙總 (集合函式) 會針對集合運作並傳回一個值。</span><span class="sxs-lookup"><span data-stu-id="6b77d-179">Collection-based aggregates (collection functions) operate on collections and return a value.</span></span> <span data-ttu-id="6b77d-180">例如如果 ORDERS 是所有訂單的集合，您可以計算最早出貨日期使用下列運算式：</span><span class="sxs-lookup"><span data-stu-id="6b77d-180">For example if ORDERS is a collection of all orders, you can calculate the earliest ship date with the following expression:</span></span>


<span data-ttu-id="6b77d-181">以集合為基礎之彙總內部的運算式是在目前的環境名稱解析範圍內評估。</span><span class="sxs-lookup"><span data-stu-id="6b77d-181">Expressions inside collection-based aggregates are evaluated within the current ambient name-resolution scope.</span></span>

## <a name="group-based-aggregates"></a><span data-ttu-id="6b77d-182">群組為基礎的彙總</span><span class="sxs-lookup"><span data-stu-id="6b77d-182">Group-based aggregates</span></span>

<span data-ttu-id="6b77d-183">以群組為基礎的彙總會計算 GROUP BY 子句所定義的整個群組。</span><span class="sxs-lookup"><span data-stu-id="6b77d-183">Group-based aggregates are calculated over a group as defined by the GROUP BY clause.</span></span> <span data-ttu-id="6b77d-184">系統會針對結果中的每個群組，使用每個群組中的項目當做彙總計算的輸入來計算個別的彙總。</span><span class="sxs-lookup"><span data-stu-id="6b77d-184">For each group in the result, a separate aggregate is calculated by using the elements in each group as input to the aggregate calculation.</span></span> <span data-ttu-id="6b77d-185">當 group-by 子句用於 select 運算式時，只有群組運算式名稱、彙總或常數運算式可以出現在投影或 order-by 子句中。</span><span class="sxs-lookup"><span data-stu-id="6b77d-185">When a group-by clause is used in a select expression, only grouping expression names, aggregates, or constant expressions can be present in the projection or order-by clause.</span></span>

<span data-ttu-id="6b77d-186">下列範例會計算針對每個產品訂購的平均數量：</span><span class="sxs-lookup"><span data-stu-id="6b77d-186">The following example calculates the average quantity ordered for each product:</span></span>


<span data-ttu-id="6b77d-187">很可能在 SELECT 運算式中有沒有明確的群組依據子句以群組為基礎的彙總。</span><span class="sxs-lookup"><span data-stu-id="6b77d-187">It's possible to have a group-based aggregate without an explicit group-by clause in the SELECT expression.</span></span> <span data-ttu-id="6b77d-188">在此情況下，所有元素都會都視為單一群組。</span><span class="sxs-lookup"><span data-stu-id="6b77d-188">In this case, all elements are treated as a single group.</span></span> <span data-ttu-id="6b77d-189">這是指定群組，根據常數的對等項目。</span><span class="sxs-lookup"><span data-stu-id="6b77d-189">This is equivalent of specifying a grouping based on a constant.</span></span> <span data-ttu-id="6b77d-190">以下列運算式為例：</span><span class="sxs-lookup"><span data-stu-id="6b77d-190">Take, for example, the following expression:</span></span>

```sql
select avg(ol.Quantity) from LOB.OrderLines as ol
```

<span data-ttu-id="6b77d-191">這個運算式就相當於下列運算式：</span><span class="sxs-lookup"><span data-stu-id="6b77d-191">This is equivalent to the following:</span></span>

```sql
select avg(ol.Quantity) from LOB.OrderLines as ol group by 1
```

<span data-ttu-id="6b77d-192">以群組為基礎之彙總內部的運算式是在 WHERE 子句運算式可見的名稱解析範圍內評估。</span><span class="sxs-lookup"><span data-stu-id="6b77d-192">Expressions inside the group-based aggregate are evaluated within the name-resolution scope that would be visible to the WHERE clause expression.</span></span>

<span data-ttu-id="6b77d-193">在[!INCLUDE[tsql](../../../../../../includes/tsql-md.md)]，以群組為基礎的彙總也可以指定 ALL 或 DISTINCT 修飾詞。</span><span class="sxs-lookup"><span data-stu-id="6b77d-193">As in [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)], group-based aggregates can also specify an ALL or DISTINCT modifier.</span></span> <span data-ttu-id="6b77d-194">如果有指定 DISTINCT 修飾詞，系統就會在計算彙總之前，從彙總輸入集合中刪除重複項目。</span><span class="sxs-lookup"><span data-stu-id="6b77d-194">If the DISTINCT modifier is specified, duplicates are eliminated from the aggregate input collection, before the aggregate is computed.</span></span> <span data-ttu-id="6b77d-195">如果指定的是 ALL 修飾詞 (或沒有指定任何修飾詞)，就不會執行重複項目刪除作業。</span><span class="sxs-lookup"><span data-stu-id="6b77d-195">If the ALL modifier is specified (or if no modifier is specified), no duplicate elimination is performed.</span></span>
