 <span data-ttu-id="96c2e-103">現今高度連接的電腦系統，經常會暴露於源自各種可能未知來源的程式碼。</span><span class="sxs-lookup"><span data-stu-id="96c2e-103">Today's highly connected computer systems are frequently exposed to code originating from various, possibly unknown sources.</span></span> <span data-ttu-id="96c2e-104">程式碼可以附加至電子郵件、 包含在文件，或透過網際網路下載。</span><span class="sxs-lookup"><span data-stu-id="96c2e-104">Code can be attached to email, contained in documents, or downloaded over the Internet.</span></span> <span data-ttu-id="96c2e-105">不幸的是，許多電腦使用者都直接經歷過惡意行動程式碼的效應，包括病毒和蠕蟲，它們可能會損壞或摧毀資料，並造成時間和金錢上的損失。</span><span class="sxs-lookup"><span data-stu-id="96c2e-105">Unfortunately, many computer users have experienced firsthand the effects of malicious mobile code, including viruses and worms, which can damage or destroy data and cost time and money.</span></span>  
  
 <span data-ttu-id="96c2e-106">最常見的安全性機制是根據使用者的登入認證 (通常是密碼) 來提供權限給使用者，並限制使用者可以存取的資源 (通常是目錄和檔案)。</span><span class="sxs-lookup"><span data-stu-id="96c2e-106">Most common security mechanisms give rights to users based on their logon credentials (usually a password) and restrict resources (often directories and files) that the user is allowed to access.</span></span> <span data-ttu-id="96c2e-107">不過，這個方法無法解決幾個問題：使用者從許多來源取得程式碼，其中有些可能不可靠；程式碼可能包含 Bug 或安全性弱點，而被惡意程式碼利用；而程式碼有時會做使用者不知道它會做的事情。</span><span class="sxs-lookup"><span data-stu-id="96c2e-107">However, this approach fails to address several issues: users obtain code from many sources, some of which might be unreliable; code can contain bugs or vulnerabilities that enable it to be exploited by malicious code; and code sometimes does things that the user does not know it will do.</span></span> <span data-ttu-id="96c2e-108">如此一來，當謹慎且值得信賴的使用者執行惡意或充滿錯誤的軟體時，就可能損害電腦系統和竊取私人資料。</span><span class="sxs-lookup"><span data-stu-id="96c2e-108">As a result, computer systems can be damaged and private data can be leaked when cautious and trustworthy users run malicious or error-filled software.</span></span> <span data-ttu-id="96c2e-109">大部分作業系統安全性機制都會要求每個程式碼片段必須受到完全信任才能執行，但是網頁上的指令碼可能除外。</span><span class="sxs-lookup"><span data-stu-id="96c2e-109">Most operating system security mechanisms require that every piece of code must be completely trusted in order to run, except perhaps for scripts on a Web page.</span></span> <span data-ttu-id="96c2e-110">因此，還是需要可廣泛套用的安全性機制，讓源自於一部電腦系統的程式碼，在另一個系統上執行時可以受到保護，即使系統之間沒有信任關係也一樣。</span><span class="sxs-lookup"><span data-stu-id="96c2e-110">Therefore, there is still a need for a widely applicable security mechanism that allows code originating from one computer system to execute with protection on another system, even when there is no trust relationship between the systems.</span></span>  
  
 <span data-ttu-id="96c2e-111">.NET Framework 提供一種名為程式碼存取安全性的安全性機制，以協助保護電腦系統免於惡意行動程式碼的傷害、允許來自未知來源的程式碼在受到防護的情況下執行，並協助防止受信任的程式碼有意或無意地危及安全性。</span><span class="sxs-lookup"><span data-stu-id="96c2e-111">The .NET Framework provides a security mechanism called code access security to help protect computer systems from malicious mobile code, to allow code from unknown origins to run with protection, and to help prevent trusted code from intentionally or accidentally compromising security.</span></span> <span data-ttu-id="96c2e-112">程式碼存取安全性依程式碼的來源和程式碼本身的其他部分，分別給與程式碼不同程度的信任等級。</span><span class="sxs-lookup"><span data-stu-id="96c2e-112">Code access security enables code to be trusted to varying degrees depending on where the code originates and on other aspects of the code's identity.</span></span> <span data-ttu-id="96c2e-113">程式碼存取安全性也會在程式碼上強制執行各種信任層級，這會將必須完全信任才能執行的程式碼數量降到最低。</span><span class="sxs-lookup"><span data-stu-id="96c2e-113">Code access security also enforces the varying levels of trust on code, which minimizes the amount of code that must be fully trusted in order to run.</span></span> <span data-ttu-id="96c2e-114">使用程式碼存取安全性可以減少您的程式碼被惡意或充滿錯誤的程式碼誤用的可能性。</span><span class="sxs-lookup"><span data-stu-id="96c2e-114">Using code access security can reduce the likelihood that your code will be misused by malicious or error-filled code.</span></span> <span data-ttu-id="96c2e-115">它可減少您的負擔，因為您可以指定一組允許您的程式碼執行的作業。</span><span class="sxs-lookup"><span data-stu-id="96c2e-115">It can reduce your liability, because you can specify the set of operations your code should be allowed to perform.</span></span> <span data-ttu-id="96c2e-116">程式碼存取安全性也有助於減少因為您的程式碼中的安全性弱點而導致的損害。</span><span class="sxs-lookup"><span data-stu-id="96c2e-116">Code access security can also help minimize the damage that can result from security vulnerabilities in your code.</span></span>

## <a name="walking-the-call-stack"></a><span data-ttu-id="96c2e-140">查核呼叫堆疊</span><span class="sxs-lookup"><span data-stu-id="96c2e-140">Walking the Call Stack</span></span>  
 <span data-ttu-id="96c2e-141">為了判斷程式碼是否經授權可存取資源或執行某項作業，執行階段的安全性系統會查核呼叫堆疊，並比較每個呼叫端被授與的權限與要求的權限。</span><span class="sxs-lookup"><span data-stu-id="96c2e-141">To determine whether code is authorized to access a resource or perform an operation, the runtime's security system walks the call stack, comparing the granted permissions of each caller to the permission being demanded.</span></span> <span data-ttu-id="96c2e-142">如果呼叫堆疊中的任何呼叫端沒有所要求的權限，系統就會擲回安全性例外狀況，並拒絕存取。</span><span class="sxs-lookup"><span data-stu-id="96c2e-142">If any caller in the call stack does not have the demanded permission, a security exception is thrown and access is refused.</span></span> <span data-ttu-id="96c2e-143">堆疊查核行程的設計是為了協助防止引誘攻擊，在這些攻擊中，信任度較低的程式碼會呼叫受高度信任的程式碼，並使用它來執行未經授權的動作。</span><span class="sxs-lookup"><span data-stu-id="96c2e-143">The stack walk is designed to help prevent luring attacks, in which less-trusted code calls highly trusted code and uses it to perform unauthorized actions.</span></span> <span data-ttu-id="96c2e-144">在執行階段要求所有呼叫端的權限會影響效能，但這是基本的防護措施，可協助保護程式碼不會引誘信任度較低的程式碼攻擊。</span><span class="sxs-lookup"><span data-stu-id="96c2e-144">Demanding permissions of all callers at run time affects performance, but it is essential to help protect code from luring attacks by less-trusted code.</span></span> <span data-ttu-id="96c2e-145">為了最佳化效能，您可以讓您的程式碼執行較少堆疊查核行程；不過，您必須確認，每當您執行這項操作時，不會暴露安全性弱點。</span><span class="sxs-lookup"><span data-stu-id="96c2e-145">To optimize performance, you can have your code perform fewer stack walks; however, you must be sure that you do not expose a security weakness whenever you do this.</span></span>
