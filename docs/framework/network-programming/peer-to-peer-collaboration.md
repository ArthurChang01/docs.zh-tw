# 對等共同作業
對等網路位於網際網路邊緣超過用戶端架構的計算工作相當強大的電腦 \(個人電腦\) 的操作。  現代個人電腦的 \(PC\) 非常快速處理器、浩大的記憶體和大型的硬碟，完全不套用任何，執行計算時一般配置 \(例如瀏覽的電子郵件地址和網路。  現代電腦可輕易地為用戶端和伺服器 \(對等\) 應用程式的許多型別。  
  
-   對等共同作業基礎結構是在 Windows Vista 和最新平台支援"近端分享"服務 Microsoft Windows 對等基礎結構的簡單實作。  它最適合用來在"近端分享"服務運作的子網路內的對等電腦可以讓應用程式使用，不過，它也可以用於網際網路端點或 Contact 服務。  它結合了 Live Messenger 和其他使用明確的應用程式使用判斷連絡人端點、可用性和存在的常見的連絡人管理員。  
  
## 共同作業應用程式  
 典型的對等共同作業應用程式包含下列步驟:  
  
-   對等判斷是裝載共同作業工作階段所需的對等電腦的識別。  
  
-   需要裝載工作階段傳送，由於某種未知的原因而，，和主機對等電腦授與管理共同作業活動。  
  
-   主應用程式邀請在子網路的連絡人 \(包括要求者\) 加入至工作階段。  
  
-   若要共同作業的對等電腦註冊可能將主機加入至連絡人管理員。  
  
-   大部分的對等電腦是否要傳送邀請回應，接受或及時遞減，回主機對等電腦。  
  
-   若要共同作業的對等電腦註冊會為訂閱主機對等電腦。  
  
-   當對等執行其初始共同作業活動時，主機對等電腦可以將遠端對等電腦加入至自己的連絡人管理員。  它也會處理所有的邀請回應判斷使用者接受，拒絕，，且未回應。  它可能會取消邀請傳送至未回應的人，或執行其他動作。  
  
-   此時，主機對等電腦可以開始使用所有已邀請的對等電腦的共同作業工作階段，或向共同作業基礎結構的應用程式。  接下來應用程式使用對等共同作業基礎結構和 <xref:System.Net.PeerToPeer.Collaboration> 命名空間座標相符項目、電子佈告欄、工作階段和其他伺服器存在應用程式進行通訊。  
  
## 對等網路安全性  
 使用，在 Active Directory 網域中，網域控制站提供驗證服務。  在一個無伺服器對等環境，對等物件必須提供自己的驗證。  對等網路的，任何節點都可以做為 CA，移除單一根憑證的要求在每一個對等的受信任的根存放區。  使用自我簽署憑證，驗證提供，格式化為 X.509 憑證。  這些是使用私密金鑰，由每個對等建立，產生公開金鑰\/私密金鑰組和憑證簽署的憑證。  自我簽署憑證來驗證和提供有關對等實體 \(Entity\) 的資訊。  就像 X.509 驗證，對等網路驗證的方式追蹤回到信任的公開金鑰的憑證鏈結。
