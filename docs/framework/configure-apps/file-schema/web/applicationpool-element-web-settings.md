<span data-ttu-id="44875-128">當您在整合模式中執行 IIS 7.0 或更新版本時，此元素組合可讓您設定當應用程式裝載于 IIS 應用程式集區時，ASP.NET 管理執行緒和佇列要求的方式。</span><span class="sxs-lookup"><span data-stu-id="44875-128">When you run IIS 7.0 or a later version in Integrated mode, this element combination lets you configure how ASP.NET manages threads and queues requests when the application is hosted in an IIS application pool.</span></span> <span data-ttu-id="44875-129">如果您執行 IIS 6，或以傳統模式或在 ISAPI 模式中執行 IIS 7.0，則會忽略這些設定。</span><span class="sxs-lookup"><span data-stu-id="44875-129">If you run IIS 6 or you run IIS 7.0 in Classic mode or in ISAPI mode, these settings are ignored.</span></span>

> [!IMPORTANT]
> <span data-ttu-id="44875-134">如果您在 [!INCLUDE[win7](../../../../../includes/win7-md.md)]上執行 IIS 7.0，您可以為每個應用程式集區設定個別的 aspnet .config 檔案。</span><span class="sxs-lookup"><span data-stu-id="44875-134">If you run IIS 7.0 on [!INCLUDE[win7](../../../../../includes/win7-md.md)], you can configure a separate aspnet.config file for every application pool.</span></span> <span data-ttu-id="44875-135">這可讓您針對每個應用程式集區量身訂做執行緒的效能。</span><span class="sxs-lookup"><span data-stu-id="44875-135">This lets you tailor the performance of the threads for each application pool.</span></span>
