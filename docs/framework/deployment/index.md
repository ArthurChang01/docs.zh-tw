# 部署 .NET Framework
這一節的.NET Framework 文件提供給想要在其應用程式安裝 .NET Framework 的開發人員和想要在整個網路上部署 .NET Framework 的系統管理員相關的資訊。  它也同時說明啟動與重新啟動的問題，以及如何部署、監視您 .NET Framework 安裝的進度。  
  
## 在本節中  
 [開發人員部署手冊](../../../docs/framework/deployment/deployment-guide-for-developers.md)  
 說明開發人員如何將 .NET Framework 隨自己的應用程式安裝在使用者的電腦上。  
  
 [系統管理員部署手冊](../../../docs/framework/deployment/guide-for-administrators.md)  
 說明系統管理員如何使用 System Center Configuration Manager \(SCCM\)，在整個網路上部署 .NET Framework 及其系統相依性。  
  
 [在 .NET Framework 4.5 安裝期間減少系統重新啟動的次數](../../../docs/framework/deployment/reducing-system-restarts.md)  
 描述重新啟動管理員，它可防止在任何可能的情況下重新開機，並說明應用程式以無訊息模式安裝 .NET Framework 時我們該如何使用它。  
  
 [如何：取得 .NET Framework 4.5 安裝程式的進度](../../../docs/framework/deployment/how-to-get-progress-from-the-dotnet-installer.md)  
 說明如何以無訊息模式啟動並追蹤 .NET Framework 安裝程序，並同時顯示安裝進度的檢視。  
  
 [.NET Framework 初始化錯誤：管理使用者經驗](../../../docs/framework/deployment/initialization-errors-managing-the-user-experience.md)  
 說明當.NET Framework 應用程式需要一個 CLR 版本，但是這個CLR版本是無效或未安裝在使用者的電腦上，該如何解決這些錯誤和如何控制給使用者的錯誤訊息。  
  
 [如何：偵錯 CLR 啟用問題](../../../docs/framework/deployment/how-to-debug-clr-activation-issues.md)  
 說明如何檢視以及偵錯 CLR 啟動記錄以解決當應用程式都是以 CLR 的正確版本運行時，可能會遇到的問題。
