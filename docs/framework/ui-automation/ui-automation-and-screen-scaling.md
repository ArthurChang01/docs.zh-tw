 <span data-ttu-id="c7e1d-113">變更 DPI 設定與變更螢幕解析度不同。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-113">Changing the dpi setting is not the same as changing the screen resolution.</span></span> <span data-ttu-id="c7e1d-114">使用 DPI 縮放比例時，螢幕上的實體圖元數目會維持不變。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-114">With dpi scaling, the number of physical pixels on the screen remains the same.</span></span> <span data-ttu-id="c7e1d-115">不過，縮放比例會套用至 [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] 項目的大小和位置。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-115">However, scaling is applied to the size and location of [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] elements.</span></span> <span data-ttu-id="c7e1d-116">針對未明確要求不縮放的桌面和應用程式，桌面視窗管理員 (DWM) 會自動執行縮放。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-116">This scaling can be performed automatically by the Desktop Window Manager (DWM) for the desktop and for applications that do not explicitly ask not to be scaled.</span></span>  
  
 <span data-ttu-id="c7e1d-117">實際上，當使用者將縮放比例設定為 120 DPI 時，螢幕上的垂直或水準英寸會變大達25%。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-117">In effect, when the user sets the scale factor to 120 dpi, a vertical or horizontal inch on the screen becomes bigger by 25 percent.</span></span> <span data-ttu-id="c7e1d-118">所有維度都會因此調整。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-118">All dimensions are scaled accordingly.</span></span> <span data-ttu-id="c7e1d-119">來自畫面上方和左邊緣的應用程式視窗位移會增加 25%。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-119">The offset of an application window from the top and left edges of the screen increases by 25 percent.</span></span> <span data-ttu-id="c7e1d-120">如果應用程式調整已啟用，且應用程式不能感知 DPI，則視窗的大小會以相同比例增加，連同其包含的所有 [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] 元素的位移和大小。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-120">If application scaling is enabled and the application is not dpi-aware, the size of the window increases in the same proportion, along with the offsets and sizes of all [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] elements it contains.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="c7e1d-121">根據預設，當使用者將 DPI 設定為120時，DWM 不會針對非 DPI 感知的應用程式執行調整，但會在 DPI 設定為144或更高的自訂值時執行。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-121">By default, the DWM does not perform scaling for non-dpi-aware applications when the user sets the dpi to 120, but does perform it when the dpi is set to a custom value of 144 or higher.</span></span> <span data-ttu-id="c7e1d-122">不過，使用者可以覆寫此預設行為。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-122">However, the user can override the default behavior.</span></span>  
  
 <span data-ttu-id="c7e1d-123">針對重視畫面座標的應用程式，畫面縮放比例會產生新挑戰。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-123">Screen scaling creates new challenges for applications that are concerned in any way with screen coordinates.</span></span> <span data-ttu-id="c7e1d-124">畫面現在包含兩個座標系統：實體和邏輯。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-124">The screen now contains two coordinate systems: physical and logical.</span></span> <span data-ttu-id="c7e1d-125">點的實體座標是來自原點左上方的實際位移 (以像素為單位)。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-125">The physical coordinates of a point are the actual offset in pixels from the top left of the origin.</span></span> <span data-ttu-id="c7e1d-126">邏輯座標則是像素本身縮放時，跟著縮放的位移。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-126">The logical coordinates are the offsets as they would be if the pixels themselves were scaled.</span></span>  
  
 <span data-ttu-id="c7e1d-127">假設您設計的對話方塊在座標 (100, 48) 上有一個按鈕。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-127">Suppose you design a dialog box with a button at coordinates (100, 48).</span></span> <span data-ttu-id="c7e1d-128">以預設 96 DPI 顯示此對話方塊時，按鈕會位於（100，48）的實體座標中。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-128">When this dialog box is displayed at the default 96 dpi, the button is located at physical coordinates of (100, 48).</span></span> <span data-ttu-id="c7e1d-129">在 120 DPI 中，它位於（125，60）的實體座標。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-129">At 120 dpi, it is located at physical coordinates of (125, 60).</span></span> <span data-ttu-id="c7e1d-130">但是邏輯座標在任何 DPI 設定上都相同：（100，48）。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-130">But the logical coordinates are the same at any dpi setting: (100, 48).</span></span>  
  
 <span data-ttu-id="c7e1d-131">邏輯座標很重要，因為不論 DPI 設定為何，它們都會讓作業系統和應用程式的行為保持一致。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-131">Logical coordinates are important, because they make the behavior of the operating system and applications consistent regardless of the dpi setting.</span></span> <span data-ttu-id="c7e1d-132">例如， <xref:System.Windows.Forms.Cursor.Position%2A?displayProperty=nameWithType> 一般會傳回邏輯座標。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-132">For example, <xref:System.Windows.Forms.Cursor.Position%2A?displayProperty=nameWithType> normally returns the logical coordinates.</span></span> <span data-ttu-id="c7e1d-133">如果您將游標移到對話方塊中的專案上方，則會傳回相同的座標，不論 DPI 設定為何。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-133">If you move the cursor over an element in a dialog box, the same coordinates are returned regardless of the dpi setting.</span></span> <span data-ttu-id="c7e1d-134">如果您在（100，100）繪製控制項，則會將它繪製到那些邏輯座標，而且會佔用與任何 DPI 設定相同的相對位置。</span><span class="sxs-lookup"><span data-stu-id="c7e1d-134">If you draw a control at (100, 100), it is drawn to those logical coordinates, and will occupy the same relative position at any dpi setting.</span></span>
