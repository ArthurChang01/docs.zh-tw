## <a name="scaling-in-windows-vista"></a><span data-ttu-id="f9d92-109">Windows Vista 中的縮放比例</span><span class="sxs-lookup"><span data-stu-id="f9d92-109">Scaling in Windows Vista</span></span>  
 <span data-ttu-id="f9d92-110">預設 DPI 設置為 96，這意味著 96 圖元佔用一個名義英寸的寬度或高度。</span><span class="sxs-lookup"><span data-stu-id="f9d92-110">The default dpi setting is 96, which means that 96 pixels occupy a width or height of one notional inch.</span></span> <span data-ttu-id="f9d92-111">「英吋」的確切測量值取決於監視器的大小和實體解析度。</span><span class="sxs-lookup"><span data-stu-id="f9d92-111">The exact measure of an "inch" depends on the size and physical resolution of the monitor.</span></span> <span data-ttu-id="f9d92-112">例如，在 12 英吋寬、1280 像素水平解析度的監視器上，96 像素的水平線長度約一英吋的 9/10。</span><span class="sxs-lookup"><span data-stu-id="f9d92-112">For example, on a monitor 12 inches wide, at a horizontal resolution of 1280 pixels, a horizontal line of 96 pixels extends about 9/10 of an inch.</span></span>  
  
 <span data-ttu-id="f9d92-113">更改 DPI 設置與更改螢幕解析度不同。</span><span class="sxs-lookup"><span data-stu-id="f9d92-113">Changing the dpi setting is not the same as changing the screen resolution.</span></span> <span data-ttu-id="f9d92-114">使用 DPI 縮放時，螢幕上的物理圖元數保持不變。</span><span class="sxs-lookup"><span data-stu-id="f9d92-114">With dpi scaling, the number of physical pixels on the screen remains the same.</span></span> <span data-ttu-id="f9d92-115">不過，縮放比例會套用至 [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] 項目的大小和位置。</span><span class="sxs-lookup"><span data-stu-id="f9d92-115">However, scaling is applied to the size and location of [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] elements.</span></span> <span data-ttu-id="f9d92-116">針對未明確要求不縮放的桌面和應用程式，桌面視窗管理員 (DWM) 會自動執行縮放。</span><span class="sxs-lookup"><span data-stu-id="f9d92-116">This scaling can be performed automatically by the Desktop Window Manager (DWM) for the desktop and for applications that do not explicitly ask not to be scaled.</span></span>  
  
 <span data-ttu-id="f9d92-117">實際上，當使用者將比例因數設置為 120 DPI 時，螢幕上的垂直或水準英寸會變大 25%。</span><span class="sxs-lookup"><span data-stu-id="f9d92-117">In effect, when the user sets the scale factor to 120 dpi, a vertical or horizontal inch on the screen becomes bigger by 25 percent.</span></span> <span data-ttu-id="f9d92-118">所有維度都會因此調整。</span><span class="sxs-lookup"><span data-stu-id="f9d92-118">All dimensions are scaled accordingly.</span></span> <span data-ttu-id="f9d92-119">來自畫面上方和左邊緣的應用程式視窗位移會增加 25%。</span><span class="sxs-lookup"><span data-stu-id="f9d92-119">The offset of an application window from the top and left edges of the screen increases by 25 percent.</span></span> <span data-ttu-id="f9d92-120">如果啟用了應用程式縮放，並且應用程式不具有 DPI 感知功能，則視窗的大小將隨其比例的增加，以及它包含的所有[!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)]元素的偏移量和大小。</span><span class="sxs-lookup"><span data-stu-id="f9d92-120">If application scaling is enabled and the application is not dpi-aware, the size of the window increases in the same proportion, along with the offsets and sizes of all [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] elements it contains.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="f9d92-121">預設情況下，當使用者將 DPI 設置為 120 時，DWM 不會對非 DPI 感知應用程式執行縮放，但在 DPI 設置為 144 或更高的自訂值時確實執行縮放。</span><span class="sxs-lookup"><span data-stu-id="f9d92-121">By default, the DWM does not perform scaling for non-dpi-aware applications when the user sets the dpi to 120, but does perform it when the dpi is set to a custom value of 144 or higher.</span></span> <span data-ttu-id="f9d92-122">不過，使用者可以覆寫此預設行為。</span><span class="sxs-lookup"><span data-stu-id="f9d92-122">However, the user can override the default behavior.</span></span>  
  
 <span data-ttu-id="f9d92-123">針對重視畫面座標的應用程式，畫面縮放比例會產生新挑戰。</span><span class="sxs-lookup"><span data-stu-id="f9d92-123">Screen scaling creates new challenges for applications that are concerned in any way with screen coordinates.</span></span> <span data-ttu-id="f9d92-124">畫面現在包含兩個座標系統：實體和邏輯。</span><span class="sxs-lookup"><span data-stu-id="f9d92-124">The screen now contains two coordinate systems: physical and logical.</span></span> <span data-ttu-id="f9d92-125">點的實體座標是來自原點左上方的實際位移 (以像素為單位)。</span><span class="sxs-lookup"><span data-stu-id="f9d92-125">The physical coordinates of a point are the actual offset in pixels from the top left of the origin.</span></span> <span data-ttu-id="f9d92-126">邏輯座標則是像素本身縮放時，跟著縮放的位移。</span><span class="sxs-lookup"><span data-stu-id="f9d92-126">The logical coordinates are the offsets as they would be if the pixels themselves were scaled.</span></span>  
  
 <span data-ttu-id="f9d92-127">假設您設計的對話方塊在座標 (100, 48) 上有一個按鈕。</span><span class="sxs-lookup"><span data-stu-id="f9d92-127">Suppose you design a dialog box with a button at coordinates (100, 48).</span></span> <span data-ttu-id="f9d92-128">當此對話方塊顯示在預設的 96 DPI 時，該按鈕位於物理座標 （100， 48）。</span><span class="sxs-lookup"><span data-stu-id="f9d92-128">When this dialog box is displayed at the default 96 dpi, the button is located at physical coordinates of (100, 48).</span></span> <span data-ttu-id="f9d92-129">在 120 DPI 處，它位於物理座標 （125， 60）。</span><span class="sxs-lookup"><span data-stu-id="f9d92-129">At 120 dpi, it is located at physical coordinates of (125, 60).</span></span> <span data-ttu-id="f9d92-130">但是邏輯座標在任何 DPI 設置中都是相同的：（100，48）。</span><span class="sxs-lookup"><span data-stu-id="f9d92-130">But the logical coordinates are the same at any dpi setting: (100, 48).</span></span>  
  
 <span data-ttu-id="f9d92-131">邏輯座標很重要，因為它們使作業系統和應用程式的行為一致，而不管 DPI 設置如何。</span><span class="sxs-lookup"><span data-stu-id="f9d92-131">Logical coordinates are important, because they make the behavior of the operating system and applications consistent regardless of the dpi setting.</span></span> <span data-ttu-id="f9d92-132">例如， <xref:System.Windows.Forms.Cursor.Position%2A?displayProperty=nameWithType> 一般會傳回邏輯座標。</span><span class="sxs-lookup"><span data-stu-id="f9d92-132">For example, <xref:System.Windows.Forms.Cursor.Position%2A?displayProperty=nameWithType> normally returns the logical coordinates.</span></span> <span data-ttu-id="f9d92-133">如果將游標移到對話方塊中的元素上，則無論 DPI 設置如何，都會返回相同的座標。</span><span class="sxs-lookup"><span data-stu-id="f9d92-133">If you move the cursor over an element in a dialog box, the same coordinates are returned regardless of the dpi setting.</span></span> <span data-ttu-id="f9d92-134">如果繪製控制項（100，100），則該控制項將繪製到這些邏輯座標，並將在任何 DPI 設置中佔據相同的相對位置。</span><span class="sxs-lookup"><span data-stu-id="f9d92-134">If you draw a control at (100, 100), it is drawn to those logical coordinates, and will occupy the same relative position at any dpi setting.</span></span>
