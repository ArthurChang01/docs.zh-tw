# <a name="reliable-services"></a><span data-ttu-id="d75b8-102">可靠的服務</span><span class="sxs-lookup"><span data-stu-id="d75b8-102">Reliable Services</span></span>
<span data-ttu-id="d75b8-103">佇列和可靠工作階段是實作可信賴傳訊的 Windows Communication Foundation (WCF) 功能。</span><span class="sxs-lookup"><span data-stu-id="d75b8-103">Queues and reliable sessions are the Windows Communication Foundation (WCF) features that implement reliable messaging.</span></span> <span data-ttu-id="d75b8-104">本主題說明 WCF 的可信賴傳訊功能。</span><span class="sxs-lookup"><span data-stu-id="d75b8-104">This topic explains the reliable messaging features of WCF.</span></span>  
  
 <span data-ttu-id="d75b8-105">*可信賴傳訊*是可信賴傳訊來源 (稱為*來源*) 將訊息可靠地傳輸到可信賴傳訊目的地 (稱為*目的地*)。</span><span class="sxs-lookup"><span data-stu-id="d75b8-105">*Reliable messaging* is how a reliable messaging source (called the *source*) transfers messages reliably to a reliable messaging destination (called the *destination*).</span></span>  
  
 <span data-ttu-id="d75b8-106">可信賴傳訊可執行下列功能：</span><span class="sxs-lookup"><span data-stu-id="d75b8-106">Reliable messaging performs the following functions:</span></span>  
  
-   <span data-ttu-id="d75b8-107">傳輸保證，保證無論是訊息傳輸或傳輸失敗，訊息都會從來源傳輸到目的地。</span><span class="sxs-lookup"><span data-stu-id="d75b8-107">Transfers assurances for messages sent from a source to a destination regardless of message transfer or transport failures.</span></span>  
  
-   <span data-ttu-id="d75b8-108">區隔來源和目的地。</span><span class="sxs-lookup"><span data-stu-id="d75b8-108">Separates the source and the destination from each other.</span></span> <span data-ttu-id="d75b8-109">這樣可以讓來源和目的地所發生的失敗與復原不具相關性，並且提供可靠的訊息傳輸和傳遞，即使在來源或目的地無法使用的情況下也是如此。</span><span class="sxs-lookup"><span data-stu-id="d75b8-109">This provides independent failure and recovery of the source and the destination, as well as reliable transfer and delivery of messages, even when the source or destination is unavailable.</span></span>  
  
 <span data-ttu-id="d75b8-110">可信賴傳訊通常會伴隨長延遲時間的發生。</span><span class="sxs-lookup"><span data-stu-id="d75b8-110">Reliable messaging frequently comes at the cost of high latency.</span></span> <span data-ttu-id="d75b8-111">*延遲*是要從來源到達目的地的訊息所花費的時間。</span><span class="sxs-lookup"><span data-stu-id="d75b8-111">*Latency* is the time it takes for the message to reach the destination from the source.</span></span> <span data-ttu-id="d75b8-112">WCF，因此，提供下列類型的可信賴傳訊：</span><span class="sxs-lookup"><span data-stu-id="d75b8-112">WCF, therefore, provides the following types of reliable messaging:</span></span>  
  
-   <span data-ttu-id="d75b8-113">[可靠工作階段](../../../docs/framework/wcf/feature-details/reliable-sessions.md)，提供可靠的傳輸，而不是高延遲時間的成本。</span><span class="sxs-lookup"><span data-stu-id="d75b8-113">[Reliable Sessions](../../../docs/framework/wcf/feature-details/reliable-sessions.md), which offers reliable transfer without the cost of high latency.</span></span>  
  
-   <span data-ttu-id="d75b8-114">[WCF 中的佇列](../../../docs/framework/wcf/feature-details/queues-in-wcf.md)，這可提供信賴傳訊以及來源和目的地的區隔。</span><span class="sxs-lookup"><span data-stu-id="d75b8-114">[Queues in WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md), which offers both reliable transfers and separation between the source and the destination.</span></span>  
  
## <a name="reliable-sessions"></a><span data-ttu-id="d75b8-115">可靠工作階段</span><span class="sxs-lookup"><span data-stu-id="d75b8-115">Reliable Sessions</span></span>  
 <span data-ttu-id="d75b8-116">可靠工作階段使用 WS-Reliable Messaging 通訊協定提供來源和目的地之間的端對端可靠訊息傳輸，而不論個別傳訊端點 (來源和目的地) 之間媒介的類型或數目為何。</span><span class="sxs-lookup"><span data-stu-id="d75b8-116">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-Reliable Messaging protocol, regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="d75b8-117">這包括不是使用 SOAP 的任何傳輸媒介 (例如，HTTP Proxy) 或使用 SOAP 的媒介 (例如，SOAP 架構的路由器或橋接器)，而訊息在端點之間流動時需要這些媒介。</span><span class="sxs-lookup"><span data-stu-id="d75b8-117">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="d75b8-118">可靠工作階段會使用記憶體中傳輸視窗來遮罩 SOAP 訊息層級的失敗，並在發生傳輸失敗時重新建立連線。</span><span class="sxs-lookup"><span data-stu-id="d75b8-118">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>  
  
 <span data-ttu-id="d75b8-119">可靠工作階段會提供短延遲時間的可信賴訊息傳輸。</span><span class="sxs-lookup"><span data-stu-id="d75b8-119">Reliable sessions provide low-latency reliable message transfers.</span></span> <span data-ttu-id="d75b8-120">它們可透過任何的 Proxy 或媒介提供 SOAP 訊息，而這相當於 TCP 透過 IP 橋接器為封包提供的內容。</span><span class="sxs-lookup"><span data-stu-id="d75b8-120">They provide for SOAP messages over any proxies or intermediaries, equivalent to what TCP provides for packets over IP bridges.</span></span> <span data-ttu-id="d75b8-121">如需可靠工作階段的詳細資訊，請參閱[可靠工作階段](../../../docs/framework/wcf/feature-details/reliable-sessions.md)。</span><span class="sxs-lookup"><span data-stu-id="d75b8-121">For more information about reliable sessions, see [Reliable Sessions](../../../docs/framework/wcf/feature-details/reliable-sessions.md).</span></span>  
  
### <a name="queues"></a><span data-ttu-id="d75b8-122">佇列</span><span class="sxs-lookup"><span data-stu-id="d75b8-122">Queues</span></span>  
 <span data-ttu-id="d75b8-123">在 WCF 中的佇列會提供可信賴傳訊訊息及區隔來源和目的地高延遲的情況下之間。</span><span class="sxs-lookup"><span data-stu-id="d75b8-123">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="d75b8-124">WCF 排入佇列通訊建置在訊息佇列 (MSMQ)。</span><span class="sxs-lookup"><span data-stu-id="d75b8-124">WCF queued communication is built on top of Message Queuing (MSMQ).</span></span>  
  
 <span data-ttu-id="d75b8-125">MSMQ 是 Windows 所附的選用元件。</span><span class="sxs-lookup"><span data-stu-id="d75b8-125">MSMQ ships as an optional component with Windows.</span></span> <span data-ttu-id="d75b8-126">MSMQ 服務會執行為 Windows 服務。</span><span class="sxs-lookup"><span data-stu-id="d75b8-126">The MSMQ service runs as a Windows Service.</span></span> <span data-ttu-id="d75b8-127">它會代表來源擷取傳輸佇列中要進行傳輸的訊息，並將該訊息傳遞至目標佇列。</span><span class="sxs-lookup"><span data-stu-id="d75b8-127">It captures messages for transmission in a transmission queue on behalf of the source and delivers it to a target queue.</span></span> <span data-ttu-id="d75b8-128">目標佇列會代表目的地接受訊息，以便隨時因應目的地要求訊息而進行傳遞。</span><span class="sxs-lookup"><span data-stu-id="d75b8-128">The target queue accepts messages on behalf of the destination for later delivery whenever the destination requests messages.</span></span> <span data-ttu-id="d75b8-129">MSMQ 管理員會實作可信賴傳訊通訊協定，這樣訊息就不會在傳輸期間遺失。</span><span class="sxs-lookup"><span data-stu-id="d75b8-129">The MSMQ managers implement a reliable message-transfer protocol so that messages are not lost in transmission.</span></span> <span data-ttu-id="d75b8-130">此通訊協定可以是原生 (Native)，或是稱為 SOAP Reliable Messaging Protocol (SRMP) 的 SOAP 架構通訊協定。</span><span class="sxs-lookup"><span data-stu-id="d75b8-130">The protocol can be native or a SOAP-based protocol called SOAP Reliable Messaging Protocol (SRMP).</span></span>  
  
 <span data-ttu-id="d75b8-131">佇列之間的區隔性以及可信賴傳訊，可讓鬆散耦合的應用程式進行可靠的通訊。</span><span class="sxs-lookup"><span data-stu-id="d75b8-131">The separation, coupled with reliable message transfers between queues, enables applications that are loosely coupled to communicate reliably.</span></span> <span data-ttu-id="d75b8-132">與可靠工作階段不同的是，來源和目的地不需要同時執行。</span><span class="sxs-lookup"><span data-stu-id="d75b8-132">Unlike reliable sessions, the source and destination do not have to be running at the same time.</span></span> <span data-ttu-id="d75b8-133">這個特點促使當來源的訊息生產率與目的地的訊息消耗率不相同時，作用中的佇列會被用來當做負載撫平機制。</span><span class="sxs-lookup"><span data-stu-id="d75b8-133">This implicitly enables scenarios where queues are, in effect, used as a load-leveling mechanism when the source's rate of message production and the destination's rate of the message consumption do not match.</span></span> <span data-ttu-id="d75b8-134">如需有關佇列的詳細資訊，請參閱[WCF 中的佇列](../../../docs/framework/wcf/feature-details/queues-in-wcf.md)。</span><span class="sxs-lookup"><span data-stu-id="d75b8-134">For more information about queues, see [Queues in WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span></span>
