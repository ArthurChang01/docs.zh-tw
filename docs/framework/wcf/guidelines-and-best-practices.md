# <a name="guidelines-and-best-practices"></a>方針及最佳作法
本節包含的主題提供建立 Windows Communication Foundation （WCF）應用程式的指導方針。  
  
## <a name="in-this-section"></a>本章節內容  
 [最佳做法：資料合約版本設定](best-practices-data-contract-versioning.md)  
 說明如何及何時建立不會在未來版本建立時中斷的資料合約。  
  
 [服務版本設定](service-versioning.md)  
 說明如何在 WCF 中考慮版本設定。 部署之後，服務 (及其公開的端點) 可能必須予以變更，例如，為了滿足商務需求或 IT 需求的變更，或者為了修復問題。 每個變更都會產生新的服務版本。  
  
 [負載平衡](load-balancing.md)  
 列出使用 Web 伺服陣列的負載平衡方針。  
  
 [控制資源使用並改善效能](controlling-resource-consumption-and-improving-performance.md)  
 說明為了防止過度消耗資源及改善安全性，並指出有關其使用之完整資訊而設計的屬性。  
  
 [利用 ClickOnce 部署 WCF 應用程式](deploying-wcf-applications-with-clickonce.md)  
 說明使用 ClickOnce 功能時所需考量的事項。
