### <a name="messaging"></a><span data-ttu-id="e80f1-125">訊息</span><span class="sxs-lookup"><span data-stu-id="e80f1-125">Messaging</span></span>  
 <span data-ttu-id="e80f1-126">在訊息層組成*通道*。</span><span class="sxs-lookup"><span data-stu-id="e80f1-126">The messaging layer is composed of *channels*.</span></span> <span data-ttu-id="e80f1-127">通道是指能以某種方式 (例如，藉由驗證訊息) 處理訊息的元件。</span><span class="sxs-lookup"><span data-stu-id="e80f1-127">A channel is a component that processes a message in some way, for example, by authenticating a message.</span></span> <span data-ttu-id="e80f1-128">一組通道就是所謂*通道堆疊*。</span><span class="sxs-lookup"><span data-stu-id="e80f1-128">A set of channels is also known as a *channel stack*.</span></span> <span data-ttu-id="e80f1-129">通道會在訊息和訊息標頭上運作。</span><span class="sxs-lookup"><span data-stu-id="e80f1-129">Channels operate on messages and message headers.</span></span> <span data-ttu-id="e80f1-130">這點不同於服務執行階段層，其主要著重於關於處理訊息本文的執行。</span><span class="sxs-lookup"><span data-stu-id="e80f1-130">This is different from the service runtime layer, which is primarily concerned about processing the contents of message bodies.</span></span>  
  
 <span data-ttu-id="e80f1-131">通道類型有兩種：傳輸通道和通訊協定通道。</span><span class="sxs-lookup"><span data-stu-id="e80f1-131">There are two types of channels: transport channels and protocol channels.</span></span>  
  
 <span data-ttu-id="e80f1-132">傳輸通道會從網路讀取和寫入訊息 (或是與外界連接的一些其他通訊點)。</span><span class="sxs-lookup"><span data-stu-id="e80f1-132">Transport channels read and write messages from the network (or some other communication point with the outside world).</span></span> <span data-ttu-id="e80f1-133">有些傳輸會使用編碼器在訊息 (表示為 XML Infoset) 和網路所使用的位元組資料流表示之間來回轉換。</span><span class="sxs-lookup"><span data-stu-id="e80f1-133">Some transports use an encoder to convert messages (which are represented as XML Infosets) to and from the byte stream representation used by the network.</span></span> <span data-ttu-id="e80f1-134">這類傳輸的範例包括 HTTP、具名管道 (Named Pipe)、TCP 及 MSMQ。</span><span class="sxs-lookup"><span data-stu-id="e80f1-134">Examples of transports are HTTP, named pipes, TCP, and MSMQ.</span></span> <span data-ttu-id="e80f1-135">這類編碼的範例包括 XML 和最佳化的二進位編碼。</span><span class="sxs-lookup"><span data-stu-id="e80f1-135">Examples of encodings are XML and optimized binary.</span></span>  
  
 <span data-ttu-id="e80f1-136">通訊協定通道會實作訊息處理通訊協定，而此通常是藉由讀取或寫入訊息的額外標頭來完成。</span><span class="sxs-lookup"><span data-stu-id="e80f1-136">Protocol channels implement message processing protocols, often by reading or writing additional headers to the message.</span></span> <span data-ttu-id="e80f1-137">這類通訊協定的範例包括 WS-Security 和 WS-Reliability。</span><span class="sxs-lookup"><span data-stu-id="e80f1-137">Examples of such protocols include WS-Security and WS-Reliability.</span></span>  
  
 <span data-ttu-id="e80f1-138">訊息層說明資料的可能格式與交換模式。</span><span class="sxs-lookup"><span data-stu-id="e80f1-138">The messaging layer illustrates the possible formats and exchange patterns of the data.</span></span> <span data-ttu-id="e80f1-139">WS-Security 是 WS-Security 規格的實作 (Implementation)，此規格會啟用訊息層的安全性。</span><span class="sxs-lookup"><span data-stu-id="e80f1-139">WS-Security is an implementation of the WS-Security specification enabling security at the message layer.</span></span> <span data-ttu-id="e80f1-140">WS-Reliable 訊息通道可保證訊息的傳遞。</span><span class="sxs-lookup"><span data-stu-id="e80f1-140">The WS-Reliable Messaging channel enables the guarantee of message delivery.</span></span> <span data-ttu-id="e80f1-141">編碼器會提供可以用來滿足訊息需要的各種編碼方式。</span><span class="sxs-lookup"><span data-stu-id="e80f1-141">The encoders present a variety of encodings that can be used to suit the needs of the message.</span></span> <span data-ttu-id="e80f1-142">HTTP 通道會指定對訊息傳遞使用超文字傳輸協定 (Hypertext Transfer Protocol，HTTP)。</span><span class="sxs-lookup"><span data-stu-id="e80f1-142">The HTTP channel specifies that the HyperText Transport Protocol is used for message delivery.</span></span> <span data-ttu-id="e80f1-143">同樣地，TCP 通道會指定 TCP 通訊協定。</span><span class="sxs-lookup"><span data-stu-id="e80f1-143">The TCP channel similarly specifies the TCP protocol.</span></span> <span data-ttu-id="e80f1-144">交易流程通道會管理交易的訊息模式。</span><span class="sxs-lookup"><span data-stu-id="e80f1-144">The Transaction Flow channel governs transacted message patterns.</span></span> <span data-ttu-id="e80f1-145">具名管道通道會啟用處理序間通訊。</span><span class="sxs-lookup"><span data-stu-id="e80f1-145">The Named Pipe channel enables interprocess communication.</span></span> <span data-ttu-id="e80f1-146">MSMQ 通道會啟用與 MSMQ 應用程式的互通。</span><span class="sxs-lookup"><span data-stu-id="e80f1-146">The MSMQ channel enables interoperation with MSMQ applications.</span></span>
