# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="91a30-102">採用 Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="91a30-102">Adopting Windows Communication Foundation</span></span>

<span data-ttu-id="91a30-103">您可以選擇使用新的程式開發的 Windows Communication Foundation (WCF)，而繼續維護現有的應用程式使用 ASP.NET 開發的。</span><span class="sxs-lookup"><span data-stu-id="91a30-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="91a30-104">WCF 的設計是要使用.NET Framework 中的任何案例建置應用程式簡化通訊最適合的選擇，因為它可做為標準的工具來解決各種軟體通訊問題的方式，讓 ASP.NET不能。</span><span class="sxs-lookup"><span data-stu-id="91a30-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="91a30-105">新的 WCF 應用程式可以部署與現有的 ASP.NET Web 服務的同一部電腦上。</span><span class="sxs-lookup"><span data-stu-id="91a30-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="91a30-106">如果現有的 ASP.NET Web 服務會使用.NET Framework 2.0 版之前的版本，您可以選擇性地將.NET Framework 2.0 部署至新的 WCF 應用程式要裝載的 IIS 應用程式使用 ASP.NET IIS 註冊工具。</span><span class="sxs-lookup"><span data-stu-id="91a30-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="91a30-107">該工具的說明位於[ASP.NET IIS 註冊工具 (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687)，和已建置到 IIS 6.0 管理主控台的使用者介面。</span><span class="sxs-lookup"><span data-stu-id="91a30-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="91a30-108">WCF 可以用來加入現有的 ASP.NET Web 服務中的新功能，加上設定為在 ASP.NET 相容性模式中執行現有的 ASP.NET Web 服務應用程式在 IIS 中的 WCF 服務中。</span><span class="sxs-lookup"><span data-stu-id="91a30-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="91a30-109">由於 ASP.NET 相容性模式中，新的 WCF 服務的程式碼可以存取與更新現有的 ASP.NET 程式碼中，相同的應用程式狀態資訊使用<xref:System.Web.HttpContext>類別。</span><span class="sxs-lookup"><span data-stu-id="91a30-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="91a30-110">應用程式也可以共用相同的類別庫。</span><span class="sxs-lookup"><span data-stu-id="91a30-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="91a30-111">WCF 用戶端可以使用 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="91a30-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="91a30-112">使用所設定的 WCF 服務<xref:System.ServiceModel.BasicHttpBinding>可由 ASP.NET Web 服務用戶端。</span><span class="sxs-lookup"><span data-stu-id="91a30-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="91a30-113">ASP.NET Web 服務可以同時存在與 WCF 應用程式，並甚至可以使用 WCF 將功能加入至現有的 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="91a30-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="91a30-114">假設這些所有方式可以一起使用 WCF 和 ASP.NET Web 服務，您可能都想要將 ASP.NET Web 服務移轉至 WCF，只有在需要 WCF 和不是 ASP.NET Web 服務所提供的功能。</span><span class="sxs-lookup"><span data-stu-id="91a30-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="91a30-115">甚至在需要在少數情況下，移轉的程式碼從一種技術之間很少是正確的方法。</span><span class="sxs-lookup"><span data-stu-id="91a30-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="91a30-116">採用新技術的原因在於符合使用稍早技術無法滿足的新需求，在該情況下，正確的做法是設計新的解決方案以符合一組新擴展的需求。</span><span class="sxs-lookup"><span data-stu-id="91a30-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="91a30-117">新設計的優點在於採用現有系統的經驗，以及設計該系統以來所獲取的智慧。</span><span class="sxs-lookup"><span data-stu-id="91a30-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="91a30-118">新設計也可以使用新技術的完整功能，而不必在新平台上重現舊設計。</span><span class="sxs-lookup"><span data-stu-id="91a30-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="91a30-119">製作新設計主要元素的原型之後，在新系統中重複使用現有系統的程式碼就會變得更容易。</span><span class="sxs-lookup"><span data-stu-id="91a30-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>
