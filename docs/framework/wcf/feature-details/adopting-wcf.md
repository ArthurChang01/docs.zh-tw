# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="ba207-102">採用 Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="ba207-102">Adopting Windows Communication Foundation</span></span>
<span data-ttu-id="ba207-103">您可以選擇進行新開發，使用 Windows Communication Foundation (WCF)，而繼續維護現有應用程式使用 ASP.NET 開發。</span><span class="sxs-lookup"><span data-stu-id="ba207-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="ba207-104">WCF 要用最適合的選擇，在任何案例中的.NET framework 建置之應用程式簡化通訊，因為它可以做為用來解決各種軟體通訊問題的方式的標準工具 ASP.NET不能。</span><span class="sxs-lookup"><span data-stu-id="ba207-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>  
  
 <span data-ttu-id="ba207-105">可以為現有的 ASP.NET Web 服務相同的電腦上部署新的 WCF 應用程式。</span><span class="sxs-lookup"><span data-stu-id="ba207-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="ba207-106">如果現有的 ASP.NET Web 服務會使用.NET Framework 2.0 版之前的版本，您可以選擇性地將.NET Framework 2.0 部署至新的 WCF 應用程式要裝載的 IIS 應用程式使用 ASP.NET IIS 註冊工具。</span><span class="sxs-lookup"><span data-stu-id="ba207-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="ba207-107">該工具已記錄在[ASP.NET IIS 註冊工具 (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687)，和已建置到 IIS 6.0 管理主控台的使用者介面。</span><span class="sxs-lookup"><span data-stu-id="ba207-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>  
  
 <span data-ttu-id="ba207-108">WCF 可以用來加入設定為在 IIS 中的現有 ASP.NET Web 服務應用程式在 ASP.NET 相容性模式中執行的 WCF 服務，將新功能加入至現有的 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="ba207-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="ba207-109">由於 ASP.NET 相容性模式中，新的 WCF 服務的程式碼可以存取與更新相同的應用程式狀態資訊已存在的 ASP.NET 程式碼使用<xref:System.Web.HttpContext>類別。</span><span class="sxs-lookup"><span data-stu-id="ba207-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="ba207-110">應用程式也可以共用相同的類別庫。</span><span class="sxs-lookup"><span data-stu-id="ba207-110">The applications can also share the same class libraries.</span></span>  
  
 <span data-ttu-id="ba207-111">WCF 用戶端可以使用 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="ba207-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="ba207-112">使用所設定的 WCF 服務<xref:System.ServiceModel.BasicHttpBinding>可由 ASP.NET Web 服務用戶端。</span><span class="sxs-lookup"><span data-stu-id="ba207-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="ba207-113">與 WCF 應用程式，可以共存 ASP.NET Web 服務和 WCF 甚至可以將功能加入現有的 ASP.NET Web 服務使用。</span><span class="sxs-lookup"><span data-stu-id="ba207-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="ba207-114">假設這些所有方式可以一起使用 WCF 和 ASP.NET Web 服務，您可能都想要將 ASP.NET Web 服務移轉至 WCF 中，只有在需要 WCF 和不是 ASP.NET Web 服務所提供的功能。</span><span class="sxs-lookup"><span data-stu-id="ba207-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>  
  
 <span data-ttu-id="ba207-115">即使是在少數必要的情況下，謹慎考慮將程式碼在兩個技術之間移轉也不見得是正確的方法。</span><span class="sxs-lookup"><span data-stu-id="ba207-115">Even in the few cases where it is necessary, carefully consider that migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="ba207-116">採用新技術的原因在於符合使用稍早技術無法滿足的新需求，在該情況下，正確的做法是設計新的解決方案以符合一組新擴展的需求。</span><span class="sxs-lookup"><span data-stu-id="ba207-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="ba207-117">新設計的優點在於採用現有系統的經驗，以及設計該系統以來所獲取的智慧。</span><span class="sxs-lookup"><span data-stu-id="ba207-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="ba207-118">新設計也可以使用新技術的完整功能，而不必在新平台上重現舊設計。</span><span class="sxs-lookup"><span data-stu-id="ba207-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="ba207-119">製作新設計主要元素的原型之後，在新系統中重複使用現有系統的程式碼就會變得更容易。</span><span class="sxs-lookup"><span data-stu-id="ba207-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>  
  
 <span data-ttu-id="ba207-120">在少數情況下從 ASP.NET Web 服務移轉至 WCF 其中為正確的解決方案下, 一節提供一些指引要如何繼續執行。</span><span class="sxs-lookup"><span data-stu-id="ba207-120">For the few cases where porting from ASP.NET Web services to WCF is the correct solution, the following section provides some guidance on how to proceed.</span></span> <span data-ttu-id="ba207-121">其中包含如何移轉服務以及如何移轉用戶端的建議。</span><span class="sxs-lookup"><span data-stu-id="ba207-121">There is advice on how to migrate services, and how to migrate clients.</span></span>  
  
 <span data-ttu-id="ba207-122">如需如何將現有的 ASP.NET Web 服務移轉至 WCF 完整分析，請參閱[ASP.NET Web 服務和 Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761)。</span><span class="sxs-lookup"><span data-stu-id="ba207-122">For a complete analysis on how to migrate existing ASP.NET Web Services to WCF please see [ASP.NET Web Services and the Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761).</span></span> <span data-ttu-id="ba207-123">本章節描述如何為您的 ASP.NET Web 服務，實作相容的 WCF 服務的中繼資料以及如何將 ASP.NET Web 服務和用戶端程式碼移轉至 WCF。</span><span class="sxs-lookup"><span data-stu-id="ba207-123">This section describes how to implement a compliant WCF service from the metadata for you ASP.NET Web Service, and how to migrate ASP.NET Web service and client code to WCF.</span></span>
