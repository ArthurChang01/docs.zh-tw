<span data-ttu-id="779fe-105">裝載在網際網路資訊服務 (IIS) 中的 WCF 服務可以位於與。ASPX 頁面和 ASMX Web 服務位於單一、 共同的應用程式定義域。</span><span class="sxs-lookup"><span data-stu-id="779fe-105">WCF services hosted in Internet Information Services (IIS) can be located with .ASPX pages and ASMX Web services inside of a single, common Application Domain.</span></span> <span data-ttu-id="779fe-106">ASP.NET 提供通用的基礎結構服務，例如 AppDomain 管理與 WCF 和 ASP.NET HTTP 執行階段動態編譯。</span><span class="sxs-lookup"><span data-stu-id="779fe-106">ASP.NET provides common infrastructure services such as AppDomain management and dynamic compilation for both WCF and the ASP.NET HTTP runtime.</span></span> <span data-ttu-id="779fe-107">WCF 的預設組態會並排顯示使用 ASP.NET。</span><span class="sxs-lookup"><span data-stu-id="779fe-107">The default configuration for WCF is side-by-side with ASP.NET.</span></span>

<span data-ttu-id="779fe-109">ASP.NET HTTP 執行階段會處理 ASP.NET 要求，但不會參與處理要求目的地的 WCF 服務，即使這些服務裝載於相同的 AppDomain，與是 ASP.NET 的內容。</span><span class="sxs-lookup"><span data-stu-id="779fe-109">The ASP.NET HTTP runtime handles ASP.NET requests but does not participate in the processing of requests destined for WCF services, even though these services are hosted in the same AppDomain as is the ASP.NET content.</span></span> <span data-ttu-id="779fe-110">相反地，WCF 服務模型會攔截訊息定址到 WCF 服務，並透過 WCF 傳輸/通道堆疊路由傳送。</span><span class="sxs-lookup"><span data-stu-id="779fe-110">Instead, the WCF Service Model intercepts messages addressed to WCF services and routes them through the WCF transport/channel stack.</span></span>

## <a name="hosting-wcf-services-in-aspnet-compatibility-mode"></a><span data-ttu-id="779fe-136">在 ASP.NET 相容性模式中裝載 WCF 服務</span><span class="sxs-lookup"><span data-stu-id="779fe-136">Hosting WCF services in ASP.NET compatibility mode</span></span>

<span data-ttu-id="779fe-137">雖然 WCF 模型設計成各種裝載環境與傳輸有一致的行為，是彈性的通常應用程式不需要這麼大的案例。</span><span class="sxs-lookup"><span data-stu-id="779fe-137">Although the WCF model is designed to behave consistently across hosting environments and transports, there are often scenarios where an application does not require this degree of flexibility.</span></span> <span data-ttu-id="779fe-138">WCF 的 ASP.NET 相容性模式是適用於不需要主機 IIS 外部或透過 HTTP 以外的通訊協定進行通訊的能力，但使用的所有 ASP.NET Web 應用程式平台功能的案例。</span><span class="sxs-lookup"><span data-stu-id="779fe-138">WCF’s ASP.NET compatibility mode is suitable for scenarios that do not require the ability to host outside of IIS or to communicate over protocols other than HTTP, but that use all of features of the ASP.NET Web application platform.</span></span>

<span data-ttu-id="779fe-139">不同於預設並排顯示設定，其中 WCF 裝載基礎結構會攔截 WCF 訊息，並將它們路由傳送至 HTTP 管線外，在 ASP.NET 相容性模式中執行的 WCF 服務充分參與 ASP.NET HTTP 要求的生命週期。</span><span class="sxs-lookup"><span data-stu-id="779fe-139">Unlike the default side-by-side configuration, where the WCF hosting infrastructure intercepts WCF messages and routes them out of the HTTP pipeline, WCF services running in ASP.NET Compatibility Mode participate fully in the ASP.NET HTTP request lifecycle.</span></span> <span data-ttu-id="779fe-140">在相容性模式中，WCF 服務會使用透過 HTTP 管線<xref:System.Web.IHttpHandler>實作，如處理 ASPX 頁面和 ASMX Web 服務的類似方式要求。</span><span class="sxs-lookup"><span data-stu-id="779fe-140">In compatibility mode, WCF services use the HTTP pipeline through an <xref:System.Web.IHttpHandler> implementation, similar to the way requests for ASPX pages and ASMX Web services are handled.</span></span> <span data-ttu-id="779fe-141">如此一來，WCF 行為與相同 ASMX 對於下列的 ASP.NET 功能：</span><span class="sxs-lookup"><span data-stu-id="779fe-141">As a result, WCF behaves identically to ASMX with respect to the following ASP.NET features:</span></span>
