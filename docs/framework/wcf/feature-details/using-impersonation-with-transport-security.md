# <a name="using-impersonation-with-transport-security"></a><span data-ttu-id="40cd6-102">使用模擬搭配傳輸安全性</span><span class="sxs-lookup"><span data-stu-id="40cd6-102">Using Impersonation with Transport Security</span></span>
<span data-ttu-id="40cd6-103">*模擬*是伺服器應用程式才會在用戶端的身分識別的能力。</span><span class="sxs-lookup"><span data-stu-id="40cd6-103">*Impersonation* is the ability of a server application to take on the identity of the client.</span></span> <span data-ttu-id="40cd6-104">對服務而言，驗證存取資源時使用模擬是很常見的。</span><span class="sxs-lookup"><span data-stu-id="40cd6-104">It is common for services to use impersonation when validating access to resources.</span></span> <span data-ttu-id="40cd6-105">伺服器應用程式使用服務帳戶執行，但是伺服器在接受用戶端連線會模擬用戶端，藉此使用用戶端的認證執行存取檢查。</span><span class="sxs-lookup"><span data-stu-id="40cd6-105">The server application runs using a service account, but when the server accepts a client connection, it impersonates the client so that access checks are performed using the client's credentials.</span></span> <span data-ttu-id="40cd6-106">傳輸安全性是一種用於傳遞認證與使用該認證保護通訊安全性的機制。</span><span class="sxs-lookup"><span data-stu-id="40cd6-106">Transport security is a mechanism both for passing credentials and securing communication using those credentials.</span></span> <span data-ttu-id="40cd6-107">本主題描述使用傳輸安全性的設定，在 Windows Communication Foundation (WCF) 搭配模擬功能。</span><span class="sxs-lookup"><span data-stu-id="40cd6-107">This topic describes using transport security in Windows Communication Foundation (WCF) with the impersonation feature.</span></span> <span data-ttu-id="40cd6-108">如需有關使用訊息安全性模擬的詳細資訊，請參閱[委派和模擬](../../../../docs/framework/wcf/feature-details/delegation-and-impersonation-with-wcf.md)。</span><span class="sxs-lookup"><span data-stu-id="40cd6-108">For more information about impersonation using message security, see [Delegation and Impersonation](../../../../docs/framework/wcf/feature-details/delegation-and-impersonation-with-wcf.md).</span></span>  
  
## <a name="five-impersonation-levels"></a><span data-ttu-id="40cd6-109">五個模擬等級</span><span class="sxs-lookup"><span data-stu-id="40cd6-109">Five Impersonation Levels</span></span>  
 <span data-ttu-id="40cd6-110">傳輸安全性運用如下表所述的五種模擬等級。</span><span class="sxs-lookup"><span data-stu-id="40cd6-110">Transport security makes use of five levels of impersonation, as described in the following table.</span></span>  
  
|<span data-ttu-id="40cd6-111">模擬等級</span><span class="sxs-lookup"><span data-stu-id="40cd6-111">Impersonation level</span></span>|<span data-ttu-id="40cd6-112">描述</span><span class="sxs-lookup"><span data-stu-id="40cd6-112">Description</span></span>|  
|-------------------------|-----------------|  
|<span data-ttu-id="40cd6-113">None</span><span class="sxs-lookup"><span data-stu-id="40cd6-113">None</span></span>|<span data-ttu-id="40cd6-114">伺服器不嘗試模擬用戶端。</span><span class="sxs-lookup"><span data-stu-id="40cd6-114">The server application does not attempt to impersonate the client.</span></span>|  
|<span data-ttu-id="40cd6-115">匿名</span><span class="sxs-lookup"><span data-stu-id="40cd6-115">Anonymous</span></span>|<span data-ttu-id="40cd6-116">伺服器應用程式可以根據用戶端的認證執行存取檢查，但不會接收任何有關用戶端身分識別的資訊。</span><span class="sxs-lookup"><span data-stu-id="40cd6-116">The server application can perform access checks against the client's credentials, but does not receive any information about the client's identity.</span></span> <span data-ttu-id="40cd6-117">模擬等級只對電腦上的通訊有意義，例如具名管道。</span><span class="sxs-lookup"><span data-stu-id="40cd6-117">This impersonation level is meaningful only for on-machine communication, such as named pipes.</span></span> <span data-ttu-id="40cd6-118">使用 `Anonymous` 搭配遠端連線將模擬等級提升至識別。</span><span class="sxs-lookup"><span data-stu-id="40cd6-118">Using `Anonymous` with a remote connection promotes the impersonation level to Identify.</span></span>|  
|<span data-ttu-id="40cd6-119">識別</span><span class="sxs-lookup"><span data-stu-id="40cd6-119">Identify</span></span>|<span data-ttu-id="40cd6-120">伺服器應用程式知道用戶端的身分，而且可以根據用戶端的認證執行存取驗證，但無法模擬用戶端。</span><span class="sxs-lookup"><span data-stu-id="40cd6-120">The server application knows the client's identity and can perform access validation against the client's credentials, but cannot impersonate the client.</span></span> <span data-ttu-id="40cd6-121">識別是除非權杖提供者提供不同的模擬等級，WCF 中搭配 SSPI 認證使用的預設模擬等級。</span><span class="sxs-lookup"><span data-stu-id="40cd6-121">Identify is the default impersonation level used with SSPI credentials in WCF unless the token provider provides a different impersonation level.</span></span>|  
|<span data-ttu-id="40cd6-122">Impersonate</span><span class="sxs-lookup"><span data-stu-id="40cd6-122">Impersonate</span></span>|<span data-ttu-id="40cd6-123">除了執行存取檢查外，伺服器應用程式可以存取做為用戶端的伺服器電腦上的資源。</span><span class="sxs-lookup"><span data-stu-id="40cd6-123">The server application can access resources on the server machine as the client in addition to performing access checks.</span></span> <span data-ttu-id="40cd6-124">伺服器應用程式無法使用用戶端識別存取遠端電腦上的資源，因為模擬的權杖沒有網路認證。</span><span class="sxs-lookup"><span data-stu-id="40cd6-124">The server application cannot access resources on remote machines using the client's identity because the impersonated token does not have network credentials</span></span>|  
|<span data-ttu-id="40cd6-125">Delegate - 委派</span><span class="sxs-lookup"><span data-stu-id="40cd6-125">Delegate</span></span>|<span data-ttu-id="40cd6-126">除了擁有與 `Impersonate` 相同的功能外，「委派」模擬等級也可以使用用戶端識別讓伺服器應用程式存取遠端電腦上的資訊，並且將識別傳遞到其他應用程式。</span><span class="sxs-lookup"><span data-stu-id="40cd6-126">In addition to having the same capabilities as `Impersonate`, the Delegate impersonation level also enables the server application to access resources on remote machines using the client's identity and to pass the identity to other applications.</span></span><br /><br /> <span data-ttu-id="40cd6-127">**重要**伺服器網域帳戶必須標示為受信任的網域控制站，才能使用這些額外的功能上的委派。</span><span class="sxs-lookup"><span data-stu-id="40cd6-127">**Important** The server domain account must be marked as trusted for delegation on the domain controller to use these additional features.</span></span> <span data-ttu-id="40cd6-128">這個模擬等級無法搭配標示為敏感的用戶端網域帳戶使用。</span><span class="sxs-lookup"><span data-stu-id="40cd6-128">This level of impersonation cannot be used with client domain accounts marked as sensitive.</span></span>|  
  
 <span data-ttu-id="40cd6-129">最常搭配傳輸安全性層級`Identify`和`Impersonate`。</span><span class="sxs-lookup"><span data-stu-id="40cd6-129">The levels most commonly used with transport security are `Identify` and `Impersonate`.</span></span> <span data-ttu-id="40cd6-130">不建議將 `None` 和 `Anonymous` 等級使用於一般用途，而且很多傳輸不支援使用這些等級進行驗證。</span><span class="sxs-lookup"><span data-stu-id="40cd6-130">The levels `None` and `Anonymous` are not recommended for typical use, and many transports do not support using those levels with authentication.</span></span> <span data-ttu-id="40cd6-131">請小心使用 `Delegate` 等級的強大功能，</span><span class="sxs-lookup"><span data-stu-id="40cd6-131">The `Delegate` level is a powerful feature that should be used with care.</span></span> <span data-ttu-id="40cd6-132">僅將委派憑證的使用權限提供給受信任的伺服器應用程式。</span><span class="sxs-lookup"><span data-stu-id="40cd6-132">Only trusted server applications should be given the permission to delegate credentials.</span></span>  
  
 <span data-ttu-id="40cd6-133">伺服器應用程式需要有 `Impersonate` 權限，才能在 `Delegate` 或 `SeImpersonatePrivilege` 等級使用模擬。</span><span class="sxs-lookup"><span data-stu-id="40cd6-133">Using impersonation at the `Impersonate` or `Delegate` levels requires the server application to have the `SeImpersonatePrivilege` privilege.</span></span> <span data-ttu-id="40cd6-134">如果應用程式在管理員群組或有服務 SID (網路服務、本機服務或本機系統) 的帳戶上執行，該應用程式預設為擁有此權限。</span><span class="sxs-lookup"><span data-stu-id="40cd6-134">An application has this privilege by default if it is running on an account in the Administrators group or on an account with a Service SID (Network Service, Local Service, or Local System).</span></span> <span data-ttu-id="40cd6-135">伺服器不需要用戶端和伺服器的相互驗證。</span><span class="sxs-lookup"><span data-stu-id="40cd6-135">Impersonation does not require mutual authentication of the client and server.</span></span> <span data-ttu-id="40cd6-136">有些支援模擬的驗證配置 (例如 NTLM) 無法搭配相互驗證使用。</span><span class="sxs-lookup"><span data-stu-id="40cd6-136">Some authentication schemes that support impersonation, such as NTLM, cannot be used with mutual authentication.</span></span>
