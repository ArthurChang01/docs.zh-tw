## <a name="application-specific-dead-letter-queue"></a><span data-ttu-id="6eebd-104">應用程式特定之寄不出的信件佇列</span><span class="sxs-lookup"><span data-stu-id="6eebd-104">Application-Specific Dead-Letter Queue</span></span>

 [!INCLUDE[wv](../../../../includes/wv-md.md)] <span data-ttu-id="6eebd-115">提供應用程式特定之寄不出的信件佇列。</span><span class="sxs-lookup"><span data-stu-id="6eebd-115">provides for application-specific dead-letter queues.</span></span> <span data-ttu-id="6eebd-116">應用程式特定之寄不出的信件佇列無法在 [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 和 [!INCLUDE[wxp](../../../../includes/wxp-md.md)] 中使用，且應用程式必須使用整個系統之寄不出的信件佇列。</span><span class="sxs-lookup"><span data-stu-id="6eebd-116">Application-specific dead-letter queues are not available in [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] and [!INCLUDE[wxp](../../../../includes/wxp-md.md)], and applications must use the system-wide dead-letter queue.</span></span>  
  
## <a name="poison-message-handling"></a><span data-ttu-id="6eebd-117">有害訊息處理</span><span class="sxs-lookup"><span data-stu-id="6eebd-117">Poison-Message Handling</span></span>  
 <span data-ttu-id="6eebd-118">有害訊息是超過嘗試傳遞至接收應用程式之次數上限的訊息。</span><span class="sxs-lookup"><span data-stu-id="6eebd-118">A poison message is a message that has exceeded the maximum number of delivery attempts to the receiving application.</span></span> <span data-ttu-id="6eebd-119">當從異動式佇列讀取訊息的應用程式因錯誤無法立即處理訊息時，便會引起這種情況。</span><span class="sxs-lookup"><span data-stu-id="6eebd-119">This situation can arise when an application that reads a message from a transactional queue cannot process the message immediately because of errors.</span></span> <span data-ttu-id="6eebd-120">如果應用程式中止收到佇列訊息的交易，會將訊息傳回佇列。</span><span class="sxs-lookup"><span data-stu-id="6eebd-120">If the application aborts the transaction in which the queued message was received, it returns the message to the queue.</span></span> <span data-ttu-id="6eebd-121">然後應用程式會嘗試在新的異動中再次擷取訊息。</span><span class="sxs-lookup"><span data-stu-id="6eebd-121">The application then tries to retrieve the message again in a new transaction.</span></span> <span data-ttu-id="6eebd-122">如果沒有更正造成錯誤的問題，接收應用程式會卡在接收及中止相同訊息的迴圈中，直到超過傳遞嘗試次數上限為止，而形成有害訊息。</span><span class="sxs-lookup"><span data-stu-id="6eebd-122">If the problem that causes the error is not corrected, the receiving application can get stuck in a loop receiving and aborting the same message until it exceeds the maximum number of delivery attempts, and a poison message results.</span></span>  
  
 <span data-ttu-id="6eebd-123">在 [!INCLUDE[wv](../../../../includes/wv-md.md)]、[!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 和 [!INCLUDE[wxp](../../../../includes/wxp-md.md)] 上的訊息佇列 (MSMQ) 之間，與有害處理有關的主要差別包括下列：</span><span class="sxs-lookup"><span data-stu-id="6eebd-123">The key differences between Message Queuing (MSMQ) on [!INCLUDE[wv](../../../../includes/wv-md.md)], [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)], and [!INCLUDE[wxp](../../../../includes/wxp-md.md)] that are relevant to poison handling include the following:</span></span>  
  
- <span data-ttu-id="6eebd-124">[!INCLUDE[wv](../../../../includes/wv-md.md)] 中的 MSMQ 支援子佇列，而 [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 和 [!INCLUDE[wxp](../../../../includes/wxp-md.md)] 不支援子佇列。</span><span class="sxs-lookup"><span data-stu-id="6eebd-124">MSMQ in [!INCLUDE[wv](../../../../includes/wv-md.md)] supports subqueues, while [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] and [!INCLUDE[wxp](../../../../includes/wxp-md.md)] do not support subqueues.</span></span> <span data-ttu-id="6eebd-125">子佇列是在有害訊息處理中使用。</span><span class="sxs-lookup"><span data-stu-id="6eebd-125">Subqueues are used in poison-message handling.</span></span> <span data-ttu-id="6eebd-126">重試佇列和有害佇列都是應用程式佇列的子佇列，應用程式佇列是根據有害訊息處理設定而建立的。</span><span class="sxs-lookup"><span data-stu-id="6eebd-126">The retry queues and the poison queue are subqueues to the application queue that is created based on the poison-message handling settings.</span></span> <span data-ttu-id="6eebd-127">`MaxRetryCycles` 會指示要建立多少重試子佇列。</span><span class="sxs-lookup"><span data-stu-id="6eebd-127">The `MaxRetryCycles` dictates how many retry subqueues to create.</span></span> <span data-ttu-id="6eebd-128">因此，當在 [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 或 [!INCLUDE[wxp](../../../../includes/wxp-md.md)] 上執行時，會略過 `MaxRetryCycles` 並且不允許 `ReceiveErrorHandling.Move`。</span><span class="sxs-lookup"><span data-stu-id="6eebd-128">Therefore, when running on [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] or [!INCLUDE[wxp](../../../../includes/wxp-md.md)], `MaxRetryCycles` are ignored and `ReceiveErrorHandling.Move` is not allowed.</span></span>  
  
- <span data-ttu-id="6eebd-129">[!INCLUDE[wv](../../../../includes/wv-md.md)] 中的 MSMQ 支援負認可，而 [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 和 [!INCLUDE[wxp](../../../../includes/wxp-md.md)] 則否。</span><span class="sxs-lookup"><span data-stu-id="6eebd-129">MSMQ in [!INCLUDE[wv](../../../../includes/wv-md.md)] supports negative acknowledgment, while [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] and [!INCLUDE[wxp](../../../../includes/wxp-md.md)] do not.</span></span> <span data-ttu-id="6eebd-130">來自接收佇列管理員的負認可會造成傳送佇列管理員將拒絕的訊息放在寄不出的信件佇列中。</span><span class="sxs-lookup"><span data-stu-id="6eebd-130">A negative acknowledgment from the receiving queue manager causes the sending queue manager to place the rejected message in the dead-letter queue.</span></span> <span data-ttu-id="6eebd-131">因此，`ReceiveErrorHandling.Reject` 和 [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 不可使用 [!INCLUDE[wxp](../../../../includes/wxp-md.md)]。</span><span class="sxs-lookup"><span data-stu-id="6eebd-131">As such, `ReceiveErrorHandling.Reject` is not allowed with [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] and [!INCLUDE[wxp](../../../../includes/wxp-md.md)].</span></span>  
  
- <span data-ttu-id="6eebd-132">[!INCLUDE[wv](../../../../includes/wv-md.md)] 中的 MSMQ 支援訊息屬性，持續計算嘗試傳遞訊息的次數。</span><span class="sxs-lookup"><span data-stu-id="6eebd-132">MSMQ in [!INCLUDE[wv](../../../../includes/wv-md.md)] supports a message property that keeps count of the number of times message delivery is attempted.</span></span> <span data-ttu-id="6eebd-133">這個中止計數屬性無法在 [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] 和 [!INCLUDE[wxp](../../../../includes/wxp-md.md)] 上使用。</span><span class="sxs-lookup"><span data-stu-id="6eebd-133">This abort count property is not available on [!INCLUDE[ws2003](../../../../includes/ws2003-md.md)] and [!INCLUDE[wxp](../../../../includes/wxp-md.md)].</span></span> <span data-ttu-id="6eebd-134">WCF 會維護記憶體中的中止計數，因此它是這個屬性可能不包含精確的值相同的訊息讀取的 Web 伺服陣列中的多個 WCF 服務時。</span><span class="sxs-lookup"><span data-stu-id="6eebd-134">WCF maintains the abort count in memory, so it is possible that this property may not contain an accurate value when the same message is read by more than one WCF service in a Web farm.</span></span>
