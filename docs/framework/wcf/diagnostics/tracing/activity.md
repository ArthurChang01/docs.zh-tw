 <span data-ttu-id="8912c-169">「開始」與「停止」追蹤對相互關聯並不重要。</span><span class="sxs-lookup"><span data-stu-id="8912c-169">Start and Stop traces are not critical for correlation.</span></span> <span data-ttu-id="8912c-170">然而，這兩項追蹤有助於提升效能、進行分析，以及驗證活動範圍。</span><span class="sxs-lookup"><span data-stu-id="8912c-170">However, they can help in increasing performance, profiling, and activity scope validation.</span></span>  
  
 <span data-ttu-id="8912c-171">透過這些類型，工具可以最佳化追蹤記錄的瀏覽工作，以找到與相同活動立即相關的事件，或是相關活動中的事件 (如果工具遵循傳輸追蹤的話)。</span><span class="sxs-lookup"><span data-stu-id="8912c-171">Using these types, the tools can optimize navigating the trace logs to find the immediately related events of the same activity, or events in related activities if the tool follows transfer traces.</span></span> <span data-ttu-id="8912c-172">例如，當工具看到「開始」與「停止」追蹤時，就會針對特定活動停止剖析記錄。</span><span class="sxs-lookup"><span data-stu-id="8912c-172">For example, the tools will stop parsing the logs for a given activity when they see a Start/Stop trace.</span></span>  
  
 <span data-ttu-id="8912c-173">這些追蹤類型也可用來進行分析。</span><span class="sxs-lookup"><span data-stu-id="8912c-173">These trace types can also be used for profiling.</span></span> <span data-ttu-id="8912c-174">開始與停止資料標記之間所使用的資源代表活動的內含時間 (包括內含的邏輯活動)。</span><span class="sxs-lookup"><span data-stu-id="8912c-174">Resources consumed between the start and stop markers represent the activity’s inclusive time including contained logical activities.</span></span> <span data-ttu-id="8912c-175">減去「暫停」與「繼續」追蹤之間的時間間隔便能提供實際的活動時間。</span><span class="sxs-lookup"><span data-stu-id="8912c-175">Subtracting the time intervals between the Suspend and Resume traces provides the actual activity time.</span></span>  
  
 <span data-ttu-id="8912c-176">「停止」追蹤也很適合用來驗證已實作之活動的範圍。</span><span class="sxs-lookup"><span data-stu-id="8912c-176">The Stop trace is also particularly useful for validating the scope of the implemented activities.</span></span> <span data-ttu-id="8912c-177">如果某些處理中的追蹤在「停止」追蹤之後出現，而不是在特定活動中出現，表示可能出現程式碼缺失。</span><span class="sxs-lookup"><span data-stu-id="8912c-177">If some processing traces appear after the Stop trace instead of inside a given activity, this can suggests code defect.</span></span>  
  
## <a name="guidelines-for-using-activity-tracing"></a><span data-ttu-id="8912c-178">使用活動追蹤的方針</span><span class="sxs-lookup"><span data-stu-id="8912c-178">Guidelines for Using Activity Tracing</span></span>  
 <span data-ttu-id="8912c-179">下列為使用 ActivityTracing 追蹤 (開始、停止、暫停、繼續與傳輸) 的方針。</span><span class="sxs-lookup"><span data-stu-id="8912c-179">The following is a guideline of using ActivityTracing traces (Start, Stop, Suspend, Resume, and Transfer).</span></span>  
  
- <span data-ttu-id="8912c-180">追蹤是直接循環的圖形，而不是樹狀。</span><span class="sxs-lookup"><span data-stu-id="8912c-180">Tracing is a directed cyclic graph, not a tree.</span></span> <span data-ttu-id="8912c-181">您可以將控制項傳回已繁衍 (Spawn) 活動的活動。</span><span class="sxs-lookup"><span data-stu-id="8912c-181">You can return control to an activity which spawned an activity.</span></span>  
  
- <span data-ttu-id="8912c-182">活動代表處理界限，這對系統管理員或支援能力而言十分重要。</span><span class="sxs-lookup"><span data-stu-id="8912c-182">An activity denotes a processing boundary which can be meaningful to the administrator of the system or for supportability.</span></span>  
  
- <span data-ttu-id="8912c-183">用戶端和伺服器上，每個 WCF 方法已受到限制，以開始新的活動，然後 （在工作完成時） 之後結束新活動，並返回環境活動。</span><span class="sxs-lookup"><span data-stu-id="8912c-183">Each WCF method, both on the client and server, is bounded by beginning a new activity, then (after work is done) ending the new activity and returning to the ambient activity.</span></span>  
  
- <span data-ttu-id="8912c-184">例如，接聽連線或等候訊息的長時間 (持續) 活動，將以相對應的開始/停止資料標記來表示。</span><span class="sxs-lookup"><span data-stu-id="8912c-184">Long running (ongoing) activities such as listening for connections or waiting for messages are represented by corresponding start/stop markers.</span></span>  
  
- <span data-ttu-id="8912c-185">因為接收或處理訊息而觸發的活動將由追蹤介面來表示。</span><span class="sxs-lookup"><span data-stu-id="8912c-185">Activities triggered by the receipt or processing of a message are represented by trace boundaries.</span></span>  
  
- <span data-ttu-id="8912c-186">活動就是代表活動，但不一定代表物件。</span><span class="sxs-lookup"><span data-stu-id="8912c-186">Activities represent activities, not necessarily objects.</span></span> <span data-ttu-id="8912c-187">活動應該解讀為 「 它的發生時。</span><span class="sxs-lookup"><span data-stu-id="8912c-187">An activity should be interpreted as "this was happening when .</span></span> <span data-ttu-id="8912c-188">。</span><span class="sxs-lookup"><span data-stu-id="8912c-188">.</span></span> <span data-ttu-id="8912c-189">。</span><span class="sxs-lookup"><span data-stu-id="8912c-189">.</span></span> <span data-ttu-id="8912c-190">(已發出有意義的追蹤)」。</span><span class="sxs-lookup"><span data-stu-id="8912c-190">(meaningful trace emission occurred)."</span></span>
