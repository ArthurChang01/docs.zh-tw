# 端對端追蹤
「端對端 \(e2e\) 追蹤」可讓開發人員追蹤 [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] 基礎結構中程式碼執行的情況，以便調查程式碼路徑失敗的原因，或是針對容量規劃與效能分析提供詳細的追蹤。  [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] 提供三種相互關聯的機制，協助診斷錯誤的成因：活動、傳輸和傳播。  
  
 如需端對端追蹤案例及其個別活動與追蹤設計的清單，請參閱[端對端追蹤案例](../../../../../docs/framework/wcf/diagnostics/tracing/end-to-end-tracing-scenarios.md)。  
  
## 本章節內容  
 [活動](../../../../../docs/framework/wcf/diagnostics/tracing/activity.md)：描述 [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] 追蹤模型中的活動追蹤。  
  
 [傳輸](../../../../../docs/framework/wcf/diagnostics/tracing/transfer.md)：描述 [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] 追蹤模型中的傳輸，並且使用傳輸讓端點內的活動相互關聯。  
  
 [傳播](../../../../../docs/framework/wcf/diagnostics/tracing/propagation.md)：描述 [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] 追蹤模型中的活動傳播，並且使用傳播跨端點讓活動相互關聯。  
  
 [追蹤類型摘要](../../../../../docs/framework/wcf/diagnostics/tracing/trace-type-summary.md)  
  
 提供所有活動追蹤類型的摘要
