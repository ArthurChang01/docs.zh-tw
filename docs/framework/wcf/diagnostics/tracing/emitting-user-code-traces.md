## <a name="creating-activities"></a><span data-ttu-id="c679c-109">建立活動</span><span class="sxs-lookup"><span data-stu-id="c679c-109">Creating Activities</span></span>

<span data-ttu-id="c679c-110">活動是處理的邏輯單位。</span><span class="sxs-lookup"><span data-stu-id="c679c-110">Activities are logical unit of processing.</span></span> <span data-ttu-id="c679c-111">您可以為每個主要處理單位建立一個活動，而這些處理單位是您要在其中將追蹤聚集成群組的地方。</span><span class="sxs-lookup"><span data-stu-id="c679c-111">You can create one activity for each major processing unit in which you want traces to be grouped together.</span></span> <span data-ttu-id="c679c-112">例如，您可以為每一個針對服務發出的要求建立一個活動。</span><span class="sxs-lookup"><span data-stu-id="c679c-112">For example, you can create one activity for each request to the service.</span></span> <span data-ttu-id="c679c-113">若要這麼做，請執行下列步驟。</span><span class="sxs-lookup"><span data-stu-id="c679c-113">To do so, perform the following steps.</span></span>

1. <span data-ttu-id="c679c-114">儲存範圍內的活動識別碼。</span><span class="sxs-lookup"><span data-stu-id="c679c-114">Save the activity ID in scope.</span></span>

2. <span data-ttu-id="c679c-115">建立新的活動識別碼。</span><span class="sxs-lookup"><span data-stu-id="c679c-115">Create a new activity ID.</span></span>

3. <span data-ttu-id="c679c-116">從範圍內的活動傳輸到新活動、設定範圍內的新活動，然後發出該活動的「開始追蹤」。</span><span class="sxs-lookup"><span data-stu-id="c679c-116">Transfer from the activity in scope to the new one, set the new activity in scope and emit a start trace for that activity.</span></span>

<span data-ttu-id="c679c-117">下列程式碼會示範如何執行此項作業。</span><span class="sxs-lookup"><span data-stu-id="c679c-117">The following code demonstrates how to do this.</span></span>

```csharp
Guid oldID = Trace.CorrelationManager.ActivityId;

ts.TraceTransfer(0, "transfer", traceID);
Trace.CorrelationManager.ActivityId = traceID; // Trace is static
ts.TraceEvent(TraceEventType.Start, 0, "Add request");
```

## <a name="emitting-traces-within-a-user-activity"></a><span data-ttu-id="c679c-118">在使用者活動內發出追蹤</span><span class="sxs-lookup"><span data-stu-id="c679c-118">Emitting Traces within a User Activity</span></span>

double value2 = 15.99D;
ts.TraceInformation("Client sends message to Add " + value1 + ", " + value2);
double result = client.Add(value1, value2);
ts.TraceInformation("Client receives Add response '" + result + "'");
```

## <a name="stopping-the-activities"></a><span data-ttu-id="c679c-120">停止活動</span><span class="sxs-lookup"><span data-stu-id="c679c-120">Stopping the Activities</span></span>

<span data-ttu-id="c679c-121">若要停止活動，請傳輸回到舊活動、停止目前的活動識別碼，然後重設範圍內的舊活動識別碼。</span><span class="sxs-lookup"><span data-stu-id="c679c-121">To stop the activities, transfer back to the old activity, stop the current activity id, and reset the old activity id in scope.</span></span>

```

## <a name="propagating-the-activity-id-to-a-service"></a><span data-ttu-id="c679c-123">將活動識別碼傳播至服務</span><span class="sxs-lookup"><span data-stu-id="c679c-123">Propagating the Activity ID to A Service</span></span>

<span data-ttu-id="c679c-124">如果同時在用戶端和服務的組態檔中，將其 `propagateActivity` 追蹤來源的 `true` 屬性設定為 `System.ServiceModel`，則會在用戶端定義的同一個活動中進行「加法」要求的服務處理。</span><span class="sxs-lookup"><span data-stu-id="c679c-124">If you set the `propagateActivity` attribute to `true` for the `System.ServiceModel` trace source in both the client and service configuration files, the service processing for the Add request occurs in the same activity as the one defined in the client.</span></span> <span data-ttu-id="c679c-125">如果服務定義了本身的活動和傳輸，服務追蹤就不會出現在用戶端傳播的活動中，</span><span class="sxs-lookup"><span data-stu-id="c679c-125">If the service defines its own activities and transfers, the service traces do not appear in the client-propagated activity.</span></span> <span data-ttu-id="c679c-126">而會出現在和用戶端傳播的識別碼所代表之活動相互關聯的活動中 (此相互關聯是由傳輸追蹤建立)。</span><span class="sxs-lookup"><span data-stu-id="c679c-126">Instead, they appear in an activity correlated by transfer traces to the activity whose ID is propagated by the client.</span></span>

> [!NOTE]
> <span data-ttu-id="c679c-127">如果`propagateActivity`屬性設為`true`WCF 用戶端與服務上，設定環境活動中的服務作業範圍。</span><span class="sxs-lookup"><span data-stu-id="c679c-127">If the `propagateActivity` attribute is set to `true` on both the client and service, the ambient activity in the operation scope of the service is set by WCF.</span></span>

<span data-ttu-id="c679c-128">您可以使用下列程式碼來檢查活動是否已由 WCF 所設定範圍中。</span><span class="sxs-lookup"><span data-stu-id="c679c-128">You can use the following code to check whether an activity was set in scope by WCF.</span></span>

```csharp
// Check if an activity was set in scope by WCF, if it was
// propagated from the client. If not, ( ambient activity is
// equal to Guid.Empty), create a new one.
if(Trace.CorrelationManager.ActivityId == Guid.Empty)
{
    Guid newGuid = Guid.NewGuid();
    Trace.CorrelationManager.ActivityId = newGuid;
}
// Emit your Start trace.
ts.TraceEvent(TraceEventType.Start, 0, "Add Activity");

// Emit the processing traces for that request.
serviceTs.TraceInformation("Service receives Add "
                            + n1 + ", " + n2);
// double result = n1 + n2;
serviceTs.TraceInformation("Service sends Add result" + result);

// Emit the Stop trace and exit the method scope.
ts.TraceEvent(TraceEventType.Stop, 0, "Add Activity");
// return result;
```
