# <a name="basic-resource-service"></a><span data-ttu-id="25a0d-102">基本資源服務</span><span class="sxs-lookup"><span data-stu-id="25a0d-102">Basic Resource Service</span></span>
<span data-ttu-id="25a0d-103">這個範例示範如何實作以 HTTP 為基礎的服務使用 Windows Communication Foundation (WCF) REST 程式設計模型會公開客戶的一組支援擷取、 加入、 刪除和取代作業。</span><span class="sxs-lookup"><span data-stu-id="25a0d-103">This sample demonstrates how to implement a HTTP-based service using the Windows Communication Foundation (WCF) REST Programming model that exposes a collection of customers that supports the retrieve, add, delete and replace operations.</span></span> <span data-ttu-id="25a0d-104">此範例包含 2-自我裝載的 WCF HTTP 服務 (Service.cs) 和元件的主控台應用程式 (program.cs) 建立服務並呼叫它。</span><span class="sxs-lookup"><span data-stu-id="25a0d-104">This sample consists of 2 components - a self-hosted WCF HTTP service (Service.cs) and a console application (program.cs) that creates the service and makes calls to it.</span></span>  
  
## <a name="sample-details"></a><span data-ttu-id="25a0d-105">範例詳細資料</span><span class="sxs-lookup"><span data-stu-id="25a0d-105">Sample Details</span></span>  
 <span data-ttu-id="25a0d-106">WCF 服務會以資源導向 /REST 的方式公開客戶集合。</span><span class="sxs-lookup"><span data-stu-id="25a0d-106">The WCF service exposes a collection of customers in a resource-oriented/REST manner.</span></span> <span data-ttu-id="25a0d-107">簡單地說，這牽涉到客戶集合和集合中的每個客戶都擁有唯一的 URI。</span><span class="sxs-lookup"><span data-stu-id="25a0d-107">In short, this involves having unique URIs for the collection of customers and every customer in the collection.</span></span> <span data-ttu-id="25a0d-108">此服務支援傳送集合 URI 的 HTTP `GET` 以擷取整個集合，也支援傳送集合 URI 的 HTTP `POST` 以便將新的客戶加入至集合。</span><span class="sxs-lookup"><span data-stu-id="25a0d-108">The service supports sending an HTTP `GET` at the collection URI to retrieve the entire collection and HTTP `POST` at the collection URI to add a new customer to the collection.</span></span> <span data-ttu-id="25a0d-109">此外，在個別客戶的 URI 上，它也支援 HTTP `GET` 取得客戶詳細資料、支援 HTTP `PUT` 取代客戶的詳細資料，以及支援 HTTP `DELETE` 從集合移除客戶。</span><span class="sxs-lookup"><span data-stu-id="25a0d-109">Also at the URI for an individual customer, it supports HTTP `GET` to get the customer details, HTTP `PUT` to replace the details of the customer and HTTP `DELETE` to remove the customer from the collection.</span></span> <span data-ttu-id="25a0d-110">當新客戶加入至集合時，服務會指派一個唯一的 URI 給它，並將 URI 儲存為客戶詳細資料的一部分。</span><span class="sxs-lookup"><span data-stu-id="25a0d-110">When a new customer is added to the collection, the service assigns it a unique URI and stores the URI as part of the customer’s details.</span></span> <span data-ttu-id="25a0d-111">此外，它會使用回應的 Location HTTP 標頭，與用戶端的 URI 進行通訊。</span><span class="sxs-lookup"><span data-stu-id="25a0d-111">Also, it communicates the URI to the client using the Location HTTP header of the response.</span></span>  
  
 <span data-ttu-id="25a0d-112">App.config 檔案會以預設的 <xref:System.ServiceModel.Description.WebHttpEndpoint> 設定 WCF 服務，且 <xref:System.ServiceModel.Description.WebHttpEndpoint.HelpEnabled%2A> 屬性設定為 `true`。</span><span class="sxs-lookup"><span data-stu-id="25a0d-112">The App.config file configures the WCF service with a default <xref:System.ServiceModel.Description.WebHttpEndpoint> that has the <xref:System.ServiceModel.Description.WebHttpEndpoint.HelpEnabled%2A> property set to `true`.</span></span> <span data-ttu-id="25a0d-113">如此一來，WCF 會建立自動 html 說明網頁`http://localhost:8000/Customers/help`會提供如何建構 HTTP 資訊服務，以及如何存取服務的 HTTP 回應執行個體，對客戶詳細資料的方式的範例要求以 XML 和 JSON 表示。</span><span class="sxs-lookup"><span data-stu-id="25a0d-113">As a result, the WCF creates an automatic HTML-based help page at `http://localhost:8000/Customers/help` that provides information about how to construct HTTP requests to the service and how to access the service’s HTTP response – for instance, an example of how the customer details is represented in XML and JSON.</span></span>  
  
 <span data-ttu-id="25a0d-114">以此方式公開客戶集合 (以及更普遍的任何資源) 可讓用戶端以統一的方式，使用 URI 和 HTTP `GET`、`PUT`、`DELETE` 以及 `POST` 與其互動。</span><span class="sxs-lookup"><span data-stu-id="25a0d-114">Exposing the customer collection (and more generally, any resource) in this manner allows the client to interact with it in a uniform way using URIs and HTTP `GET`, `PUT`, `DELETE` and `POST`.</span></span> <span data-ttu-id="25a0d-115">Program.cs 會示範如何使用 <xref:System.Net.HttpWebRequest> 授權此種用戶端。</span><span class="sxs-lookup"><span data-stu-id="25a0d-115">Program.cs demonstrates how such a client can be authored using <xref:System.Net.HttpWebRequest>.</span></span> <span data-ttu-id="25a0d-116">請注意，這是只有一種來存取 WCF REST 服務。</span><span class="sxs-lookup"><span data-stu-id="25a0d-116">Note that this is just one way to access a WCF REST service.</span></span> <span data-ttu-id="25a0d-117">您也可以使用其他 .NET Framework 類別 (例如 <xref:System.ServiceModel.ChannelFactory> 和 <xref:System.Net.WebClient>) 來存取服務。</span><span class="sxs-lookup"><span data-stu-id="25a0d-117">It is also possible to access the service using other .NET Framework classes like the <xref:System.ServiceModel.ChannelFactory> and <xref:System.Net.WebClient>.</span></span> <span data-ttu-id="25a0d-118">SDK 中的其他範例 (例如[基本 HTTP 服務](../../../../docs/framework/wcf/samples/basic-http-service.md)範例和[自動格式選取](../../../../docs/framework/wcf/samples/automatic-format-selection.md)範例) 示範如何使用這些類別，與 WCF 服務進行通訊。</span><span class="sxs-lookup"><span data-stu-id="25a0d-118">Other samples in the SDK (such as the [Basic HTTP Service](../../../../docs/framework/wcf/samples/basic-http-service.md) sample and the [Automatic Format Selection](../../../../docs/framework/wcf/samples/automatic-format-selection.md) sample) show how to use these classes to communicate with a WCF service.</span></span>  
  
 <span data-ttu-id="25a0d-119">此範例包含同時在主控台應用程式中執行的自我裝載服務和用戶端。</span><span class="sxs-lookup"><span data-stu-id="25a0d-119">The sample consists a self-hosted service and a client that both run within a console application.</span></span> <span data-ttu-id="25a0d-120">當主控台應用程式執行時，用戶端會對服務發出要求，然後將相關的資訊從回應寫入至主控台視窗。</span><span class="sxs-lookup"><span data-stu-id="25a0d-120">As the console application runs, the client makes requests to the service and writes the pertinent information from the responses to the console window.</span></span>
