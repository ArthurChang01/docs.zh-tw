 <span data-ttu-id="86fea-250">指定由傳輸層機制 (例如 HTTPS) 提供機密性、完整性和驗證。</span><span class="sxs-lookup"><span data-stu-id="86fea-250">Specifies that confidentiality, integrity, and authentication are provided by the transport layer mechanisms (such as HTTPS).</span></span> <span data-ttu-id="86fea-251">在使用如 HTTPS 的傳輸時，這個模式的優點在於效能會更有效率，而且易於理解 (因為普遍使用在網際網路上)。</span><span class="sxs-lookup"><span data-stu-id="86fea-251">When using a transport like HTTPS, this mode has the advantage of being efficient in its performance, and well understood because of its prevalence on the Internet.</span></span> <span data-ttu-id="86fea-252">缺點是，這種類型的安全性是個別套用在通訊路徑上的每個躍點，因此通訊容易受到「攔截式」攻擊。</span><span class="sxs-lookup"><span data-stu-id="86fea-252">The disadvantage is that this kind of security is applied separately on each hop in the communication path, making the communication susceptible to a "man in the middle" attack.</span></span>  
  
 <span data-ttu-id="86fea-253">訊息安全性模式</span><span class="sxs-lookup"><span data-stu-id="86fea-253">message security mode</span></span>  
 <span data-ttu-id="86fea-254">指定安全性提供藉由實作一或多個安全性規格，例如規格名為[Web 服務安全性：SOAP 訊息安全性](https://go.microsoft.com/fwlink/?LinkId=94684)。</span><span class="sxs-lookup"><span data-stu-id="86fea-254">Specifies that security is provided by implementing one or more of the security specifications, such as the specification named [Web Services Security: SOAP Message Security](https://go.microsoft.com/fwlink/?LinkId=94684).</span></span> <span data-ttu-id="86fea-255">每個訊息都會包含必要機制，以便在訊息傳輸期間提供安全性，並讓接收者可以偵測竄改及解密訊息。</span><span class="sxs-lookup"><span data-stu-id="86fea-255">Each message contains the necessary mechanisms to provide security during its transit, and to enable the receivers to detect tampering and to decrypt the messages.</span></span> <span data-ttu-id="86fea-256">就這個意義而言，安全性是封裝在每個訊息內，可以在多個躍點間提供端對端安全性。</span><span class="sxs-lookup"><span data-stu-id="86fea-256">In this sense, the security is encapsulated within every message, providing end-to-end security across multiple hops.</span></span> <span data-ttu-id="86fea-257">因為安全性資訊成為訊息的一部分，它也可包含多種認證包含在訊息 (這些指*宣告*)。</span><span class="sxs-lookup"><span data-stu-id="86fea-257">Because security information becomes part of the message, it is also possible to include multiple kinds of credentials with the message (these are referred to as *claims*).</span></span> <span data-ttu-id="86fea-258">這個方法的優點是，訊息可以透過任何傳輸安全地傳送，包括來源和目的地之間的多個傳輸。</span><span class="sxs-lookup"><span data-stu-id="86fea-258">This approach also has the advantage of enabling the message to travel securely over any transport, including multiple transports between its origin and destination.</span></span> <span data-ttu-id="86fea-259">缺點是使用的密碼編譯機制很複雜，因此會影響效能。</span><span class="sxs-lookup"><span data-stu-id="86fea-259">The disadvantage of this approach is the complexity of the cryptographic mechanisms employed, resulting in performance implications.</span></span>  
  
 <span data-ttu-id="86fea-260">使用訊息認證的傳輸安全性模式</span><span class="sxs-lookup"><span data-stu-id="86fea-260">transport with message credential security mode</span></span>  
 <span data-ttu-id="86fea-261">指定使用傳輸層來提供訊息的機密性、驗證和完整性，而每個訊息都包含訊息接收者所需的多個認證 (宣告)。</span><span class="sxs-lookup"><span data-stu-id="86fea-261">Specifies the use of the transport layer to provide confidentiality, authentication, and integrity of the messages, while each of the messages can contain multiple credentials (claims) required by the receivers of the message.</span></span>  
  
 <span data-ttu-id="86fea-262">WS-\*</span><span class="sxs-lookup"><span data-stu-id="86fea-262">WS-\*</span></span>  
 <span data-ttu-id="86fea-263">一組發展中 Web 服務 (WS) 規格的縮寫，例如 WCF 中所實作的 WS-Security 與 WS-ReliableMessaging 等等。</span><span class="sxs-lookup"><span data-stu-id="86fea-263">Shorthand for the growing set of Web Service (WS) specifications, such as WS-Security, WS-ReliableMessaging, and so on, that are implemented in WCF.</span></span>
