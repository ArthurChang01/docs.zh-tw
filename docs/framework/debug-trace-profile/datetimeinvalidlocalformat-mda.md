## <a name="symptom"></a><span data-ttu-id="3ad49-105">徵狀</span><span class="sxs-lookup"><span data-stu-id="3ad49-105">Symptom</span></span>

### <a name="cause"></a><span data-ttu-id="3ad49-107">原因</span><span class="sxs-lookup"><span data-stu-id="3ad49-107">Cause</span></span>  
 <span data-ttu-id="3ad49-108"><xref:System.DateTime.ToString%2A?displayProperty=nameWithType> 方法的 'z' 格式包含當地時區位移，例如，"+10:00" 表示雪梨時間。</span><span class="sxs-lookup"><span data-stu-id="3ad49-108">The 'z' format for the <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> method includes the local time zone offset, for example, "+10:00" for Sydney time.</span></span> <span data-ttu-id="3ad49-109">因此，如果 <xref:System.DateTime> 的值是當地時間，則它只會產生有意義的結果。</span><span class="sxs-lookup"><span data-stu-id="3ad49-109">As such, it will only produce a meaningful result if the value of the <xref:System.DateTime> is local.</span></span> <span data-ttu-id="3ad49-110">如果值是 UTC 時間，則 <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> 會包含當地時區位移，但不會顯示或調整時區規範。</span><span class="sxs-lookup"><span data-stu-id="3ad49-110">If the value is UTC time, <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> includes the local time zone offset, but it does not display or adjust the time zone specifier.</span></span>  
  
### <a name="resolution"></a><span data-ttu-id="3ad49-111">解決方案</span><span class="sxs-lookup"><span data-stu-id="3ad49-111">Resolution</span></span>  
 <span data-ttu-id="3ad49-112">UTC <xref:System.DateTime> 執行個體應該使用指出它們為 UTC 的方式進行格式化。</span><span class="sxs-lookup"><span data-stu-id="3ad49-112">UTC <xref:System.DateTime> instances should be formatted in a way that indicates that they are UTC.</span></span> <span data-ttu-id="3ad49-113">UTC 時間的建議格式是使用 'Z' 表示 UTC 時間：</span><span class="sxs-lookup"><span data-stu-id="3ad49-113">The recommended format for UTC times to use a 'Z' to denote UTC time:</span></span>  
  
```csharp

Serialize(myDateTime.ToString("yyyy-MM-dd'T'HH:mm:ss.fffffffZ"));  
```  
  
 <span data-ttu-id="3ad49-114">也有 "o" 格式可序列化利用正確序列化之 <xref:System.DateTime.Kind%2A> 屬性的 <xref:System.DateTime>，不論執行個體是當地時間、UTC 還是未指定都一樣：</span><span class="sxs-lookup"><span data-stu-id="3ad49-114">There is also an "o" format that serializes a <xref:System.DateTime> making use of the <xref:System.DateTime.Kind%2A> property that serializes correctly regardless of whether the instance is local, UTC, or unspecified:</span></span>  
  
```csharp
DateTime myDateTime = DateTime.UtcNow;  
Serialize(myDateTime.ToString("o"));  
```

## <a name="example"></a><span data-ttu-id="3ad49-120">範例</span><span class="sxs-lookup"><span data-stu-id="3ad49-120">Example</span></span>  
 <span data-ttu-id="3ad49-121">請以下列方式考慮使用利用 <xref:System.Xml.XmlConvert> 或 <xref:System.Data.DataSet> 類別間接序列化 UTC <xref:System.DateTime> 值的應用程式。</span><span class="sxs-lookup"><span data-stu-id="3ad49-121">Consider an application that is indirectly serializing a UTC <xref:System.DateTime> value by using the <xref:System.Xml.XmlConvert> or <xref:System.Data.DataSet> class, in the following manner.</span></span>

 <span data-ttu-id="3ad49-126">如果使用 <xref:System.Data.DataSet>，請將 <xref:System.Data.DataColumn> 物件上的 <xref:System.Data.DataColumn.DateTimeMode%2A> 屬性設定為 <xref:System.Data.DataSetDateTime.Utc>。</span><span class="sxs-lookup"><span data-stu-id="3ad49-126">If using a <xref:System.Data.DataSet>, set the <xref:System.Data.DataColumn.DateTimeMode%2A> property on the <xref:System.Data.DataColumn> object to <xref:System.Data.DataSetDateTime.Utc>.</span></span>
