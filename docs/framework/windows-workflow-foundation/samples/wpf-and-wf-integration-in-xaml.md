視窗。</span><span class="sxs-lookup"><span data-stu-id="7d587-108">The `ShowWindow` activity displays a [!INCLUDE[avalon2](../../../../includes/avalon2-md.md)] window as part of its execution logic.</span></span> <span data-ttu-id="7d587-109">視窗的 <xref:System.Activities.ActivityContext.DataContext%2A> 包含序列中宣告的變數。</span><span class="sxs-lookup"><span data-stu-id="7d587-109">The 活動中宣告的視窗控制項使用資料繫結來操作這些變數。</span><span class="sxs-lookup"><span data-stu-id="7d587-110">The controls of the window declared in the `ShowWindow` activity use data binding to manipulate those  <span data-ttu-id="7d587-124">在這個範例中值得注意的最後一點是名為 <xref:System.Workflow.ComponentModel.Serialization.MarkupExtension> 的 `DelegateActivityExtension`。</span><span class="sxs-lookup"><span data-stu-id="7d587-124">The last point of interest in this sample is a <xref:System.Workflow.ComponentModel.Serialization.MarkupExtension> called `DelegateActivityExtension`.</span></span> <span data-ttu-id="7d587-125">這個標記延伸的 `ProvideValue` 方法會傳回可叫用內嵌活動的委派。</span><span class="sxs-lookup"><span data-stu-id="7d587-125">The `ProvideValue` method of this markup extension returns a delegate that invokes an embedded activity.</span></span> <span data-ttu-id="7d587-126">在包含 [!INCLUDE[avalon2](../../../../includes/avalon2-md.md)] 資料內容和任何 `x:Name` 範圍中值的環境中執行這個活動。</span><span class="sxs-lookup"><span data-stu-id="7d587-126">This activity runs in an environment that includes the [!INCLUDE[avalon2](../../../../includes/avalon2-md.md)] data context and any `x:Name` values in scope.</span></span> <span data-ttu-id="7d587-127">在 `GenericInvoke` 方法中，這個環境是透過 <xref:System.Activities.Hosting.SymbolResolver> 延伸提供給活動。</span><span class="sxs-lookup"><span data-stu-id="7d587-127">In the `GenericInvoke` method, this environment is provided to the activity through a <xref:System.Activities.Hosting.SymbolResolver> extension.</span></span> <span data-ttu-id="7d587-128">這個延伸會加入至 <xref:System.Activities.WorkflowInvoker>，後者可在標記延伸的委派叫用時用來叫用內嵌活動。</span><span class="sxs-lookup"><span data-stu-id="7d587-128">This extension is added to a <xref:System.Activities.WorkflowInvoker> that is then used to invoke the embedded activity whenever the markup extension’s delegate is invoked.</span></span>
