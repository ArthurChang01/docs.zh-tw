## <a name="remarks"></a><span data-ttu-id="66517-108">備註</span><span class="sxs-lookup"><span data-stu-id="66517-108">Remarks</span></span>  
 <span data-ttu-id="66517-109">在 [MS-XAML] 中，`x:Uid` 會定義為指示詞。</span><span class="sxs-lookup"><span data-stu-id="66517-109">In [MS-XAML], `x:Uid` is defined as a directive.</span></span> <span data-ttu-id="66517-110">如需詳細資訊，請參閱[\[MS-XAML\] 區段 5.3.6](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10))。</span><span class="sxs-lookup"><span data-stu-id="66517-110">For more information, see [\[MS-XAML\] Section 5.3.6](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10)).</span></span>  
  
 <span data-ttu-id="66517-111">`x:Uid` 與 `x:Name` 的不同之處在于，這兩者都是因為所述的 XAML 當地語系化案例，因此用於當地語系化的識別碼與 `x:Name`的程式設計模型含意沒有相依性。</span><span class="sxs-lookup"><span data-stu-id="66517-111">`x:Uid` is discrete from `x:Name` both because of the stated XAML localization scenario and so that identifiers that are used for localization have no dependencies on the programming model implications of `x:Name`.</span></span> <span data-ttu-id="66517-112">此外，`x:Name` 是由 XAML 名稱範圍所控制;不過，`x:Uid` 不受任何 XAML 語言定義的唯一性強制概念所規範。</span><span class="sxs-lookup"><span data-stu-id="66517-112">Also, `x:Name` is governed by the XAML namescope; however, `x:Uid` is not governed by any XAML language defined concept of uniqueness enforcement.</span></span> <span data-ttu-id="66517-113">XAML 處理器的廣泛意義（不屬於當地語系化進程的處理器）不應該強制執行 `x:Uid` 值的唯一性。</span><span class="sxs-lookup"><span data-stu-id="66517-113">XAML processors in a broad sense (processors that are not part of the localization process) are not expected to enforce uniqueness of `x:Uid` values.</span></span> <span data-ttu-id="66517-114">這項責任在概念上是在值的建立者上。</span><span class="sxs-lookup"><span data-stu-id="66517-114">That responsibility is conceptually on the originator of the values.</span></span> <span data-ttu-id="66517-115">在單一 XAML 來源中，`x:Uid` 值的唯一性，對於值的取用者而言是合理的，例如專用的全球化處理常式或工具。</span><span class="sxs-lookup"><span data-stu-id="66517-115">The expectation of uniqueness of `x:Uid` values within a single XAML source is reasonable for consumers of the values, such as dedicated globalization processes or tools.</span></span> <span data-ttu-id="66517-116">一般的唯一性模型是，在代表 XAML 的 XML 編碼檔案中，`x:Uid` 值是唯一的。</span><span class="sxs-lookup"><span data-stu-id="66517-116">The typical uniqueness model is that `x:Uid` values are unique within an XML-encoded file that represents XAML.</span></span>  
  
 <span data-ttu-id="66517-117">對於特定 XAML 架構具有重要知識的工具，可以選擇僅針對真正可當地語系化的字串套用 `x:Uid`，而不是針對在標記中遇到文字字串值的所有情況。</span><span class="sxs-lookup"><span data-stu-id="66517-117">Tools that have significant knowledge of a particular XAML schema can choose to apply `x:Uid` only for true localizable strings, instead of for all cases where a text string value is encountered in markup.</span></span>  
  
 <span data-ttu-id="66517-118">架構可以將屬性 <xref:System.Windows.Markup.UidPropertyAttribute> 套用至定義類型，藉此指定物件模型中的特定屬性，使其成為 `x:Uid` 的別名。</span><span class="sxs-lookup"><span data-stu-id="66517-118">Frameworks can specify a particular property in their object model to be an alias for `x:Uid` by applying the attribute <xref:System.Windows.Markup.UidPropertyAttribute> to the defining type.</span></span> <span data-ttu-id="66517-119">如果架構指定特定屬性，則在相同物件上同時指定 `x:Uid` 和別名成員是不正確。</span><span class="sxs-lookup"><span data-stu-id="66517-119">If a framework specifies a particular property, it is not valid to specify both `x:Uid` and the aliased member on the same object.</span></span> <span data-ttu-id="66517-120">如果同時指定 `x:Uid` 和別名成員，則 .NET Framework XAML 服務 API 通常會在此情況下擲回 <xref:System.Xaml.XamlDuplicateMemberException>。</span><span class="sxs-lookup"><span data-stu-id="66517-120">If both `x:Uid` and the aliased member are specified, the .NET Framework XAML Services API typically throws <xref:System.Xaml.XamlDuplicateMemberException> for this case.</span></span>
