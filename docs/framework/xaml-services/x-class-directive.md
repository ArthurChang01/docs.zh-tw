 <span data-ttu-id="f408c-121">在使用 `x:Class`的現有程式設計模型中，`x:Class` 是選擇性的，因為具有不具有程式碼後置的 XAML 頁面是完全有效的。</span><span class="sxs-lookup"><span data-stu-id="f408c-121">In existing programming models that use `x:Class`, `x:Class` is optional in the sense that it is entirely valid to have a XAML page that has no code-behind.</span></span> <span data-ttu-id="f408c-122">不過，這項功能會與使用 XAML 的架構所執行的組建動作互動。</span><span class="sxs-lookup"><span data-stu-id="f408c-122">However, that capability interacts with the build actions as implemented by frameworks that use XAML.</span></span> <span data-ttu-id="f408c-123">`x:Class` 功能也會受到 XAML 指定內容的各種分類在應用程式模型和對應的組建動作中的角色所影響。</span><span class="sxs-lookup"><span data-stu-id="f408c-123">`x:Class` capability is also influenced by the roles that various classifications of XAML-specified content have in an application model and in the corresponding build actions.</span></span> <span data-ttu-id="f408c-124">如果您的 XAML 宣告事件處理屬性值或具現化自訂專案，其中定義的類別位於程式碼後置類別中，您必須提供 `x:Class` 指示詞參考（或[x:Subclass](x-subclass-directive.md)）給程式碼後置的適當類別。</span><span class="sxs-lookup"><span data-stu-id="f408c-124">If your XAML declares event-handling attribute values or instantiates custom elements where the defining classes are in the code-behind class, you have to provide the `x:Class` directive reference (or [x:Subclass](x-subclass-directive.md)) to the appropriate class for code-behind.</span></span>  
  
 <span data-ttu-id="f408c-125">`x:Class` 指示詞的值必須是指定類別之完整名稱的字串，但不含任何元件資訊（相當於 <xref:System.Type.FullName%2A?displayProperty=nameWithType>）。</span><span class="sxs-lookup"><span data-stu-id="f408c-125">The value of the `x:Class` directive must be a string that specifies the fully qualified name of a class but without any assembly information (equivalent to the <xref:System.Type.FullName%2A?displayProperty=nameWithType>).</span></span> <span data-ttu-id="f408c-126">對於簡單的應用程式，如果程式碼後置也以該方式結構化，您就可以省略 CLR 命名空間資訊（在類別層級開始程式碼定義）。</span><span class="sxs-lookup"><span data-stu-id="f408c-126">For simple applications, you can omit CLR namespace information if the code-behind is also structured in that manner (code definition starts at the class level).</span></span>

 <span data-ttu-id="f408c-131">這項 `x:Class` 屬性的解讀僅適用于以 CLR 為基礎的 XAML 執行，特別是 .NET Framework XAML 服務。</span><span class="sxs-lookup"><span data-stu-id="f408c-131">This interpretation of the `x:Class` attribute applies only to a CLR-based XAML implementation, in particular to .NET Framework XAML Services.</span></span> <span data-ttu-id="f408c-132">其他不是以 CLR 為基礎且未使用 .NET Framework XAML 服務的 XAML 執行，可能會使用不同的解析公式來連接 XAML 標記和支援執行時間程式碼。</span><span class="sxs-lookup"><span data-stu-id="f408c-132">Other XAML implementations that are not based on CLR and that do not use .NET Framework XAML Services might use a different resolution formula for connecting XAML markup and backing run-time code.</span></span> <span data-ttu-id="f408c-133">如需更多 `x:Class`一般解讀的詳細資訊，請參閱[\[MS-XAML\]](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10))。</span><span class="sxs-lookup"><span data-stu-id="f408c-133">For more information about more general interpretations of `x:Class`, see [\[MS-XAML\]](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10)).</span></span>
