 <span data-ttu-id="5d25d-113">XAML 命名空間與 XML 命名空間的區別在於，XML 命名空間可能會用來參考架構，也可能用來區別實體。</span><span class="sxs-lookup"><span data-stu-id="5d25d-113">The distinction of a XAML namespace versus an XML namespace is that an XML namespace might be used to reference a schema, or might be used to simply differentiate entities.</span></span> <span data-ttu-id="5d25d-114">對於 XAML，XAML 中所使用的類型和成員最終必須解析成支援類型，而 XML 架構概念不適用於這項功能。</span><span class="sxs-lookup"><span data-stu-id="5d25d-114">For XAML, the types and members as used in XAML must ultimately be resolved to backing types, and XML schema concepts do not apply well to this capability.</span></span> <span data-ttu-id="5d25d-115">XAML 命名空間包含 XAML 架構內容必須可用才能執行此類型對應的資訊。</span><span class="sxs-lookup"><span data-stu-id="5d25d-115">The XAML namespace contains information that the XAML schema context must have available in order to perform this type mapping.</span></span>

 <span data-ttu-id="5d25d-125">此識別碼可以是[XML 1.0 規格中 W3C 命名空間](https://www.w3.org/TR/REC-xml-names/)所允許的任何字串。</span><span class="sxs-lookup"><span data-stu-id="5d25d-125">The identifier can be any string as allowed by the [W3C Namespaces in XML 1.0 specification](https://www.w3.org/TR/REC-xml-names/).</span></span> <span data-ttu-id="5d25d-126">依照慣例，XML 命名空間或 XAML 命名空間的識別碼通常是以 URI 形式提供，通常是以通訊協定限定的絕對 URI。</span><span class="sxs-lookup"><span data-stu-id="5d25d-126">By convention, identifiers for either XML namespaces or XAML namespaces are often given in URI form, typically as a protocol-qualified absolute URI.</span></span> <span data-ttu-id="5d25d-127">通常，定義特定 XAML 詞彙的版本資訊會隱含為路徑字串的一部分。</span><span class="sxs-lookup"><span data-stu-id="5d25d-127">Often, version information that defines a particular XAML vocabulary is implied as part of the path string.</span></span> <span data-ttu-id="5d25d-128">XAML 命名空間會在 XML URI 慣例以外新增額外的識別碼慣例。</span><span class="sxs-lookup"><span data-stu-id="5d25d-128">XAML namespaces add an additional identifier convention beyond the XML URI convention.</span></span> <span data-ttu-id="5d25d-129">對於 XAML 命名空間，識別碼會傳達 XAML 架構內容所需的資訊，以便解析指定為該 XAML 命名空間下之元素的類型，或將屬性解析為成員。</span><span class="sxs-lookup"><span data-stu-id="5d25d-129">For XAML namespaces, the identifier communicates information that is needed by a XAML schema context in order to resolve the types that are specified as elements under that XAML namespace, or to resolve attributes to members.</span></span>
