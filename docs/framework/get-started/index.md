## <a name="Introducing"></a> <span data-ttu-id="723fe-107">什麼是 .NET Framework？</span><span class="sxs-lookup"><span data-stu-id="723fe-107">What is the .NET Framework?</span></span>

<span data-ttu-id="723fe-108">.NET Framework 是適用於 Windows 的 受控執行環境，可為執行中的應用程式提供多樣的服務。</span><span class="sxs-lookup"><span data-stu-id="723fe-108">The .NET Framework is a managed execution environment for Windows that provides a variety of services to its running apps.</span></span> <span data-ttu-id="723fe-109">其由兩個主要元件組成：通用語言執行平台 (CLR) 和 .NET Framework 類別庫，前者是負責處理執行中應用程式的執行引擎，後者提供通過測試、可重複使用的程式碼程式庫，讓開發人員可從自己的應用程式中呼叫。</span><span class="sxs-lookup"><span data-stu-id="723fe-109">It consists of two major components: the common language runtime (CLR), which is the execution engine that handles running apps, and the .NET Framework Class Library, which provides a library of tested, reusable code that developers can call from their own apps.</span></span> <span data-ttu-id="723fe-110">.NET Framework 提供給執行中應用程式的服務包括：</span><span class="sxs-lookup"><span data-stu-id="723fe-110">The services that the .NET Framework provides to running apps include the following:</span></span>

- <span data-ttu-id="723fe-111">記憶體管理。</span><span class="sxs-lookup"><span data-stu-id="723fe-111">Memory management.</span></span> <span data-ttu-id="723fe-112">在許多程式設計語言中，程式設計人員負責配置和釋放記憶體，以及處理物件存留期。</span><span class="sxs-lookup"><span data-stu-id="723fe-112">In many programming languages, programmers are responsible for allocating and releasing memory and for handling object lifetimes.</span></span> <span data-ttu-id="723fe-113">在 .NET Framework 應用程式中，CLR 會代表應用程式提供這些服務。</span><span class="sxs-lookup"><span data-stu-id="723fe-113">In .NET Framework apps, the CLR provides these services on behalf of the app.</span></span>

- <span data-ttu-id="723fe-114">一般型別系統。</span><span class="sxs-lookup"><span data-stu-id="723fe-114">A common type system.</span></span> <span data-ttu-id="723fe-115">在傳統的程式語言中，基本型別是由編譯器所定義，這會讓跨語言互通性變得很複雜。</span><span class="sxs-lookup"><span data-stu-id="723fe-115">In traditional programming languages, basic types are defined by the compiler, which complicates cross-language interoperability.</span></span> <span data-ttu-id="723fe-116">在 .NET Framework 中，基底型別是由 .NET Framework 型別系統所定義，可在所有以 .NET Framework 為目標的語言之間通用。</span><span class="sxs-lookup"><span data-stu-id="723fe-116">In the .NET Framework, basic types are defined by the .NET Framework type system and are common to all languages that target the .NET Framework.</span></span>

- <span data-ttu-id="723fe-117">廣泛類別庫。</span><span class="sxs-lookup"><span data-stu-id="723fe-117">An extensive class library.</span></span> <span data-ttu-id="723fe-118">程式設計人員不再需要撰寫大量的程式碼來處理常見的低階程式設計作業，而能夠使用 .NET Framework 類別庫中可立即存取的類型程式庫及其成員來完成這項工作。</span><span class="sxs-lookup"><span data-stu-id="723fe-118">Instead of having to write vast amounts of code to handle common low-level programming operations, programmers use a readily accessible library of types and their members from the .NET Framework Class Library.</span></span>

- <span data-ttu-id="723fe-119">開發架構和技術。</span><span class="sxs-lookup"><span data-stu-id="723fe-119">Development frameworks and technologies.</span></span> <span data-ttu-id="723fe-120">.NET Framework 包含特定應用程式開發領域所需的程式庫，例如 Web 應用程式所需的 ASP.NET、資料存取所需的 ADO.NET、服務導向應用程式所需的 Windows Communication Foundation，以及 Windows 傳統型應用程式所需的 Windows Presentation Foundation。</span><span class="sxs-lookup"><span data-stu-id="723fe-120">The .NET Framework includes libraries for specific areas of app development, such as ASP.NET for web apps, ADO.NET for data access, Windows Communication Foundation for service-oriented apps, and Windows Presentation Foundation for Windows desktop apps.</span></span>

- <span data-ttu-id="723fe-121">語言互通性。</span><span class="sxs-lookup"><span data-stu-id="723fe-121">Language interoperability.</span></span> <span data-ttu-id="723fe-122">以 .NET Framework 為目標的語言編譯器會發出名為通用中間語言 (CIL) 的中繼程式碼，這個程式碼接著會在執行階段由通用語言執行平台進行編譯。</span><span class="sxs-lookup"><span data-stu-id="723fe-122">Language compilers that target the .NET Framework emit an intermediate code named Common Intermediate Language (CIL), which, in turn, is compiled at runtime by the common language runtime.</span></span> <span data-ttu-id="723fe-123">有了此功能，使用某一種語言撰寫的常式就能供其他語言存取，而程式設計人員也能使用自己慣用的語言專心建立應用程式。</span><span class="sxs-lookup"><span data-stu-id="723fe-123">With this feature, routines written in one language are accessible to other languages, and programmers focus on creating apps in their preferred languages.</span></span>

- <span data-ttu-id="723fe-124">版本相容性。</span><span class="sxs-lookup"><span data-stu-id="723fe-124">Version compatibility.</span></span> <span data-ttu-id="723fe-125">在極少數例外狀況下，使用某一特定 .NET Framework 版本開發的應用程式可不經修改直接在較新的版本上執行。</span><span class="sxs-lookup"><span data-stu-id="723fe-125">With rare exceptions, apps that are developed by using a particular version of the .NET Framework run without modification on a later version.</span></span>

- <span data-ttu-id="723fe-126">並存執行。</span><span class="sxs-lookup"><span data-stu-id="723fe-126">Side-by-side execution.</span></span> <span data-ttu-id="723fe-127">.NET Framework 允許同一部電腦上存在多個版本的 Common Language Runtime，藉此協助解決版本衝突。</span><span class="sxs-lookup"><span data-stu-id="723fe-127">The .NET Framework helps resolve version conflicts by allowing multiple versions of the common language runtime to exist on the same computer.</span></span> <span data-ttu-id="723fe-128">這表示，多個應用程式版本不僅可共存，應用程式也可以建置時使用的 .NET Framework 版本上執行。</span><span class="sxs-lookup"><span data-stu-id="723fe-128">This means that multiple versions of apps can coexist and that an app can run on the version of the .NET Framework with which it was built.</span></span> <span data-ttu-id="723fe-129">並存執行適用於.NET Framework 版本群組 1.0/1.1、2.0/3.0/3.5 和 4/4.5.x/4.6.x/4.7.x/4.8。</span><span class="sxs-lookup"><span data-stu-id="723fe-129">Side-by-side execution applies to the .NET Framework version groups 1.0/1.1, 2.0/3.0/3.5, and 4/4.5.x/4.6.x/4.7.x/4.8.</span></span>

- <span data-ttu-id="723fe-130">多目標。</span><span class="sxs-lookup"><span data-stu-id="723fe-130">Multitargeting.</span></span> <span data-ttu-id="723fe-131">將目標設為 [.NET Standard](../../standard/net-standard.md)，開發人員就能建立可在該標準版本所支援的多個 .NET Framework 平台上運作的類別庫。</span><span class="sxs-lookup"><span data-stu-id="723fe-131">By targeting [.NET Standard](../../standard/net-standard.md), developers create class libraries that work on multiple .NET Framework platforms supported by that version of the standard.</span></span> <span data-ttu-id="723fe-132">例如，以 .NET Standard 2.0 為目標的類別庫，可由目標為 .NET Framework 4.6.1、.NET Core 2.0 和 UWP 10.0.16299 的應用程式使用。</span><span class="sxs-lookup"><span data-stu-id="723fe-132">For example, libraries that target the .NET Standard 2.0 can be used by apps that target the .NET Framework 4.6.1, .NET Core 2.0, and UWP 10.0.16299.</span></span>
