# <a name="reflection-in-the-net-framework-for-windows-store-apps"></a><span data-ttu-id="108e8-102">適用於 Windows 市集應用程式之 .NET Framework 中的反映</span><span class="sxs-lookup"><span data-stu-id="108e8-102">Reflection in the .NET Framework for Windows Store Apps</span></span>
<span data-ttu-id="108e8-103">從 [!INCLUDE[net_v45](../../../includes/net-v45-md.md)] 開始，.NET Framework 包含一組反映類型和成員，以便用於 [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] 應用程式。</span><span class="sxs-lookup"><span data-stu-id="108e8-103">Starting with the [!INCLUDE[net_v45](../../../includes/net-v45-md.md)], the .NET Framework includes a set of reflection types and members for use in [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] apps.</span></span> <span data-ttu-id="108e8-104">這些類型和成員提供於完整 .NET Framework，以及[適用於 Windows 市集應用程式的 .NET](https://go.microsoft.com/fwlink/?LinkID=225700)。</span><span class="sxs-lookup"><span data-stu-id="108e8-104">These types and members are available in the full .NET Framework as well as in the [.NET for Windows Store apps](https://go.microsoft.com/fwlink/?LinkID=225700).</span></span> <span data-ttu-id="108e8-105">本文件說明這些與其對應項目在 .NET Framework 4 和舊版之間的主要差異。</span><span class="sxs-lookup"><span data-stu-id="108e8-105">This document explains the major differences between these and their counterparts in the .NET Framework 4 and earlier versions.</span></span>  
  
 <span data-ttu-id="108e8-106">如果您要建立 [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] 應用程式，必須使用 [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] 中的反映類型和成員。</span><span class="sxs-lookup"><span data-stu-id="108e8-106">If you are creating a [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] app, you must use the reflection types and members in the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)].</span></span> <span data-ttu-id="108e8-107">這些類型和成員也可用於 (但不是必要) 桌面應用程式，因此您可以對這兩種應用程式使用相同的程式碼。</span><span class="sxs-lookup"><span data-stu-id="108e8-107">These types and members are also available, but not required, for use in desktop apps, so you can use the same code for both types of apps.</span></span>  
  
## <a name="typeinfo-and-assembly-loading"></a><span data-ttu-id="108e8-108">TypeInfo 和組件載入</span><span class="sxs-lookup"><span data-stu-id="108e8-108">TypeInfo and Assembly Loading</span></span>  
 <span data-ttu-id="108e8-109">在 [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] 中，<xref:System.Reflection.TypeInfo> 類別包含 .NET Framework 4 <xref:System.Type> 類別的某些功能。</span><span class="sxs-lookup"><span data-stu-id="108e8-109">In the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)], the <xref:System.Reflection.TypeInfo> class contains some of the functionality of the .NET Framework 4 <xref:System.Type> class.</span></span> <span data-ttu-id="108e8-110"><xref:System.Type> 物件代表類型定義的參考，而 <xref:System.Reflection.TypeInfo> 物件則代表類型定義本身。</span><span class="sxs-lookup"><span data-stu-id="108e8-110">A <xref:System.Type> object represents a reference to a type definition, whereas a <xref:System.Reflection.TypeInfo> object represents the type definition itself.</span></span> <span data-ttu-id="108e8-111">這可讓您操作 <xref:System.Type> 物件，而不一定需要執行階段載入它們所參考的組件。</span><span class="sxs-lookup"><span data-stu-id="108e8-111">This enables you to manipulate <xref:System.Type> objects without necessarily requiring the runtime to load the assembly they reference.</span></span> <span data-ttu-id="108e8-112">取得相關聯的 <xref:System.Reflection.TypeInfo> 物件會強制載入組件。</span><span class="sxs-lookup"><span data-stu-id="108e8-112">Getting the associated <xref:System.Reflection.TypeInfo> object forces the assembly to load.</span></span>  
  
 <span data-ttu-id="108e8-113"><xref:System.Reflection.TypeInfo> 包含許多 <xref:System.Type> 上可用的成員，而 [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] 中的許多反映屬性會傳回 <xref:System.Reflection.TypeInfo> 物件的集合。</span><span class="sxs-lookup"><span data-stu-id="108e8-113"><xref:System.Reflection.TypeInfo> contains many of the members available on <xref:System.Type>, and many of the reflection properties in the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] return collections of <xref:System.Reflection.TypeInfo> objects.</span></span> <span data-ttu-id="108e8-114">若要從 <xref:System.Type> 物件取得 <xref:System.Reflection.TypeInfo> 物件，請使用 <xref:System.Reflection.IReflectableType.GetTypeInfo%2A> 方法。</span><span class="sxs-lookup"><span data-stu-id="108e8-114">To get a <xref:System.Reflection.TypeInfo> object from a <xref:System.Type> object, use the <xref:System.Reflection.IReflectableType.GetTypeInfo%2A> method.</span></span>  
  
## <a name="query-methods"></a><span data-ttu-id="108e8-115">查詢方法</span><span class="sxs-lookup"><span data-stu-id="108e8-115">Query Methods</span></span>  
 <span data-ttu-id="108e8-116">在 [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] 中，請使用會傳回 <xref:System.Collections.Generic.IEnumerable%601> 集合的反映屬性，而不要使用傳回陣列的方法。</span><span class="sxs-lookup"><span data-stu-id="108e8-116">In the [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)], you use the reflection properties that return <xref:System.Collections.Generic.IEnumerable%601> collections instead of methods that return arrays.</span></span> <span data-ttu-id="108e8-117">反映內容可以針對大型組件或類型實作這些集合的消極式周遊。</span><span class="sxs-lookup"><span data-stu-id="108e8-117">Reflection contexts can implement lazy traversal of these collections for large assemblies or types.</span></span>  
  
 <span data-ttu-id="108e8-118">反映屬性只傳回特定物件上宣告的方法，而不會周遊繼承樹狀結構。</span><span class="sxs-lookup"><span data-stu-id="108e8-118">The reflection properties return only the declared methods on a particular object instead of traversing the inheritance tree.</span></span> <span data-ttu-id="108e8-119">此外，它們不會使用 <xref:System.Reflection.BindingFlags> 參數進行篩選。</span><span class="sxs-lookup"><span data-stu-id="108e8-119">Moreover, they do not use <xref:System.Reflection.BindingFlags> parameters for filtering.</span></span> <span data-ttu-id="108e8-120">相反地，篩選發生在使用者程式碼中，對傳回的集合使用 LINQ 查詢。</span><span class="sxs-lookup"><span data-stu-id="108e8-120">Instead, filtering takes place in user code, by using LINQ queries on the returned collections.</span></span> <span data-ttu-id="108e8-121">對於因執行階段而產生的反映物件 (例如，因為 `typeof(Object)`)，周遊繼承樹狀結構最佳的執行方式是使用 <xref:System.Reflection.RuntimeReflectionExtensions> 類別的 helper 方法。</span><span class="sxs-lookup"><span data-stu-id="108e8-121">For reflection objects that originate with the runtime (for example, as the result of `typeof(Object)`), traversing the inheritance tree is best accomplished by using the helper methods of the <xref:System.Reflection.RuntimeReflectionExtensions> class.</span></span> <span data-ttu-id="108e8-122">來自自訂反映內容之物件的取用者不能使用這些方法，而且必須自行周遊繼承樹狀結構。</span><span class="sxs-lookup"><span data-stu-id="108e8-122">Consumers of objects from customized reflection contexts cannot use these methods, and must traverse the inheritance tree themselves.</span></span>  
  
## <a name="restrictions"></a><span data-ttu-id="108e8-123">限制</span><span class="sxs-lookup"><span data-stu-id="108e8-123">Restrictions</span></span>  
 <span data-ttu-id="108e8-124">在 [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] 應用程式中，對於某些 .NET Framework 類型和成員的存取會受到限制。</span><span class="sxs-lookup"><span data-stu-id="108e8-124">In a [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] app, access to some .NET Framework types and members is restricted.</span></span> <span data-ttu-id="108e8-125">例如，您不能藉由使用 <xref:System.Reflection.MethodInfo> 物件，呼叫未包含在 [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] 的 .NET Framework 方法。</span><span class="sxs-lookup"><span data-stu-id="108e8-125">For example, you cannot call .NET Framework methods that are not included in [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)], by using a <xref:System.Reflection.MethodInfo> object.</span></span> <span data-ttu-id="108e8-126">此外，在 [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] 應用程式的內容中不被視為安全的特定類型和成員會被封鎖，就如同 <xref:System.Runtime.InteropServices.Marshal> 和 <xref:System.Runtime.InteropServices.WindowsRuntime.WindowsRuntimeMarshal> 成員一樣。</span><span class="sxs-lookup"><span data-stu-id="108e8-126">In addition, certain types and members that are not considered safe within the context of a [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] app are blocked, as are <xref:System.Runtime.InteropServices.Marshal> and <xref:System.Runtime.InteropServices.WindowsRuntime.WindowsRuntimeMarshal> members.</span></span> <span data-ttu-id="108e8-127">此限制只會影響 .NET Framework 類型和成員；您可以如常呼叫您的程式碼或協力廠商程式碼。</span><span class="sxs-lookup"><span data-stu-id="108e8-127">This restriction affects only .NET Framework types and members; you can call your code or third-party code as you normally would.</span></span>
