# <a name="assemblies-in-the-common-language-runtime"></a><span data-ttu-id="f4806-102">Common Language Runtime 中的組件</span><span class="sxs-lookup"><span data-stu-id="f4806-102">Assemblies in the Common Language Runtime</span></span>
<span data-ttu-id="f4806-103">組件是 .NET Framework 應用程式的建置組塊；它們構成部署、版本控制、重複使用、啟用範圍和安全性權限的基礎單位。</span><span class="sxs-lookup"><span data-stu-id="f4806-103">Assemblies are the building blocks of .NET Framework applications; they form the fundamental unit of deployment, version control, reuse, activation scoping, and security permissions.</span></span> <span data-ttu-id="f4806-104">組件是建置來共同運作及構成一個功能邏輯單位的型別與資源集合。</span><span class="sxs-lookup"><span data-stu-id="f4806-104">An assembly is a collection of types and resources that are built to work together and form a logical unit of functionality.</span></span> <span data-ttu-id="f4806-105">組件為通用語言執行平台提供了感知型別實作所需的資訊。</span><span class="sxs-lookup"><span data-stu-id="f4806-105">An assembly provides the common language runtime with the information it needs to be aware of type implementations.</span></span> <span data-ttu-id="f4806-106">對於執行階段而言，型別不會存在於組件的內容以外。</span><span class="sxs-lookup"><span data-stu-id="f4806-106">To the runtime, a type does not exist outside the context of an assembly.</span></span>  
  
 <span data-ttu-id="f4806-107">組件可以執行以下功能：</span><span class="sxs-lookup"><span data-stu-id="f4806-107">An assembly performs the following functions:</span></span>  
  
- <span data-ttu-id="f4806-108">組件包含通用語言執行平台執行的程式碼。</span><span class="sxs-lookup"><span data-stu-id="f4806-108">It contains code that the common language runtime executes.</span></span> <span data-ttu-id="f4806-109">如果組件沒有相關聯的組件資訊清單，則不會執行可攜式執行檔 (PE) 中的 Microsoft 中繼語言 (MSIL) 程式碼。</span><span class="sxs-lookup"><span data-stu-id="f4806-109">Microsoft intermediate language (MSIL) code in a portable executable (PE) file will not be executed if it does not have an associated assembly manifest.</span></span> <span data-ttu-id="f4806-110">請注意，每個組件只能有一個進入點 (`DllMain`、`WinMain` 或 `Main`)。</span><span class="sxs-lookup"><span data-stu-id="f4806-110">Note that each assembly can have only one entry point (that is, `DllMain`, `WinMain`, or `Main`).</span></span>  
  
- <span data-ttu-id="f4806-111">組件可構成安全性界限。</span><span class="sxs-lookup"><span data-stu-id="f4806-111">It forms a security boundary.</span></span> <span data-ttu-id="f4806-112">組件是要求和授與權限的單位。</span><span class="sxs-lookup"><span data-stu-id="f4806-112">An assembly is the unit at which permissions are requested and granted.</span></span> <span data-ttu-id="f4806-113">如需套用至組件之安全性界限的詳細資訊，請參閱[組件安全性考量](../../../docs/framework/app-domains/assembly-security-considerations.md)。</span><span class="sxs-lookup"><span data-stu-id="f4806-113">For more information about security boundaries as they apply to assemblies, see [Assembly Security Considerations](../../../docs/framework/app-domains/assembly-security-considerations.md).</span></span>  
  
- <span data-ttu-id="f4806-114">組件可構成型別界限。</span><span class="sxs-lookup"><span data-stu-id="f4806-114">It forms a type boundary.</span></span> <span data-ttu-id="f4806-115">每種型別的識別都包含該型別所在之組件的名稱。</span><span class="sxs-lookup"><span data-stu-id="f4806-115">Every type's identity includes the name of the assembly in which it resides.</span></span> <span data-ttu-id="f4806-116">在某個組件範圍中載入的型別 `MyType` 與在另一個組件範圍中載入的型別 `MyType` 不同。</span><span class="sxs-lookup"><span data-stu-id="f4806-116">A type called `MyType` that is loaded in the scope of one assembly is not the same as a type called `MyType` that is loaded in the scope of another assembly.</span></span>  
  
- <span data-ttu-id="f4806-117">組件可構成參考範圍界限。</span><span class="sxs-lookup"><span data-stu-id="f4806-117">It forms a reference scope boundary.</span></span> <span data-ttu-id="f4806-118">組件的資訊清單含有用來解析型別和滿足資源要求的組件中繼資料。</span><span class="sxs-lookup"><span data-stu-id="f4806-118">The assembly's manifest contains assembly metadata that is used for resolving types and satisfying resource requests.</span></span> <span data-ttu-id="f4806-119">它會指定公開於組件之外的型別和資源。</span><span class="sxs-lookup"><span data-stu-id="f4806-119">It specifies the types and resources that are exposed outside the assembly.</span></span> <span data-ttu-id="f4806-120">資訊清單也會列舉組件相依的其他組件。</span><span class="sxs-lookup"><span data-stu-id="f4806-120">The manifest also enumerates other assemblies on which it depends.</span></span>  
  
- <span data-ttu-id="f4806-121">組件可構成版本界限。</span><span class="sxs-lookup"><span data-stu-id="f4806-121">It forms a version boundary.</span></span> <span data-ttu-id="f4806-122">組件是通用語言執行平台中可控制版本的最小單位；在同一組件中的所有型別和資源會當做一個單位來控制版本。</span><span class="sxs-lookup"><span data-stu-id="f4806-122">The assembly is the smallest versionable unit in the common language runtime; all types and resources in the same assembly are versioned as a unit.</span></span> <span data-ttu-id="f4806-123">組件的資訊清單描述您對任何相依組件所指定的版本相依性。</span><span class="sxs-lookup"><span data-stu-id="f4806-123">The assembly's manifest describes the version dependencies you specify for any dependent assemblies.</span></span> <span data-ttu-id="f4806-124">如需版本控制的詳細資訊，請參閱[組件版本控制](../../../docs/framework/app-domains/assembly-versioning.md)。</span><span class="sxs-lookup"><span data-stu-id="f4806-124">For more information about versioning, see [Assembly Versioning](../../../docs/framework/app-domains/assembly-versioning.md).</span></span>  
  
- <span data-ttu-id="f4806-125">組件可構成部署單位。</span><span class="sxs-lookup"><span data-stu-id="f4806-125">It forms a deployment unit.</span></span> <span data-ttu-id="f4806-126">當應用程式啟動時，一定要有應用程式一開始所呼叫的組件。</span><span class="sxs-lookup"><span data-stu-id="f4806-126">When an application starts, only the assemblies that the application initially calls must be present.</span></span> <span data-ttu-id="f4806-127">至於其他組件 (例如當地語系化資源或含有公用程式類別的組件)，則可視需要擷取。</span><span class="sxs-lookup"><span data-stu-id="f4806-127">Other assemblies, such as localization resources or assemblies containing utility classes, can be retrieved on demand.</span></span> <span data-ttu-id="f4806-128">如此可讓應用程式在第一次下載時保持最簡狀態。</span><span class="sxs-lookup"><span data-stu-id="f4806-128">This allows applications to be kept simple and thin when first downloaded.</span></span> <span data-ttu-id="f4806-129">如需部署組件的詳細資訊，請參閱[部署應用程式](../../../docs/framework/deployment/index.md)。</span><span class="sxs-lookup"><span data-stu-id="f4806-129">For more information about deploying assemblies, see [Deploying Applications](../../../docs/framework/deployment/index.md).</span></span>  
  
- <span data-ttu-id="f4806-130">組件是支援並存執行的單位。</span><span class="sxs-lookup"><span data-stu-id="f4806-130">It is the unit at which side-by-side execution is supported.</span></span> <span data-ttu-id="f4806-131">如需執行多個組件版本的詳細資訊，請參閱[組件和並存執行](../../../docs/framework/app-domains/assemblies-and-side-by-side-execution.md)。</span><span class="sxs-lookup"><span data-stu-id="f4806-131">For more information about running multiple versions of an assembly, see [Assemblies and Side-by-Side Execution](../../../docs/framework/app-domains/assemblies-and-side-by-side-execution.md).</span></span>  
  
 <span data-ttu-id="f4806-132">組件可以是靜態，也可以是動態。</span><span class="sxs-lookup"><span data-stu-id="f4806-132">Assemblies can be static or dynamic.</span></span> <span data-ttu-id="f4806-133">靜態組件可以包含 .NET Framework 型別 (介面和類別)，以及組件的資源 (點陣圖、JPEG 檔案、資源檔案等)。</span><span class="sxs-lookup"><span data-stu-id="f4806-133">Static assemblies can include .NET Framework types (interfaces and classes), as well as resources for the assembly (bitmaps, JPEG files, resource files, and so on).</span></span> <span data-ttu-id="f4806-134">靜態組件儲存在磁碟上的可攜式執行檔 (PE) 中。</span><span class="sxs-lookup"><span data-stu-id="f4806-134">Static assemblies are stored on disk in portable executable (PE) files.</span></span> <span data-ttu-id="f4806-135">您也可以使用 .NET Framework 建立動態組件，直接從記憶體執行，並且執行前不會儲存至磁碟。</span><span class="sxs-lookup"><span data-stu-id="f4806-135">You can also use the .NET Framework to create dynamic assemblies, which are run directly from memory and are not saved to disk before execution.</span></span> <span data-ttu-id="f4806-136">您可以在執行動態組件之後，再將其儲存至磁碟。</span><span class="sxs-lookup"><span data-stu-id="f4806-136">You can save dynamic assemblies to disk after they have executed.</span></span>  
  
 <span data-ttu-id="f4806-137">您可以使用幾種方式建立組件。</span><span class="sxs-lookup"><span data-stu-id="f4806-137">There are several ways to create assemblies.</span></span> <span data-ttu-id="f4806-138">您可以使用過去用來建立 .dll 或 .exe 檔案的開發工具，例如 Visual Studio。</span><span class="sxs-lookup"><span data-stu-id="f4806-138">You can use development tools, such as Visual Studio, that you have used in the past to create .dll or .exe files.</span></span> <span data-ttu-id="f4806-139">您可以使用 Windows 軟體開發套件 (SDK) 提供的工具，搭配其他開發環境中建立的模組來建立組件。</span><span class="sxs-lookup"><span data-stu-id="f4806-139">You can use tools provided in the Windows Software Development Kit (SDK) to create assemblies with modules created in other development environments.</span></span> <span data-ttu-id="f4806-140">您也可以使用 Common Language Runtime API (例如 <xref:System.Reflection.Emit?displayProperty=nameWithType>) 來建立動態組件。</span><span class="sxs-lookup"><span data-stu-id="f4806-140">You can also use common language runtime APIs, such as <xref:System.Reflection.Emit?displayProperty=nameWithType>, to create dynamic assemblies.</span></span>
