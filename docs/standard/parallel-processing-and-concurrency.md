# <a name="parallel-processing-and-concurrency-in-the-net-framework"></a>.NET Framework 中的平行處理和並行機制
.NET Framework 提供數種方式，讓您可以使用多執行緒的執行，以保持應用程式回應您的使用者，同時讓使用者電腦的效能極大化。  
  
## <a name="in-this-section"></a>本章節內容  
 [執行緒處理](../../docs/standard/threading/index.md)  
 描述 .NET Framework 所提供的基本並行和同步處理機制。  
  
 [非同步程式設計模式](../../docs/standard/asynchronous-programming-patterns/index.md)  
 提供 .NET Framework 所支援之三個非同步程式設計模式的簡短概觀：  
  
-   [非同步程式設計模型 (APM)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md) (舊版)  
  
-   [事件架構非同步模式 (EAP)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md) (舊版)  
  
-   [工作架構非同步模式 (TAP)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md) (新的開發建議)  
  
 [平行程式設計](../../docs/standard/parallel-programming/index.md)  
 描述工作架構程式設計模型，該模型簡化了平行開發作業，讓您能夠利用簡單常見的語法，撰寫有效率、精細且具彈性的平行程式碼，而不需要直接使用執行緒或執行緒集區。
