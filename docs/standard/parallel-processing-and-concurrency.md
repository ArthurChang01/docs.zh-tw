# Parallel Processing and Concurrency in the .NET Framework
.NET Framework 提供數種方式，讓您可以使用多執行緒的執行，以保持應用程式回應您的使用者，同時讓使用者電腦的效能極大化。  
  
## 在本節中  
 [Threading](../../docs/standard/threading/index.md)  
 將告訴您 .NET Framework.所提供的基本並行處理和同步處理機制。  
  
 [Asynchronous Programming Patterns](../../docs/standard/asynchronous-programming-patterns/index.md)  
 提供 .NET Framework 支援三個非同步程式設計模式的簡短概觀:  
  
-   [Asynchronous Programming Model \(APM\)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md)舊版  
  
-   [Event\-based Asynchronous Pattern \(EAP\)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md)舊版  
  
-   [Task\-based Asynchronous Pattern \(TAP\)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md) \(新的程式開發建議\)  
  
 [Parallel Programming](../../docs/standard/parallel-programming/index.md)  
 告訴您工作架構程式撰寫模型可簡化平行開發，讓您撰寫有效、 精密，並可調整的平行處理程式碼，使用自然的慣用句，而不需要直接使用執行緒或執行緒集區。
