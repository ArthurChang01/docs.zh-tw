# <a name="conclusion"></a><span data-ttu-id="2ce45-104">結論</span><span class="sxs-lookup"><span data-stu-id="2ce45-104">Conclusion</span></span>

<span data-ttu-id="2ce45-105">下列的重點是從本指南最重要的結論。</span><span class="sxs-lookup"><span data-stu-id="2ce45-105">The following key takeaways are the most important conclusions from this guide.</span></span>

<span data-ttu-id="2ce45-106">**使用無伺服器的優點。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-106">**Benefits of using serverless.**</span></span> <span data-ttu-id="2ce45-107">無伺服器解決方案提供節省成本的重要優點，因為無伺服器實作在每個使用次數付費模型中。</span><span class="sxs-lookup"><span data-stu-id="2ce45-107">Serverless solutions provide the important benefit of cost savings because serverless is implemented in a pay-per-use model.</span></span> <span data-ttu-id="2ce45-108">無伺服器可讓您能夠獨立調整規模、 測試及部署您的應用程式的個別元件。</span><span class="sxs-lookup"><span data-stu-id="2ce45-108">Serverless makes it possible to independently scale, test, and deploy individual components of your application.</span></span> <span data-ttu-id="2ce45-109">無伺服器唯一適合實作微服務架構，並完全整合至 DevOps 管線。</span><span class="sxs-lookup"><span data-stu-id="2ce45-109">Serverless is uniquely suited to implement microservices architectures and integrates fully into a DevOps pipeline.</span></span>

<span data-ttu-id="2ce45-110">**程式碼作為部署單位。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-110">**Code as a unit of deployment.**</span></span> <span data-ttu-id="2ce45-111">無伺服器提取硬體、 OS 和執行階段從應用程式。</span><span class="sxs-lookup"><span data-stu-id="2ce45-111">Serverless abstracts the hardware, OS, and runtime away from the application.</span></span> <span data-ttu-id="2ce45-112">無伺服器可讓您將焦點放在程式碼中的商務邏輯部署單位。</span><span class="sxs-lookup"><span data-stu-id="2ce45-112">Serverless enables focusing on business logic in code as the unit of deployment.</span></span>

<span data-ttu-id="2ce45-113">**觸發程序和繫結。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-113">**Triggers and bindings.**</span></span> <span data-ttu-id="2ce45-114">與儲存體、 Api 和其他雲端資源的無伺服器 eases 整合。</span><span class="sxs-lookup"><span data-stu-id="2ce45-114">Serverless eases integration with storage, APIs, and other cloud resources.</span></span> <span data-ttu-id="2ce45-115">Azure Functions 提供觸發程序來執行程式碼和繫結與資源互動。</span><span class="sxs-lookup"><span data-stu-id="2ce45-115">Azure Functions provides triggers to execute code and bindings to interact with resources.</span></span>

<span data-ttu-id="2ce45-116">**微服務。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-116">**Microservices.**</span></span> <span data-ttu-id="2ce45-117">微服務架構逐漸成為分散式大型或複雜關鍵任務應用程式以自發性服務的形式的多個獨立子系統為基礎的慣用的方法。</span><span class="sxs-lookup"><span data-stu-id="2ce45-117">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="2ce45-118">在微服務型架構中，應用程式會建立可開發、 測試、 建立版本、 部署及調整獨立服務的集合。</span><span class="sxs-lookup"><span data-stu-id="2ce45-118">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently.</span></span> <span data-ttu-id="2ce45-119">無伺服器是適合用來建置這些服務的架構。</span><span class="sxs-lookup"><span data-stu-id="2ce45-119">Serverless is an architecture well-suited for building these services.</span></span>

<span data-ttu-id="2ce45-120">**無伺服器平台。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-120">**Serverless platforms.**</span></span> <span data-ttu-id="2ce45-121">無伺服器並不只是程式碼。</span><span class="sxs-lookup"><span data-stu-id="2ce45-121">Serverless isn't just about the code.</span></span> <span data-ttu-id="2ce45-122">支援無伺服器架構的平台包括無伺服器工作流程和協調流程、 無伺服器訊息事件服務和無伺服器資料庫。</span><span class="sxs-lookup"><span data-stu-id="2ce45-122">Platforms that support serverless architectures include serverless workflows and orchestration, serverless messaging and event services, and serverless databases.</span></span>

<span data-ttu-id="2ce45-123">**無伺服器的挑戰。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-123">**Serverless challenges.**</span></span> <span data-ttu-id="2ce45-124">挑戰分散式應用程式開發，例如分散式和獨立的資料模型、 復原、 版本控制和服務探索，導入了無伺服器。</span><span class="sxs-lookup"><span data-stu-id="2ce45-124">Serverless introduces challenges related to distributed application development, such as fragmented and independent data models, resiliency, versioning, and service discovery.</span></span> <span data-ttu-id="2ce45-125">無伺服器可能不在理想情況下適合長時間執行的處理程序或受益於更緊密的結合性的元件。</span><span class="sxs-lookup"><span data-stu-id="2ce45-125">Serverless may not be ideally suited to long running processes or components that benefit from tighter coupling.</span></span>

<span data-ttu-id="2ce45-126">**無伺服器作為工具，不是 [工具箱]。**</span><span class="sxs-lookup"><span data-stu-id="2ce45-126">**Serverless as a tool, not the toolbox.**</span></span> <span data-ttu-id="2ce45-127">無伺服器是不應用程式架構的專屬解決方案。</span><span class="sxs-lookup"><span data-stu-id="2ce45-127">Serverless is not the exclusive solution to application architecture.</span></span> <span data-ttu-id="2ce45-128">它是一種工具，可利用混合式應用程式可能會包含傳統的層級、 單體後端，以及容器的一部分。</span><span class="sxs-lookup"><span data-stu-id="2ce45-128">It is a tool that can be leveraged as part of a hybrid application that may contain traditional tiers, monolith back ends, and containers.</span></span> <span data-ttu-id="2ce45-129">無伺服器可用來增強現有的解決方案，並不是孤注一擲的方法，應用程式開發。</span><span class="sxs-lookup"><span data-stu-id="2ce45-129">Serverless can be used to enhance existing solutions and is not an all-or-nothing approach to application development.</span></span>
