## <a name="windows-container-types"></a><span data-ttu-id="a0424-209">Windows 容器類型</span><span class="sxs-lookup"><span data-stu-id="a0424-209">Windows container types</span></span>

<span data-ttu-id="a0424-210">Linux 容器，例如 Windows Server 容器會管理使用 Docker 引擎。</span><span class="sxs-lookup"><span data-stu-id="a0424-210">Like Linux containers, Windows Server containers are managed by using Docker Engine.</span></span> <span data-ttu-id="a0424-211">與 Linux 容器，不同的是 Windows 容器包含兩種不同的容器類型，或執行時間-Windows Server 容器和 HYPER-V 隔離。</span><span class="sxs-lookup"><span data-stu-id="a0424-211">Unlike Linux containers, Windows containers include two different container types, or run times-Windows Server containers and Hyper-V isolation.</span></span>

<span data-ttu-id="a0424-212">**Windows Server 容器**:提供透過程序和命名空間隔離技術的應用程式隔離。</span><span class="sxs-lookup"><span data-stu-id="a0424-212">**Windows Server containers**: Provides application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="a0424-213">Windows Server 容器與容器主機和主機執行的所有容器共用核心。</span><span class="sxs-lookup"><span data-stu-id="a0424-213">A Windows Server container shares a kernel with the container host and all containers that are running on the host.</span></span> <span data-ttu-id="a0424-214">這些容器不提供惡意的安全性界限，而且不應該用來隔離不受信任的程式碼。</span><span class="sxs-lookup"><span data-stu-id="a0424-214">These containers do not provide a hostile security boundary and should not be used to isolate untrusted code.</span></span> <span data-ttu-id="a0424-215">因為共用的核心空間，而這些容器會要求相同的核心版本和組態。</span><span class="sxs-lookup"><span data-stu-id="a0424-215">Because of the shared kernel space, these containers require the same kernel version and configuration.</span></span>

<span data-ttu-id="a0424-216">**HYPER-V 隔離**:擴充了 Windows Server 容器提供高度最佳化的 VM 上執行每個容器的隔離。</span><span class="sxs-lookup"><span data-stu-id="a0424-216">**Hyper-V isolation**: Expands on the isolation provided by Windows Server Containers by running each container on a highly optimized VM.</span></span> <span data-ttu-id="a0424-217">在此組態中，容器主機的核心不會與相同主機上的其他容器共用。</span><span class="sxs-lookup"><span data-stu-id="a0424-217">In this configuration, the kernel of the container host is not shared with other containers on the same host.</span></span> <span data-ttu-id="a0424-218">這些容器是針對惡意多租用戶裝載，使用相同的安全性保證，VM 的設計。</span><span class="sxs-lookup"><span data-stu-id="a0424-218">These containers are designed for hostile multitenant hosting, with the same security assurances of a VM.</span></span> <span data-ttu-id="a0424-219">因為這些容器不會與主機或主機上的其他容器共用核心，可以使用不同的版本和設定 （具有支援的版本） 執行的核心。</span><span class="sxs-lookup"><span data-stu-id="a0424-219">Because these containers don't share the kernel with the host or other containers on the host, they can run kernels with different versions and configurations (with supported versions).</span></span> <span data-ttu-id="a0424-220">比方說，Windows 10 上的所有 Windows 容器會都使用 HYPER-V 隔離，利用 Windows Server 的核心版本和組態。</span><span class="sxs-lookup"><span data-stu-id="a0424-220">For example, all Windows containers on Windows 10 use Hyper-V isolation to utilize the Windows Server kernel version and configuration.</span></span>

<span data-ttu-id="a0424-221">在 Windows 上執行的容器，包含或不含 HYPER-V 隔離是執行階段的決策。</span><span class="sxs-lookup"><span data-stu-id="a0424-221">Running a container on Windows with or without Hyper-V isolation is a run-time decision.</span></span> <span data-ttu-id="a0424-222">您可以選擇一開始，建立容器與 HYPER-V 隔離，並在執行階段，選擇改為執行 Windows Server 容器。</span><span class="sxs-lookup"><span data-stu-id="a0424-222">You might choose to create the container with Hyper-V isolation initially, and at run time, choose to run it as a Windows Server container instead.</span></span>
