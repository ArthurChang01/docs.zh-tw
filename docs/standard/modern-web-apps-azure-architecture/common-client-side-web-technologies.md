<span data-ttu-id="9d0a2-107">ASP.NET Core 應用程式是 Web 應用程式，通常依賴於如 HTML、CSS 和 JavaScript 等用戶端 Web 技術。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-107">ASP.NET Core applications are web applications and they typically rely on client-side web technologies like HTML, CSS, and JavaScript.</span></span> <span data-ttu-id="9d0a2-108">藉由將頁面 (HTML) 的內容與其版面配置和樣式 (CSS) 分離開來，以及其行為 (透過 JavaScript)，複雜的 Web 應用程式可以利用關注點分離原則。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-108">By separating the content of the page (the HTML) from its layout and styling (the CSS), and its behavior (via JavaScript), complex web apps can leverage the Separation of Concerns principle.</span></span> <span data-ttu-id="9d0a2-109">當這些問題並非密不可分時，未來對應用程式結構、設計或行為的變更，可以更容易進行。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-109">Future changes to the structure, design, or behavior of the application can be made more easily when these concerns are not intertwined.</span></span>

<span data-ttu-id="9d0a2-110">雖然 HTML 和 CSS 相對穩定，但 JavaScript 透過開發人員使用應用程式架構和公用程式來建置 Web 型應用程式，正以驚人的速度發展。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-110">While HTML and CSS are relatively stable, JavaScript, by means of the application frameworks and utilities developers work with to build web-based applications, is evolving at breakneck speed.</span></span> <span data-ttu-id="9d0a2-111">本章介紹 Web 開發人員使用 JavaScript 作為部分開發應用程式的一些方式，提供對 Angular 和 React 用戶端程式庫的高階概觀。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-111">This chapter looks at a few ways JavaScript is used by web developers as part of developing applications, as provides a high-level overview of the Angular and React client side libraries.</span></span>

## <a name="html"></a><span data-ttu-id="9d0a2-112">HTML</span><span class="sxs-lookup"><span data-stu-id="9d0a2-112">HTML</span></span>

<span data-ttu-id="9d0a2-113">HTML (超文字標記語言) 是用來建立網頁和 Web 應用程式的標準標記語言。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-113">HTML (HyperText Markup Language) is the standard markup language used to create web pages and web applications.</span></span> <span data-ttu-id="9d0a2-114">其項目形成頁面的建置組塊、表示格式化的文字、影像、表單輸入和其他結構。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-114">Its elements form the building blocks of pages, representing formatted text, images, form inputs, and other structures.</span></span> <span data-ttu-id="9d0a2-115">當瀏覽器向 URL 提出要求時，無論是擷取頁面或應用程式，傳回的第一件事都是 HTML 文件。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-115">When a browser makes a request to a URL, whether fetching a page or an application, the first thing that is returned is an HTML document.</span></span> <span data-ttu-id="9d0a2-116">此 HTML 文件可能會以 CSS 的形式參考或包含其外觀和版面配置的其他資訊，或以 JavaScript 的形式表現行為。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-116">This HTML document may reference or include additional information about its look and layout in the form of CSS, or behavior in the form of JavaScript.</span></span>

<span data-ttu-id="9d0a2-118">CSS (階層式樣式表) 用來控制 HTML 項目的外觀和版面配置。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-118">CSS (Cascading Style Sheets) is used to control the look and layout of HTML elements.</span></span> <span data-ttu-id="9d0a2-119">CSS 樣式可直接套用至 HTML 項目，在同一頁面上個別定義或在不同的檔案中定義，並由頁面所參考。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-119">CSS styles can be applied directly to an HTML element, defined separately on the same page, or defined in a separate file and referenced by the page.</span></span> <span data-ttu-id="9d0a2-120">樣式串聯基於如何用於選擇指定的 HTML 項目。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-120">Styles cascade based on how they are used to select a given HTML element.</span></span> <span data-ttu-id="9d0a2-121">比方說，樣式可能適用於整個文件，但套用於特定項目的樣式可能會將其覆寫。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-121">For instance, a style might apply to an entire document, but would be overridden by a style that applied to a particular element.</span></span> <span data-ttu-id="9d0a2-122">同樣地，項目特定樣式會受到套用於 CSS 類別 (套用於該項目) 的樣式所覆寫，而此樣式又會受針對該項目的特定執行個體之樣式所覆寫 (透過其識別碼)。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-122">Likewise, an element-specific style would be overridden by a style that applied to a CSS class that was applied to the element, which in turn would be overridden by a style targeting a specific instance of that element (via its id).</span></span> <span data-ttu-id="9d0a2-123">圖 6-1</span><span class="sxs-lookup"><span data-stu-id="9d0a2-123">Figure 6-1</span></span>

<span data-ttu-id="9d0a2-126">建議您保留樣式於各自的樣式表檔案中，並且使用基於選取範圍的串聯，來實作應用程式中一致且可重複使用的樣式。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-126">It's best to keep styles in their own separate stylesheet files, and to use selection-based cascading to implement consistent and reusable styles within the application.</span></span> <span data-ttu-id="9d0a2-127">應避免在 HTML 中放置樣式規則；並且將樣式套用於特定的單個項目 (而不是整個項目類別或套用特定 CSS 類別的項目) 應屬例外情況，而非規則。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-127">Placing style rules within HTML should be avoided, and applying styles to specific individual elements (rather than whole classes of elements, or elements that have had a particular CSS class applied to them) should be the exception, not the rule.</span></span>

### <a name="css-preprocessors"></a><span data-ttu-id="9d0a2-128">CSS 前置處理器</span><span class="sxs-lookup"><span data-stu-id="9d0a2-128">CSS Preprocessors</span></span>

<span data-ttu-id="9d0a2-129">CSS 樣式表缺少對條件邏輯、變數和其他程式設計語言功能的支援。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-129">CSS stylesheets lack support for conditional logic, variables, and other programming language features.</span></span> <span data-ttu-id="9d0a2-130">因此，大型樣式表通常包含大量的重複，因為相同的顏色、字型或其他設定會套用至 HTML 項目和 CSS 類別的許多不同變化。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-130">Thus, large stylesheets often include a lot of repetition, as the same color, font, or other setting is applied to many different variations of HTML elements and CSS classes.</span></span> <span data-ttu-id="9d0a2-131">透過新增對變數和邏輯的支援，CSS 前置處理器可以幫助您的樣式表遵循 [DRY Principle](http://deviq.com/don-t-repeat-yourself/) (DRY 準則)。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-131">CSS preprocessors can help your stylesheets follow the [DRY principle](http://deviq.com/don-t-repeat-yourself/) by adding support for variables and logic.</span></span>

<span data-ttu-id="9d0a2-132">最熱門的 CSS 前置處理器是 Sass 和 LESS。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-132">The most popular CSS preprocessors are Sass and LESS.</span></span> <span data-ttu-id="9d0a2-133">兩者都擴充 CSS 並回溯相容，這表示一般的 CSS 檔案即為有效的 Sass 或 LESS 檔案。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-133">Both extend CSS and are backward compatible with it, meaning that a plain CSS file is a valid Sass or LESS file.</span></span> <span data-ttu-id="9d0a2-134">Sass 基於 Ruby，LESS 基於 JavaScript，而兩者通常都是作為本機開發程序的一部分執行。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-134">Sass is Ruby-based and LESS is JavaScript based, and both typically run as part of your local development process.</span></span> <span data-ttu-id="9d0a2-135">兩者都提供命令列工具以及 Visual Studio 中的內建支援，以使用 Gulp 或 Grunt 工作來執行。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-135">Both have command line tools available, as well as built-in support in Visual Studio for running them using Gulp or Grunt tasks.</span></span>

## <a name="javascript"></a><span data-ttu-id="9d0a2-136">JavaScript</span><span class="sxs-lookup"><span data-stu-id="9d0a2-136">JavaScript</span></span>

<span data-ttu-id="9d0a2-137">JavaScript 是一種動態、解譯的程式設計語言，已在 ECMAScript 語言規格中進行標準化。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-137">JavaScript is a dynamic, interpreted programming language that has been standardized in the ECMAScript language specification.</span></span> <span data-ttu-id="9d0a2-138">其為 Web 程式設計語言。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-138">It is the programming language of the web.</span></span> <span data-ttu-id="9d0a2-139">與 CSS 一樣，JavaScript 可以在 HTML 項目中 (如同頁面內的指令碼區塊) 或在個別的檔案中定義為屬性。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-139">Like CSS, JavaScript can be defined as attributes within HTML elements, as blocks of script within a page, or in separate files.</span></span> <span data-ttu-id="9d0a2-140">就像 CSS 一樣，通常建議您將 JavaScript 組織成不同的檔案，盡可能將其與個別網頁或應用程式檢視的 HTML 相隔。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-140">Just like CSS, it's generally recommended to organize JavaScript into separate files, keeping it separated as much as possible from the HTML found on individual web pages or application views.</span></span>

<span data-ttu-id="9d0a2-141">在 Web 應用程式中使用 JavaScript 時，您通常需要執行幾項工作：</span><span class="sxs-lookup"><span data-stu-id="9d0a2-141">When working with JavaScript in your web application, there are a few tasks that you'll commonly need to perform:</span></span>

-   <span data-ttu-id="9d0a2-142">選取 HTML 項目並擷取及/或 更新其值</span><span class="sxs-lookup"><span data-stu-id="9d0a2-142">Selecting an HTML element and retrieving and/or updating its value</span></span>

-   <span data-ttu-id="9d0a2-143">查詢 Web API 以取得資料</span><span class="sxs-lookup"><span data-stu-id="9d0a2-143">Querying a Web API for data</span></span>

-   <span data-ttu-id="9d0a2-144">將命令傳送至 Web API (並回應回呼及其結果)</span><span class="sxs-lookup"><span data-stu-id="9d0a2-144">Sending a command to a Web API (and responding to a callback with its result)</span></span>

### <a name="legacy-web-apps-with-jquery"></a><span data-ttu-id="9d0a2-149">使用 jQuery 的傳統 Web 應用程式</span><span class="sxs-lookup"><span data-stu-id="9d0a2-149">Legacy Web Apps with jQuery</span></span>

<span data-ttu-id="9d0a2-150">儘管 JavaScript 架構標準已老舊，jQuery 仍然是一個相當常用的程式庫，用於處理 HTML/CSS 以及建置對 Web API 進行 AJAX 呼叫的應用程式。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-150">Although ancient by JavaScript framework standards, jQuery continues to be a very commonly used library for working with HTML/CSS and building applications that make AJAX calls to web APIs.</span></span> <span data-ttu-id="9d0a2-151">然而，jQuery 在瀏覽器文件物件模型 (DOM) 層級上運作，且預設只提供命令式模型，而非宣告式模型。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-151">However, jQuery operates at the level of the browser document object model (DOM), and by default offers only an imperative, rather than declarative, model.</span></span>

<span data-ttu-id="9d0a2-152">例如，假設文字方塊的值超過 10，則頁面上的項目應該可見。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-152">For example, imagine that if a textbox's value exceeds 10, an element on the page should be made visible.</span></span> <span data-ttu-id="9d0a2-153">在 jQuery 中，這通常是透過撰寫事件處理常式來實作，其程式碼會檢查文字方塊的值，並根據該值設定目標項目的可見性。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-153">In jQuery, this would typically be implemented by writing an event handler with code that would inspect the textbox's value and set the visibility of the target element based on that value.</span></span> <span data-ttu-id="9d0a2-154">這是基於程式碼的命令式方法。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-154">This is an imperative, code-based approach.</span></span> <span data-ttu-id="9d0a2-155">另一個架構則可能會使用資料繫結，以宣告方式將項目可見性繫結於文字方塊的值。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-155">Another framework might instead use databinding to bind the visibility of the element to the value of the textbox declaratively.</span></span> <span data-ttu-id="9d0a2-156">這不需要撰寫任何程式碼，而只需要修飾與資料繫結屬性有關的項目。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-156">This would not require writing any code, but instead only requires decorating the elements involved with data binding attributes.</span></span> <span data-ttu-id="9d0a2-157">隨著客戶端行為越來越複雜，資料繫結方法經常會導致更簡單的解決方案，程式碼更少且具有條件的複雜性。</span><span class="sxs-lookup"><span data-stu-id="9d0a2-157">As client side behaviors grow more complex, data binding approaches frequently result in simpler solutions with less code and conditional complexity.</span></span>
