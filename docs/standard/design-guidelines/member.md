# 成員設計方針
方法、 屬性、 事件、 建構函式和欄位統稱為成員。 成員是最終的架構功能公開給架構的使用者。  
  
 成員可以是虛擬或非虛擬、 實體或抽象，靜態或執行個體，而且可以有數個不同的範圍的存取範圍。 這樣的複雜性提供驚人的表達，但同時需要針對架構設計師特別小心。  
  
 本章提供設計任何型別的成員時，應遵循的基本指導方針。
