# 命名方針
遵循一組一致的開發架構的命名慣例，可以是主要的投稿內容架構的使用性。 它可讓廣泛分隔專案的開發人員所使用的架構。 表單的一致性，超過架構項目的名稱必須很容易瞭解和必須傳遞每個項目的函式。  
  
 本指南的目標是提供一組一致的命名慣例會導致對開發人員的即時運算的有意義的名稱。  
  
 雖然採用下列命名慣例，因為一般程式碼開發指導方針會導致更一致的命名，在整個程式碼，您僅需要將它們套用至會對外公開的 Api （公用或受保護的型別和成員，以及明確實作的介面）。
