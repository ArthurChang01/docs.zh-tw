## <a name="implementing-custom-collections"></a><span data-ttu-id="dab30-169">執行自訂集合</span><span class="sxs-lookup"><span data-stu-id="dab30-169">Implementing Custom Collections</span></span>  
 <span data-ttu-id="dab30-170">✓在設計新的集合時，**請考慮**從 `Collection<T>`、`ReadOnlyCollection<T>`或 `KeyedCollection<TKey,TItem>` 繼承。</span><span class="sxs-lookup"><span data-stu-id="dab30-170">**✓ CONSIDER** inheriting from `Collection<T>`, `ReadOnlyCollection<T>`, or `KeyedCollection<TKey,TItem>` when designing new collections.</span></span>  
  
 <span data-ttu-id="dab30-171">**✓**會在設計新的集合時，執行 `IEnumerable<T>`。</span><span class="sxs-lookup"><span data-stu-id="dab30-171">**✓ DO** implement `IEnumerable<T>` when designing new collections.</span></span> <span data-ttu-id="dab30-172">請考慮在合理的地方執行 `ICollection<T>` 或甚至 `IList<T>`。</span><span class="sxs-lookup"><span data-stu-id="dab30-172">Consider implementing `ICollection<T>` or even `IList<T>` where it makes sense.</span></span>  
  
 <span data-ttu-id="dab30-173">在執行這類自訂集合時，請遵循 `Collection<T>` 所建立的 API 模式，並盡可能 `ReadOnlyCollection<T>`。</span><span class="sxs-lookup"><span data-stu-id="dab30-173">When implementing such custom collection, follow the API pattern established by `Collection<T>` and `ReadOnlyCollection<T>` as closely as possible.</span></span> <span data-ttu-id="dab30-174">也就是說，請明確地執行相同的成員，將這些參數命名為這兩個集合的名稱，依此類推。</span><span class="sxs-lookup"><span data-stu-id="dab30-174">That is, implement the same members explicitly, name the parameters like these two collections name them, and so on.</span></span>  
  
 <span data-ttu-id="dab30-175">如果集合通常會傳遞給採用這些介面做為輸入的 Api， **✓請考慮**執行非泛型集合介面（`IList` 和 `ICollection`）。</span><span class="sxs-lookup"><span data-stu-id="dab30-175">**✓ CONSIDER** implementing nongeneric collection interfaces (`IList` and `ICollection`) if the collection will often be passed to APIs taking these interfaces as input.</span></span>  
  
 <span data-ttu-id="dab30-176">**X 避免**在具有與集合概念無關之複雜 api 的類型上，執行集合介面。</span><span class="sxs-lookup"><span data-stu-id="dab30-176">**X AVOID** implementing collection interfaces on types with complex APIs unrelated to the concept of a collection.</span></span>  
  
 <span data-ttu-id="dab30-177">**X 不會**繼承自非泛型基底集合，例如 `CollectionBase`。</span><span class="sxs-lookup"><span data-stu-id="dab30-177">**X DO NOT** inherit from nongeneric base collections such as `CollectionBase`.</span></span> <span data-ttu-id="dab30-178">請改用 `Collection<T>`、`ReadOnlyCollection<T>` 和 `KeyedCollection<TKey,TItem>`。</span><span class="sxs-lookup"><span data-stu-id="dab30-178">Use `Collection<T>`, `ReadOnlyCollection<T>`, and `KeyedCollection<TKey,TItem>` instead.</span></span>  
  
### <a name="naming-custom-collections"></a><span data-ttu-id="dab30-179">命名自訂集合</span><span class="sxs-lookup"><span data-stu-id="dab30-179">Naming Custom Collections</span></span>  
 <span data-ttu-id="dab30-180">集合（實 `IEnumerable`的型別）主要是為了兩個原因而建立：（1）建立具有結構特定作業的新資料結構，以及通常與現有資料結構（例如，<xref:System.Collections.Generic.List%601>、<xref:System.Collections.Generic.LinkedList%601>、<xref:System.Collections.Generic.Stack%601>）和（2）不同的效能特性，以建立用於保存特定專案集（例如 <xref:System.Collections.Specialized.StringCollection>）的特殊集合。</span><span class="sxs-lookup"><span data-stu-id="dab30-180">Collections (types that implement `IEnumerable`) are created mainly for two reasons: (1) to create a new data structure with structure-specific operations and often different performance characteristics than existing data structures (e.g.,  <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>), and (2) to create a specialized collection for holding a specific set of items (e.g.,  <xref:System.Collections.Specialized.StringCollection>).</span></span> <span data-ttu-id="dab30-181">資料結構最常用於應用程式和程式庫的內部執行。</span><span class="sxs-lookup"><span data-stu-id="dab30-181">Data structures are most often used in the internal implementation of applications and libraries.</span></span> <span data-ttu-id="dab30-182">專門的集合主要是在 Api 中公開（作為屬性和參數類型）。</span><span class="sxs-lookup"><span data-stu-id="dab30-182">Specialized collections are mainly to be exposed in APIs (as property and parameter types).</span></span>  
  
 <span data-ttu-id="dab30-183">**✓**會在用來執行 `IDictionary` 或 `IDictionary<TKey,TValue>`的抽象概念名稱中使用「字典」尾碼。</span><span class="sxs-lookup"><span data-stu-id="dab30-183">**✓ DO** use the "Dictionary" suffix in names of abstractions implementing `IDictionary` or `IDictionary<TKey,TValue>`.</span></span>  
  
 <span data-ttu-id="dab30-184">**✓確實**會在型別的名稱中使用 "Collection" 後置詞，這些型別會實作為 `IEnumerable` （或其子系的任何子系）並代表專案清單。</span><span class="sxs-lookup"><span data-stu-id="dab30-184">**✓ DO** use the "Collection" suffix in names of types implementing `IEnumerable` (or any of its descendants) and representing a list of items.</span></span>  
  
 <span data-ttu-id="dab30-185">**✓ DO**會針對自訂資料結構使用適當的資料結構名稱。</span><span class="sxs-lookup"><span data-stu-id="dab30-185">**✓ DO** use the appropriate data structure name for custom data structures.</span></span>  
  
 <span data-ttu-id="dab30-186">**X**在集合抽象的名稱中，請避免使用任何尾碼來隱含特定的執行，例如 "linkedlist<t>" 或 "Hashtable"。</span><span class="sxs-lookup"><span data-stu-id="dab30-186">**X AVOID** using any suffixes implying particular implementation, such as "LinkedList" or "Hashtable," in names of collection abstractions.</span></span>  
  
 <span data-ttu-id="dab30-187">**✓請考慮**在集合名稱前面加上專案類型的名稱。</span><span class="sxs-lookup"><span data-stu-id="dab30-187">**✓ CONSIDER** prefixing collection names with the name of the item type.</span></span> <span data-ttu-id="dab30-188">例如，儲存類型 `Address` （執行 `IEnumerable<Address>`）之專案的集合應該命名為 `AddressCollection`。</span><span class="sxs-lookup"><span data-stu-id="dab30-188">For example, a collection storing items of type `Address` (implementing `IEnumerable<Address>`) should be named `AddressCollection`.</span></span> <span data-ttu-id="dab30-189">如果專案類型是介面，則可以省略專案類型的 "I" 前置詞。</span><span class="sxs-lookup"><span data-stu-id="dab30-189">If the item type is an interface, the "I" prefix of the item type can be omitted.</span></span> <span data-ttu-id="dab30-190">因此，可以 `DisposableCollection`呼叫 <xref:System.IDisposable> 專案的集合。</span><span class="sxs-lookup"><span data-stu-id="dab30-190">Thus, a collection of <xref:System.IDisposable> items can be called `DisposableCollection`.</span></span>  
  
 <span data-ttu-id="dab30-191">如果對應的可寫入集合可能已加入或已存在於架構中，則**✓會考慮**在唯讀集合的名稱中使用 "ReadOnly" 前置詞。</span><span class="sxs-lookup"><span data-stu-id="dab30-191">**✓ CONSIDER** using the "ReadOnly" prefix in names of read-only collections if a corresponding writeable collection might be added or already exists in the framework.</span></span>  
  
 <span data-ttu-id="dab30-192">例如，應該 `ReadOnlyStringCollection`呼叫字串的唯讀集合。</span><span class="sxs-lookup"><span data-stu-id="dab30-192">For example, a read-only collection of strings should be called `ReadOnlyStringCollection`.</span></span>
