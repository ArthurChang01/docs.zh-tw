# <a name="road-to-modern-applications-based-on-containers"></a>邁向以容器為基礎的新式應用程式

您閱讀本書的理由可能是因為您打算開發新應用程式，或要評估在公司中使用 Docker、容器及新方法 (例如微服務) 所造成的影響。

在啟動專案之前，必須謹慎採用新的開發範例，以評估對您的開發小組、預算或基礎結構所帶來的影響。

Microsoft 一直致力於完善豐富的指導方針、應用程式範例及電子書套件，以便協助您制訂明智的決策，並引導您的小組成功開發、部署及操作新應用程式。

本書屬於 Microsoft 指南套件，其中涵蓋許多您在根據容器開發新式應用程式的處理序期間所面臨需求和挑戰。

您可以在下列清單中找到與 Docker 容器相關的其他 Microsoft 電子書：

- **.NET 微服務：容器化 .NET 應用程式的架構** \
  [https://docs.microsoft.com/dotnet/standard/microservices-architecture/](../microservices-architecture/index.md)

- **使用 Azure 雲端及 Windows 容器將現有 .NET 應用程式轉換成現代化的應用程式** \
  [https://docs.microsoft.com/dotnet/standard/modernize-with-azure-and-containers/](../modernize-with-azure-and-containers/index.md)
