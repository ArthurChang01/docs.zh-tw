# <a name="design-docker-applications"></a>設計 Docker 應用程式

第 1 章引進了有關容器和 Docker 的基本概念。 這項資訊是資訊的基本的層級，若要開始您需要。 但是，企業應用程式很複雜，而不是單一服務或容器的多個服務所組成。 選擇性的使用情況下，您需要知道其他方法，例如服務導向架構 (SOA) 的設計，以及更進階微服務概念和容器協調流程概念。 本文件的範圍並不限於微服務，但若要任何 Docker 應用程式的生命週期，因此，它不會不探索深入了解微服務架構，因為您也可以使用容器和 Docker 與一般聖保羅、 背景工作或作業，或甚至是使用整合型應用程式部署方法。

不過，我們的應用程式生命週期和 DevOps 之前，務必知道您要如何設計及建構您的應用程式和您的設計選擇是什麼。
