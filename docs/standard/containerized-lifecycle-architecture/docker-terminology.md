# <a name="docker-terminology"></a><span data-ttu-id="9f918-103">Docker 術語</span><span class="sxs-lookup"><span data-stu-id="9f918-103">Docker terminology</span></span>

<span data-ttu-id="9f918-104">本節列出您應該熟悉才能深入探索 Docker 的術語和定義。</span><span class="sxs-lookup"><span data-stu-id="9f918-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="9f918-105">如需進一步定義，請參閱廣泛[詞彙](https://docs.docker.com/glossary/)Docker 所提供。</span><span class="sxs-lookup"><span data-stu-id="9f918-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker.</span></span>

<span data-ttu-id="9f918-106">**容器映像**：一種套件，其中包含建立容器所需的所有相依性和資訊。</span><span class="sxs-lookup"><span data-stu-id="9f918-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="9f918-107">映像會包含所有相依性 (例如架構)，以及容器執行階段所要使用的部署和執行組態。</span><span class="sxs-lookup"><span data-stu-id="9f918-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="9f918-108">通常，一個映像會衍生自多個基底映像，這些基底映像是彼此交互堆疊以構成容器檔案系統的圖層。</span><span class="sxs-lookup"><span data-stu-id="9f918-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="9f918-109">映像一旦建立，就不可改變。</span><span class="sxs-lookup"><span data-stu-id="9f918-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="9f918-110">**Dockerfile**：文字檔案，其中包含有關如何建置 Docker 映像的指示。</span><span class="sxs-lookup"><span data-stu-id="9f918-110">**Dockerfile**: A text file that contains instructions on how to build a Docker image.</span></span> <span data-ttu-id="9f918-111">就像批次指令碼，第一行指出開頭，，然後遵循指示來安裝必要的程式，複製檔案，在基礎映像，您需要等等，直到您取得的工作環境。</span><span class="sxs-lookup"><span data-stu-id="9f918-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files, and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="9f918-112">**組建**：建立容器映像的動作，該映像會以其 Dockerfile 及建立映像之資料夾中其他檔案所提供的資訊和內容為依據。</span><span class="sxs-lookup"><span data-stu-id="9f918-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="9f918-113">您可以建置 Docker 映像 **`docker build`** 命令。</span><span class="sxs-lookup"><span data-stu-id="9f918-113">You can build images with the Docker **`docker build`** command.</span></span>

<span data-ttu-id="9f918-114">**容器**：Docker 映像的執行個體。</span><span class="sxs-lookup"><span data-stu-id="9f918-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="9f918-115">容器代表單一應用程式、處理序或服務的執行。</span><span class="sxs-lookup"><span data-stu-id="9f918-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="9f918-116">其中包含 Docker 映像的內容、執行環境和一組標準的指示。</span><span class="sxs-lookup"><span data-stu-id="9f918-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="9f918-117">擴充服務時，您會從同一個映像建立容器的多個執行個體。</span><span class="sxs-lookup"><span data-stu-id="9f918-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="9f918-118">或者，一個批次工作可以從同一個映像建立多個容器，並將不同的參數傳遞至每個執行個體。</span><span class="sxs-lookup"><span data-stu-id="9f918-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="9f918-119">**磁碟區**：提供容器可使用的可寫入檔案系統。</span><span class="sxs-lookup"><span data-stu-id="9f918-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="9f918-120">因為映像是唯讀的，但是大部分程式需要寫入到檔案系統 (磁碟區會在容器映像上方新增可寫入的層)，因此程式能夠存取可寫入的檔案系統。</span><span class="sxs-lookup"><span data-stu-id="9f918-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="9f918-121">程式並不知道它正在存取分層的檔案系統，如往常般是檔案系統。</span><span class="sxs-lookup"><span data-stu-id="9f918-121">The program doesn't know it's accessing a layered filesystem, it's just the filesystem as usual.</span></span> <span data-ttu-id="9f918-122">磁碟區位於主機系統，而且由 Docker 管理。</span><span class="sxs-lookup"><span data-stu-id="9f918-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="9f918-123">**標籤**：您可以套用至映像的標記或標籤，以便識別相同映像的不同映像版本 (視版本號碼或目標環境而定)。</span><span class="sxs-lookup"><span data-stu-id="9f918-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="9f918-124">**多階段組建**：這是 Docker 17.05 或更高版本中的功能，可協助減少最終映像的大小。</span><span class="sxs-lookup"><span data-stu-id="9f918-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="9f918-125">簡單幾句話來說，有了多階段建置，您可以使用例如大型基底映像 (包含 SDK)，來編譯和發行應用程式，然後使用發行資料夾和小型僅執行階段基底映像，以產生更小的最終映像</span><span class="sxs-lookup"><span data-stu-id="9f918-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image</span></span>

<span data-ttu-id="9f918-126">**存放庫**：相關的 Docker 映像集合，已加上標籤指出映像版本。</span><span class="sxs-lookup"><span data-stu-id="9f918-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="9f918-127">某些存放庫包含特定映像的多種變化，例如包含 SDK 的映像 (較大)、只包含執行階段的映像 (較小) 等等。這些變化可以加上標記。</span><span class="sxs-lookup"><span data-stu-id="9f918-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="9f918-128">一個存放庫可以包含多種平台變化，例如 Linux 映像和 Windows 映像。</span><span class="sxs-lookup"><span data-stu-id="9f918-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="9f918-129">**登錄**：提供存放庫存取權的服務。</span><span class="sxs-lookup"><span data-stu-id="9f918-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="9f918-130">大多數公用映像的預設登錄是 [Docker Hub](https://hub.docker.com/) (以組織形式為 Docker 所擁有）。</span><span class="sxs-lookup"><span data-stu-id="9f918-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="9f918-131">登錄通常會包含來自多個小組的存放庫。</span><span class="sxs-lookup"><span data-stu-id="9f918-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="9f918-132">公司通常會有私人登錄來儲存及管理其所建立的映像。</span><span class="sxs-lookup"><span data-stu-id="9f918-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="9f918-133">Azure Container Registry 是另一個範例。</span><span class="sxs-lookup"><span data-stu-id="9f918-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="9f918-134">**多架構映像**：針對多重架構，是一項功能，可簡化選取適當的映像，根據 Docker 正在執行，例如，當 Dockerfile 要求的基底映像的平台 **`FROM mcr.microsoft.com/dotnet/core/sdk:2.1`** 從登錄它實際上會取得 **`2.1-nanoserver-1709`**， **`2.1-nanoserver-1803`**， **`2.1-nanoserver-1809`** 或 **`2.1-stretch`**，根據作業系統和執行 Docker 的版本。</span><span class="sxs-lookup"><span data-stu-id="9f918-134">**Multi-arch image**: For multi-architecture, is a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running, for example, when a Dockerfile requests a base image **`FROM mcr.microsoft.com/dotnet/core/sdk:2.1`** from the registry it actually gets **`2.1-nanoserver-1709`**, **`2.1-nanoserver-1803`**, **`2.1-nanoserver-1809`** or **`2.1-stretch`**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="9f918-135">**Docker Hub**：上傳並使用映像的公開登錄。</span><span class="sxs-lookup"><span data-stu-id="9f918-135">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="9f918-136">Docker Hub 提供 Docker 映像裝載、公開或私人登錄、組建觸發程序和 Webhook，以及與 GitHub 和 Bitbucket 的整合。</span><span class="sxs-lookup"><span data-stu-id="9f918-136">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="9f918-137">**Azure Container Registry**：Azure 中使用 Docker 映像及其元件的公用資源。</span><span class="sxs-lookup"><span data-stu-id="9f918-137">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="9f918-138">這提供的登錄，即將在 Azure 中部署，並可讓您控制存取權，讓您可以使用您的 Azure Active Directory 群組和權限。</span><span class="sxs-lookup"><span data-stu-id="9f918-138">This provides a registry that's close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="9f918-139">**Docker Trusted Registry (DTR)**：可在內部部署安裝的 Docker 登錄服務 (來自 Docker)，以便存在於組織的資料中心和網路內。</span><span class="sxs-lookup"><span data-stu-id="9f918-139">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="9f918-140">這對很方便應管理企業內的私人映像。</span><span class="sxs-lookup"><span data-stu-id="9f918-140">It's convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="9f918-141">Docker Trusted Registry 隨附於 Docker Datacenter 產品中。</span><span class="sxs-lookup"><span data-stu-id="9f918-141">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="9f918-142">如需詳細資訊，請參閱 [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/)。</span><span class="sxs-lookup"><span data-stu-id="9f918-142">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="9f918-143">**Dotfuscator Community Edition (CE)**：適用於 Windows 和 macOS 的開發工具，可在本機建置、執行及測試容器。</span><span class="sxs-lookup"><span data-stu-id="9f918-143">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="9f918-144">Docker CE for Windows 提供適用於 Linux 和 Windows 容器的開發環境。</span><span class="sxs-lookup"><span data-stu-id="9f918-144">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="9f918-145">Windows 上的 Linux Docker 主機是以 [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) 虛擬機器為基礎。</span><span class="sxs-lookup"><span data-stu-id="9f918-145">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="9f918-146">Windows 容器的主機則是直接以 Windows 為基礎。</span><span class="sxs-lookup"><span data-stu-id="9f918-146">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="9f918-147">Docker CE for Mac 是以 Apple Hypervisor 架構和 [xhyve hypervisor](https://github.com/mist64/xhyve) 為基礎，其提供 Mac OS X 版的 Linux Docker 主機虛擬機器。Docker CE for Windows 和 Docker CE for Mac 取代以 Oracle VirtualBox 為基礎的 Docker Toolbox。</span><span class="sxs-lookup"><span data-stu-id="9f918-147">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="9f918-148">**Docker Enterprise Edition (EE)**：適用於 Linux 和 Windows 開發之企業級版本的 Docker 工具。</span><span class="sxs-lookup"><span data-stu-id="9f918-148">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="9f918-149">**撰寫**：命令列工具和 YAML 檔案格式，其中繼資料可定義及執行多容器應用程式。</span><span class="sxs-lookup"><span data-stu-id="9f918-149">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="9f918-150">您可以使用一或多個 .yml 檔案，來定義以多個映像為基礎的單一應用程式，這些檔案可能會覆寫相依於環境的值。</span><span class="sxs-lookup"><span data-stu-id="9f918-150">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="9f918-151">您已建立的定義之後，您可以部署整個多容器應用程式使用單一命令 (docker-docker-compose up) Docker 主機上建立一個容器，每個影像。</span><span class="sxs-lookup"><span data-stu-id="9f918-151">After you've created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="9f918-152">**叢集**：以單一虛擬 Docker 主機形式公開的 Docker 主機集合，讓應用程式可以擴充為分散到叢集內多部主機的多個服務執行個體。</span><span class="sxs-lookup"><span data-stu-id="9f918-152">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="9f918-153">您可以使用 Kubernetes、Azure Service Fabric、Docker Swarm 和 Mesosphere DC/OS 來建立 Docker 叢集。</span><span class="sxs-lookup"><span data-stu-id="9f918-153">Docker clusters can be created with Kubernetes, Azure Service Fabric, Docker Swarm and Mesosphere DC/OS.</span></span>

<span data-ttu-id="9f918-154">**協調器**：可簡化叢集和 Docker 主機管理的工具。</span><span class="sxs-lookup"><span data-stu-id="9f918-154">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="9f918-155">協調器可讓您管理其映像、 容器和主機透過命令列介面 (CLI) 或圖形化使用者介面。</span><span class="sxs-lookup"><span data-stu-id="9f918-155">Orchestrators enable you to manage their images, containers, and hosts through a command-line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="9f918-156">您可以管理容器網路功能、組態、負載平衡、服務探索、高可用性、Docker 主機組態等等。</span><span class="sxs-lookup"><span data-stu-id="9f918-156">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="9f918-157">協調器會負責跨節點集合執行、散發、擴充及修復工作負載。</span><span class="sxs-lookup"><span data-stu-id="9f918-157">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="9f918-158">一般而言，協調器產品與提供叢集基礎結構的產品相同，例如在市場中其他供應項目之間的 Kubernetes 和 Azure Service Fabric。</span><span class="sxs-lookup"><span data-stu-id="9f918-158">Typically, orchestrator products are the same products that provide cluster infrastructure, like Kubernetes and Azure Service Fabric, among other offerings in the market.</span></span>
