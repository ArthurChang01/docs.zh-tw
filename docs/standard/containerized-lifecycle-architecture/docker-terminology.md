# <a name="docker-terminology"></a><span data-ttu-id="8c274-103">Docker 術語</span><span class="sxs-lookup"><span data-stu-id="8c274-103">Docker terminology</span></span>

<span data-ttu-id="8c274-104">此區段會列出詞彙和定義，您應該熟悉在深入探究 Docker 之前 (如需進一步定義，請參閱廣泛[詞彙](https://docs.docker.com/glossary/)提供在 Docker <https://docs.docker.com/glossary/>:</span><span class="sxs-lookup"><span data-stu-id="8c274-104">This section lists terms and definitions with which you should become familiar with before delving deeper into Docker (for further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker at <https://docs.docker.com/glossary/>:</span></span>

-   <span data-ttu-id="8c274-105">**容器映像** 封裝的所有相依性和建立容器所需的資訊。</span><span class="sxs-lookup"><span data-stu-id="8c274-105">**Container image** A package with all of the dependencies and information needed to create a container.</span></span> <span data-ttu-id="8c274-106">映像會包含所有的相依性 （例如架構） 以及部署和設定，以供容器執行階段。</span><span class="sxs-lookup"><span data-stu-id="8c274-106">An image includes all of the dependencies (such as frameworks) plus deployment and configuration to be used by a container runtime.</span></span> <span data-ttu-id="8c274-107">通常，映像衍生自多個基底映像，層級堆疊在另一個則以構成容器檔案系統之上的其中一個。</span><span class="sxs-lookup"><span data-stu-id="8c274-107">Usually, an image derives from multiple base images that are layers stacked one atop the other to form the container's file system.</span></span> <span data-ttu-id="8c274-108">一旦建立後，影像是不可變。</span><span class="sxs-lookup"><span data-stu-id="8c274-108">An image is immutable after it has been created.</span></span>

-   <span data-ttu-id="8c274-109">**容器** Docker 映像執行個體。</span><span class="sxs-lookup"><span data-stu-id="8c274-109">**Container** An instance of a Docker image.</span></span> <span data-ttu-id="8c274-110">容器代表單一應用程式、 處理序或服務的執行階段。</span><span class="sxs-lookup"><span data-stu-id="8c274-110">A container represents a runtime for a single application, process, or service.</span></span> <span data-ttu-id="8c274-111">它包含 Docker 映像，執行階段環境和一組標準的指示的內容。</span><span class="sxs-lookup"><span data-stu-id="8c274-111">It consists of the contents of a Docker image, a runtime environment, and a standard set of instructions.</span></span> <span data-ttu-id="8c274-112">擴充服務時，您會從同一個映像建立容器的多個執行個體。</span><span class="sxs-lookup"><span data-stu-id="8c274-112">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="8c274-113">或者，以批次工作可以建立多個容器從相同的映像，並將不同的參數傳遞給每個執行個體。</span><span class="sxs-lookup"><span data-stu-id="8c274-113">Or, a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

-   <span data-ttu-id="8c274-114">**標記** 的標記或，如此就可以識別不同的映像或版本 （取決於版本號碼或目的地環境） 的相同映像，您可以套用至映像的標籤。</span><span class="sxs-lookup"><span data-stu-id="8c274-114">**Tag** A mark or label that you can apply to images so that different images or versions of the same image (depending on the version number or the destination environment) can be identified.</span></span>

-   <span data-ttu-id="8c274-115">**Dockerfile** 文字檔案，其中包含如何建置 Docker 映像的指示。</span><span class="sxs-lookup"><span data-stu-id="8c274-115">**Dockerfile** A text file that contains instructions for how to build a Docker image.</span></span>

-   <span data-ttu-id="8c274-116">**建置** 建置容器映像的動作會根據其 Dockerfile，以及其他檔案的映像的建立位置的資料夾中所提供的內容與資訊。</span><span class="sxs-lookup"><span data-stu-id="8c274-116">**Build** The action of building a container image based on the information and context provided by its Dockerfile as well as additional files in the folder where the image is built.</span></span> <span data-ttu-id="8c274-117">您可以使用 Docker 的 docker build 命令建置映像。</span><span class="sxs-lookup"><span data-stu-id="8c274-117">You can build images by using the Docker docker build command.</span></span>

-   <span data-ttu-id="8c274-118">**存放庫 (也稱為 repo)** 加上標記指出映像版本相關的 Docker 映像的集合。</span><span class="sxs-lookup"><span data-stu-id="8c274-118">**Repository (also known as repo)** A collection of related Docker images labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="8c274-119">某些存放庫包含多個特定的映像，例如包含 Sdk （更繁重），其中包含只執行階段 （較淺） 映像的映像變體，依此類推。</span><span class="sxs-lookup"><span data-stu-id="8c274-119">Some repositories contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), and so on.</span></span> <span data-ttu-id="8c274-120">這些變化可以加上標記。</span><span class="sxs-lookup"><span data-stu-id="8c274-120">Those variants can be marked with tags.</span></span> <span data-ttu-id="8c274-121">單一存放庫可以包含多種平台變化，例如 Linux 映像和 Windows 映像。</span><span class="sxs-lookup"><span data-stu-id="8c274-121">A single repository can contain platform variants, such as a Linux image and a Windows image.</span></span>

-   <span data-ttu-id="8c274-122">**登錄** 服務可提供存取權儲存機制。</span><span class="sxs-lookup"><span data-stu-id="8c274-122">**Registry** A service that provides access to repositories.</span></span> <span data-ttu-id="8c274-123">大多數公用映像的預設登錄是 [Docker Hub](https://hub.docker.com/) (以組織形式為 Docker 所擁有）。</span><span class="sxs-lookup"><span data-stu-id="8c274-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="8c274-124">登錄通常會包含來自多個小組的存放庫。</span><span class="sxs-lookup"><span data-stu-id="8c274-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="8c274-125">公司通常會有私人登錄來儲存和管理他們所建立的映像。</span><span class="sxs-lookup"><span data-stu-id="8c274-125">Companies often have private registries to store and manage images that they've created.</span></span> <span data-ttu-id="8c274-126">*Azure Container Registry*是另一個範例。</span><span class="sxs-lookup"><span data-stu-id="8c274-126">*Azure Container Registry* is another example.</span></span>

-   <span data-ttu-id="8c274-127">**Docker Hub** 影像上傳並使用它們的公開登錄。</span><span class="sxs-lookup"><span data-stu-id="8c274-127">**Docker Hub** A public registry to upload images and work with them.</span></span> <span data-ttu-id="8c274-128">Docker Hub 提供 Docker 映像裝載、公開或私人登錄、組建觸發程序和 Webhook，以及與 GitHub 和 Bitbucket 的整合。</span><span class="sxs-lookup"><span data-stu-id="8c274-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

-   <span data-ttu-id="8c274-129">**Azure Container Registry** 使用 Docker 映像和其元件在 Azure 中的公用資源。</span><span class="sxs-lookup"><span data-stu-id="8c274-129">**Azure Container Registry** A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="8c274-130">這會提供接近 Azure 部署的登錄，並可讓您控制存取權，以便使用您的 Azure Active Directory 群組和權限。</span><span class="sxs-lookup"><span data-stu-id="8c274-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

-   <span data-ttu-id="8c274-131">**Docker Trusted Registry (DTR)** （來自 Docker) 的 Docker 登錄服務，您可以安裝在內部部署，讓它位於組織的資料中心和網路。</span><span class="sxs-lookup"><span data-stu-id="8c274-131">**Docker Trusted Registry (DTR)** A Docker registry service (from Docker) that you can install on-premises so that it resides within the organization's datacenter and network.</span></span> <span data-ttu-id="8c274-132">這會方便在企業內管理私人映像。</span><span class="sxs-lookup"><span data-stu-id="8c274-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="8c274-133">Docker Trusted Registry 隨附於 Docker Datacenter 產品中。</span><span class="sxs-lookup"><span data-stu-id="8c274-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="8c274-134">如需詳細資訊，請移至[ https://docs.docker.com/docker-trusted-registry/overview/ ](https://docs.docker.com/docker-trusted-registry/overview/)。</span><span class="sxs-lookup"><span data-stu-id="8c274-134">For more information, go to [https://docs.docker.com/docker-trusted-registry/overview/](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

-   <span data-ttu-id="8c274-135">**Docker Community Edition (CE)** 適用於 Windows 和 macOS 的建置、 執行和測試在本機容器開發工具。</span><span class="sxs-lookup"><span data-stu-id="8c274-135">**Docker Community Edition (CE)** Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="8c274-136">Docker CE for Windows 提供適用於 Linux 和 Windows 容器的開發環境。</span><span class="sxs-lookup"><span data-stu-id="8c274-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="8c274-137">在 Windows 上的 Linux Docker 主機根據[HYPER-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM。</span><span class="sxs-lookup"><span data-stu-id="8c274-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM.</span></span> <span data-ttu-id="8c274-138">Windows 容器的主機則是直接以 Windows 為基礎。</span><span class="sxs-lookup"><span data-stu-id="8c274-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="8c274-139">Docker CE for Mac 以 Apple Hypervisor 架構為基礎而[xhyve hypervisor](https://github.com/mist64/xhyve)，以提供 Linux Docker 主機 VM，在 Mac OS x。 Docker CE for Windows 和 Mac 取代以 Oracle VirtualBox 為基礎的 Docker Toolbox。</span><span class="sxs-lookup"><span data-stu-id="8c274-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host VM on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

-   <span data-ttu-id="8c274-140">**Docker Enterprise Edition (EE)** 是企業級的舊版適用於 Linux 和 Windows 開發的 Docker 工具。</span><span class="sxs-lookup"><span data-stu-id="8c274-140">**Docker Enterprise Edition (EE)** An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

-   <span data-ttu-id="8c274-141">**Compose** 命令列工具和 YAML 檔案來定義和執行多容器應用程式的中繼資料格式。</span><span class="sxs-lookup"><span data-stu-id="8c274-141">**Compose** A command-line tool and YAML file format with metadata for defining and running multicontainer applications.</span></span> <span data-ttu-id="8c274-142">您可以使用一或多個 .yml 檔案，來定義以多個映像為基礎的單一應用程式，這些檔案可能會覆寫相依於環境的值。</span><span class="sxs-lookup"><span data-stu-id="8c274-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="8c274-143">您已建立的定義之後，您可以部署整個多容器應用程式使用單一命令 (docker-docker-compose up) Docker 主機上建立一個容器，每個影像。</span><span class="sxs-lookup"><span data-stu-id="8c274-143">After you have created the definitions, you can deploy the entire multicontainer application by using a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

-   <span data-ttu-id="8c274-144">**叢集** 如同它們是單一虛擬 Docker 主機，確保應用程式可以擴展多個執行個體的服務所公開的 Docker 主機集合分散在叢集內的多部主機。</span><span class="sxs-lookup"><span data-stu-id="8c274-144">**Cluster** A collection of Docker hosts exposed as if they were a single virtual Docker host so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="8c274-145">您可以使用 Docker Swarm、 Mesosphere DC/OS、 Kubernetes 和 Azure Service Fabric 來建立 Docker 叢集。</span><span class="sxs-lookup"><span data-stu-id="8c274-145">You can create Docker clusters by using Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="8c274-146">(如果您使用 Docker Swarm 來管理叢集，您通常會將叢集稱為「群集」而不是叢集)。</span><span class="sxs-lookup"><span data-stu-id="8c274-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

-   <span data-ttu-id="8c274-147">**Orchestrator** 可簡化叢集和 Docker 主機的管理工具。</span><span class="sxs-lookup"><span data-stu-id="8c274-147">**Orchestrator** A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="8c274-148">使用協調器，您可以管理其映像、 容器和主機透過 CLI 或圖形化使用者介面。</span><span class="sxs-lookup"><span data-stu-id="8c274-148">Using orchestrators, you can manage their images, containers, and hosts through a CLI or a graphical user interface.</span></span> <span data-ttu-id="8c274-149">您可以管理容器網路功能、組態、負載平衡、服務探索、高可用性、Docker 主機組態等等。</span><span class="sxs-lookup"><span data-stu-id="8c274-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="8c274-150">協調器會負責跨節點集合執行、散發、擴充及修復工作負載。</span><span class="sxs-lookup"><span data-stu-id="8c274-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="8c274-151">一般而言，協調器產品與提供叢集基礎結構的產品相同，例如 Mesosphere DC/OS、Kubernetes、Docker Swarm 和 Azure Service Fabric。</span><span class="sxs-lookup"><span data-stu-id="8c274-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
