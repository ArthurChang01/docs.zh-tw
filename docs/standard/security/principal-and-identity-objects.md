## <a name="identity-objects"></a><span data-ttu-id="38e0f-109">Identity 物件</span><span class="sxs-lookup"><span data-stu-id="38e0f-109">Identity Objects</span></span>

 <span data-ttu-id="38e0f-115"><xref:System.Security.Principal.IIdentity>介面定義屬性來存取名稱和驗證類型，例如 Kerberos V5 或 NTLM。</span><span class="sxs-lookup"><span data-stu-id="38e0f-115">The <xref:System.Security.Principal.IIdentity> interface defines properties for accessing a name and an authentication type, such as Kerberos V5 or NTLM.</span></span> <span data-ttu-id="38e0f-116">所有**身分識別**類別會實作 **IIdentity** 介面。</span><span class="sxs-lookup"><span data-stu-id="38e0f-116">All **Identity** classes implement the **IIdentity** interface.</span></span> <span data-ttu-id="38e0f-117">**Identity** 物件和 Windows NT 處理序語彙基元 (執行緒目前在其底下執行) 之間沒有任何必要的關聯性。</span><span class="sxs-lookup"><span data-stu-id="38e0f-117">There is no required relationship between an **Identity** object and the Windows NT process token under which a thread is currently executing.</span></span> <span data-ttu-id="38e0f-118">不過，如果 **Identity** 物件是 **WindowsIdentity** 物件時，身分識別會假設為代表 Windows NT 安全性語彙基元。</span><span class="sxs-lookup"><span data-stu-id="38e0f-118">However, if the **Identity** object is a **WindowsIdentity** object, the identity is assumed to represent a Windows NT security token.</span></span>  
  
## <a name="principal-objects"></a><span data-ttu-id="38e0f-119">Principal 物件</span><span class="sxs-lookup"><span data-stu-id="38e0f-119">Principal Objects</span></span>  
 <span data-ttu-id="38e0f-120">Principal 物件代表執行程式碼所在的安全性內容。</span><span class="sxs-lookup"><span data-stu-id="38e0f-120">The principal object represents the security context under which code is running.</span></span> <span data-ttu-id="38e0f-121">實作以角色為基礎的安全性之應用程式，會根據與 Principal 物件相關聯的角色授與權限。</span><span class="sxs-lookup"><span data-stu-id="38e0f-121">Applications that implement role-based security grant rights based on the role associated with a principal object.</span></span> <span data-ttu-id="38e0f-122">類似於 identity 物件，.NET Framework 會提供<xref:System.Security.Principal.GenericPrincipal>物件和<xref:System.Security.Principal.WindowsPrincipal>物件。</span><span class="sxs-lookup"><span data-stu-id="38e0f-122">Similar to identity objects, the .NET Framework provides a <xref:System.Security.Principal.GenericPrincipal> object and a <xref:System.Security.Principal.WindowsPrincipal> object.</span></span> <span data-ttu-id="38e0f-123">您也可以定義自己的自訂主體類別。</span><span class="sxs-lookup"><span data-stu-id="38e0f-123">You can also define your own custom principal classes.</span></span>  
  
 <span data-ttu-id="38e0f-124"><xref:System.Security.Principal.IPrincipal>介面會定義屬性，以存取相關聯**身分識別**物件的方法來判斷使用者是否由識別**主體**物件是的成員指定的角色。</span><span class="sxs-lookup"><span data-stu-id="38e0f-124">The <xref:System.Security.Principal.IPrincipal> interface defines a property for accessing an associated **Identity** object as well as a method for determining whether the user identified by the **Principal** object is a member of a given role.</span></span> <span data-ttu-id="38e0f-125">所有 **Principal** 類別會實作 **IPrincipal** 介面以及任何其他需要的屬性和方法。</span><span class="sxs-lookup"><span data-stu-id="38e0f-125">All **Principal** classes implement the **IPrincipal** interface as well as any additional properties and methods that are necessary.</span></span> <span data-ttu-id="38e0f-126">例如，通用語言執行平台提供 **WindowsPrincipal** 類別，實作其他功能以將 Windows NT 或 Windows 2000 群組成員資格對應至角色。</span><span class="sxs-lookup"><span data-stu-id="38e0f-126">For example, the common language runtime provides the **WindowsPrincipal** class, which implements additional functionality for mapping Windows NT or Windows 2000 group membership to roles.</span></span>  
  
 <span data-ttu-id="38e0f-127">A**主體**物件所繫結至呼叫內容 (<xref:System.Runtime.Remoting.Messaging.CallContext>) 應用程式定義域中的物件 (<xref:System.AppDomain>)。</span><span class="sxs-lookup"><span data-stu-id="38e0f-127">A **Principal** object is bound to a call context (<xref:System.Runtime.Remoting.Messaging.CallContext>) object within an application domain (<xref:System.AppDomain>).</span></span> <span data-ttu-id="38e0f-128">預設呼叫內容一律使用每個新的 **AppDomain** 建立，因此永遠有呼叫內容可以接受 **Principal** 物件。</span><span class="sxs-lookup"><span data-stu-id="38e0f-128">A default call context is always created with each new **AppDomain**, so there is always a call context available to accept the **Principal** object.</span></span> <span data-ttu-id="38e0f-129">建立新的執行緒時，也會針對執行緒建立 **CallContext** 物件。</span><span class="sxs-lookup"><span data-stu-id="38e0f-129">When a new thread is created, a **CallContext** object is also created for the thread.</span></span> <span data-ttu-id="38e0f-130">**Principal** 物件參考會自動從建立的執行緒複製到新執行緒的 **CallContext**。</span><span class="sxs-lookup"><span data-stu-id="38e0f-130">The **Principal** object reference is automatically copied from the creating thread to the new thread's **CallContext**.</span></span> <span data-ttu-id="38e0f-131">如果執行階段無法判斷哪個 **Principal** 物件屬於執行緒的建立者，它會依照 **Principal** 和 **Identity** 物件建立的預設原則。</span><span class="sxs-lookup"><span data-stu-id="38e0f-131">If the runtime cannot determine which **Principal** object belongs to the creator of the thread, it follows the default policy for **Principal** and **Identity** object creation.</span></span>  
  
 <span data-ttu-id="38e0f-132">可設定的應用程式定義域特定原則會定義規則，來決定哪種類型的 **Principal** 物件與新的應用程式定義域相關聯。</span><span class="sxs-lookup"><span data-stu-id="38e0f-132">A configurable application domain-specific policy defines the rules for deciding what type of **Principal** object to associate with a new application domain.</span></span> <span data-ttu-id="38e0f-133">安全性原則允許時，執行階段可以建立 **Principal** 和 **Identity** 物件，反映與目前執行緒相關聯的作業系統語彙基元。</span><span class="sxs-lookup"><span data-stu-id="38e0f-133">Where security policy permits, the runtime can create **Principal** and **Identity** objects that reflect the operating system token associated with the current thread of execution.</span></span> <span data-ttu-id="38e0f-134">根據預設，執行階段會使用 **Principal** 和 **Identity** 物件，代表未經授權的使用者。</span><span class="sxs-lookup"><span data-stu-id="38e0f-134">By default, the runtime uses **Principal** and **Identity** objects that represent unauthenticated users.</span></span> <span data-ttu-id="38e0f-135">執行階段不會建立這些預設 **Principal** 和 **Identity** 物件，直到程式碼嘗試存取它們為止。</span><span class="sxs-lookup"><span data-stu-id="38e0f-135">The runtime does not create these default **Principal** and **Identity** objects until the code attempts to access them.</span></span>  
  
 <span data-ttu-id="38e0f-136">會建立應用程式定義域之信任的程式碼，可以設定應用程式定義域原則，控制預設 **Principal** 和 **Identity** 物件的建構。</span><span class="sxs-lookup"><span data-stu-id="38e0f-136">Trusted code that creates an application domain can set the application domain policy that controls construction of the default **Principal** and **Identity** objects.</span></span> <span data-ttu-id="38e0f-137">此應用程式定義域專屬的原則適用於所有應用程式定義域中的執行緒。</span><span class="sxs-lookup"><span data-stu-id="38e0f-137">This application domain-specific policy applies to all execution threads in that application domain.</span></span> <span data-ttu-id="38e0f-138">未受管理、 受信任主機本來就具有能夠設定此原則，但必須設定此原則的 managed 程式碼<xref:System.Security.Permissions.SecurityPermission?displayProperty=nameWithType>以控制定義域原則。</span><span class="sxs-lookup"><span data-stu-id="38e0f-138">An unmanaged, trusted host inherently has the ability to set this policy, but managed code that sets this policy must have the <xref:System.Security.Permissions.SecurityPermission?displayProperty=nameWithType> for controlling domain policy.</span></span>  
  
 <span data-ttu-id="38e0f-139">跨應用程式定義域但是在相同處理序內傳輸 **Principal** 物件時 (因此是在同一部電腦上)，遠端基礎結構會將參考複製到 **Principal** 物件，該物件將呼叫者的內容與被呼叫端的內容產生關聯。</span><span class="sxs-lookup"><span data-stu-id="38e0f-139">When transmitting a **Principal** object across application domains but within the same process (and therefore on the same computer), the remoting infrastructure copies a reference to the **Principal** object associated with the caller's context to the callee's context.</span></span>
