## <a name="fundamentals-of-memory"></a><span data-ttu-id="c48bd-122">記憶體的基本概念</span><span class="sxs-lookup"><span data-stu-id="c48bd-122">Fundamentals of memory</span></span>  
 <span data-ttu-id="c48bd-123">下列清單摘要說明重要的 CLR 記憶體概念。</span><span class="sxs-lookup"><span data-stu-id="c48bd-123">The following list summarizes important CLR memory concepts.</span></span>  
  
-   <span data-ttu-id="c48bd-124">每個處理序都有各自獨立的虛擬位址空間。</span><span class="sxs-lookup"><span data-stu-id="c48bd-124">Each process has its own, separate virtual address space.</span></span> <span data-ttu-id="c48bd-125">同一部電腦上的所有處理序會共用相同的實體記憶體，並且共用分頁檔 (如果有的話)。</span><span class="sxs-lookup"><span data-stu-id="c48bd-125">All processes on the same computer share the same physical memory, and share the page file if there is one.</span></span>  
  
-   <span data-ttu-id="c48bd-126">根據預設，在 32 位元電腦上，每個處理序都有 2 GB 使用者模式虛擬位址空間。</span><span class="sxs-lookup"><span data-stu-id="c48bd-126">By default, on 32-bit computers, each process has a 2-GB user-mode virtual address space.</span></span>  
  
-   <span data-ttu-id="c48bd-127">身為應用程式開發人員，您只會處理虛擬位址空間，絕不會直接操作實體記憶體。</span><span class="sxs-lookup"><span data-stu-id="c48bd-127">As an application developer, you work only with virtual address space and never manipulate physical memory directly.</span></span> <span data-ttu-id="c48bd-128">記憶體回收行程會在 Managed 堆積上自動配置和釋出虛擬記憶體。</span><span class="sxs-lookup"><span data-stu-id="c48bd-128">The garbage collector allocates and frees virtual memory for you on the managed heap.</span></span>  
  
     <span data-ttu-id="c48bd-129">如果您要撰寫原生程式碼，可使用 Win32 函式來處理虛擬位址空間。</span><span class="sxs-lookup"><span data-stu-id="c48bd-129">If you are writing native code, you use Win32 functions to work with the virtual address space.</span></span> <span data-ttu-id="c48bd-130">這些函式會在原生堆積上自動配置和釋出虛擬記憶體。</span><span class="sxs-lookup"><span data-stu-id="c48bd-130">These functions allocate and free virtual memory for you on native heaps.</span></span>  
  
-   <span data-ttu-id="c48bd-131">虛擬記憶體可以有三種狀態：</span><span class="sxs-lookup"><span data-stu-id="c48bd-131">Virtual memory can be in three states:</span></span>

-   <span data-ttu-id="c48bd-143">如果您用盡保留用的虛擬位址空間或認可用的實體空間，則可能會用盡記憶體。</span><span class="sxs-lookup"><span data-stu-id="c48bd-143">You can run out of memory if you run out of virtual address space to reserve or physical space to commit.</span></span>

 <span data-ttu-id="c48bd-332">背景記憶體回收期間，暫時層代的回收稱為前景記憶體回收。</span><span class="sxs-lookup"><span data-stu-id="c48bd-332">A collection on ephemeral generations during background garbage collection is known as foreground garbage collection.</span></span> <span data-ttu-id="c48bd-333">進行前景記憶體回收時，所有 Managed 執行緒都會暫停。</span><span class="sxs-lookup"><span data-stu-id="c48bd-333">When foreground garbage collections occur, all managed threads are suspended.</span></span>

 <span data-ttu-id="c48bd-338">背景記憶體回收會移除並行記憶體回收所加諸的配置限制，因為暫時記憶體回收可能會在背景記憶體回收期間進行。</span><span class="sxs-lookup"><span data-stu-id="c48bd-338">Background garbage collection removes allocation restrictions imposed by concurrent garbage collection, because ephemeral garbage collections can occur during background garbage collection.</span></span> <span data-ttu-id="c48bd-339">這表示，背景記憶體回收可以移除暫時層代中的無作用物件，而且也可以在層代 1 記憶體回收期間視需要擴展堆積。</span><span class="sxs-lookup"><span data-stu-id="c48bd-339">This means that background garbage collection can remove dead objects in ephemeral generations and can also expand the heap if needed during a generation 1 garbage collection.</span></span>
