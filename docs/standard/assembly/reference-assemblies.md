## <a name="reference-assemblies-structure"></a><span data-ttu-id="6bbb1-154">參考元件結構</span><span class="sxs-lookup"><span data-stu-id="6bbb1-154">Reference assemblies structure</span></span>

<span data-ttu-id="6bbb1-155">參考元件是相關概念的擴充，*僅限中繼資料的元件*。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-155">Reference assemblies are an expansion of the related concept, *metadata-only assemblies*.</span></span> <span data-ttu-id="6bbb1-156">僅中繼資料組件以單一的 `throw null` 主體取代其方法主體，但包含匿名型別以外的所有成員。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-156">Metadata-only assemblies have their method bodies replaced with a single `throw null` body, but include all members except anonymous types.</span></span> <span data-ttu-id="6bbb1-157">使用 `throw null` 主體（而不是任何主體）的原因，是為了讓**PEVerify**能夠執行和傳遞（藉此驗證中繼資料的完整性）。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-157">The reason for using `throw null` bodies (as opposed to no bodies) is so that **PEVerify** can run and pass (thus validating the completeness of the metadata).</span></span>

<span data-ttu-id="6bbb1-158">參考組件會進一步移除來自僅中繼資料組件的中繼資料 (私用成員)：</span><span class="sxs-lookup"><span data-stu-id="6bbb1-158">Reference assemblies further remove metadata (private members) from metadata-only assemblies:</span></span>

- <span data-ttu-id="6bbb1-159">參考組件只有在 API 介面中所需項目的參考。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-159">A reference assembly only has references for what it needs in the API surface.</span></span> <span data-ttu-id="6bbb1-160">實際的組件可能有與特定實作相關的其他參考。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-160">The real assembly may have additional references related to specific implementations.</span></span> <span data-ttu-id="6bbb1-161">例如，`class C { private void M() { dynamic d = 1; ... } }` 的參考元件不會參考 `dynamic`所需的任何類型。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-161">For instance, the reference assembly for `class C { private void M() { dynamic d = 1; ... } }` doesn't reference any types required for `dynamic`.</span></span>
- <span data-ttu-id="6bbb1-162">如果移除它們不會明顯影響編譯的話，則會移除私用函式的成員 (方法、屬性和事件)。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-162">Private function-members (methods, properties, and events) are removed in cases where their removal doesn't observably impact compilation.</span></span> <span data-ttu-id="6bbb1-163">如果沒有[InternalsVisibleTo](xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute)屬性，內部函式成員也會一併移除。</span><span class="sxs-lookup"><span data-stu-id="6bbb1-163">If there are no [InternalsVisibleTo](xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute) attributes, internal function members are also removed.</span></span>
