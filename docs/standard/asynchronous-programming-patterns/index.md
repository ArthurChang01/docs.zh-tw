# <a name="asynchronous-programming-patterns"></a><span data-ttu-id="ba3ce-102">非同步程式設計模式</span><span class="sxs-lookup"><span data-stu-id="ba3ce-102">Asynchronous programming patterns</span></span>

<span data-ttu-id="ba3ce-103">.NET 提供三種模式來執行非同步作業：</span><span class="sxs-lookup"><span data-stu-id="ba3ce-103">.NET provides three patterns for performing asynchronous operations:</span></span>  

- <span data-ttu-id="ba3ce-104">**基於任務的非同步模式 （TAP），** 它使用單個方法表示非同步作業的啟動和完成。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-104">**Task-based Asynchronous Pattern (TAP)**, which uses a single method to represent the initiation and completion of an asynchronous operation.</span></span> <span data-ttu-id="ba3ce-105">TAP 是在 .NET Framework 4 中引進。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-105">TAP was introduced in the .NET Framework 4.</span></span> <span data-ttu-id="ba3ce-106">**它是在 .NET 中進行非同步程式設計的建議方法。**</span><span class="sxs-lookup"><span data-stu-id="ba3ce-106">**It's the recommended approach to asynchronous programming in .NET.**</span></span> <span data-ttu-id="ba3ce-107">C# 中的 [async](../../csharp/language-reference/keywords/async.md) 與 [await](../../csharp/language-reference/operators/await.md) 關鍵字，以及 Visual Basic 中的 [Async](../../visual-basic/language-reference/modifiers/async.md) 與 [Await](../../visual-basic/language-reference/operators/await-operator.md) 運算子，都加入對 TAP 的語言支援。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-107">The [async](../../csharp/language-reference/keywords/async.md) and [await](../../csharp/language-reference/operators/await.md) keywords in C# and the [Async](../../visual-basic/language-reference/modifiers/async.md) and [Await](../../visual-basic/language-reference/operators/await-operator.md) operators in Visual Basic add language support for TAP.</span></span> <span data-ttu-id="ba3ce-108">如需詳細資訊，請參閱[以工作為基礎的非同步模式 (TAP)](task-based-asynchronous-pattern-tap.md)</span><span class="sxs-lookup"><span data-stu-id="ba3ce-108">For more information, see [Task-based Asynchronous Pattern (TAP)](task-based-asynchronous-pattern-tap.md).</span></span>  

- <span data-ttu-id="ba3ce-109">**事件架構非同步模式 (EAP)**，這是用於提供非同步行為的事件架構傳統模型。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-109">**Event-based Asynchronous Pattern (EAP)**, which is the event-based legacy model for providing asynchronous behavior.</span></span> <span data-ttu-id="ba3ce-110">它需要一個具有 `Async` 尾碼的方法、一或多個事件，事件處理常式委派類型，以及 `EventArg` 衍生類型。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-110">It requires a method that has the `Async` suffix and one or more events, event handler delegate types, and `EventArg`-derived types.</span></span> <span data-ttu-id="ba3ce-111">在 .NET Framework 2.0 中採用了 EAP。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-111">EAP was introduced in the .NET Framework 2.0.</span></span> <span data-ttu-id="ba3ce-112">在新的程式開發時，不再建議使用它。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-112">It's no longer recommended for new development.</span></span> <span data-ttu-id="ba3ce-113">如需詳細資訊，請參閱[事件架構非同步模式 (EAP)](event-based-asynchronous-pattern-eap.md)。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-113">For more information, see [Event-based Asynchronous Pattern (EAP)](event-based-asynchronous-pattern-eap.md).</span></span>  

- <span data-ttu-id="ba3ce-114">**非同步程式設計模型 (APM)** 模式 (亦稱為 <xref:System.IAsyncResult> 模式)，它是使用 <xref:System.IAsyncResult> 介面來提供非同步行為的傳統模型。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-114">**Asynchronous Programming Model (APM)** pattern (also called the <xref:System.IAsyncResult> pattern), which is the legacy model that uses the <xref:System.IAsyncResult> interface to provide asynchronous behavior.</span></span> <span data-ttu-id="ba3ce-115">在此模式中，同步作業需要 `Begin` 與 `End` 方法 (例如，`BeginWrite` 與 `EndWrite` 來實作非同步寫入作業)。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-115">In this pattern, synchronous operations require `Begin` and `End` methods (for example, `BeginWrite` and `EndWrite` to implement an asynchronous write operation).</span></span> <span data-ttu-id="ba3ce-116">在新的程式開發時，不建議使用此模式。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-116">This pattern is no longer recommended for new development.</span></span> <span data-ttu-id="ba3ce-117">如需詳細資訊，請參閱[非同步程式設計模型 (APM)](asynchronous-programming-model-apm.md)。</span><span class="sxs-lookup"><span data-stu-id="ba3ce-117">For more information, see [Asynchronous Programming Model (APM)](asynchronous-programming-model-apm.md).</span></span>  
  
## <a name="comparison-of-patterns"></a><span data-ttu-id="ba3ce-118">模式的比較</span><span class="sxs-lookup"><span data-stu-id="ba3ce-118">Comparison of patterns</span></span>

<span data-ttu-id="ba3ce-119">若要快速比較三個模式如何建立非同步作業的模型，請考慮 `Read` 方法，它會將指定的資料量讀取到所提供的緩衝區，並且在指定的位移處開始：</span><span class="sxs-lookup"><span data-stu-id="ba3ce-119">For a quick comparison of how the three patterns model asynchronous operations, consider a `Read` method that reads a specified amount of data into a provided buffer starting at a specified offset:</span></span>  
  
```csharp  
public class MyClass  
{  
    public int Read(byte [] buffer, int offset, int count);  
}  
```  

<span data-ttu-id="ba3ce-120">此方法的 TAP 對應項會公開下列單一 `ReadAsync` 方法：</span><span class="sxs-lookup"><span data-stu-id="ba3ce-120">The TAP counterpart of this method would expose the following single `ReadAsync` method:</span></span>  
  
```csharp
public class MyClass  
{  
    public Task<int> ReadAsync(byte [] buffer, int offset, int count);  
}  
```

<span data-ttu-id="ba3ce-121">EAP 對應項會公開下列類型和成員的集合：</span><span class="sxs-lookup"><span data-stu-id="ba3ce-121">The EAP counterpart would expose the following set of types and members:</span></span>  
  
```csharp  
public class MyClass  
{  
    public void ReadAsync(byte [] buffer, int offset, int count);  
    public event ReadCompletedEventHandler ReadCompleted;  
}  
```  
  
<span data-ttu-id="ba3ce-122">APM 對應項會公開 `BeginRead` 與 `EndRead` 方法：</span><span class="sxs-lookup"><span data-stu-id="ba3ce-122">The APM counterpart would expose the `BeginRead` and `EndRead` methods:</span></span>  
  
```csharp  
public class MyClass  
{  
    public IAsyncResult BeginRead(  
        byte [] buffer, int offset, int count,
        AsyncCallback callback, object state);  
    public int EndRead(IAsyncResult asyncResult);  
}  
```
