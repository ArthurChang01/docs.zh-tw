# <a name="state-and-data-in-docker-applications"></a><span data-ttu-id="1d0cd-104">狀態和 Docker 應用程式中的資料</span><span class="sxs-lookup"><span data-stu-id="1d0cd-104">State and data in Docker applications</span></span>

<span data-ttu-id="1d0cd-105">在大部分情況下，您可以將做為處理序的執行個體的容器。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-105">In most cases, you can think of a container as an instance of a process.</span></span> <span data-ttu-id="1d0cd-106">處理程序不會維護永續性狀態。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-106">A process does not maintain persistent state.</span></span> <span data-ttu-id="1d0cd-107">假設執行個體將會無限期周圍是容器可以寫入其本機儲存體，而就像假設記憶體中的單一位置將會永久性。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-107">While a container can write to its local storage, assuming that an instance will be around indefinitely would be like assuming that a single location in memory will be durable.</span></span> <span data-ttu-id="1d0cd-108">容器映像，例如處理程序，應該假設有多個執行個體，或它們就最終會終止。如果它們透過容器 orchestrator 管理，它應該假設，它們可能會取得一個節點或 VM 間移動。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-108">Container images, like processes, should be assumed to have multiple instances or that they will eventually be killed; if they’re managed with a container orchestrator, it should be assumed that they might get moved from one node or VM to another.</span></span>

<span data-ttu-id="1d0cd-109">Docker 提供功能，稱為*重疊檔案系統*。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-109">Docker provides a feature named the *overlay file system*.</span></span> <span data-ttu-id="1d0cd-110">這會實作寫入時複製工作存放區更新容器的根檔案系統的資訊。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-110">This implements a copy-on-write task that stores updated information to the root file system of the container.</span></span> <span data-ttu-id="1d0cd-111">該資訊是另外為原始的映像所依據的容器。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-111">That information is in addition to the original image on which the container is based.</span></span> <span data-ttu-id="1d0cd-112">如果從系統刪除容器，則會失去這些變更。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-112">If the container is deleted from the system, those changes are lost.</span></span> <span data-ttu-id="1d0cd-113">因此，雖然也可以儲存在其本機儲存體容器的狀態，但是設計系統，以解決此衝突與容器的設計，其預設值是無狀態的內部部署。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-113">Therefore, while it is possible to save the state of a container within its local storage, designing a system around this would conflict with the premise of container design, which by default is stateless.</span></span>

<span data-ttu-id="1d0cd-114">下列解決方案可用於管理 Docker 應用程式中的永續性資料：</span><span class="sxs-lookup"><span data-stu-id="1d0cd-114">The following solutions are used to manage persistent data in Docker applications:</span></span>

-   <span data-ttu-id="1d0cd-115">[資料磁碟區](https://docs.docker.com/engine/tutorials/dockervolumes/)，掛接至主機。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-115">[Data volumes](https://docs.docker.com/engine/tutorials/dockervolumes/) that mount to the host.</span></span>

-   <span data-ttu-id="1d0cd-116">[資料磁碟區容器](https://docs.docker.com/engine/tutorials/dockervolumes/#creating-and-mounting-a-data-volume-container)可提供共用存放裝置使用的外部容器的容器。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-116">[Data volume containers](https://docs.docker.com/engine/tutorials/dockervolumes/#creating-and-mounting-a-data-volume-container) that provide shared storage across containers using an external container.</span></span>

-   <span data-ttu-id="1d0cd-117">[磁碟區外掛程式](https://docs.docker.com/engine/tutorials/dockervolumes/)的掛接磁碟區，以遠端服務，才能提供長期持續性。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-117">[Volume plugins](https://docs.docker.com/engine/tutorials/dockervolumes/) that mount volumes to remote services, providing long-term persistence.</span></span>

-   <span data-ttu-id="1d0cd-118">[Azure 儲存體](https://docs.microsoft.com/azure/storage/)，這樣會提供可散布異地儲存體，容器提供很好的長期持續性解決方案。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-118">[Azure Storage](https://docs.microsoft.com/azure/storage/), which provides geo-distributable storage, providing a good long-term persistence solution for containers.</span></span>

-   <span data-ttu-id="1d0cd-119">遠端關聯式資料庫喜歡[Azure SQL Database](https://azure.microsoft.com/services/sql-database/)或是想要的 NoSQL 資料庫[Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction)，或是快取服務，例如[Redis](https://redis.io/)。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-119">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/) or NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="1d0cd-120">以下提供有關這些選項的更多詳細資料。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-120">The following provides more detail about these options.</span></span>

<span data-ttu-id="1d0cd-121">**資料磁碟區**主機 OS 從對應到容器中的目錄的目錄。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-121">**Data volumes** are directories mapped from the host OS to directories in containers.</span></span> <span data-ttu-id="1d0cd-122">當容器中的程式碼可存取主機作業系統上的目錄實際上是存取的目錄。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-122">When code in the container has access to the directory, that access is actually to a directory on the host OS.</span></span> <span data-ttu-id="1d0cd-123">此目錄未繫結至容器本身的存留期和從直接在主機作業系統上執行的程式碼可以存取目錄，或由另一個容器對應相同的主機目錄至本身。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-123">This directory is not tied to the lifetime of the container itself, and the directory can be accessed from code running directly on the host OS or by another container that maps the same host directory to itself.</span></span> <span data-ttu-id="1d0cd-124">因此，資料磁碟區被設計來保存資料與容器的生命週期無關。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-124">Thus, data volumes are designed to persist data independently of the life of the container.</span></span> <span data-ttu-id="1d0cd-125">如果您刪除容器或映像從 Docker 主機時，將資料保存在資料磁碟區不會刪除。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-125">If you delete a container or an image from the Docker host, the data persisted in the data volume is not deleted.</span></span> <span data-ttu-id="1d0cd-126">從主機作業系統，以及可以存取磁碟區中的資料。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-126">The data in a volume can be accessed from the host OS as well.</span></span>

<span data-ttu-id="1d0cd-127">**資料磁碟區容器**是演進而來的一般資料磁碟區。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-127">**Data volume containers** are an evolution of regular data volumes.</span></span> <span data-ttu-id="1d0cd-128">資料磁碟區容器是簡單的容器具有中的一個或多個資料磁碟。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-128">A data volume container is a simple container that has one or more data volumes within it.</span></span> <span data-ttu-id="1d0cd-129">從中央的掛接點，資料磁碟區容器會提供存取至容器。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-129">The data volume container provides access to containers from a central mount point.</span></span> <span data-ttu-id="1d0cd-130">這個方法的資料存取是很方便，因為它擷取原始資料的位置。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-130">This method of data access is convenient because it abstracts the location of the original data.</span></span> <span data-ttu-id="1d0cd-131">除此之外，其行為未類似於一般資料磁碟區，讓資料保存於獨立應用程式的容器的生命週期此專用的容器。</span><span class="sxs-lookup"><span data-stu-id="1d0cd-131">Other than that, its behavior is similar to that of a regular data volume, so data is persisted in this dedicated container independently of the lifecycle of the application’s containers.</span></span>
