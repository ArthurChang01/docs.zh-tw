<span data-ttu-id="6dfb5-114">相較之下，複合 UI 精確地產生並由 microservices 本身組成。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-114">In contrast, a composite UI is precisely generated and composed by the microservices themselves.</span></span> <span data-ttu-id="6dfb5-115">某些 microservices 磁碟機視覺 UI 的特定區域的形狀。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-115">Some of the microservices drive the visual shape of specific areas of the UI.</span></span> <span data-ttu-id="6dfb5-116">主要差異是，您必須根據範本，用戶端 UI 元件 （例如 TS 類別），而且這些範本資料成形 UI ViewModel 來自每個微服務。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-116">The key difference is that you have client UI components (TS classes, for example) based on templates, and the data-shaping-UI ViewModel for those templates comes from each microservice.</span></span>

<span data-ttu-id="6dfb5-117">在用戶端應用程式啟動時，每個用戶端 UI 元件 （例如 TypeScript 類別） 註冊其本身與基礎結構微服務能夠提供 ViewModels 特定案例。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-117">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="6dfb5-118">如果微服務變更形狀，UI 也會變更。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-118">If the microservice changes the shape, the UI changes also.</span></span>

<span data-ttu-id="6dfb5-119">圖 4-21 顯示此複合 UI 方法的版本。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-119">Figure 4-21 shows a version of this composite UI approach.</span></span> <span data-ttu-id="6dfb5-120">這已經過簡化，因為您可能會有彙總不同的技術為基礎的精細的部分其他 microservices — 這取決於您要建置的傳統 web 方法 (ASP.NET MVC) 或 SPA （單一頁面應用程式）。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-120">This is simplified, because you might have other microservices that are aggregating granular parts based on different techniques—it depends on whether you are building a traditional web approach (ASP.NET MVC) or an SPA (Single Page Application).</span></span>

<span data-ttu-id="6dfb5-125">由 microservices 所驅動的複合 UI 方法可以是更具挑戰性或較少，依據哪些 UI 技術使用。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-125">A composite UI approach that is driven by microservices can be more challenging or less so, depending on what UI technologies you are using.</span></span> <span data-ttu-id="6dfb5-126">比方說，您將不使用相同的技術來建置建置 SPA 或原生行動裝置應用程式使用的傳統 web 應用程式 （做為開發的 Xamarin 應用程式，它可以是這種方法更具挑戰性）。</span><span class="sxs-lookup"><span data-stu-id="6dfb5-126">For instance, you will not use the same techniques for building a traditional web application that you use for building an SPA or for native mobile app (as when developing Xamarin apps, which can be more challenging for this approach).</span></span>
