# <a name="docker-terminology"></a><span data-ttu-id="f7c4d-103">Docker 術語</span><span class="sxs-lookup"><span data-stu-id="f7c4d-103">Docker terminology</span></span>

<span data-ttu-id="f7c4d-104">本節列出您應該熟悉才能深入探索 Docker 的術語和定義。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="f7c4d-105">如需進一步定義，請參閱 Docker 所提供的詳盡[字彙表](https://docs.docker.com/glossary/)。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker .</span></span>

<span data-ttu-id="f7c4d-106">**容器映像**：包含建立容器所需之所有相依性和資訊的封裝。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="f7c4d-107">映像會包含所有相依性 (例如架構)，以及容器執行階段所要使用的部署和執行組態。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="f7c4d-108">通常，一個映像會衍生自多個基底映像，這些基底映像是彼此交互堆疊以構成容器檔案系統的圖層。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="f7c4d-109">映像一旦建立，就不可改變。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="f7c4d-110">**容器**：Docker 映像的執行個體。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-110">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="f7c4d-111">容器代表單一應用程式、處理序或服務的執行。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-111">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="f7c4d-112">其中包含 Docker 映像的內容、執行環境和一組標準的指示。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-112">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="f7c4d-113">擴充服務時，您會從同一個映像建立容器的多個執行個體。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-113">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="f7c4d-114">或者，一個批次工作可以從同一個映像建立多個容器，並將不同的參數傳遞至每個執行個體。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-114">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="f7c4d-115">**標記**：您可以套用至映像的標記或標籤，以便識別相同映像的不同映像版本 (視版本號碼或目標環境而定)。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-115">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="f7c4d-116">**Dockerfile**：文字檔，其中包含有關如何建立 Docker 映像的指示。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-116">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="f7c4d-117">**建立**：建立容器映像的動作，該映像會以其 Dockerfile 及建立映像之資料夾中其他檔案所提供的資訊和內容為依據。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-117">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="f7c4d-118">您可以使用 Docker 的 docker build 命令來建立映像。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-118">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="f7c4d-119">**存放庫 (Repository 或 Repo)**：相關的 Docker 映像集合，已加上標記指出映像版本。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-119">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="f7c4d-120">某些存放庫包含特定映像的多種變化，例如包含 SDK 的映像 (較大)、只包含執行階段的映像 (較小) 等等。這些變化可以加上標記。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-120">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="f7c4d-121">一個存放庫可以包含多種平台變化，例如 Linux 映像和 Windows 映像。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-121">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="f7c4d-122">**登錄**：提供存放庫存取權的服務。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-122">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="f7c4d-123">大多數公用映像的預設登錄是 [Docker Hub](https://hub.docker.com/) (以組織形式為 Docker 所擁有）。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="f7c4d-124">登錄通常會包含來自多個小組的存放庫。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="f7c4d-125">公司通常會有私人登錄來儲存及管理其所建立的映像。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-125">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="f7c4d-126">Azure Container Registry 是另一個範例。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-126">Azure Container Registry is another example.</span></span>

<span data-ttu-id="f7c4d-127">**Docker Hub**：上傳並使用映像的公開登錄。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-127">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="f7c4d-128">Docker Hub 提供 Docker 映像裝載、公開或私人登錄、組建觸發程序和 Webhook，以及與 GitHub 和 Bitbucket 的整合。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="f7c4d-129">**Azure Container Registry**：Azure 中使用 Docker 映像及其元件的公用資源。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-129">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="f7c4d-130">這會提供接近 Azure 部署的登錄，並可讓您控制存取權，以便使用您的 Azure Active Directory 群組和權限。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="f7c4d-131">**Docker Trusted Registry (DTR)**：可在內部部署安裝的 Docker 登錄服務 (來自 Docker)，以便存在於組織的資料中心和網路內。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-131">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="f7c4d-132">這會方便在企業內管理私人映像。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="f7c4d-133">Docker Trusted Registry 隨附於 Docker Datacenter 產品中。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="f7c4d-134">如需詳細資訊，請參閱 [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/)。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-134">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="f7c4d-135">**Docker Community Edition (CE)**：適用於 Windows 和 macOS 的開發工具，可在本機建置、執行及測試容器。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-135">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="f7c4d-136">Docker CE for Windows 提供適用於 Linux 和 Windows 容器的開發環境。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="f7c4d-137">Windows 上的 Linux Docker 主機是以 [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) 虛擬機器為基礎。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="f7c4d-138">Windows 容器的主機則是直接以 Windows 為基礎。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="f7c4d-139">Docker CE for Mac 是以 Apple Hypervisor 架構和 [xhyve hypervisor](https://github.com/mist64/xhyve) 為基礎，其提供 Mac OS X 版的 Linux Docker 主機虛擬機器。Docker CE for Windows 和 Docker CE for Mac 取代以 Oracle VirtualBox 為基礎的 Docker Toolbox。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="f7c4d-140">**Docker Enterprise Edition (EE)**：適用於 Linux 和 Windows 開發之企業級版本的 Docker 工具。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-140">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="f7c4d-141">**撰寫**：命令列工具和 YAML 檔案格式，其中繼資料可定義及執行多容器應用程式。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-141">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="f7c4d-142">您可以使用一或多個 .yml 檔案，來定義以多個映像為基礎的單一應用程式，這些檔案可能會覆寫相依於環境的值。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="f7c4d-143">建立定義之後，您可以使用單一命令 (docker-compose up) 來部署整個多容器應用程式，以在 Docker 主機上針對每個映像建立一個容器。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-143">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="f7c4d-144">**叢集**：以單一虛擬 Docker 主機形式公開的 Docker 主機集合，讓應用程式可以擴充為分散到叢集內多部主機的多個服務執行個體。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-144">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="f7c4d-145">您可以使用 Docker Swarm、Mesosphere DC/OS、Kubernetes 和 Azure Service Fabric 來建立 Docker 叢集</span><span class="sxs-lookup"><span data-stu-id="f7c4d-145">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="f7c4d-146">(如果您使用 Docker Swarm 來管理叢集，您通常會將叢集稱為「群集」而不是叢集)。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="f7c4d-147">**協調器**：可簡化叢集和 Docker 主機管理的工具。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-147">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="f7c4d-148">協調器可讓您透過命令列介面 (CLI) 或圖形化 UI 來管理其映像、容器和主機。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-148">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="f7c4d-149">您可以管理容器網路功能、組態、負載平衡、服務探索、高可用性、Docker 主機組態等等。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="f7c4d-150">協調器會負責跨節點集合執行、散發、擴充及修復工作負載。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="f7c4d-151">一般而言，協調器產品與提供叢集基礎結構的產品相同，例如 Mesosphere DC/OS、Kubernetes、Docker Swarm 和 Azure Service Fabric。</span><span class="sxs-lookup"><span data-stu-id="f7c4d-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
