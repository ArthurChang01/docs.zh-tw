# <a name="docker-terminology"></a><span data-ttu-id="36d6b-104">Docker 術語</span><span class="sxs-lookup"><span data-stu-id="36d6b-104">Docker terminology</span></span>

<span data-ttu-id="36d6b-105">此區段會列出詞彙和定義，您應該要熟悉之前深入 Docker。</span><span class="sxs-lookup"><span data-stu-id="36d6b-105">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="36d6b-106">如需進一步定義，請參閱廣泛[詞彙](https://docs.docker.com/v1.11/engine/reference/glossary/)Docker 所提供。</span><span class="sxs-lookup"><span data-stu-id="36d6b-106">For further definitions, see the extensive [glossary](https://docs.docker.com/v1.11/engine/reference/glossary/) provided by Docker .</span></span>

<span data-ttu-id="36d6b-107">**容器映像**: 套件與所有相依性和建立容器所需的資訊。</span><span class="sxs-lookup"><span data-stu-id="36d6b-107">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="36d6b-108">映像包括所有相依性 （例如架構），再加上部署和執行容器執行階段所使用的組態。</span><span class="sxs-lookup"><span data-stu-id="36d6b-108">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="36d6b-109">通常，映像會衍生自會彼此交互堆疊以構成容器的檔案系統層級的多個基底映像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-109">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="36d6b-110">一旦建立之後，映像不變。</span><span class="sxs-lookup"><span data-stu-id="36d6b-110">An image is immutable once it has been created.</span></span>

<span data-ttu-id="36d6b-111">**容器**： 的 Docker 映像的執行個體。</span><span class="sxs-lookup"><span data-stu-id="36d6b-111">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="36d6b-112">容器代表執行單一應用程式、 處理序或服務。</span><span class="sxs-lookup"><span data-stu-id="36d6b-112">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="36d6b-113">其中包含內容的 Docker 映像、 執行環境，以及一組標準的指示。</span><span class="sxs-lookup"><span data-stu-id="36d6b-113">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="36d6b-114">擴充服務，當您從相同的映像建立容器的多個執行個體。</span><span class="sxs-lookup"><span data-stu-id="36d6b-114">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="36d6b-115">或批次工作可以建立多個容器從相同的映像，將不同的參數傳遞至每個執行個體。</span><span class="sxs-lookup"><span data-stu-id="36d6b-115">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="36d6b-116">**標記**： 記號或標籤，如此就可以識別不同的影像或相同的映像 （依據版本號碼或目標環境） 的版本，您可以套用至映像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-116">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="36d6b-117">**Dockerfile**： 文字檔案，其中包含有關如何建立 Docker 映像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-117">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="36d6b-118">**建置**： 建立容器映像的動作為基礎的資訊和提供其 Dockerfile 中，以及映像的建立位置的資料夾中的其他檔案的內容。</span><span class="sxs-lookup"><span data-stu-id="36d6b-118">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="36d6b-119">您可以建立 Docker docker 建置命令的影像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-119">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="36d6b-120">**儲存機制 （儲存機制）**： 相關的 Docker 映像，加上標記，指出映像版本的集合。</span><span class="sxs-lookup"><span data-stu-id="36d6b-120">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="36d6b-121">某些儲存機制包含多個變異的特定映像，例如包含 Sdk （高），其中包含唯一的執行階段，（輕） 映像的映像等等。這些變異可以使用標記來標記。</span><span class="sxs-lookup"><span data-stu-id="36d6b-121">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="36d6b-122">單一儲存機制可以包含平台的變異，例如 Linux 映像和 Windows 映像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-122">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="36d6b-123">**登錄**： 服務可提供存取至儲存機制。</span><span class="sxs-lookup"><span data-stu-id="36d6b-123">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="36d6b-124">最公用映像的預設登錄是[Docker Hub](https://hub.docker.com/) （Docker 做為組織所擁有）。</span><span class="sxs-lookup"><span data-stu-id="36d6b-124">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="36d6b-125">登錄通常包含多個小組的儲存機制。</span><span class="sxs-lookup"><span data-stu-id="36d6b-125">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="36d6b-126">公司通常具有私人登錄來儲存和管理他們所建立的映像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-126">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="36d6b-127">Azure 容器登錄中是另一個範例。</span><span class="sxs-lookup"><span data-stu-id="36d6b-127">Azure Container Registry is another example.</span></span>

<span data-ttu-id="36d6b-128">**Docker Hub**： 上傳映像，並使用它們的公用登錄。</span><span class="sxs-lookup"><span data-stu-id="36d6b-128">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="36d6b-129">Docker Hub 提供 Docker 映像裝載、 公用或私人登錄、 組建觸發程序和 web 連結，以及與 GitHub 和 Bitbucket 整合。</span><span class="sxs-lookup"><span data-stu-id="36d6b-129">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="36d6b-130">**Azure 容器登錄中**： 公用使用 Docker 映像和其元件在 Azure 中的資源。</span><span class="sxs-lookup"><span data-stu-id="36d6b-130">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="36d6b-131">這可提供的登錄，這是您在 Azure 中部署接近和，可讓您控制存取權，讓您可以使用您的 Azure Active Directory 群組和權限。</span><span class="sxs-lookup"><span data-stu-id="36d6b-131">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="36d6b-132">**Docker 信任登錄 (DTR)**: （從 Docker) Docker 登錄服務，可以是安裝在內部部署，讓它都位於組織的資料中心和網路內。</span><span class="sxs-lookup"><span data-stu-id="36d6b-132">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="36d6b-133">很方便應管理企業內的私人映像。</span><span class="sxs-lookup"><span data-stu-id="36d6b-133">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="36d6b-134">Docker Trusted Registry 是 Docker Datacenter 產品的一部分。</span><span class="sxs-lookup"><span data-stu-id="36d6b-134">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="36d6b-135">如需詳細資訊，請參閱[Docker 信任登錄 (DTR)](https://docs.docker.com/docker-trusted-registry/overview/)。</span><span class="sxs-lookup"><span data-stu-id="36d6b-135">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="36d6b-136">**Docker Community Edition (CE)**： 適用於 Windows 和 macOS 如建置、 執行和測試容器在本機開發工具。</span><span class="sxs-lookup"><span data-stu-id="36d6b-136">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="36d6b-137">用於 Windows CE docker 會提供開發環境適用於 Linux 和 Windows 容器。</span><span class="sxs-lookup"><span data-stu-id="36d6b-137">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="36d6b-138">在 Windows 上的 Linux Docker 主機根據[HYPER-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx)虛擬機器。</span><span class="sxs-lookup"><span data-stu-id="36d6b-138">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="36d6b-139">Windows 容器主機直接以 Windows 為基礎。</span><span class="sxs-lookup"><span data-stu-id="36d6b-139">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="36d6b-140">適用於 Mac 的 docker CE 根據 Apple Hypervisor 架構和[xhyve hypervisor](https://github.com/mist64/xhyve)，文中提供 Linux Docker 主機的虛擬機器在 Mac OS X Docker CE for Windows 和 Mac 的取代以 Oracle 為基礎的 Docker 工具箱VirtualBox。</span><span class="sxs-lookup"><span data-stu-id="36d6b-140">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="36d6b-141">**Docker Enterprise Edition (EE)**： 適用於 Linux 和 Windows 開發的 Docker 工具的企業規模版本。</span><span class="sxs-lookup"><span data-stu-id="36d6b-141">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="36d6b-142">**撰寫**： 命令列工具及 YAML 檔案格式定義和執行多個容器應用程式的中繼資料。</span><span class="sxs-lookup"><span data-stu-id="36d6b-142">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="36d6b-143">您定義的一個或多個.yml 檔案，可以覆寫值根據環境的多個映像為基礎的單一應用程式。</span><span class="sxs-lookup"><span data-stu-id="36d6b-143">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="36d6b-144">您已建立定義之後，您可以部署整個多容器應用程式使用單一命令 (docker-撰寫總) 會建立 Docker 主機上的每個影像的容器。</span><span class="sxs-lookup"><span data-stu-id="36d6b-144">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="36d6b-145">**叢集**： 的 Docker 主機，讓應用程式可以擴充至多個執行個體的服務公開好像單一虛擬 Docker 主機時，集合會分散在叢集中的多個主機。</span><span class="sxs-lookup"><span data-stu-id="36d6b-145">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="36d6b-146">Docker 叢集可以使用 Docker Swarm、 Mesosphere DC/OS、 Kubernetes 和 Azure Service Fabric 建立。</span><span class="sxs-lookup"><span data-stu-id="36d6b-146">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="36d6b-147">(如果您使用 Docker Swarm 來管理叢集，您通常參照為叢集*廣域*而不是叢集。)</span><span class="sxs-lookup"><span data-stu-id="36d6b-147">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="36d6b-148">**Orchestrator**： 一種工具，可簡化管理的叢集與 Docker 主機。</span><span class="sxs-lookup"><span data-stu-id="36d6b-148">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="36d6b-149">Orchestrators 可讓您管理其影像、 容器和主機透過命令列介面 (CLI) 或圖形化使用者介面。</span><span class="sxs-lookup"><span data-stu-id="36d6b-149">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="36d6b-150">您可以管理容器網路功能、 設定、 負載平衡，服務探索、 高可用性、 Docker 主機組態，等等。</span><span class="sxs-lookup"><span data-stu-id="36d6b-150">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="36d6b-151">協調者負責執行、 散發、 調整及修復的節點集合之間的工作負載。</span><span class="sxs-lookup"><span data-stu-id="36d6b-151">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="36d6b-152">一般而言，orchestrator 產品是提供叢集基礎結構，例如 Mesosphere DC/OS、 Kubernetes、 Docker Swarm，和 Azure Service Fabric 的相同產品。</span><span class="sxs-lookup"><span data-stu-id="36d6b-152">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
