<span data-ttu-id="a97d2-119">[Windows 容器](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview)使用兩種執行階段：</span><span class="sxs-lookup"><span data-stu-id="a97d2-119">[Windows Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview) work with two types of runtimes:</span></span>

-   <span data-ttu-id="a97d2-122">Hyper-V 容器可在 Windows Server 容器提供的隔離上展開，方法是在高度最佳化的虛擬機器中執行每個容器。</span><span class="sxs-lookup"><span data-stu-id="a97d2-122">Hyper-V Containers expand on the isolation provided by Windows Server Containers by running each container in a highly optimized virtual machine.</span></span> <span data-ttu-id="a97d2-123">在此組態中，容器主機的核心不會與 Hyper-V 容器共用，以提供更好的隔離。</span><span class="sxs-lookup"><span data-stu-id="a97d2-123">In this configuration, the kernel of the container host is not shared with the Hyper-V Containers, providing better isolation.</span></span> <span data-ttu-id="a97d2-124">Hyper-V 容器允許未受信任的和「惡意的多租用戶」應用程式在相同的主機上執行。</span><span class="sxs-lookup"><span data-stu-id="a97d2-124">Hyper-V Containers allow untrusted and *hostile multi-tenant* applications to run on the same host.</span></span> <span data-ttu-id="a97d2-125">Hyper-V 容器的啟動時間和密度，效率略遜於 Windows Server 容器。</span><span class="sxs-lookup"><span data-stu-id="a97d2-125">Hyper-V Containers have a bit less efficiency in startup times and density than Windows Server Containers.</span></span>

<span data-ttu-id="a97d2-126">這些容器映像的建立及運作方式都相同。</span><span class="sxs-lookup"><span data-stu-id="a97d2-126">The images for these containers are created and function the same way.</span></span> <span data-ttu-id="a97d2-127">只容器建立方式有異。</span><span class="sxs-lookup"><span data-stu-id="a97d2-127">They differ in how the container is created.</span></span> <span data-ttu-id="a97d2-128">如需詳細資料，請參閱 [Hyper-V 容器](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview)。</span><span class="sxs-lookup"><span data-stu-id="a97d2-128">For details, see [Hyper-V Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview).</span></span>
