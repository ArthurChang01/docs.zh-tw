# <a name="handling-and-throwing-exceptions-in-net"></a><span data-ttu-id="9c8ca-102">在 .NET 中處理和擲回例外狀況</span><span class="sxs-lookup"><span data-stu-id="9c8ca-102">Handling and throwing exceptions in .NET</span></span>

<span data-ttu-id="9c8ca-103">應用程式必須能以一致的方式處理執行期間發生的錯誤。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-103">Applications must be able to handle errors that occur during execution in a consistent manner.</span></span> <span data-ttu-id="9c8ca-104">.NET 提供模型，可以統一的方式通知應用程式的錯誤：.NET 作業會藉由擲回例外狀況指出失敗。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-104">.NET provides a model for notifying applications of errors in a uniform way: .NET operations indicate failure by throwing exceptions.</span></span>

## <a name="exceptions"></a><span data-ttu-id="9c8ca-105">例外狀況</span><span class="sxs-lookup"><span data-stu-id="9c8ca-105">Exceptions</span></span>

<span data-ttu-id="9c8ca-106">例外狀況是執行程式所遇到的錯誤狀況或未預期的行為。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-106">An exception is any error condition or unexpected behavior that is encountered by an executing program.</span></span> <span data-ttu-id="9c8ca-107">在發生程式碼或您呼叫的程式碼 (例如共用程式庫) 中有錯誤、無法使用作業系統資源、執行階段遇到非預期的狀況 (例如無法驗證的程式碼) 等情況時，就可能會擲回例外狀況。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-107">Exceptions can be thrown because of a fault in your code or in code that you call (such as a shared library), unavailable operating system resources, unexpected conditions that the runtime encounters (such as code that can't be verified), and so on.</span></span> <span data-ttu-id="9c8ca-108">您的應用程式可從一些狀況中復原，但有些狀況就無法復原。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-108">Your application can recover from some of these conditions, but not from others.</span></span> <span data-ttu-id="9c8ca-109">雖然您可以從大部分的應用程式例外狀況中復原，但是無法從大部分的執行階段例外狀況中復原。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-109">Although you can recover from most application exceptions, you can't recover from most runtime exceptions.</span></span>

<span data-ttu-id="9c8ca-110">在 .NET 中，例外狀況是繼承自 <xref:System.Exception?displayProperty=nameWithType> 類別的物件。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-110">In .NET, an exception is an object that inherits from the <xref:System.Exception?displayProperty=nameWithType> class.</span></span> <span data-ttu-id="9c8ca-111">從發生問題的程式碼區域擲回例外狀況。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-111">An exception is thrown from an area of code where a problem has occurred.</span></span> <span data-ttu-id="9c8ca-112">例外狀況會向上傳遞堆疊，直到應用程式處理或程式終止它。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-112">The exception is passed up the stack until the application handles it or the program terminates.</span></span>

## <a name="exceptions-vs-traditional-error-handling-methods"></a><span data-ttu-id="9c8ca-113">例外狀況與傳統錯誤處理方法的比較</span><span class="sxs-lookup"><span data-stu-id="9c8ca-113">Exceptions vs. traditional error-handling methods</span></span>

<span data-ttu-id="9c8ca-114">傳統上，一種語言的錯誤處理模型不是依賴語言唯一偵測錯誤的方式與尋找處理常式，就是依賴作業系統所提供的錯誤處理機制。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-114">Traditionally, a language's error-handling model relied on either the language's unique way of detecting errors and locating handlers for them, or on the error-handling mechanism provided by the operating system.</span></span> <span data-ttu-id="9c8ca-115">.NET 實作例外狀況處理的方式具有下列優點：</span><span class="sxs-lookup"><span data-stu-id="9c8ca-115">The way .NET implements exception handling provides the following advantages:</span></span>

- <span data-ttu-id="9c8ca-116">.NET 程式語言擲回和處理例外狀況的方式都相同。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-116">Exception throwing and handling works the same for .NET programming languages.</span></span>

- <span data-ttu-id="9c8ca-117">不需要任何特定語言語法以處理例外狀況，但可讓每一種語言定義屬於自己的語法。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-117">Doesn't require any particular language syntax for handling exceptions, but allows each language to define its own syntax.</span></span>

- <span data-ttu-id="9c8ca-118">例外狀況可跨處理序，甚至是跨電腦界限擲回。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-118">Exceptions can be thrown across process and even machine boundaries.</span></span>

- <span data-ttu-id="9c8ca-119">可將例外狀況處理程式碼加入應用程式，以增加程式可靠性。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-119">Exception-handling code can be added to an application to increase program reliability.</span></span>

<span data-ttu-id="9c8ca-120">例外狀況優於其他錯誤通知方法，例如傳回碼。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-120">Exceptions offer advantages over other methods of error notification, such as return codes.</span></span> <span data-ttu-id="9c8ca-121">不會發生未注意到失敗的情況，因為如果系統擲出例外狀況且您未加以處理，執行階段就會終止您的應用程式。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-121">Failures don't go unnoticed because if an exception is thrown and you don't handle it, the runtime terminates your application.</span></span> <span data-ttu-id="9c8ca-122">無效值不會因為程式碼無法檢查失敗傳回碼，而持續在系統中散佈。</span><span class="sxs-lookup"><span data-stu-id="9c8ca-122">Invalid values don't continue to propagate through the system as a result of code that fails to check for a failure return code.</span></span>
