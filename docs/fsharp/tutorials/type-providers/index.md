# <a name="type-providers"></a><span data-ttu-id="50db3-103">型別提供者</span><span class="sxs-lookup"><span data-stu-id="50db3-103">Type Providers</span></span>

<span data-ttu-id="50db3-104">F# 型別提供者是一個元件，該元件會提供類型、屬性和方法讓您在程式中使用。</span><span class="sxs-lookup"><span data-stu-id="50db3-104">An F# type provider is a component that provides types, properties, and methods for use in your program.</span></span> <span data-ttu-id="50db3-105">型別提供者產生所謂**提供型別**，F # 編譯器所產生以及外部資料來源為基礎。</span><span class="sxs-lookup"><span data-stu-id="50db3-105">Type Providers generate what are known as **Provided Types**, which are generated by the F# compiler and are based on an external data source.</span></span>

<span data-ttu-id="50db3-106">例如，F # 類型 SQL 的提供者可以產生型別代表資料表與關聯式資料庫中的資料行。</span><span class="sxs-lookup"><span data-stu-id="50db3-106">For example, an F# Type Provider for SQL can generate types representing tables and columns in a relational database.</span></span> <span data-ttu-id="50db3-107">事實上，這是什麼[根據 SQLProvider](https://fsprojects.github.io/SQLProvider/)型別提供者。</span><span class="sxs-lookup"><span data-stu-id="50db3-107">In fact, this is what the [SQLProvider](https://fsprojects.github.io/SQLProvider/) Type Provider does.</span></span>

<span data-ttu-id="50db3-108">提供類型取決於型別提供者的輸入參數。</span><span class="sxs-lookup"><span data-stu-id="50db3-108">Provided Types depend on input parameters to a Type Provider.</span></span> <span data-ttu-id="50db3-109">這類的輸入可以是範例資料來源 （例如 JSON 結構描述檔案），直接指向外部服務或資料來源的連接字串的 URL。</span><span class="sxs-lookup"><span data-stu-id="50db3-109">Such input can be a sample data source (such as a JSON schema file), a URL pointing directly to an external service, or a connection string to a data source.</span></span> <span data-ttu-id="50db3-110">型別提供者也可以確保類型群組只會展開視;也就是說，它們會展開如果您的程式實際參考類型。</span><span class="sxs-lookup"><span data-stu-id="50db3-110">A Type Provider can also ensure that groups of types are only expanded on demand; that is, they are expanded if the types are actually referenced by your program.</span></span> <span data-ttu-id="50db3-111">這樣就能以強類型的方式視需要直接整合大型的資訊空間，例如線上資料市場。</span><span class="sxs-lookup"><span data-stu-id="50db3-111">This allows for the direct, on-demand integration of large-scale information spaces such as online data markets in a strongly typed way.</span></span>

## <a name="generative-and-erased-type-providers"></a><span data-ttu-id="50db3-112">Generative 和清除的型別提供者</span><span class="sxs-lookup"><span data-stu-id="50db3-112">Generative and Erased Type Providers</span></span>

<span data-ttu-id="50db3-113">型別提供者有兩種形式： Generative 和清除。</span><span class="sxs-lookup"><span data-stu-id="50db3-113">Type Providers come in two forms: Generative and Erased.</span></span>

<span data-ttu-id="50db3-114">Generative 型別提供者會產生可以寫成.NET 型別，會產生這些組件的類型。</span><span class="sxs-lookup"><span data-stu-id="50db3-114">Generative Type Providers produce types that can be written as .NET types into the assembly in which they are produced.</span></span> <span data-ttu-id="50db3-115">這可讓它們使用從其他組件中的程式碼。</span><span class="sxs-lookup"><span data-stu-id="50db3-115">This allows them to be consumed from code in other assemblies.</span></span> <span data-ttu-id="50db3-116">這表示資料來源的型別的表示通常必須是可搭配.NET 類型所代表的其中一個。</span><span class="sxs-lookup"><span data-stu-id="50db3-116">This means that the typed representation of the data source must generally be one that is feasible to represent with .NET types.</span></span>

<span data-ttu-id="50db3-117">清除的型別提供者產生才可以使用組件或從產生的專案中的類型。</span><span class="sxs-lookup"><span data-stu-id="50db3-117">Erasing Type Providers produce types that can only be consumed in the assembly or project they are generated from.</span></span> <span data-ttu-id="50db3-118">型別是暫時的。也就是說，它們不會寫入至組件，而且不能使用的其他組件中的程式碼。</span><span class="sxs-lookup"><span data-stu-id="50db3-118">The types are ephemeral; that is, they are not written into an assembly and cannot be consumed by code in other assemblies.</span></span> <span data-ttu-id="50db3-119">它們可以包含*延遲*成員，可讓您將使用提供的型別從可能無限的資訊空間。</span><span class="sxs-lookup"><span data-stu-id="50db3-119">They can contain *delayed* members, allowing you to use provided types from a potentially infinite information space.</span></span> <span data-ttu-id="50db3-120">它們可用於使用很大且相互連結的資料來源的一小部分。</span><span class="sxs-lookup"><span data-stu-id="50db3-120">They are useful for using a small subset of a large and interconnected data source.</span></span>

- <span data-ttu-id="50db3-123">[FSharp.Data](https://fsharp.github.io/FSharp.Data/)的 JSON、 XML、 CSV 及 HTML 文件格式和資源，包含型別提供者。</span><span class="sxs-lookup"><span data-stu-id="50db3-123">[FSharp.Data](https://fsharp.github.io/FSharp.Data/) includes Type Providers for JSON, XML, CSV, and HTML document formats and resources.</span></span>
- <span data-ttu-id="50db3-124">[根據 SQLProvider](https://fsprojects.github.io/SQLProvider/)提供強型別存取透過物件對應和 F # LINQ 關聯資料庫對這些資料來源的查詢。</span><span class="sxs-lookup"><span data-stu-id="50db3-124">[SQLProvider](https://fsprojects.github.io/SQLProvider/) provides strongly-typed access to relation databases through object mapping and F# LINQ queries against these data sources.</span></span>
- <span data-ttu-id="50db3-125">[FSharp.Data.SqlClient](https://fsprojects.github.io/FSharp.Data.SqlClient/)一組型別提供者的編譯時間檢查 T-SQL F # 中的內嵌。</span><span class="sxs-lookup"><span data-stu-id="50db3-125">[FSharp.Data.SqlClient](https://fsprojects.github.io/FSharp.Data.SqlClient/) has a set of type providers for compile-time checked embedding of T-SQL in F#.</span></span>
- <span data-ttu-id="50db3-126">[Azure 儲存體的型別提供者](https://fsprojects.github.io/AzureStorageTypeProvider/)提供 Azure Blob、 資料表和佇列，可讓您存取這些資源，而不需要資源名稱指定為字串，在整個程式的類型。</span><span class="sxs-lookup"><span data-stu-id="50db3-126">[Azure Storage Type provider](https://fsprojects.github.io/AzureStorageTypeProvider/) provides types for Azure Blobs, Tables, and Queues, allowing you to access these resources without needing to specify resource names as strings throughout your program.</span></span>
- <span data-ttu-id="50db3-127">[FSharp.Data.GraphQL](https://fsprojects.github.io/FSharp.Data.GraphQL/index.html)包含**GraphQLProvider**，可由 URL 指定 GraphQL server 為基礎的類型。</span><span class="sxs-lookup"><span data-stu-id="50db3-127">[FSharp.Data.GraphQL](https://fsprojects.github.io/FSharp.Data.GraphQL/index.html) contains the **GraphQLProvider**, which provides types based on a GraphQL server specified by URL.</span></span>

<span data-ttu-id="50db3-128">在需要時，您可以[建立您自己的自訂型別提供者](creating-a-type-provider.md)，或參考其他人所建立的型別提供者。</span><span class="sxs-lookup"><span data-stu-id="50db3-128">Where necessary, you can [create your own custom type providers](creating-a-type-provider.md), or reference type providers that have been created by others.</span></span> <span data-ttu-id="50db3-129">例如，假設您的組織的資料服務提供大量且不斷增加的具名資料集，各資料集都有自己的穩定資料結構描述。</span><span class="sxs-lookup"><span data-stu-id="50db3-129">For example, assume your organization has a data service providing a large and growing number of named data sets, each with its own stable data schema.</span></span> <span data-ttu-id="50db3-130">您可能選擇建立一個類型提供者，以強類型方式讀取結構描述並對程式設計人員呈現最新的可用資料集。</span><span class="sxs-lookup"><span data-stu-id="50db3-130">You may choose to create a type provider that reads the schemas and presents the latest available data sets to the programmer in a strongly typed way.</span></span>
