# <a name="type-extensions"></a><span data-ttu-id="efc03-103">類型延伸模組</span><span class="sxs-lookup"><span data-stu-id="efc03-103">Type extensions</span></span>

<span data-ttu-id="efc03-109">每個都可以用於不同的案例，而且有不同的取捨。</span><span class="sxs-lookup"><span data-stu-id="efc03-109">Each can be used in different scenarios and has different tradeoffs.</span></span>

## <a name="syntax"></a><span data-ttu-id="efc03-110">語法</span><span class="sxs-lookup"><span data-stu-id="efc03-110">Syntax</span></span>

```fsharp
// Intrinsic and optional extensions
type typename with

    ...

// Extension methods
open System.Runtime.CompilerServices

[<Extension>]
type Extensions() =
    [static] member self-identifier.extension-name (ty: typename, [args]) =
        body
    ...
```

## <a name="intrinsic-type-extensions"></a><span data-ttu-id="efc03-111">內建類型延伸模組</span><span class="sxs-lookup"><span data-stu-id="efc03-111">Intrinsic type extensions</span></span>

<span data-ttu-id="efc03-112">內建類型延伸模組是擴充使用者定義類型的類型延伸。</span><span class="sxs-lookup"><span data-stu-id="efc03-112">An intrinsic type extension is a type extension that extends a user-defined type.</span></span>

<span data-ttu-id="efc03-113">內建類型延伸模組必須在相同的檔案中 **，以及**與所擴充之類型相同的命名空間或模組中定義。</span><span class="sxs-lookup"><span data-stu-id="efc03-113">Intrinsic type extensions must be defined in the same file **and** in the same namespace or module as the type they're extending.</span></span> <span data-ttu-id="efc03-114">任何其他定義都會導致它們成為[選擇性的類型延伸](type-extensions.md#optional-type-extensions)。</span><span class="sxs-lookup"><span data-stu-id="efc03-114">Any other definition will result in them being [optional type extensions](type-extensions.md#optional-type-extensions).</span></span>

<span data-ttu-id="efc03-115">內建類型延伸有時是將功能與類型宣告分開的更清楚方式。</span><span class="sxs-lookup"><span data-stu-id="efc03-115">Intrinsic type extensions are sometimes a cleaner way to separate functionality from the type declaration.</span></span> <span data-ttu-id="efc03-116">下列範例顯示如何定義內建類型延伸模組：</span><span class="sxs-lookup"><span data-stu-id="efc03-116">The following example shows how to define an intrinsic type extension:</span></span>

```fsharp
namespace Example

type Variant =
    | Num of int
    | Str of string
  
module Variant =
    let print v =
        match v with
        | Num n -> printf "Num %d" n
        | Str s -> printf "Str %s" s

// Add a member to Variant as an extension
type Variant with
    member x.Print() = Variant.print x
```

<span data-ttu-id="efc03-117">使用類型延伸可讓您分隔下列各項：</span><span class="sxs-lookup"><span data-stu-id="efc03-117">Using a type extension allows you to separate each of the following:</span></span>

- <span data-ttu-id="efc03-118">`Variant` 類型的宣告</span><span class="sxs-lookup"><span data-stu-id="efc03-118">The declaration of a `Variant` type</span></span>
- <span data-ttu-id="efc03-119">根據其「圖形」來列印 `Variant` 類別的功能</span><span class="sxs-lookup"><span data-stu-id="efc03-119">Functionality to print the `Variant` class depending on its "shape"</span></span>
- <span data-ttu-id="efc03-120">使用物件樣式 `.`標記法來存取列印功能的方式</span><span class="sxs-lookup"><span data-stu-id="efc03-120">A way to access the printing functionality with object-style `.`-notation</span></span>

<span data-ttu-id="efc03-121">這是在 `Variant`上將所有專案定義為成員的替代方法。</span><span class="sxs-lookup"><span data-stu-id="efc03-121">This is an alternative to defining everything as a member on `Variant`.</span></span> <span data-ttu-id="efc03-122">雖然它不是原本較好的方法，但在某些情況下，它可能是更清楚的功能表示。</span><span class="sxs-lookup"><span data-stu-id="efc03-122">Although it is not an inherently better approach, it can be a cleaner representation of functionality in some situations.</span></span>

<span data-ttu-id="efc03-123">內建類型延伸模組會編譯為它們所擴充之類型的成員，而且會在反映的類型檢查時出現在類型上。</span><span class="sxs-lookup"><span data-stu-id="efc03-123">Intrinsic type extensions are compiled as members of the type they augment, and appear on the type when the type is examined by reflection.</span></span>

## <a name="optional-type-extensions"></a><span data-ttu-id="efc03-124">選擇性類型擴充功能</span><span class="sxs-lookup"><span data-stu-id="efc03-124">Optional type extensions</span></span>

<span data-ttu-id="efc03-125">選擇性的類型延伸模組是一種延伸模組，它會出現在要擴充之類型的原始模組、命名空間或元件外部。</span><span class="sxs-lookup"><span data-stu-id="efc03-125">An optional type extension is an extension that appears outside the original module, namespace, or assembly of the type being extended.</span></span>

<span data-ttu-id="efc03-126">選擇性類型延伸模組適用于擴充您尚未自行定義的類型。</span><span class="sxs-lookup"><span data-stu-id="efc03-126">Optional type extensions are useful for extending a type that you have not defined yourself.</span></span> <span data-ttu-id="efc03-127">例如:</span><span class="sxs-lookup"><span data-stu-id="efc03-127">For example:</span></span>

```fsharp
module Extensions

type IEnumerable<'T> with
    /// Repeat each element of the sequence n times
    member xs.RepeatElements(n: int) =
        seq {
            for x in xs do
                for _ in 1 .. n -> x
        }
```

<span data-ttu-id="efc03-128">您現在可以存取 `RepeatElements`，如同它是 <xref:System.Collections.Generic.IEnumerable%601> 的成員，只要在您工作的範圍中開啟 `Extensions` 模組即可。</span><span class="sxs-lookup"><span data-stu-id="efc03-128">You can now access `RepeatElements` as if it's a member of <xref:System.Collections.Generic.IEnumerable%601> as long as the `Extensions` module is opened in the scope that you are working in.</span></span>

<span data-ttu-id="efc03-129">當反映進行檢查時，選擇性擴充功能不會出現在擴充類型上。</span><span class="sxs-lookup"><span data-stu-id="efc03-129">Optional extensions do not appear on the extended type when examined by reflection.</span></span> <span data-ttu-id="efc03-130">選擇性擴充功能必須在模組中，而且只有在包含擴充功能的模組已開啟或在範圍內時，才會在範圍中。</span><span class="sxs-lookup"><span data-stu-id="efc03-130">Optional extensions must be in modules, and they're only in scope when the module that contains the extension is open or is otherwise in scope.</span></span>

<span data-ttu-id="efc03-131">選擇性的延伸模組成員會編譯成靜態成員，其中會以隱含方式將物件實例當做第一個參數傳遞。</span><span class="sxs-lookup"><span data-stu-id="efc03-131">Optional extension members are compiled to static members for which the object instance is passed implicitly as the first parameter.</span></span> <span data-ttu-id="efc03-132">不過，它們的行為就像是實例成員或靜態成員（根據它們的宣告方式而定）。</span><span class="sxs-lookup"><span data-stu-id="efc03-132">However, they act as if they're instance members or static members according to how they're declared.</span></span>

<span data-ttu-id="efc03-133">選擇性的擴充成員也不會對C#或 VB 取用者顯示。</span><span class="sxs-lookup"><span data-stu-id="efc03-133">Optional extension members are also not visible to C# or VB consumers.</span></span> <span data-ttu-id="efc03-134">它們只能在其他F#程式碼中使用。</span><span class="sxs-lookup"><span data-stu-id="efc03-134">They can only be consumed in other F# code.</span></span>

## <a name="generic-limitation-of-intrinsic-and-optional-type-extensions"></a><span data-ttu-id="efc03-135">內建和選擇性類型延伸的一般限制</span><span class="sxs-lookup"><span data-stu-id="efc03-135">Generic limitation of intrinsic and optional type extensions</span></span>

<span data-ttu-id="efc03-136">在型別變數受條件約束的泛型型別上，可以宣告型別延伸。</span><span class="sxs-lookup"><span data-stu-id="efc03-136">It's possible to declare a type extension on a generic type where the type variable is constrained.</span></span> <span data-ttu-id="efc03-137">其需求是延伸模組宣告的條件約束符合宣告類型的條件約束。</span><span class="sxs-lookup"><span data-stu-id="efc03-137">The requirement is that the constraint of the extension declaration matches the constraint of the declared type.</span></span>

<span data-ttu-id="efc03-138">不過，即使在宣告的型別與型別延伸之間有相符的條件約束，條件約束也可能會受到擴充成員的主體的推斷，而這項要求在型別參數上會有不同的需求，而不是宣告的型別。</span><span class="sxs-lookup"><span data-stu-id="efc03-138">However, even when constraints are matched between a declared type and a type extension, it's possible for a constraint to be inferred by the body of an extended member that imposes a different requirement on the type parameter than the declared type.</span></span> <span data-ttu-id="efc03-139">例如:</span><span class="sxs-lookup"><span data-stu-id="efc03-139">For example:</span></span>

```fsharp
open System.Collections.Generic

// NOT POSSIBLE AND FAILS TO COMPILE!
//
// The member 'Sum' has a different requirement on 'T than the type IEnumerable<'T>
type IEnumerable<'T> with
    member this.Sum() = Seq.sum this
```

<span data-ttu-id="efc03-140">沒有任何方法可以取得此程式碼以使用選擇性的類型延伸模組：</span><span class="sxs-lookup"><span data-stu-id="efc03-140">There is no way to get this code to work with an optional type extension:</span></span>

- <span data-ttu-id="efc03-141">也就是，`Sum` 成員在 `'T` （`static member get_Zero` 和 `static member (+)`）上有不同的條件約束，而不是類型延伸模組所定義的。</span><span class="sxs-lookup"><span data-stu-id="efc03-141">As is, the `Sum` member has a different constraint on `'T` (`static member get_Zero` and `static member (+)`) than what the type extension defines.</span></span>
- <span data-ttu-id="efc03-142">將類型延伸模組修改為具有與 `Sum` 相同的條件約束，將不再符合 `IEnumerable<'T>`上定義的條件約束。</span><span class="sxs-lookup"><span data-stu-id="efc03-142">Modifying the type extension to have the same constraint as `Sum` will no longer match the defined constraint on `IEnumerable<'T>`.</span></span>
- <span data-ttu-id="efc03-143">將 `member this.Sum` 變更為 `member inline this.Sum` 會提供類型條件約束不相符的錯誤。</span><span class="sxs-lookup"><span data-stu-id="efc03-143">Changing `member this.Sum` to `member inline this.Sum` will give an error that type constraints are mismatched.</span></span>

<span data-ttu-id="efc03-144">所需的是「以空格浮點數」的靜態方法，可以如同擴充類型般呈現。</span><span class="sxs-lookup"><span data-stu-id="efc03-144">What is desired are static methods that "float in space" and can be presented as if they're extending a type.</span></span> <span data-ttu-id="efc03-145">這是必要的擴充方法。</span><span class="sxs-lookup"><span data-stu-id="efc03-145">This is where extension methods become necessary.</span></span>

## <a name="extension-methods"></a><span data-ttu-id="efc03-146">擴充方法</span><span class="sxs-lookup"><span data-stu-id="efc03-146">Extension methods</span></span>

<span data-ttu-id="efc03-147">最後，擴充方法（有時稱為「C#樣式延伸成員」）可以在中F#宣告為類別的靜態成員方法。</span><span class="sxs-lookup"><span data-stu-id="efc03-147">Finally, extension methods (sometimes called "C# style extension members") can be declared in F# as a static member method on a class.</span></span>

<span data-ttu-id="efc03-148">當您想要在將限制型別變數的泛型型別上定義延伸模組時，擴充方法很有用。</span><span class="sxs-lookup"><span data-stu-id="efc03-148">Extension methods are useful for when you wish to define extensions on a generic type that will constrain the type variable.</span></span> <span data-ttu-id="efc03-149">例如:</span><span class="sxs-lookup"><span data-stu-id="efc03-149">For example:</span></span>

```fsharp
namespace Extensions

open System.Runtime.CompilerServices

[<Extension>]
type IEnumerableExtensions() =
    [<Extension>]
    static member inline Sum(xs: IEnumerable<'T>) = Seq.sum xs
```

<span data-ttu-id="efc03-150">使用時，此程式碼會使其看起來像是在 <xref:System.Collections.Generic.IEnumerable%601>上定義 `Sum`，只要 `Extensions` 已開啟或在範圍內即可。</span><span class="sxs-lookup"><span data-stu-id="efc03-150">When used, this code will make it appear as if `Sum` is defined on <xref:System.Collections.Generic.IEnumerable%601>, so long as `Extensions` has been opened or is in scope.</span></span>
