# <a name="net-core-distribution-packaging"></a><span data-ttu-id="5e293-104">.NET Core 發佈封裝</span><span class="sxs-lookup"><span data-stu-id="5e293-104">.NET Core distribution packaging</span></span>

<span data-ttu-id="5e293-105">隨著 .NET Core 可在越來越多的平台上使用，了解如何封裝、命名和建立 .NET Core 版本是有用的。</span><span class="sxs-lookup"><span data-stu-id="5e293-105">As .NET Core becomes available on more and more platforms, it's useful to learn how to package, name, and version it.</span></span> <span data-ttu-id="5e293-106">如此一來，套件維護人員可以協助確保一致的體驗，無論使用者選擇在何處執行 .NET。</span><span class="sxs-lookup"><span data-stu-id="5e293-106">This way, package maintainers can help ensure a consistent experience no matter where users choose to run .NET.</span></span>

## <a name="disk-layout"></a><span data-ttu-id="5e293-107">磁碟配置</span><span class="sxs-lookup"><span data-stu-id="5e293-107">Disk layout</span></span>

<span data-ttu-id="5e293-108">安裝時，.NET Core 包含在檔案系統中進行如下配置的數個元件：</span><span class="sxs-lookup"><span data-stu-id="5e293-108">When installed, .NET Core consists of several components that are layed out as follows in the filesystem:</span></span>

```
.
├── dotnet                       (1)
├── LICENSE.txt                  (8)
├── ThirdPartyNotices.txt        (8)
├── host
│   └── fxr
│       └── <fxr version>        (2)
├── sdk
│   ├── <sdk version>            (3)
│   └── NuGetFallbackFolder      (4)
└── shared
    ├── Microsoft.NETCore.App
    │   └── <runtime version>    (5)
    └── Microsoft.AspNetCore.App
        └── <aspnetcore version> (6)
    └── Microsoft.AspNetCore.All
        └── <aspnetcore version> (7)
/
├─usr/share/man/man1
│       └── dotnet.1.gz          (9)
└─usr/bin
        └── dotnet               (10)
```

- <span data-ttu-id="5e293-109">(1) **dotnet** 主機 (也稱為"muxer") 有兩個不同的角色：啟用執行階段以啟動應用程式，以及啟用 SDK 將命令分派給它。</span><span class="sxs-lookup"><span data-stu-id="5e293-109">(1) **dotnet** The host (also known as the "muxer") has two distinct roles: activate a runtime to launch an application, and activate an SDK to dispatch commands to it.</span></span> <span data-ttu-id="5e293-110">主機是原生可執行檔 (`dotnet.exe`)。</span><span class="sxs-lookup"><span data-stu-id="5e293-110">The host is a native executable (`dotnet.exe`).</span></span>

<span data-ttu-id="5e293-111">有單一主機時，大部分的其他元件都會位在已建立版本的目錄 (2,3,5,6) 中。</span><span class="sxs-lookup"><span data-stu-id="5e293-111">While there is a single host, most of the other components are in versioned directories (2,3,5,6).</span></span> <span data-ttu-id="5e293-112">這些表示系統上可以有多個版本，因為它們是並排安裝。</span><span class="sxs-lookup"><span data-stu-id="5e293-112">These means multiple versions can be present on the system since they are installed side-by-side.</span></span>

- <span data-ttu-id="5e293-113">(2) **host/fxr/\<fxr 版本>** 包含主機所使用的架構解析邏輯。</span><span class="sxs-lookup"><span data-stu-id="5e293-113">(2) **host/fxr/\<fxr version>** contains the framework resolution logic used by the host.</span></span> <span data-ttu-id="5e293-114">主機會使用已安裝的最新 hostfxr。</span><span class="sxs-lookup"><span data-stu-id="5e293-114">The host uses the latest hostfxr that is installed.</span></span> <span data-ttu-id="5e293-115">hostfxr 負責在執行 .NET Core 應用程式時選取適當的執行階段。</span><span class="sxs-lookup"><span data-stu-id="5e293-115">The hostfxr is responsible for selecting the appropriate runtime when executing a .NET Core application.</span></span> <span data-ttu-id="5e293-116">例如，針對 .NET Core 2.0.0 所建置的應用程式將會使用可用的 2.0.5 執行階段。</span><span class="sxs-lookup"><span data-stu-id="5e293-116">For example, an application built for .NET Core 2.0.0 will use the 2.0.5 runtime when it is available.</span></span> <span data-ttu-id="5e293-117">同樣地，hostfxr 會在開發期間選取適當的 SDK。</span><span class="sxs-lookup"><span data-stu-id="5e293-117">Similarly, hostfxr selects the appropriate SDK during development.</span></span>

- <span data-ttu-id="5e293-118">(3) **sdk/\<SDK 版本>** SDK (也稱為「工具」) 是一組受控工具，可用來撰寫與建置 .NET Core 程式庫和應用程式。</span><span class="sxs-lookup"><span data-stu-id="5e293-118">(3) **sdk/\<sdk version>** The SDK (also known as "the tooling") is a set of managed tools that can be used to write and build .NET Core libraries and applications.</span></span> <span data-ttu-id="5e293-119">SDK 包含 CLI、Roslyn 編譯器、MSBuild，以及相關聯的建置工作與目標、NuGet 及新的專案範本等。</span><span class="sxs-lookup"><span data-stu-id="5e293-119">The SDK includes the CLI, the Roslyn compiler, MSBuild, and associated build tasks and targets, NuGet, new project templates, etc.</span></span>

- <span data-ttu-id="5e293-120">(4) **sdk/NuGetFallbackFolder** 包含 SDK 在 `dotnet restore` 步驟期間所使用的 NuGet 套件快取。</span><span class="sxs-lookup"><span data-stu-id="5e293-120">(4) **sdk/NuGetFallbackFolder** contains a cache of NuGet packages used by an SDK during the `dotnet restore` step.</span></span>

<span data-ttu-id="5e293-121">**共用**資料夾包含架構。</span><span class="sxs-lookup"><span data-stu-id="5e293-121">The **shared** folder contains frameworks.</span></span> <span data-ttu-id="5e293-122">共用架構在集中位置提供一組程式庫，以供不同的應用程式使用。</span><span class="sxs-lookup"><span data-stu-id="5e293-122">A shared framework provides a set of libraries at a central location so they can be used by different applications.</span></span>

- <span data-ttu-id="5e293-123">(5) **shared/Microsoft.NETCore.App/\<執行階段版本>** 此架構包含 .NET Core 執行階段和支援受控程式庫。</span><span class="sxs-lookup"><span data-stu-id="5e293-123">(5) **shared/Microsoft.NETCore.App/\<runtime version>** This framework contains the .NET Core runtime and supporting managed libraries.</span></span>

- <span data-ttu-id="5e293-124">(6,7) **shared/Microsoft.AspNetCore.{App,All}/\<aspnetcore 版本>** 包含 ASP.NET Core 程式庫。</span><span class="sxs-lookup"><span data-stu-id="5e293-124">(6,7) **shared/Microsoft.AspNetCore.{App,All}/\<aspnetcore version>** contains the ASP.NET Core libraries.</span></span> <span data-ttu-id="5e293-125">在 .NET Core 專案期間，開發並支援 `Microsoft.AspNetCore.App` 下的程式庫。</span><span class="sxs-lookup"><span data-stu-id="5e293-125">The libraries under `Microsoft.AspNetCore.App` are developed and supported as part of the .NET Core project.</span></span> <span data-ttu-id="5e293-126">`Microsoft.AspNetCore.All` 下的程式庫是也包含協力廠商程式庫的超集。</span><span class="sxs-lookup"><span data-stu-id="5e293-126">The libraries under `Microsoft.AspNetCore.All` are a superset which also contains 3rd party libraries.</span></span>

- <span data-ttu-id="5e293-127">(8) **LICENSE.txt,ThirdPartyNotices.txt** 是 .NET Core 授權以及 .NET Core 中所使用協力廠商程式庫的授權。</span><span class="sxs-lookup"><span data-stu-id="5e293-127">(8) **LICENSE.txt,ThirdPartyNotices.txt** are the .NET Core license and licenses of third-party libraries used in .NET Core.</span></span>

- <span data-ttu-id="5e293-128">(9,10) **dotnet.1.gz, dotnet** `dotnet.1.gz` 是 dotnet 手冊頁。</span><span class="sxs-lookup"><span data-stu-id="5e293-128">(9,10) **dotnet.1.gz, dotnet** `dotnet.1.gz` is the dotnet man page.</span></span> <span data-ttu-id="5e293-129">`dotnet` 是 dotnet host(1) 的符號連結。</span><span class="sxs-lookup"><span data-stu-id="5e293-129">`dotnet` is a symlink to the dotnet host(1).</span></span> <span data-ttu-id="5e293-130">這些檔案會安裝在已知位置，以進行系統整合。</span><span class="sxs-lookup"><span data-stu-id="5e293-130">These files are installed at well known locations for system integration.</span></span>
