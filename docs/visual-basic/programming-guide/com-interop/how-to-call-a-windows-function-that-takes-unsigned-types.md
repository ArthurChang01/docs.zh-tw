<span data-ttu-id="a669a-103">如果您使用的類別、模組或結構具有不帶正負號整數類型的成員，您可以使用 Visual Basic 來存取這些成員。</span><span class="sxs-lookup"><span data-stu-id="a669a-103">If you are consuming a class, module, or structure that has members of unsigned integer types, you can access these members with Visual Basic.</span></span>

## <a name="to-call-a-windows-function-that-takes-an-unsigned-type"></a><span data-ttu-id="a669a-104">呼叫採用不帶正負號類型的 Windows 函式</span><span class="sxs-lookup"><span data-stu-id="a669a-104">To call a Windows function that takes an unsigned type</span></span>

1. <span data-ttu-id="a669a-105">使用[Declare 語句](../../../visual-basic/language-reference/statements/declare-statement.md)來告訴 Visual Basic 哪一個程式庫包含函式、它在該程式庫中的名稱、其呼叫順序為何，以及如何在呼叫它時轉換字串。</span><span class="sxs-lookup"><span data-stu-id="a669a-105">Use a [Declare Statement](../../../visual-basic/language-reference/statements/declare-statement.md) to tell Visual Basic which library holds the function, what its name is in that library, what its calling sequence is, and how to convert strings when calling it.</span></span>

2. <span data-ttu-id="a669a-106">在 `Declare` 語句中，針對具有不帶正負號類型的每個參數，使用適當的 `UInteger`、`ULong`、`UShort`或 `Byte`。</span><span class="sxs-lookup"><span data-stu-id="a669a-106">In the `Declare` statement, use `UInteger`, `ULong`, `UShort`, or `Byte` as appropriate for each parameter with an unsigned type.</span></span>

3. <span data-ttu-id="a669a-107">請參閱您所呼叫之 Windows 函式的檔，以尋找其所使用之常數的名稱和值。</span><span class="sxs-lookup"><span data-stu-id="a669a-107">Consult the documentation for the Windows function you are calling to find the names and values of the constants it uses.</span></span> <span data-ttu-id="a669a-108">其中有許多都是在 WinUser 檔案中定義。</span><span class="sxs-lookup"><span data-stu-id="a669a-108">Many of these are defined in the WinUser.h file.</span></span>

4. <span data-ttu-id="a669a-109">在您的程式碼中宣告必要的常數。</span><span class="sxs-lookup"><span data-stu-id="a669a-109">Declare the necessary constants in your code.</span></span> <span data-ttu-id="a669a-110">許多 Windows 常數都是32位不帶正負號的值，您應該宣告這些 `As UInteger`。</span><span class="sxs-lookup"><span data-stu-id="a669a-110">Many Windows constants are 32-bit unsigned values, and you should declare these `As UInteger`.</span></span>

5. <span data-ttu-id="a669a-111">以正常方式呼叫函式。</span><span class="sxs-lookup"><span data-stu-id="a669a-111">Call the function in the normal way.</span></span> <span data-ttu-id="a669a-112">下列範例會呼叫 Windows 函數 `MessageBox`，此函式會採用不帶正負號的整數引數。</span><span class="sxs-lookup"><span data-stu-id="a669a-112">The following example calls the Windows function `MessageBox`, which takes an unsigned integer argument.</span></span>

    ```vb
    Public Class windowsMessage
        Private Declare Auto Function mb Lib "user32.dll" Alias "MessageBox" (
            ByVal hWnd As Integer,
            ByVal lpText As String,
            ByVal lpCaption As String,
            ByVal uType As UInteger) As Integer
        Private Const MB_OK As UInteger = 0
        Private Const MB_ICONEXCLAMATION As UInteger = &H30
        Private Const IDOK As UInteger = 1
        Private Const IDCLOSE As UInteger = 8
        Private Const c As UInteger = MB_OK Or MB_ICONEXCLAMATION
        Public Function messageThroughWindows() As String
            Dim r As Integer = mb(0, "Click OK if you see this!",
                "Windows API call", c)
            Dim s As String = "Windows API MessageBox returned " &
                 CStr(r)& vbCrLf & "(IDOK = " & CStr(IDOK) &
                 ", IDCLOSE = " & CStr(IDCLOSE) & ")"
            Return s
        End Function
    End Class
    ```

     <span data-ttu-id="a669a-113">您可以使用下列程式碼來測試函數 `messageThroughWindows`。</span><span class="sxs-lookup"><span data-stu-id="a669a-113">You can test the function `messageThroughWindows` with the following code.</span></span>

    ```vb
    Public Sub consumeWindowsMessage()
        Dim w As New windowsMessage
        w.messageThroughWindows()
    End Sub
    ```

    > [!CAUTION]
    > <span data-ttu-id="a669a-114">`UInteger`、`ULong`、`UShort`和 `SByte` 資料類型不屬於[語言獨立性和與語言無關的元件](../../../standard/language-independence-and-language-independent-components.md)（CLS）的一部分，因此符合 CLS 標準的程式碼無法取用使用它們的元件。</span><span class="sxs-lookup"><span data-stu-id="a669a-114">The `UInteger`, `ULong`, `UShort`, and `SByte` data types are not part of the [Language Independence and Language-Independent Components](../../../standard/language-independence-and-language-independent-components.md) (CLS), so CLS-compliant code cannot consume a component that uses them.</span></span>

    > [!IMPORTANT]
    > <span data-ttu-id="a669a-115">呼叫非受控碼（例如 Windows 應用程式開發介面（API））會讓您的程式碼暴露于潛在的安全性風險下。</span><span class="sxs-lookup"><span data-stu-id="a669a-115">Making a call to unmanaged code, such as the Windows application programming interface (API), exposes your code to potential security risks.</span></span>

    > [!IMPORTANT]
    > <span data-ttu-id="a669a-116">呼叫 Windows API 需要未受管理的程式碼許可權，這可能會影響在部分信任情況下的執行。</span><span class="sxs-lookup"><span data-stu-id="a669a-116">Calling the Windows API requires unmanaged code permission, which might affect its execution in partial-trust situations.</span></span> <span data-ttu-id="a669a-117">如需詳細資訊，請參閱 <xref:System.Security.Permissions.SecurityPermission> 和程式[代碼存取權限](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/h846e9b3(v=vs.100))。</span><span class="sxs-lookup"><span data-stu-id="a669a-117">For more information, see <xref:System.Security.Permissions.SecurityPermission> and [Code Access Permissions](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/h846e9b3(v=vs.100)).</span></span>
