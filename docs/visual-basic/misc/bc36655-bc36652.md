<span data-ttu-id="b9aa2-103">中的擴充方法的型別參數的資料類型 '\<方法名稱 >' 中定義'\<類型名稱 >' 無法從這些引數推斷，因為可能會有一個以上的類型。</span><span class="sxs-lookup"><span data-stu-id="b9aa2-103">Data type(s) of the type parameter(s) in extension method '\<methodname>' defined in '\<typename>' cannot be inferred from these arguments because more than one type is possible.</span></span> <span data-ttu-id="b9aa2-104">明確指定資料類型或許可以更正這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="b9aa2-104">Specifying the data type(s) explicitly might correct this error.</span></span>  
  
 <span data-ttu-id="b9aa2-105">嘗試使用類型推斷來判斷泛型擴充方法呼叫中之類型參數的類型。</span><span class="sxs-lookup"><span data-stu-id="b9aa2-105">An attempt has been made to use type inference to determine the type (or types) of the type parameter (or parameters) in a call to a generic extension method.</span></span> <span data-ttu-id="b9aa2-106">編譯器發現一或多個類型參數的多個可能資料類型，並報告這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="b9aa2-106">The compiler finds more than one possible data type for one or more of the type parameters, and it reports this error.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="b9aa2-107">當無法指定引數時 (例如，針對查詢運算式中的查詢運算子)，會顯示不含第二句的錯誤訊息。</span><span class="sxs-lookup"><span data-stu-id="b9aa2-107">When specifying arguments is not an option (for example, for query operators in query expressions), the error message appears without the second sentence.</span></span>  
  
 <span data-ttu-id="b9aa2-108">下列程式碼示範這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="b9aa2-108">The following code demonstrates the error.</span></span>  
  
```vb  
Option Strict Off  
Imports System.Runtime.CompilerServices  
Module Module1  
    Sub Main()  
  
        Dim caller As New Class1  
        '' Not valid.  
        'caller.targetExtension(1, "2")  
  
    End Sub  
  
    <Extension()> _  
    Sub targetExtension(Of T)(ByVal p0 As Class1, ByVal p1 As T, ByVal p2 As T)  
    End Sub  
  
    Class Class1  
    End Class  
  
End Module  
```
