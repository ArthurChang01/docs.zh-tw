-   <span data-ttu-id="12303-169">**包含擴充邏輯。**</span><span class="sxs-lookup"><span data-stu-id="12303-169">**Include only the extension logic.**</span></span> <span data-ttu-id="12303-170">中所包含的邏輯`My`命名空間擴充功能應該包含來公開必要的功能中的程式碼`My`命名空間。</span><span class="sxs-lookup"><span data-stu-id="12303-170">The logic included in the `My` namespace extension should include only the code that is needed to expose the required functionality in the `My` namespace.</span></span> <span data-ttu-id="12303-171">因為您的擴充功能會以原始程式碼存在使用者專案中，更新擴充功能元件成本也較高的維護，應儘可能避免。</span><span class="sxs-lookup"><span data-stu-id="12303-171">Because your extension will reside in user projects as source code, updating the extension component incurs a high maintenance cost and should be avoided if possible.</span></span>  
  
-   <span data-ttu-id="12303-172">**最小化專案的假設。**</span><span class="sxs-lookup"><span data-stu-id="12303-172">**Minimize project assumptions.**</span></span> <span data-ttu-id="12303-173">當您建立您的擴充功能的`My`命名空間，請不要假設一組參考、 匯入專案層級或特定的編譯器設定 (例如，`Option Strict`關閉)。</span><span class="sxs-lookup"><span data-stu-id="12303-173">When you create your extensions of the `My` namespace, do not assume a set of references, project-level imports, or specific compiler settings (for example, `Option Strict` off).</span></span> <span data-ttu-id="12303-174">相反地，最小化的相依性，並使用完整限定所有型別參考`Global`關鍵字。</span><span class="sxs-lookup"><span data-stu-id="12303-174">Instead, minimize dependencies and fully qualify all type references by using the `Global` keyword.</span></span> <span data-ttu-id="12303-175">此外，請確定延伸模組編譯與`Option Strict`上延伸模組中的錯誤降至最低。</span><span class="sxs-lookup"><span data-stu-id="12303-175">Also, ensure that the extension compiles with `Option Strict` on to minimize errors in the extension.</span></span>  
  
-   <span data-ttu-id="12303-176">**隔離的延伸模組程式碼。**</span><span class="sxs-lookup"><span data-stu-id="12303-176">**Isolate the extension code.**</span></span> <span data-ttu-id="12303-177">將程式碼放在單一檔案，可讓您擴充功能可方便部署以 Visual Studio 項目範本。</span><span class="sxs-lookup"><span data-stu-id="12303-177">Placing the code in a single file makes your extension easily deployable as a Visual Studio item template.</span></span> <span data-ttu-id="12303-178">如需詳細資訊，請參閱本主題稍後的 「 封裝及部署擴充 」。</span><span class="sxs-lookup"><span data-stu-id="12303-178">For more information, see "Packaging and Deploying Extensions" later in this topic.</span></span> <span data-ttu-id="12303-179">將所有`My`單一檔案中的命名空間延伸模組程式碼或個別的資料夾，在專案中，也有助於找出使用者`My`命名空間擴充。</span><span class="sxs-lookup"><span data-stu-id="12303-179">Placing all the `My` namespace extension code in a single file or a separate folder in a project will also help users locate the `My` namespace extension.</span></span>  
  
##  <span data-ttu-id="12303-180"><a name="designing"></a>設計的類別庫我</span><span class="sxs-lookup"><span data-stu-id="12303-180"><a name="designing"></a> Designing Class Libraries for My</span></span>  
 <span data-ttu-id="12303-181">在此情況下，與大多數的物件模型，一些設計模式中正常運作`My`命名空間和其他人則否。</span><span class="sxs-lookup"><span data-stu-id="12303-181">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="12303-182">設計的擴充功能時`My`命名空間，請考慮下列原則︰</span><span class="sxs-lookup"><span data-stu-id="12303-182">When designing an extension to the `My` namespace, consider the following principles:</span></span>  
  
-   <span data-ttu-id="12303-183">**無狀態的方法。**</span><span class="sxs-lookup"><span data-stu-id="12303-183">**Stateless methods.**</span></span> <span data-ttu-id="12303-184">中的方法`My`命名空間應該提供特定工作的完整解決方案。</span><span class="sxs-lookup"><span data-stu-id="12303-184">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="12303-185">請確定傳遞至方法的參數值，提供完成特定工作所需的所有輸入。</span><span class="sxs-lookup"><span data-stu-id="12303-185">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="12303-186">請避免建立依賴前一個狀態，例如資源的開啟連接的方法。</span><span class="sxs-lookup"><span data-stu-id="12303-186">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>  
  
-   <span data-ttu-id="12303-187">**全域執行個體。**</span><span class="sxs-lookup"><span data-stu-id="12303-187">**Global instances.**</span></span> <span data-ttu-id="12303-188">唯一的狀態會保留在`My`命名空間是專案的全域。</span><span class="sxs-lookup"><span data-stu-id="12303-188">The only state that is maintained in the `My` namespace is global to the project.</span></span> <span data-ttu-id="12303-189">例如，`My.Application.Info`封裝共用整個應用程式的狀態。</span><span class="sxs-lookup"><span data-stu-id="12303-189">For example, `My.Application.Info` encapsulates state that is shared throughout the application.</span></span>  
  
-   <span data-ttu-id="12303-190">**簡單的參數型別。**</span><span class="sxs-lookup"><span data-stu-id="12303-190">**Simple parameter types.**</span></span> <span data-ttu-id="12303-191">簡單起見，避免複雜的參數型別。</span><span class="sxs-lookup"><span data-stu-id="12303-191">Keep things simple by avoiding complex parameter types.</span></span> <span data-ttu-id="12303-192">相反地，建立 不要使用輸入參數的方法，或只使用簡單的輸入的類型，例如字串、 基本型別，等等。</span><span class="sxs-lookup"><span data-stu-id="12303-192">Instead, create methods that either take no parameter input or that take simple input types such as strings, primitive types, and so on.</span></span>  
  
-   <span data-ttu-id="12303-193">**原廠方法。**</span><span class="sxs-lookup"><span data-stu-id="12303-193">**Factory methods.**</span></span> <span data-ttu-id="12303-194">某些類型非常難以具現化。</span><span class="sxs-lookup"><span data-stu-id="12303-194">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="12303-195">提供 factory 方法當做擴充`My`命名空間可讓您更輕鬆地探索及使用屬於此分類的類型。</span><span class="sxs-lookup"><span data-stu-id="12303-195">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="12303-196">適合的 factory 方法的範例是`My.Computer.FileSystem.OpenTextFileReader`。</span><span class="sxs-lookup"><span data-stu-id="12303-196">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="12303-197">沒有可用的.NET Framework 中的數個資料流類型。</span><span class="sxs-lookup"><span data-stu-id="12303-197">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="12303-198">具體來說，指定文字檔案`OpenTextFileReader`可協助使用者了解要使用的資料流。</span><span class="sxs-lookup"><span data-stu-id="12303-198">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>  
  
 <span data-ttu-id="12303-199">這些指導方針並未預先排除類別庫的一般設計原則。</span><span class="sxs-lookup"><span data-stu-id="12303-199">These guidelines do not preclude general design principles for class libraries.</span></span> <span data-ttu-id="12303-200">而是在最適合用於使用 Visual Basic 開發人員的建議和`My`命名空間。</span><span class="sxs-lookup"><span data-stu-id="12303-200">Rather, they are recommendations that are optimized for developers who are using Visual Basic and the `My` namespace.</span></span> <span data-ttu-id="12303-201">建立類別庫的一般設計原則，請參閱[Framework 設計方針](http://msdn.microsoft.com/library/5fbcaf4f-ea2a-4d20-b0d6-e61dee202b4b)。</span><span class="sxs-lookup"><span data-stu-id="12303-201">For general design principles for creating class libraries, see [Framework Design Guidelines](http://msdn.microsoft.com/library/5fbcaf4f-ea2a-4d20-b0d6-e61dee202b4b).</span></span>
