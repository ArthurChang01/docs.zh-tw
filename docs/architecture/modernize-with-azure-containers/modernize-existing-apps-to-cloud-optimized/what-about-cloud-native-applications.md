## <a name="what-about-microservices"></a><span data-ttu-id="a5d1a-137">微服務呢？</span><span class="sxs-lookup"><span data-stu-id="a5d1a-137">What about microservices?</span></span>

<span data-ttu-id="a5d1a-138">當您為組織考慮雲原生應用程式時，瞭解微服務及其工作方式非常重要。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-138">Understanding microservices and how they work is important when you are considering cloud-native applications for your organization.</span></span>

<span data-ttu-id="a5d1a-139">微服務體系結構是一種高級方法，可用於從頭開始創建的應用程式，或者當您將現有應用程式發展到雲原生應用程式時。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-139">The microservices architecture is an advanced approach that you can use for applications that are created from scratch or when you evolve existing applications toward cloud-native applications.</span></span> <span data-ttu-id="a5d1a-140">您可以首先向現有應用程式添加一些微服務，以瞭解新的微服務範例。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-140">You can start by adding a few microservices to existing applications to learn about the new microservices paradigms.</span></span> <span data-ttu-id="a5d1a-141">但顯然，您需要構建和編碼，尤其是對於這種類型的體系結構方法。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-141">But clearly, you need to architect and code, especially for this type of architectural approach.</span></span>

<span data-ttu-id="a5d1a-142">但是，對於任何新的或現代的應用程式來說，微服務並不是強制性的。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-142">However, microservices are not mandatory for any new or modern application.</span></span> <span data-ttu-id="a5d1a-143">微服務不是"靈丹妙藥"，它們不是創建每個應用程式的單一最佳方法。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-143">Microservices are not a "magic bullet," and they aren't the single, best way to create every application.</span></span> <span data-ttu-id="a5d1a-144">使用微服務的方式和時間取決於需要構建的應用程式類型。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-144">How and when you use microservices depends on the type of application that you need to build.</span></span>

<span data-ttu-id="a5d1a-145">微服務體系結構正在成為分散式、大型或複雜任務關鍵型應用程式的首選方法，這些應用程式基於多個獨立子系統，以自治服務的形式存在。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-145">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple, independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="a5d1a-146">在基於微服務的體系結構中，應用程式構建為可獨立開發、測試、版本控制、部署和縮放的服務集合。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-146">In a microservices-based architecture, an application is built as a collection of services that can be independently developed, tested, versioned, deployed, and scaled.</span></span> <span data-ttu-id="a5d1a-147">這可以包括每個微服務的任何相關的自主資料庫。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-147">This can include any related, autonomous database per microservice.</span></span>

<span data-ttu-id="a5d1a-148">有關可以使用 .NET Core 實現的微服務體系結構的詳細資訊，請參閱可下載的 PDF 電子書[.NET 微服務：容器化 .NET 應用程式的體系結構](https://aka.ms/microservicesebook)。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-148">For a detailed look at a microservices architecture that you can implement by using .NET Core, see the downloadable PDF e-book [.NET microservices: Architecture for containerized .NET applications](https://aka.ms/microservicesebook).</span></span> <span data-ttu-id="a5d1a-149">指南也可[線上](../../microservices/index.md)獲取。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-149">The guide also is available [online](../../microservices/index.md).</span></span>

<span data-ttu-id="a5d1a-150">但是，即使在微服務提供與功能無關的強大部署、強大的子系統邊界和技術多樣性的情況下，它們也會帶來許多新的挑戰。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-150">But even in scenarios in which microservices offer powerful capabilities-independent deployment, strong subsystem boundaries, and technology diversity-they also raise many new challenges.</span></span> <span data-ttu-id="a5d1a-151">這些挑戰與分散式應用程式開發有關，例如零碎和獨立的資料模型;實現微服務之間的彈性通信;需要最終一致性;和操作複雜性。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-151">The challenges are related to distributed application development, such as fragmented and independent data models; achieving resilient communication between microservices; the need for eventual consistency; and operational complexity.</span></span> <span data-ttu-id="a5d1a-152">與傳統的單片應用相比，微服務引入了更高的複雜性。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-152">Microservices introduce a higher level of complexity compared to traditional monolithic applications.</span></span>

<span data-ttu-id="a5d1a-153">由於微服務體系結構的複雜性，只有特定的方案和某些應用程式類型適用于基於微服務的應用程式。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-153">Because of the complexity of a microservices architecture, only specific scenarios and certain application types are suitable for microservice-based applications.</span></span> <span data-ttu-id="a5d1a-154">其中包括具有多個不斷發展的子系統的大型複雜應用程式。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-154">These include large and complex applications that have multiple, evolving subsystems.</span></span> <span data-ttu-id="a5d1a-155">在這些情況下，值得投資更複雜的軟體體系結構，提高長期敏捷性和更高效的應用程式維護。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-155">In these cases, it's worth investing in a more complex software architecture, for increased long-term agility and more efficient application maintenance.</span></span> <span data-ttu-id="a5d1a-156">但對於不太複雜的方案，最好繼續採用整體應用程式方法或更簡單的 N-Tier 方法。</span><span class="sxs-lookup"><span data-stu-id="a5d1a-156">But for less complex scenarios, it might be better to continue with a monolithic application approach or simpler N-Tier approaches.</span></span>
