# <a name="conclusions"></a>結論

- 以容器為基礎的解決方案最終提供節省成本的優點。 容器是部署問題的解決方案, 因為它們會移除在生產環境中缺少相依性所造成的摩擦。 藉由移除這些問題, 可大幅改善開發/測試、DevOps 和生產作業。

- Docker 容器會變成任何以伺服器為基礎之應用程式或服務的標準部署單位。

- 針對生產環境, 您應該使用協調器 (例如 Kubernetes) 來裝載可擴充的容器型應用程式。

- Azure Vm 裝載容器是在雲端中建立小型開發/測試環境的快速且簡單的方式。

- 根據預設, 將關係資料庫從現有應用程式遷移至 Azure 時, 建議使用 Azure SQL Database 受控執行個體。

- Visual Studio 2017 和 Image2Docker 是基本工具, 可讓您藉由加速快速入門學習曲線, 使用 Windows 容器開始現代化現有的 .NET 應用程式。

- 將容器化應用程式放在生產環境中時, 您一律會建立或採用 CI/CD 管線的 DevOps 文化特性和 DevOps 工具, 例如 Azure DevOps Services 或 Jenkins。

- Microsoft Azure 提供最全面且完整的環境, 讓您的現有 .NET Framework 應用程式與 Windows 容器、雲端基礎結構和 PaaS 服務現代化。
