# <a name="conclusions"></a>結論

- 基於容器的解決方案最終提供成本節約優勢。 容器是解決部署問題的解決方案，因為它們消除了生產環境中缺少依賴關係造成的摩擦。 通過消除這些問題，它顯著改善了開發人員/測試、DevOps 和生產操作。

- Docker 容器會變成任何以伺服器為基礎之應用程式或服務的標準部署單位。

- 對於生產環境，應使用協調器（如 Kubernetes）來託管基於容器的可擴展應用程式。

- 託管容器的 Azure VM 是在雲中創建小型開發人員/測試環境的快速簡便方法。

- 預設情況下，建議將關係資料庫從現有應用程式遷移到 Azure 時，建議使用 Azure SQL 資料庫託管實例。

- Visual Studio 2017 和 Image2Docker 是您通過加快入門學習曲線開始使用 Windows 容器開始實現現有 .NET 應用程式現代化的基本工具。

- 將容器化應用程式放入生產中時，您始終會為 CI/CD 管道（如 Azure DevOps 服務或 Jenkins）創建或採用 DevOps 區域性和 DevOps 工具。

- Microsoft Azure 提供了最全面、最完整的環境，以便使用 Windows 容器、雲基礎結構和 PaaS 服務實現現有 .NET 框架應用程式的現代化。
