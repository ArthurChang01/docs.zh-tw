## <a name="a-simple-analogy"></a><span data-ttu-id="d4f4c-148">簡單的比喻</span><span class="sxs-lookup"><span data-stu-id="d4f4c-148">A simple analogy</span></span>

<span data-ttu-id="d4f4c-149">也許簡單的比喻可以協助您掌握 Docker 的核心概念。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-149">Perhaps a simple analogy can help getting the grasp of the core concept of Docker.</span></span>

<span data-ttu-id="d4f4c-150">讓我們暫時回到 1950 年。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-150">Let's go back in time to the 1950s for a moment.</span></span> <span data-ttu-id="d4f4c-151">當時沒有任何文書處理器，而且到處都在使用影印機 (差不多所有人都在用)。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-151">There were no word processors, and the photocopiers were used everywhere (kind of).</span></span>

<span data-ttu-id="d4f4c-152">假設您負責根據需求來快速發送大批信件，並使用信紙和信封，將信件實際投遞到每個客戶的地址 (當時並沒有電子郵件)。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-152">Imagine you're responsible for quickly issuing batches of letters as required, to mail them to customers, using real paper and envelopes, to be delivered physically to each customer's address (there was no email back then).</span></span>

<span data-ttu-id="d4f4c-153">慢慢地，您了解到這些信件的內容是大量段落的組合，只是根據信件目的從中挑選和編排需要的段落，因此您設計一個系統，可以快速地發送信件來提高效率。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-153">At some point, you realize the letters are just a composition of a large set of paragraphs, which are picked and arranged as needed, according to the purpose of the letter, so you devise a system to issue letters quickly, expecting to get a hefty raise.</span></span>

<span data-ttu-id="d4f4c-154">系統很簡單：</span><span class="sxs-lookup"><span data-stu-id="d4f4c-154">The system is simple:</span></span>

1. <span data-ttu-id="d4f4c-155">您開始準備一疊透明紙張，每一張紙都包含一個段落。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-155">You begin with a deck of transparent sheets containing one paragraph each.</span></span>

2. <span data-ttu-id="d4f4c-156">若要發送一批信件，您就挑選具有所需段落的紙張，然後將其堆疊並且排列整齊，讓它們整潔易讀。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-156">To issue a set of letters, you pick the sheets with the paragraphs you need, then you stack and align them so they look and read fine.</span></span>

3. <span data-ttu-id="d4f4c-157">最後，您將這個組合放入影印機中，然後按下開始以產生所需數量的信件。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-157">Finally, you place the set in the photocopier and press start to produce as many letters as required.</span></span>

<span data-ttu-id="d4f4c-158">因此，工作精簡了，這就是 Docker 的核心概念。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-158">So, simplifying, that's the core idea of Docker.</span></span>

<span data-ttu-id="d4f4c-159">Docker 中的每一層都是在執行命令 (例如安裝程式) 之後，檔案系統上所發生的變更結果集。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-159">In Docker, each layer is the resulting set of changes that happen to the filesystem after executing a command, such as, installing a program.</span></span>

<span data-ttu-id="d4f4c-160">因此，當您在複製層之後「查看」檔案系統時，會看到檔案整體，包含安裝程式的層。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-160">So, when you "look" at the filesystem after the layer has been copied, you see all the files, included the layer when the program was installed.</span></span>

<span data-ttu-id="d4f4c-161">您可以將映像當作是準備要在「電腦」(已安裝作業系統) 中安裝的輔助唯讀硬碟。</span><span class="sxs-lookup"><span data-stu-id="d4f4c-161">You can think of an image as an auxiliary read-only hard disk ready to be installed in a "computer" where the operating system is already installed.</span></span>
