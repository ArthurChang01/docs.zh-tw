### <a name="httpclient-lifetimes"></a><span data-ttu-id="ef955-151">HttpClient 存留期</span><span class="sxs-lookup"><span data-stu-id="ef955-151">HttpClient lifetimes</span></span>

<span data-ttu-id="ef955-154">處理常式的集合是需要的做法，因為每個處理常式通常會管理自己的基礎 HTTP 連線；建立比所需數目更多的處理常式可能會導致連線延遲。</span><span class="sxs-lookup"><span data-stu-id="ef955-154">Pooling of handlers is desirable as each handler typically manages its own underlying HTTP connections; creating more handlers than necessary can result in connection delays.</span></span> <span data-ttu-id="ef955-155">有些處理常式也會保持連線無限期地開啟，這可能導致處理常式無法對 DNS 變更回應。</span><span class="sxs-lookup"><span data-stu-id="ef955-155">Some handlers also keep connections open indefinitely, which can prevent the handler from reacting to DNS changes.</span></span>
