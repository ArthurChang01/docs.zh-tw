# <a name="introduction"></a><span data-ttu-id="b49ef-103">簡介</span><span class="sxs-lookup"><span data-stu-id="b49ef-103">Introduction</span></span>

<span data-ttu-id="b49ef-104">協助機器彼此通訊的方式，是數位年齡的主要 preoccupations 之一。</span><span class="sxs-lookup"><span data-stu-id="b49ef-104">Helping machines communicate with each other has been one of the primary preoccupations of the digital age.</span></span> <span data-ttu-id="b49ef-105">特別是，我們會持續致力於判斷符合目前基礎結構之互通性需求的最佳遠端通訊機制。</span><span class="sxs-lookup"><span data-stu-id="b49ef-105">In particular, there is an ongoing effort to determine the optimal remote communication mechanism that will suit the interoperability demands of the current infrastructure.</span></span> <span data-ttu-id="b49ef-106">您可以想像，這項機制會隨著需求或基礎結構的發展而改變。</span><span class="sxs-lookup"><span data-stu-id="b49ef-106">As you can imagine, that mechanism changes as either the demands or the infrastructure evolves.</span></span>

<span data-ttu-id="b49ef-107">.NET Core 3.0 的推出會在 Microsoft 為想要跨多種平臺提供服務的開發人員提供遠端通訊解決方案的方式。</span><span class="sxs-lookup"><span data-stu-id="b49ef-107">The release of .NET Core 3.0 marks a shift in the way that Microsoft delivers remote communication solutions to developers who want to deliver services across a range of platforms.</span></span> <span data-ttu-id="b49ef-108">.NET Core 不提供現成 Windows Communication Foundation （WCF），但在版本 ASP.NET Core 3.0 中，它提供內建的 gRPC 功能。</span><span class="sxs-lookup"><span data-stu-id="b49ef-108">.NET Core doesn't offer Windows Communication Foundation (WCF) out of the box but, with the release of version ASP.NET Core 3.0, it does provide built-in gRPC functionality.</span></span>

<span data-ttu-id="b49ef-109">gRPC 是廣泛軟體發展的熱門架構，適用于各種程式設計語言，供新式 RPC 案例使用。</span><span class="sxs-lookup"><span data-stu-id="b49ef-109">gRPC is a popular framework in the wider software community, used by developers across many programming languages for modern RPC scenarios.</span></span> <span data-ttu-id="b49ef-110">此社區和生態系統既活躍又活躍，而且支援將 gRPC 通訊協定新增至基礎結構元件，例如 Kubernetes、服務網格、負載平衡器等等。</span><span class="sxs-lookup"><span data-stu-id="b49ef-110">The community and the ecosystem are vibrant and active, with support for the gRPC protocol being added to infrastructure components like Kubernetes, service meshes, load balancers and more.</span></span> <span data-ttu-id="b49ef-111">這些因素，以及其效能、效率和跨平臺相容性，讓 gRPC 成為新應用程式的自然選擇，以及移至 .NET Core 的 WCF 應用程式。</span><span class="sxs-lookup"><span data-stu-id="b49ef-111">These factors, as well as its performance, efficiency and cross-platform compatibility, make gRPC a natural choice for new apps and WCF apps moving to .NET Core.</span></span>

## <a name="history"></a><span data-ttu-id="b49ef-112">歷程</span><span class="sxs-lookup"><span data-stu-id="b49ef-112">History</span></span>

<span data-ttu-id="b49ef-113">電腦網路的基本原則，只是一組電腦彼此交換資料，以達成一系列相互關聯的工作，而不會在其開始後變更。</span><span class="sxs-lookup"><span data-stu-id="b49ef-113">The fundamental principle of a computer network as nothing more than a group of computers exchanging data with each other to achieve a set of interrelated tasks hasn't changed since its inception.</span></span> <span data-ttu-id="b49ef-114">不過，複雜度、規模和預期已以指數方式成長。</span><span class="sxs-lookup"><span data-stu-id="b49ef-114">However, the complexity, scale, and expectations have grown exponentially.</span></span>  

<span data-ttu-id="b49ef-115">在二十世紀，重點主要著重于使用相同的語言和平臺來改善內部網路。</span><span class="sxs-lookup"><span data-stu-id="b49ef-115">During the 1990s, the emphasis had been mainly focused on improving internal networks using the same language and platforms.</span></span> <span data-ttu-id="b49ef-116">TCP/IP 已成為這種通訊類型的黃金標準。</span><span class="sxs-lookup"><span data-stu-id="b49ef-116">TCP/IP became the gold standard for this type of communication.</span></span>

<span data-ttu-id="b49ef-117">不過，此焦點很快就會轉移到跨多個平臺的最佳通訊效能，進而提升語言中立的方法。</span><span class="sxs-lookup"><span data-stu-id="b49ef-117">However, the focus soon shifted to how best to optimize communication across multiple platforms promoting a language agnostic approach.</span></span> <span data-ttu-id="b49ef-118">服務導向架構（SOA）提供了一種結構，可讓您將廣泛的服務集合提供給應用程式。</span><span class="sxs-lookup"><span data-stu-id="b49ef-118">Service-oriented architecture (SOA) provided a structure for loosely coupling a broad collection of services that could be provided to an application.</span></span>

<span data-ttu-id="b49ef-119">當所有主要平臺都能存取網際網路，但仍無法彼此互動時，就會發生*Web 服務*的開發。</span><span class="sxs-lookup"><span data-stu-id="b49ef-119">The development of *Web Services* occurred once all major platforms could access the Internet, but they still couldn’t interact with each other.</span></span> <span data-ttu-id="b49ef-120">Web 服務具有開放式標準和通訊協定，包括：</span><span class="sxs-lookup"><span data-stu-id="b49ef-120">Web services have open standards and protocols including:</span></span>

- <span data-ttu-id="b49ef-121">XML 來標記和程式碼資料;</span><span class="sxs-lookup"><span data-stu-id="b49ef-121">XML to tag and code data;</span></span>
- <span data-ttu-id="b49ef-122">簡單的物件存取通訊協定（SOAP），用來傳送資料;</span><span class="sxs-lookup"><span data-stu-id="b49ef-122">Simple Object Access Protocol (SOAP) to transfer data;</span></span>
- <span data-ttu-id="b49ef-123">Web 服務定義語言（WSDL）–描述並將 web 服務連接到用戶端應用程式;*和*</span><span class="sxs-lookup"><span data-stu-id="b49ef-123">Web Services Definition Language (WSDL) – describes and connects the web service to the client application; *and*</span></span>
- <span data-ttu-id="b49ef-124">通用描述探索整合（UDDI）-讓其他服務能夠探索 Web 服務</span><span class="sxs-lookup"><span data-stu-id="b49ef-124">Universal Description Discovery Integration (UDDI)- enables Web Services to be discoverable by other Services</span></span>

<span data-ttu-id="b49ef-125">SOAP 會定義應用程式的分散式元素可以彼此通訊的規則，即使它們在不同的平臺上也一樣。</span><span class="sxs-lookup"><span data-stu-id="b49ef-125">SOAP defines the rules by which distributed elements of an application can communicate with each other even if they are on different platforms.</span></span> <span data-ttu-id="b49ef-126">SOAP 是以 XML 為基礎，因此是人類看得懂的。</span><span class="sxs-lookup"><span data-stu-id="b49ef-126">SOAP is based on XML, so it's human-readable.</span></span> <span data-ttu-id="b49ef-127">讓 SOAP 易於瞭解的犧牲是大小;SOAP 訊息大於可比較通訊協定中的訊息。</span><span class="sxs-lookup"><span data-stu-id="b49ef-127">The sacrifice for making SOAP easily understood is size; SOAP messages are larger than messages in comparable protocols.</span></span> <span data-ttu-id="b49ef-128">SOAP 的設計是將整合型應用程式分解成多元件形式，而不會失去安全性或控制。</span><span class="sxs-lookup"><span data-stu-id="b49ef-128">SOAP was designed to break down monolithic applications into multi-component form without losing security or control.</span></span> <span data-ttu-id="b49ef-129">因此，WCF 的設計可使用這種類型的系統，與從分散式系統開始的 gRPC 不同。</span><span class="sxs-lookup"><span data-stu-id="b49ef-129">Therefore, WCF was designed to work with that kind of system, unlike gRPC, which began as a distributed system.</span></span> <span data-ttu-id="b49ef-130">WCF 藉由開發並記載 SOAP 堆疊的專屬擴充通訊協定，但代價是缺少其他平臺的支援，來解決其中一些限制。</span><span class="sxs-lookup"><span data-stu-id="b49ef-130">WCF addressed some of these limitations by developing and documenting proprietary extension protocols for the SOAP stack, but at the cost of lack of support from other platforms.</span></span>

<span data-ttu-id="b49ef-131">Windows Communication Foundation 是用來建立服務的架構。</span><span class="sxs-lookup"><span data-stu-id="b49ef-131">Windows Communication Foundation is a framework for building services.</span></span> <span data-ttu-id="b49ef-132">其設計在早期的2000s 中，可協助開發人員使用舊版 SOA 來管理與 SOAP 搭配運作的複雜性。</span><span class="sxs-lookup"><span data-stu-id="b49ef-132">It was designed in the early 2000s to help developers using early SOA to manage the complexities of working with SOAP.</span></span> <span data-ttu-id="b49ef-133">雖然它會移除開發人員撰寫自己的 SOAP 通訊協定的需求，但是 WCF 仍會使用 SOAP 來啟用與其他系統的互通性。</span><span class="sxs-lookup"><span data-stu-id="b49ef-133">Although it removes the requirement for the developer to write their own SOAP protocols, WCF still uses SOAP to enable interoperability with other systems.</span></span> <span data-ttu-id="b49ef-134">WCF 也是設計來提供跨多個通訊協定（HTTP/1.1、NetTCP 等）的解決方案。</span><span class="sxs-lookup"><span data-stu-id="b49ef-134">WCF was also designed to deliver solutions across multiple protocols (HTTP/1.1, NetTCP, and so on).</span></span>

## <a name="microservices"></a><span data-ttu-id="b49ef-135">微服務</span><span class="sxs-lookup"><span data-stu-id="b49ef-135">Microservices</span></span>

<span data-ttu-id="b49ef-136">在微服務架構中，大型應用程式是以較小的模組化服務集合來建立。</span><span class="sxs-lookup"><span data-stu-id="b49ef-136">In microservice architectures, large applications are built as a collection of smaller modular services.</span></span> <span data-ttu-id="b49ef-137">每個元件都會執行特定的工作或程式，而元件則是設計成可運作 interoperably，但可以視需要加以隔離。</span><span class="sxs-lookup"><span data-stu-id="b49ef-137">Each component does a specific task or process, and components are designed to work interoperably but can be isolated as necessary.</span></span>

<span data-ttu-id="b49ef-138">微服務的優點包括：</span><span class="sxs-lookup"><span data-stu-id="b49ef-138">Advantages to microservices include:</span></span>

- <span data-ttu-id="b49ef-139">變更和升級可以獨立處理。</span><span class="sxs-lookup"><span data-stu-id="b49ef-139">Changes and upgrades can be handled independently.</span></span>
- <span data-ttu-id="b49ef-140">錯誤處理會變得更有效率，因為可以追蹤特定服務的問題，然後獨立于其他服務之外隔離、重建、測試和重新部署。</span><span class="sxs-lookup"><span data-stu-id="b49ef-140">Error handling becomes more efficient as problems can be traced to specific services that are then isolated, rebuilt, tested, and redeployed independent of the other services.</span></span>
- <span data-ttu-id="b49ef-141">擴充性可以限制為特定的實例或服務，而不是整個應用程式。</span><span class="sxs-lookup"><span data-stu-id="b49ef-141">Scalability can be confined to the specific instances or services rather than the whole application.</span></span>
- <span data-ttu-id="b49ef-142">開發可以跨多個小組進行，相較于許多小組在單一程式碼基底上工作，會有更少的摩擦。</span><span class="sxs-lookup"><span data-stu-id="b49ef-142">Development can happen across multiple teams, with less friction than occurs when many teams work on a single codebase.</span></span>

<span data-ttu-id="b49ef-143">隨著日益增加的虛擬化、雲端運算、容器和物聯網的發展，已貢獻微服務的持續成長。</span><span class="sxs-lookup"><span data-stu-id="b49ef-143">The move towards increasing virtualization, cloud computing, containers and the Internet of Things have contributed to the ongoing rise of microservices.</span></span> <span data-ttu-id="b49ef-144">不過，微服務不會有其挑戰。</span><span class="sxs-lookup"><span data-stu-id="b49ef-144">However, microservices aren't without their challenges.</span></span> <span data-ttu-id="b49ef-145">分散/非集中式基礎結構在服務之間進行通訊時，會更重視簡單性和速度的需求。</span><span class="sxs-lookup"><span data-stu-id="b49ef-145">The fragmented/decentralized infrastructure placed more emphasis on the need for simplicity and speed when communicating between services.</span></span> <span data-ttu-id="b49ef-146">這又會特別注意 SOAP 的偶爾費力和 contorted 本質。</span><span class="sxs-lookup"><span data-stu-id="b49ef-146">This in turn drew attention to the sometimes laborious and contorted nature of SOAP.</span></span>

<span data-ttu-id="b49ef-147">在此環境中，gRPC 已啟動，Microsoft 第一次發行 WCF 之後10年。</span><span class="sxs-lookup"><span data-stu-id="b49ef-147">It was into this environment that gRPC was launched, 10 years after Microsoft first released WCF.</span></span> <span data-ttu-id="b49ef-148">直接從 Google 的內部基礎結構 RPC （Stubby）演變，gRPC 從未以已通知許多舊版 Rpc 參數的相同標準和通訊協定為基礎。</span><span class="sxs-lookup"><span data-stu-id="b49ef-148">Evolved directly from Google’s internal infrastructure RPC (Stubby), gRPC was never based on the same standards and protocols that had informed the parameters of many earlier RPCs.</span></span> <span data-ttu-id="b49ef-149">此外，gRPC 只會以 HTTP/2 為基礎，因此它可以在先進的傳輸通訊協定提供的新功能上進行繪製。</span><span class="sxs-lookup"><span data-stu-id="b49ef-149">Additionally, gRPC was only ever based on HTTP/2 and that's why it could draw on the new capabilities that advanced transport protocol provided.</span></span> <span data-ttu-id="b49ef-150">特別是雙向串流、二進位訊息和多工。</span><span class="sxs-lookup"><span data-stu-id="b49ef-150">In particular, bidirectional streaming, binary messaging, and multiplexing.</span></span>

<span data-ttu-id="b49ef-152">本指南涵蓋 gRPC 的主要功能。</span><span class="sxs-lookup"><span data-stu-id="b49ef-152">This guide covers the key features of gRPC.</span></span> <span data-ttu-id="b49ef-153">早期章節會大致瞭解 WCF 的主要功能，並將其與 gRPC 比較。</span><span class="sxs-lookup"><span data-stu-id="b49ef-153">The early chapters take a high-level look at the main features of WCF and compare them to gRPC.</span></span> <span data-ttu-id="b49ef-154">它會識別在 WCF 和 gRPC 之間直接相互關聯的位置，以及 gRPC 提供優勢的位置。</span><span class="sxs-lookup"><span data-stu-id="b49ef-154">It identifies where the are direct correlations between WCF and gRPC and also where gRPC offers an advantage.</span></span> <span data-ttu-id="b49ef-155">在 WCF 和 gRPC 之間沒有相互關聯，或 gRPC 無法提供對 WCF 的對等解決方案時，本指南將會建議因應措施或位置以取得進一步的資訊。</span><span class="sxs-lookup"><span data-stu-id="b49ef-155">Where there's no correlation between WCF and gRPC or where gRPC isn't able to offer an equivalent solution to WCF, this guide will suggest workarounds or places to go for further information.</span></span>
