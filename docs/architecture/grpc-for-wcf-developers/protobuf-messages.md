## <a name="declaring-a-message"></a><span data-ttu-id="0ceba-106">宣告訊息</span><span class="sxs-lookup"><span data-stu-id="0ceba-106">Declaring a message</span></span>

<span data-ttu-id="0ceba-107">在 WCF 中， `Stock`股票市場貿易應用程式的類別可能會如下列範例所定義：</span><span class="sxs-lookup"><span data-stu-id="0ceba-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="0ceba-108">若要在 Protobuf 中執行對等的類別，它必須在`.proto`檔案中宣告。</span><span class="sxs-lookup"><span data-stu-id="0ceba-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="0ceba-109">然後`protoc`編譯器會在組建程式中產生 .net 類別。</span><span class="sxs-lookup"><span data-stu-id="0ceba-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string displayName = 3;
    int32 marketId = 4;

}  
```

<span data-ttu-id="0ceba-110">第一行會宣告所使用的語法版本。</span><span class="sxs-lookup"><span data-stu-id="0ceba-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="0ceba-111">第3版語言已于2016發行，而且是建議的 gRPC 服務版本。</span><span class="sxs-lookup"><span data-stu-id="0ceba-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="0ceba-112">這一行會指定要用於產生C#之類型的命名空間。 `option csharp_namespace`</span><span class="sxs-lookup"><span data-stu-id="0ceba-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="0ceba-113">針對其他語言編譯檔案時， `.proto`將會忽略此選項。</span><span class="sxs-lookup"><span data-stu-id="0ceba-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="0ceba-114">Protobuf 檔案通常會包含數種語言的語言特定選項。</span><span class="sxs-lookup"><span data-stu-id="0ceba-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="0ceba-115">`Stock`訊息定義會指定四個欄位，各有一個類型、一個名稱和一個欄位編號。</span><span class="sxs-lookup"><span data-stu-id="0ceba-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="0ceba-116">欄位編號</span><span class="sxs-lookup"><span data-stu-id="0ceba-116">Field numbers</span></span>

<span data-ttu-id="0ceba-117">欄位編號是 Protobuf 中很重要的一部分。</span><span class="sxs-lookup"><span data-stu-id="0ceba-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="0ceba-118">它們是用來識別二進位編碼資料中的欄位，這表示它們無法從版本變更為您的服務版本。</span><span class="sxs-lookup"><span data-stu-id="0ceba-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="0ceba-119">優點是可以進行回溯和回溯相容性。</span><span class="sxs-lookup"><span data-stu-id="0ceba-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="0ceba-120">只要處理遺漏值的可能性，用戶端和服務就會直接忽略他們不知道的欄位編號。</span><span class="sxs-lookup"><span data-stu-id="0ceba-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>

<span data-ttu-id="0ceba-121">在二進位格式中，欄位編號會與類型識別碼結合。</span><span class="sxs-lookup"><span data-stu-id="0ceba-121">In the binary format, the field number is combined with a type identifier.</span></span> <span data-ttu-id="0ceba-122">從1到15的欄位可以用其類型編碼為單一位元組;從16到2047的數位需要2個位元組。</span><span class="sxs-lookup"><span data-stu-id="0ceba-122">Field numbers from 1 to 15 can be encoded with their type as a single byte; numbers from 16 to 2047 take 2 bytes.</span></span> <span data-ttu-id="0ceba-123">如果因任何原因而需要訊息超過2047個欄位，您就可以更高。</span><span class="sxs-lookup"><span data-stu-id="0ceba-123">You can go higher if you need more than 2047 fields on a message for any reason.</span></span> <span data-ttu-id="0ceba-124">欄位編號1到15的單一位元組識別碼提供較佳的效能，因此您應該將它們用於最基本、常用的欄位。</span><span class="sxs-lookup"><span data-stu-id="0ceba-124">The single byte identifiers for field numbers 1 to 15 offer better performance, so you should use them for the most basic, frequently used fields.</span></span>

## <a name="the-generated-code"></a><span data-ttu-id="0ceba-130">產生的程式碼</span><span class="sxs-lookup"><span data-stu-id="0ceba-130">The generated code</span></span>

<span data-ttu-id="0ceba-131">當您建立應用程式時，Protobuf 會為您的每個訊息建立類別，並將C#其原生類型對應至類型。</span><span class="sxs-lookup"><span data-stu-id="0ceba-131">When you build your application, Protobuf creates classes for each of your messages, mapping its native types to C# types.</span></span> <span data-ttu-id="0ceba-132">產生`Stock`的類型會具有下列簽章：</span><span class="sxs-lookup"><span data-stu-id="0ceba-132">The generated `Stock` type would have the following signature:</span></span>

```csharp

    public int Id { get; set; }
    public string Symbol { get; set; }
    public string DisplayName { get; set; }
    public int MarketId { get; set; }
}
```

<span data-ttu-id="0ceba-133">所產生的實際程式碼遠比這個更為複雜，因為每個類別都包含將本身序列化和還原序列化為二進位電傳格式所需的所有程式碼。</span><span class="sxs-lookup"><span data-stu-id="0ceba-133">The actual code that is generated is far more complicated than this, because each class contains all the code necessary to serialize and deserialize itself to the binary wire format.</span></span>
