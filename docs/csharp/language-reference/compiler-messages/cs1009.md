# <a name="compiler-error-cs1009"></a><span data-ttu-id="766e6-102">編譯器錯誤 CS1009</span><span class="sxs-lookup"><span data-stu-id="766e6-102">Compiler Error CS1009</span></span>
<span data-ttu-id="766e6-103">無法辨識的逸出序列</span><span class="sxs-lookup"><span data-stu-id="766e6-103">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="766e6-104">在[字串](../../../csharp/language-reference/keywords/string.md)中反斜線 (\\) 的後方出現未預期的字元。</span><span class="sxs-lookup"><span data-stu-id="766e6-104">An unexpected character follows a backslash (\\) in a [string](../../../csharp/language-reference/keywords/string.md).</span></span> <span data-ttu-id="766e6-105">編譯器需要其中一個有效的逸出字元。</span><span class="sxs-lookup"><span data-stu-id="766e6-105">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="766e6-106">如需詳細資訊，請參閱[逸出字元](../../../standard/base-types/character-escapes-in-regular-expressions.md)。</span><span class="sxs-lookup"><span data-stu-id="766e6-106">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="766e6-107">下列範例會產生 CS1009。</span><span class="sxs-lookup"><span data-stu-id="766e6-107">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";     
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="766e6-108">通常發生此錯誤的原因是在檔案名稱中使用反斜線字元，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="766e6-108">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="766e6-109">若要解決這個錯誤，請使用 "\\\\" 或 @-quoted 字串常值，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="766e6-109">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";     
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
