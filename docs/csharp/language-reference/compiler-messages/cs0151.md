# <a name="compiler-error-cs0151"></a><span data-ttu-id="034ad-102">編譯器錯誤 CS0151</span><span class="sxs-lookup"><span data-stu-id="034ad-102">Compiler Error CS0151</span></span>
<span data-ttu-id="034ad-103">必須是整數型別的值</span><span class="sxs-lookup"><span data-stu-id="034ad-103">A value of an integral type expected</span></span>  
  
 <span data-ttu-id="034ad-104">在需要整數資料型別的情況下使用了變數。</span><span class="sxs-lookup"><span data-stu-id="034ad-104">A variable was used in a situation where an integral data type was required.</span></span> <span data-ttu-id="034ad-105">如需詳細資訊，請參閱[類型](../../../csharp/programming-guide/types/index.md)。</span><span class="sxs-lookup"><span data-stu-id="034ad-105">For more information, see [Types](../../../csharp/programming-guide/types/index.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="034ad-106">範例</span><span class="sxs-lookup"><span data-stu-id="034ad-106">Example</span></span>  
 <span data-ttu-id="034ad-107">如果沒有轉換，或是可用的隱含轉換導致模稜兩可的情況，就會發生此錯誤。</span><span class="sxs-lookup"><span data-stu-id="034ad-107">This error can occur when there is no conversion or if the available implicit conversions result in an ambiguous situation.</span></span> <span data-ttu-id="034ad-108">下列範例會產生 CS0151。</span><span class="sxs-lookup"><span data-stu-id="034ad-108">The following sample generates CS0151.</span></span>  
  
```csharp  
// CS0151.cs  
public class MyClass  
{  
   public static implicit operator int (MyClass aa)  
   {  
      return 0;  
   }  
  
   public static implicit operator long (MyClass aa)  
   {  
      return 0;  
   }  
  
   public static void Main()  
   {  
      MyClass a = new MyClass();  
  
      // Compiler cannot choose between int and long  
      switch (a)   // CS0151  
      // try the following line instead  
      // switch ((int)a)  
      {  
         case 1:  
            break;  
      }  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="034ad-109">範例</span><span class="sxs-lookup"><span data-stu-id="034ad-109">Example</span></span>  
 <span data-ttu-id="034ad-110">在 Visual Studio 2008 和更新版本中，[void](../../../csharp/language-reference/keywords/void.md) 方法引動過程會產生 CS0151。</span><span class="sxs-lookup"><span data-stu-id="034ad-110">In Visual Studio 2008 and later, a [void](../../../csharp/language-reference/keywords/void.md) method invocation generates CS0151.</span></span> <span data-ttu-id="034ad-111">您可以呼叫傳回整數型別 (例如 [int](../../../csharp/language-reference/keywords/int.md) 或 [long](../../../csharp/language-reference/keywords/long.md)) 的方法，來修正此錯誤。</span><span class="sxs-lookup"><span data-stu-id="034ad-111">You can fix the error by calling a method that returns an integral type such as [int](../../../csharp/language-reference/keywords/int.md) or [long](../../../csharp/language-reference/keywords/long.md).</span></span>  
  
```csharp  
class C  
{  
    static void Main()  
    {  
  
        switch (M()) // CS0151  
        {  
            default:  
                break;  
        }  
    }  
  
    static void M()  
    {  
    }  
}  
```
