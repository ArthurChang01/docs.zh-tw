# <a name="compiler-error-cs0433"></a><span data-ttu-id="4318a-102">編譯器錯誤 CS0433</span><span class="sxs-lookup"><span data-stu-id="4318a-102">Compiler Error CS0433</span></span>
<span data-ttu-id="4318a-103">類型 TypeName1 同時存在於 TypeName2 和 TypeName3 中</span><span class="sxs-lookup"><span data-stu-id="4318a-103">The type TypeName1 exists in both TypeName2 and TypeName3</span></span>  
  
 <span data-ttu-id="4318a-104">您的應用程式中參考了包含相同命名空間和類型的兩個不同組件，因而產生模稜兩可的情況。</span><span class="sxs-lookup"><span data-stu-id="4318a-104">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>  
  
 <span data-ttu-id="4318a-105">若要解決這個錯誤，請使用 [/reference (C# 編譯器選項)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) 編譯器選項的別名功能，或者不要參考其中一個組件。</span><span class="sxs-lookup"><span data-stu-id="4318a-105">To resolve this error, use the alias feature of the [/reference (C# Compiler Options)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) compiler option or do not reference one of your assemblies.</span></span>  
  
## <a name="example"></a><span data-ttu-id="4318a-106">範例</span><span class="sxs-lookup"><span data-stu-id="4318a-106">Example</span></span>

```csharp  
}  
```

## <a name="example"></a><span data-ttu-id="4318a-110">範例</span><span class="sxs-lookup"><span data-stu-id="4318a-110">Example</span></span>  
 <span data-ttu-id="4318a-111">下列範例會產生 CS0433。</span><span class="sxs-lookup"><span data-stu-id="4318a-111">The following example generates CS0433.</span></span>  
  
```csharp  
// CS0433_3.cs  
// compile with: /reference:cs0433_1.dll /reference:cs0433_2.dll  
using TypeBindConflicts;  
public class Test   
{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();   // CS0433  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="4318a-112">範例</span><span class="sxs-lookup"><span data-stu-id="4318a-112">Example</span></span>  
 <span data-ttu-id="4318a-113">下列範例示範如何使用 **/reference** 編譯器選項的別名功能，來解決這個 CS0433 錯誤。</span><span class="sxs-lookup"><span data-stu-id="4318a-113">The following example shows how you can use the alias feature of the **/reference** compiler option to resolve this CS0433 error.</span></span>

```csharp  
// compile with: /reference:cs0433_1.dll /reference:TypeBindConflicts=cs0433_2.dll

{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();  
   }  
}  
```
