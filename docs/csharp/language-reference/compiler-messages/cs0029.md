# <a name="compiler-error-cs0029"></a><span data-ttu-id="1f5f3-102">編譯器錯誤 CS0029</span><span class="sxs-lookup"><span data-stu-id="1f5f3-102">Compiler Error CS0029</span></span>
<span data-ttu-id="1f5f3-103">無法將類型 'type' 隱含轉換成 'type'</span><span class="sxs-lookup"><span data-stu-id="1f5f3-103">Cannot implicitly convert type 'type' to 'type'</span></span>  
  
 <span data-ttu-id="1f5f3-104">編譯器需要明確轉換。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-104">The compiler requires an explicit conversion.</span></span> <span data-ttu-id="1f5f3-105">例如，您可能需要將右值轉換成與左值相同的類型。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-105">For example, you may need to cast an r-value to be the same type as an l-value.</span></span> <span data-ttu-id="1f5f3-106">否則，您必須提供轉換常式，以支援特定運算子多載。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-106">Or, you must provide conversion routines to support certain operator overloads.</span></span>  
  
 <span data-ttu-id="1f5f3-107">將某種類型的變數指派給不同類型的變數時，一定會發生轉換。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-107">Conversions must occur when assigning a variable of one type to a variable of a different type.</span></span> <span data-ttu-id="1f5f3-108">在不同類型的變數之間進行指派時，編譯器必須將指派運算子右邊的類型，轉換成指派運算子左邊的類型。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-108">When making an assignment between variables of different types, the compiler must convert the type on the right-hand side of the assignment operator to the type on the left-hand side of the assignment operator.</span></span> <span data-ttu-id="1f5f3-109">以下面這段程式碼來說：</span><span class="sxs-lookup"><span data-stu-id="1f5f3-109">Take the following the code:</span></span>  
  
```csharp  
int i = 50;  
long lng = 100;  
i = lng;  
```  
  
 <span data-ttu-id="1f5f3-110">`i = lng;` 會進行指派，但指派運算子左邊和右邊變數的資料類型不符。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-110">`i = lng;` makes an assignment, but the data types of the variables on the left and right-hand side of the assignment operator don't match.</span></span> <span data-ttu-id="1f5f3-111">編譯器會將 long 類型的變數 `lng` 隱含轉換成 int，再進行指派。因為沒有程式碼明確指示編譯器執行這項轉換，所以這是隱含的。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-111">Before making the assignment the compiler is implicitly converting the variable `lng`, which is of type long, to an int. This is implicit because no code explicitly instructed the compiler to perform this conversion.</span></span> <span data-ttu-id="1f5f3-112">此程式碼的問題在於這會視為縮小轉換，而編譯器不允許隱含的縮小轉換，因為這可能會遺失資料。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-112">The problem with this code is that this is considered a narrowing conversion, and the compiler does not allow implicit narrowing conversions because there could be a potential loss of data.</span></span>  
  
 <span data-ttu-id="1f5f3-113">當轉換的目標資料類型比轉換的來源資料類型佔用更少的記憶體儲存空間時，就是縮小轉換。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-113">A narrowing conversion exists when converting to a data type that occupies less storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="1f5f3-114">例如，將 long 轉換成 int 會視為縮小轉換。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-114">For example, converting a long to an int would be considered a narrowing conversion.</span></span> <span data-ttu-id="1f5f3-115">long 佔用 8 個位元組的記憶體，而 int 佔用 4 個位元組。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-115">A long occupies 8 bytes of memory while an int occupies 4 bytes.</span></span> <span data-ttu-id="1f5f3-116">若要了解資料遺失可能發生的情況，請考慮下列範例：</span><span class="sxs-lookup"><span data-stu-id="1f5f3-116">To see how data loss can occur, consider the following sample:</span></span>  
  
```csharp  
int i = 50;  
long lng = 3147483647;  
i = lng;  
```  
  
 <span data-ttu-id="1f5f3-117">變數 `lng` 現在包含一個值，由於太大而無法儲存在變數 `i` 中。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-117">The variable `lng` now contains a value that cannot be stored in the variable `i` because it is too large.</span></span> <span data-ttu-id="1f5f3-118">如果我們將此值轉換成 int 類型，則會遺失部分資料，而且轉換後的值與轉換前的值不同。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-118">If we were to convert this value to an int type we would be losing some of our data and the converted value would not be the same as the value before the conversion.</span></span>  
  
 <span data-ttu-id="1f5f3-119">擴展轉換與縮小轉換相反。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-119">A widening conversion would be the opposite of a narrowing conversion.</span></span> <span data-ttu-id="1f5f3-120">使用擴展轉換時，轉換的目標資料類型比轉換的來源資料類型佔用更多的記憶體儲存空間。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-120">With widening conversions, we are converting to a data type that occupies more storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="1f5f3-121">擴展轉換的範例如下：</span><span class="sxs-lookup"><span data-stu-id="1f5f3-121">Here is an example of a widening conversion:</span></span>  
  
```csharp  
int i = 50;  
long lng = 100;  
lng = i;  
```  
  
 <span data-ttu-id="1f5f3-122">注意此程式碼範例與第一個程式碼範例之間的差異。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-122">Notice the difference between this code sample and the first.</span></span> <span data-ttu-id="1f5f3-123">此時，變數 `lng` 是在指派運算子的左邊，因此是我們的指派目標。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-123">This time the variable `lng` is on the left-hand side of the assignment operator, so it is the target of our assignment.</span></span> <span data-ttu-id="1f5f3-124">編譯器必須將 int 類型的變數 `i` 隱含轉換成 long 類型，才能進行指派。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-124">Before the assignment can be made, the compiler must implicitly convert the variable `i`, which is of type int, to type long.</span></span> <span data-ttu-id="1f5f3-125">這是擴展轉換，因為我們是將佔用 4 個位元組記憶體的類型 (int)，轉換成佔用 8 個位元組記憶體的類型 (long)。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-125">This is a widening conversion since we are converting from a type that occupies 4 bytes of memory (an int) to a type that occupies 8 bytes of memory (a long).</span></span> <span data-ttu-id="1f5f3-126">因為不可能遺失資料，所以允許隱含的擴展轉換。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-126">Implicit widening conversions are allowed because there is no potential loss of data.</span></span> <span data-ttu-id="1f5f3-127">任何可以儲存在 int 中的值，也可以儲存在 long 中。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-127">Any value that can be stored in an int can also be stored in a long.</span></span>  
  
 <span data-ttu-id="1f5f3-128">我們知道不允許隱含的縮小轉換，因此若要編譯此程式碼，我們需要明確轉換資料類型。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-128">We know that implicit narrowing conversions are not allowed, so to be able to compile this code we need to explicitly convert the data type.</span></span> <span data-ttu-id="1f5f3-129">明確轉換是透過轉型來進行。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-129">Explicit conversions are done using casting.</span></span> <span data-ttu-id="1f5f3-130">「轉型」是 C# 中用來描述將某種資料類型轉換成另一種的字詞。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-130">Casting is the term used in C# to describe converting one data type to another.</span></span> <span data-ttu-id="1f5f3-131">若要讓程式碼進行編譯，我們需要使用下列語法：</span><span class="sxs-lookup"><span data-stu-id="1f5f3-131">To get the code to compile we would need to use the following syntax:</span></span>  
  
```csharp  
int i = 50;  
long lng = 100;  
i = (int) lng;   // cast to int  
```  
  
 <span data-ttu-id="1f5f3-132">程式碼的第三行會指示編譯器將 long 類型的變數 `lng` 隱含轉換成 int，再進行指派。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-132">The third line of code tells the compiler to explicitly convert the variable `lng`, which is of type long, to an int before making the assignment.</span></span> <span data-ttu-id="1f5f3-133">請記住，若使用縮小轉換，則有可能會遺失資料。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-133">Remember that with a narrowing conversion, there is a potential loss of data.</span></span> <span data-ttu-id="1f5f3-134">請謹慎使用縮小轉換，即使程式碼會編譯，您還是可能會在執行階段收到非預期的結果。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-134">Narrowing conversions should be used with caution and even though the code will compile you may get unexpected results at run-time.</span></span>  
  
 <span data-ttu-id="1f5f3-135">此討論僅適用於實值型別。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-135">This discussion has only been for value types.</span></span> <span data-ttu-id="1f5f3-136">使用實值型別時，您是直接處理儲存在變數中的資料。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-136">When working with value types you work directly with the data stored in the variable.</span></span> <span data-ttu-id="1f5f3-137">不過，.NET Framework 還有參考型別。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-137">However, the .NET Framework also has reference types.</span></span> <span data-ttu-id="1f5f3-138">使用參考型別時，您是處理變數的參考，而不是實際資料。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-138">When working with reference types you are working with a reference to a variable, not the actual data.</span></span> <span data-ttu-id="1f5f3-139">類別、介面和陣列都是參考型別的例子。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-139">Examples of reference types would be classes, interfaces and arrays.</span></span> <span data-ttu-id="1f5f3-140">除非編譯器允許特定轉換，或實作適當的轉換運算子，否則您無法將某種參考型別隱含或明確轉換成另一種。</span><span class="sxs-lookup"><span data-stu-id="1f5f3-140">You cannot implicitly or explicitly convert one reference type to another unless the compiler allows the specific conversion or the appropriate conversion operators are implemented.</span></span>  
  
 <span data-ttu-id="1f5f3-141">下列範例會產生 CS0029：</span><span class="sxs-lookup"><span data-stu-id="1f5f3-141">The following sample generates CS0029:</span></span>  
  
```csharp

    private int x = 0;

      MyInt myInt = new MyInt();
```  
