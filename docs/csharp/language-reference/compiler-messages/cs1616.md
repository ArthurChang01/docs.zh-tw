# <a name="compiler-warning-level-1-cs1616"></a>編譯器警告 (層級 1) CS1616
選項 'option' 會覆寫原始程式檔或新增的模組中指定的屬性 'attribute'  
  
 如果原始檔中的組件屬性 <xref:System.Reflection.AssemblyKeyFileAttribute> 或 <xref:System.Reflection.AssemblyKeyNameAttribute>，與 [/keyfile](../compiler-options/keyfile-compiler-option.md) 或 [/keycontainer](../compiler-options/keycontainer-compiler-option.md) 命令列選項，或是 [專案屬性] 中指定的金鑰檔名稱或金鑰容器發生衝突，就會發生此警告。  
  
 在下列範例中，假設您有名為 `cs1616.snk` 的金鑰檔。 此檔案是使用下列命令列所產生：  
  
```console  
sn -k CS1616.snk  
```  
  
 下列範例會產生 CS1616：  
  
```csharp  
// CS1616.cs  
// compile with: /keyfile:cs1616.snk  
using System.Reflection;  
  
// To fix the error, remove the next line  
[assembly: AssemblyKeyFile("cs1616b.snk")]  // CS1616  
  
class C  
{  
  public static void Main()  
  {  
  }  
}  
```
