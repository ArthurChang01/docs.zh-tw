 <span data-ttu-id="2c639-104">在 [namespace](../../../csharp/language-reference/keywords/namespace.md) 中，編譯器只會接受類別、結構、等位、列舉、介面和委派。</span><span class="sxs-lookup"><span data-stu-id="2c639-104">Inside a [namespace](../../../csharp/language-reference/keywords/namespace.md), the compiler only accepts classes, structures, unions, enumerations, interfaces, and delegates.</span></span> <span data-ttu-id="2c639-105">具有 C/C++ 背景的開發人員若忘記在 C# 中，方法和變數必須在結構或類別內宣告及定義，通常會產生此錯誤。</span><span class="sxs-lookup"><span data-stu-id="2c639-105">This error is often generated by developers from a C/C++ background who forget that in C#, methods and variables must be declared and defined within a struct or class.</span></span> <span data-ttu-id="2c639-106">如需詳細資訊，請參閱 [C# 程式的一般結構](../../../csharp/programming-guide/inside-a-program/general-structure-of-a-csharp-program.md)。</span><span class="sxs-lookup"><span data-stu-id="2c639-106">For more information, see [General Structure of a C# Program](../../../csharp/programming-guide/inside-a-program/general-structure-of-a-csharp-program.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="2c639-107">範例</span><span class="sxs-lookup"><span data-stu-id="2c639-107">Example</span></span>

 <span data-ttu-id="2c639-108">下列範例會產生 CS0116：</span><span class="sxs-lookup"><span data-stu-id="2c639-108">The following sample generates CS0116:</span></span>  

```csharp
// CS0116.cs  
namespace x  
{  
    using System;  
  
    // method must be in class/struct  
    void Method(string str) // CS0116  
    {  
        Console.WriteLine(str);  
    }  
    // To fix the error, you must  
    // enclose a method in a class:  
    class Program  
    {  
        void Method2(string str)  
        {  
            Console.WriteLine(str);  
        }  
    }  
}  
```
