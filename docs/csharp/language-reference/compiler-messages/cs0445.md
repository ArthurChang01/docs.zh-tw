# <a name="compiler-error-cs0445"></a><span data-ttu-id="0e97b-102">編譯器錯誤 CS0445</span><span class="sxs-lookup"><span data-stu-id="0e97b-102">Compiler Error CS0445</span></span>
<span data-ttu-id="0e97b-103">無法修改 Unboxing 轉換的結果</span><span class="sxs-lookup"><span data-stu-id="0e97b-103">Cannot modify the result of an unboxing conversion</span></span>  
  
 <span data-ttu-id="0e97b-104">Unboxing 轉換的結果是暫存變數。</span><span class="sxs-lookup"><span data-stu-id="0e97b-104">The result of an unboxing conversion is a temporary variable.</span></span> <span data-ttu-id="0e97b-105">編譯器會防止您修改這類變數，因為暫存變數消失時所有修改也都會消失。</span><span class="sxs-lookup"><span data-stu-id="0e97b-105">The compiler prevents you from modifying such variables because any modification would go away when the temporary variable goes away.</span></span> <span data-ttu-id="0e97b-106">若要修正此錯誤，請宣告新的實值類型變數來儲存中繼運算式，然後將 Unboxing 轉換的結果指派給該變數。</span><span class="sxs-lookup"><span data-stu-id="0e97b-106">To fix this, declare a new value-type variable to store the intermediate expression, and assign the result of the unboxing conversion to that variable.</span></span>  
  
 <span data-ttu-id="0e97b-107">下列程式碼會產生 CS0455。</span><span class="sxs-lookup"><span data-stu-id="0e97b-107">The following code generates CS0455.</span></span>  
  
```csharp  
// CS0445.CS  
class UnboxingTest  
{  
    public static void Main()  
    {  
        Point p;  
        p.x = 1;  
        p.y = 2;  
        object obj = p;  
        // The following line generates CS0445, because the result  
        // of unboxing obj is a temporary variable.  
        ((Point)obj).x = 2;  
  
        // The following lines resolve the error.  
  
        // Store the result of the unboxing conversion in p2.  
        Point p2;
        p2 = (Point)obj;  
        // Then you can modify the unboxed value.  
        p2.x = 2;  
    }  
}  
  
struct Point  
{  
    public int x, y;  
}  
```
