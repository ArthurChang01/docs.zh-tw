# <a name="compiler-error-cs0034"></a><span data-ttu-id="907f7-102">編譯器錯誤 CS0034</span><span class="sxs-lookup"><span data-stu-id="907f7-102">Compiler Error CS0034</span></span>
<span data-ttu-id="907f7-103">運算子 'operator' 用在類型 'type1' 和 'type2' 的運算元上時其意義會模稜兩可</span><span class="sxs-lookup"><span data-stu-id="907f7-103">Operator 'operator' is ambiguous on operands of type 'type1' and 'type2'</span></span>  
  
 <span data-ttu-id="907f7-104">一個運算子用於兩個物件，且編譯器發現多個轉換。</span><span class="sxs-lookup"><span data-stu-id="907f7-104">An operator was used on two objects and the compiler found more than one conversion.</span></span> <span data-ttu-id="907f7-105">因為轉換必須是唯一的，所以您必須進行轉換，或將其中一個轉換設為明確。</span><span class="sxs-lookup"><span data-stu-id="907f7-105">Because conversions have to be unique, you either have to make a cast or to make one of the conversions explicit.</span></span> <span data-ttu-id="907f7-106">如需詳細資訊，請參閱[使用者定義轉換運算子](../operators/user-defined-conversion-operators.md)。</span><span class="sxs-lookup"><span data-stu-id="907f7-106">For more information, see [User-defined conversion operators](../operators/user-defined-conversion-operators.md).</span></span>  
  
 <span data-ttu-id="907f7-107">下列範例會產生 CS0034：</span><span class="sxs-lookup"><span data-stu-id="907f7-107">The following sample generates CS0034:</span></span>  
  
```csharp  
// CS0034.cs  
public class A  
{  
   // allows for the conversion of A object to int  
   public static implicit operator int (A s)  
   {  
      return 0;  
   }  
  
   public static implicit operator string (A i)  
   {  
      return null;  
   }  
}  
  
public class B  
{  
   public static implicit operator int (B s)  
   // one way to resolve this CS0034 is to make one conversion explicit  
   // public static explicit operator int (B s)  
   {  
      return 0;  
   }  
  
   public static implicit operator string (B i)  
   {  
      return null;  
   }  
  
   public static implicit operator B (string i)  
   {  
      return null;  
   }  
  
   public static implicit operator B (int i)  
   {  
      return null;  
   }  
}  
  
public class C  
{  
   public static void Main ()  
   {  
      A a = new A();  
      B b = new B();  
      b = b + a;   // CS0034  
      // another way to resolve this CS0034 is to make a cast  
      // b = b + (int)a;  
   }  
}  
```  
  
 <span data-ttu-id="907f7-108">在 C# 1.1 中，編譯器 Bug 可讓您將具有隱含使用者定義之轉換的類別定義為 `int` 和 `bool`，並在該類型的物件上使用位元 `AND` 運算子 (`&`)。</span><span class="sxs-lookup"><span data-stu-id="907f7-108">In C# 1.1 a compiler bug made it possible to define a class that has implicit user-defined conversions to both `int` and `bool`, and to use a bitwise `AND` operator (`&`) on objects of that type.</span></span> <span data-ttu-id="907f7-109">在 C# 2.0 中已修正此 Bug，讓編譯器符合 C# 規格，因此下列程式碼現在會產生 CS0034：</span><span class="sxs-lookup"><span data-stu-id="907f7-109">In C# 2.0, this bug was fixed to bring the compiler into compliance with the C# specification, and therefore the following code will now cause CS0034:</span></span>  
  
```csharp  
namespace CS0034  
{  
    class TestClass2  
    {  
        public void Test()  
        {  
            TestClass o1 = new TestClass();  
            TestClass o2 = new TestClass();  
            TestClass o3 = o1 & o2; //CS0034  
        }  
    }  
  
    class TestClass  
    {  
        public static implicit operator int(TestClass o)  
        {  
            return 1;  
        }  
  
        public static implicit operator TestClass(int v)  
        {  
            return new TestClass();  
        }  
  
        public static implicit operator bool(TestClass o)  
        {  
            return true;  
        }  
    }  
  
}  
```
