# <a name="compiler-error-cs0173"></a><span data-ttu-id="45c93-102">編譯器錯誤 CS0173</span><span class="sxs-lookup"><span data-stu-id="45c93-102">Compiler Error CS0173</span></span>
<span data-ttu-id="45c93-103">無法判斷條件運算式的類型，因為 'class1' 和 'class2' 之間沒有隱含轉換。</span><span class="sxs-lookup"><span data-stu-id="45c93-103">Type of conditional expression cannot be determined because there is no implicit conversion between 'class1' and 'class2'</span></span>  
  
 <span data-ttu-id="45c93-104">當您希望不同類別的物件能夠使用相同的程式碼時，類別之間的轉換會很有用。</span><span class="sxs-lookup"><span data-stu-id="45c93-104">Conversions between classes are useful when you want objects of different classes to work with the same code.</span></span> <span data-ttu-id="45c93-105">不過，兩個一起運作的類別不能有相互及多餘的轉換或沒有隱含轉換。</span><span class="sxs-lookup"><span data-stu-id="45c93-105">However, two classes that work together cannot have mutual and redundant conversions, or no implicit conversions.</span></span> <span data-ttu-id="45c93-106">`class1` 和 `class2` 的類型獨立判斷，條件運算式類型會選取更一般的類型。</span><span class="sxs-lookup"><span data-stu-id="45c93-106">The types of `class1` and `class2` are determined independently, and the more general type is selected as the type of the conditional expression.</span></span> <span data-ttu-id="45c93-107">如需如何決定類型的詳細資訊，請參閱[條件運算子無法隱含轉換](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959)。</span><span class="sxs-lookup"><span data-stu-id="45c93-107">For more information about how types are determined, see [Conditional operator cannot cast implicitly](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959).</span></span>  
  
 <span data-ttu-id="45c93-108">若要解決 CS0173，請確認無論朝哪個方向轉換，也無論轉換的類別，`class1` 和 `class2` 之間只有一個隱含轉換。</span><span class="sxs-lookup"><span data-stu-id="45c93-108">To resolve CS0173, verify that there is one and only one implicit conversion between `class1` and `class2`, regardless of which direction the conversion is in and regardless of which class the conversion is in.</span></span> <span data-ttu-id="45c93-109">如需詳細資訊，請參閱[隱含數值轉換表](../../../csharp/language-reference/keywords/implicit-numeric-conversions-table.md)和[轉換運算子](../../../csharp/programming-guide/statements-expressions-operators/conversion-operators.md)。</span><span class="sxs-lookup"><span data-stu-id="45c93-109">For more information, see [Implicit Numeric Conversions Table](../../../csharp/language-reference/keywords/implicit-numeric-conversions-table.md) and [Conversion Operators](../../../csharp/programming-guide/statements-expressions-operators/conversion-operators.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="45c93-110">範例</span><span class="sxs-lookup"><span data-stu-id="45c93-110">Example</span></span>  
 <span data-ttu-id="45c93-111">下列範例會產生 CS0173：</span><span class="sxs-lookup"><span data-stu-id="45c93-111">The following sample generates CS0173:</span></span>  
  
```csharp  
// CS0173.cs  
public class C {}  
  
public class A   
{  
    //// The following code defines an implicit conversion operator from    
    //// type C to type A.  
    //public static implicit operator A(C c)  
    //{  
    //    A a = new A();  
    //    a = c;  
    //    return a;  
    //}  
}  
  
public class MyClass  
{  
    public static void F(bool b)  
    {  
        A a = new A();  
        C c = new C();  
  
        // The following line causes CS0173 because there is no implicit  
        // conversion from a to c or from c to a.  
        object o = b ? a : c;  
  
        // To resolve the error, you can cast a and c.  
        //object o = b ? (object)a : (object)c;  
  
        // Alternatively, you can add a conversion operator from class C to  
        // class A, or from class A to class C, but not both.  
    }  
  
   public static void Main()  
   {  
      F(true);  
   }  
}  
```
