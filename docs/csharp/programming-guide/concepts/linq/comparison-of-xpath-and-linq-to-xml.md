## <a name="positional-predicates"></a><span data-ttu-id="63b3d-155">位置性述詞</span><span class="sxs-lookup"><span data-stu-id="63b3d-155">Positional Predicates</span></span>  
 <span data-ttu-id="63b3d-156">在 XPath 運算式內，許多座標軸的位置性述詞都是以文件順序表示，但是反向座標軸則以反向的文件順序表示，這些反向座標軸包括 `preceding`、`preceding-sibling`、`ancestor` 和 `ancestor-or-self`。</span><span class="sxs-lookup"><span data-stu-id="63b3d-156">Within an XPath expression, positional predicates are expressed in terms of document order for many axes, but are expressed in reverse document order for reverse axes, which are `preceding`, `preceding-sibling`, `ancestor`, and `ancestor-or-self`.</span></span> <span data-ttu-id="63b3d-157">例如，XPath 運算式 `preceding-sibling::*[1]` 會傳回正前面的同層級。</span><span class="sxs-lookup"><span data-stu-id="63b3d-157">For example, the XPath expression `preceding-sibling::*[1]` returns the immediately preceding sibling.</span></span> <span data-ttu-id="63b3d-158">即使最後的結果集會以文件順序呈現，也是如此。</span><span class="sxs-lookup"><span data-stu-id="63b3d-158">This is the case even though the final result set is presented in document order.</span></span>
