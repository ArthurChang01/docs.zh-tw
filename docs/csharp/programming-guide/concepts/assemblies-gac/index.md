 <span data-ttu-id="1982f-110">組件包含下列屬性：</span><span class="sxs-lookup"><span data-stu-id="1982f-110">Assemblies have the following properties:</span></span>

## <a name="assembly-manifest"></a><span data-ttu-id="1982f-121">組件資訊清單</span><span class="sxs-lookup"><span data-stu-id="1982f-121">Assembly Manifest</span></span>  
 <span data-ttu-id="1982f-122">每個組件內都有「資訊清單」。</span><span class="sxs-lookup"><span data-stu-id="1982f-122">Within every assembly, there is an *assembly manifest*.</span></span> <span data-ttu-id="1982f-123">類似於目錄，組件資訊清單包含下列資訊︰</span><span class="sxs-lookup"><span data-stu-id="1982f-123">Similar to a table of contents, the assembly manifest contains the following:</span></span>  
  
-   <span data-ttu-id="1982f-124">組件的身分識別 (其名稱和版本)。</span><span class="sxs-lookup"><span data-stu-id="1982f-124">The assembly's identity (its name and version).</span></span>  
  
-   <span data-ttu-id="1982f-125">一個檔案表格，說明構成該組件的所有其他檔案，例如，您所建立為 .exe 或 .dll 檔依賴的任何其他組件，甚至是點陣圖或讀我檔案。</span><span class="sxs-lookup"><span data-stu-id="1982f-125">A file table describing all the other files that make up the assembly, for example, any other assemblies you created that your .exe or .dll file relies on, or even bitmap or Readme files.</span></span>  
  
-   <span data-ttu-id="1982f-126">一個「組件參考清單」，它是所有外部相依性的清單 - 應用程式所需的 .dll 檔或其他檔案，可能是由其他人所建立。</span><span class="sxs-lookup"><span data-stu-id="1982f-126">An *assembly reference list*, which is a list of all external dependencies—.dlls or other files your application needs that may have been created by someone else.</span></span> <span data-ttu-id="1982f-127">組件參考同時包含全域和私用物件的參考。</span><span class="sxs-lookup"><span data-stu-id="1982f-127">Assembly references contain references to both global and private objects.</span></span> <span data-ttu-id="1982f-128">全域物件位於全域組件快取中，此區域可供其他應用程式使用。</span><span class="sxs-lookup"><span data-stu-id="1982f-128">Global objects reside in the global assembly cache, an area available to other applications.</span></span> <span data-ttu-id="1982f-129">私用物件必須同樣位在您的應用程式安裝目錄中或位在其底下的目錄。</span><span class="sxs-lookup"><span data-stu-id="1982f-129">Private objects must be in a directory at either the same level as or below the directory in which your application is installed.</span></span>  
  
 <span data-ttu-id="1982f-130">因為組件包含內容、版本設定及相依性的相關資訊，所以您使用 C# 建立的應用程式不依賴 Windows 登錄值就能正常運作。</span><span class="sxs-lookup"><span data-stu-id="1982f-130">Because assemblies contain information about content, versioning, and dependencies, the applications you create with C# do not rely on Windows registry values to function properly.</span></span> <span data-ttu-id="1982f-131">組件可減少 .dll 衝突，並讓您的應用程式更可靠，也更容易部署。</span><span class="sxs-lookup"><span data-stu-id="1982f-131">Assemblies reduce .dll conflicts and make your applications more reliable and easier to deploy.</span></span> <span data-ttu-id="1982f-132">在許多情況下，您只要將 .NET 型應用程式的檔案複製到目標電腦，即完成安裝。</span><span class="sxs-lookup"><span data-stu-id="1982f-132">In many cases, you can install a .NET-based application simply by copying its files to the target computer.</span></span>  
  
 <span data-ttu-id="1982f-133">如需詳細資訊，請參閱[組件資訊清單](../../../../../docs/framework/app-domains/assembly-manifest.md)。</span><span class="sxs-lookup"><span data-stu-id="1982f-133">For more information see [Assembly Manifest](../../../../../docs/framework/app-domains/assembly-manifest.md).</span></span>
