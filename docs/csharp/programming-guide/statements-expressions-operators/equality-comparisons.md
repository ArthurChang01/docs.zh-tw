## <a name="value-equality"></a><span data-ttu-id="3620d-120">實值相等</span><span class="sxs-lookup"><span data-stu-id="3620d-120">Value Equality</span></span>  
 <span data-ttu-id="3620d-121">實值相等表示兩個物件包含相同的值。</span><span class="sxs-lookup"><span data-stu-id="3620d-121">Value equality means that two objects contain the same value or values.</span></span> <span data-ttu-id="3620d-122">對於 [int](../../../csharp/language-reference/keywords/int.md) 或 [bool](../../../csharp/language-reference/keywords/bool.md) 這類基本實值型別，測試實值相等十分簡單。</span><span class="sxs-lookup"><span data-stu-id="3620d-122">For primitive value types such as [int](../../../csharp/language-reference/keywords/int.md) or [bool](../../../csharp/language-reference/keywords/bool.md), tests for value equality are straightforward.</span></span> <span data-ttu-id="3620d-123">您可以使用 [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) 運算子，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="3620d-123">You can use the [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) operator, as shown in the following example.</span></span>  
  
```csharp  
int a = GetOriginalValue();  
int b = GetCurrentValue();  
  
// Test for value equality.   
if( b == a)   
{  
    // The two integers are equal.  
}  
```  
  
 <span data-ttu-id="3620d-124">對於大部分其他類型，測試實值相等較為複雜，因為您需要了解類型如何定義它。</span><span class="sxs-lookup"><span data-stu-id="3620d-124">For most other types, testing for value equality is more complex because it requires that you understand how the type defines it.</span></span> <span data-ttu-id="3620d-125">對於具有多個欄位或屬性的類別和結構，通常會定義實值相等，表示所有欄位或屬性具有相同的值。</span><span class="sxs-lookup"><span data-stu-id="3620d-125">For classes and structs that have multiple fields or properties, value equality is often defined to mean that all fields or properties have the same value.</span></span> <span data-ttu-id="3620d-126">例如，如果 pointA.X 等於 pointB.X 而且 pointA.Y 等於 pointB.Y，則兩個 `Point` 物件可能定義為相等。</span><span class="sxs-lookup"><span data-stu-id="3620d-126">For example, two `Point` objects might be defined to be equivalent if pointA.X is equal to pointB.X and pointA.Y is equal to pointB.Y.</span></span>  
  
 <span data-ttu-id="3620d-127">不過，相等性不需要根據類型中的所有欄位。</span><span class="sxs-lookup"><span data-stu-id="3620d-127">However, there is no requirement that equivalence be based on all the fields in a type.</span></span> <span data-ttu-id="3620d-128">而是根據子集。</span><span class="sxs-lookup"><span data-stu-id="3620d-128">It can be based on a subset.</span></span> <span data-ttu-id="3620d-129">當您比較不屬於您的類型時，務必特別了解如何定義該類型的相等性。</span><span class="sxs-lookup"><span data-stu-id="3620d-129">When you compare types that you do not own, you should make sure to understand specifically how equivalence is defined for that type.</span></span> <span data-ttu-id="3620d-130">如需如何在專屬類別和結構中定義實值相等的詳細資訊，請參閱[如何：定義類型的實值相等](../../../csharp/programming-guide/statements-expressions-operators/how-to-define-value-equality-for-a-type.md)。</span><span class="sxs-lookup"><span data-stu-id="3620d-130">For more information about how to define value equality in your own classes and structs, see [How to: Define Value Equality for a Type](../../../csharp/programming-guide/statements-expressions-operators/how-to-define-value-equality-for-a-type.md).</span></span>
