## <a name="general-guidelines"></a><span data-ttu-id="4b19f-169">一般準則</span><span class="sxs-lookup"><span data-stu-id="4b19f-169">General Guidelines</span></span>

<span data-ttu-id="4b19f-170">雖然仍然認為最好通過修改物件的代碼或在合理和可能時派生新類型來添加功能，但擴充方法已成為在整個 .NET 中創建可重用功能的關鍵選項生態。</span><span class="sxs-lookup"><span data-stu-id="4b19f-170">While it's still considered preferable to add functionality by modifying an object's code or deriving a new type whenever it's reasonable and possible to do so, extension methods have become a crucial option for creating reusable functionality throughout the .NET ecosystem.</span></span> <span data-ttu-id="4b19f-171">對於原始源不在您控制範圍內、派生物件不恰當或不可能或功能不應超出其適用範圍的情況，擴充方法是一個絕佳的選擇。</span><span class="sxs-lookup"><span data-stu-id="4b19f-171">For those occasions when the original source isn't under your control, when a derived object is inappropriate or impossible, or when the functionality shouldn't be exposed beyond its applicable scope, Extension methods are an excellent choice.</span></span>

<span data-ttu-id="4b19f-172">有關派生類型的詳細資訊，請參閱[繼承](./inheritance.md)。</span><span class="sxs-lookup"><span data-stu-id="4b19f-172">For more information on derived types, see [Inheritance](./inheritance.md).</span></span>

<span data-ttu-id="4b19f-173">當使用擴充方法擴展原始程式碼無法控制的類型時，您會運行類型實現中的更改會導致擴充方法中斷的風險。</span><span class="sxs-lookup"><span data-stu-id="4b19f-173">When using an extension method to extend a type whose source code you aren't in control of, you run the risk that a change in the implementation of the type will cause your extension method to break.</span></span>

<span data-ttu-id="4b19f-174">如果您要實作所指定類型的擴充方法，請記住下列幾點：</span><span class="sxs-lookup"><span data-stu-id="4b19f-174">If you do implement extension methods for a given type, remember the following points:</span></span>

- <span data-ttu-id="4b19f-175">如果擴充方法的簽章與類型中定義的方法相同，則絕不會呼叫擴充方法。</span><span class="sxs-lookup"><span data-stu-id="4b19f-175">An extension method will never be called if it has the same signature as a method defined in the type.</span></span>
- <span data-ttu-id="4b19f-176">擴充方法是帶入命名空間層級的範圍。</span><span class="sxs-lookup"><span data-stu-id="4b19f-176">Extension methods are brought into scope at the namespace level.</span></span> <span data-ttu-id="4b19f-177">例如，如果多個靜態類在名為 的`Extensions`單個命名空間中包含擴充方法，則`using Extensions;`指令將所有這些類都引入作用域。</span><span class="sxs-lookup"><span data-stu-id="4b19f-177">For example, if you have multiple static classes that contain extension methods in a single namespace named `Extensions`, they'll all be brought into scope by the `using Extensions;` directive.</span></span>

<span data-ttu-id="4b19f-178">針對實作的類別庫，您不應該使用擴充方法阻止組件的版本號碼遞增。</span><span class="sxs-lookup"><span data-stu-id="4b19f-178">For a class library that you implemented, you shouldn't use extension methods to avoid incrementing the version number of an assembly.</span></span> <span data-ttu-id="4b19f-179">如果您要在擁有其原始程式碼的程式庫中加入重要功能，則應遵循組件版本控制的標準 .NET Framework 方針。</span><span class="sxs-lookup"><span data-stu-id="4b19f-179">If you want to add significant functionality to a library for which you own the source code, you should follow the standard .NET Framework guidelines for assembly versioning.</span></span> <span data-ttu-id="4b19f-180">如需詳細資訊，請參閱[組件版本控制](../../../standard/assembly/versioning.md)。</span><span class="sxs-lookup"><span data-stu-id="4b19f-180">For more information, see [Assembly Versioning](../../../standard/assembly/versioning.md).</span></span>
