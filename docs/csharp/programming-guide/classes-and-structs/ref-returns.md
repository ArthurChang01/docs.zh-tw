# <a name="ref-returns-and-ref-locals"></a><span data-ttu-id="6872c-103">ref 傳回值和 ref 區域變數</span><span class="sxs-lookup"><span data-stu-id="6872c-103">Ref returns and ref locals</span></span>

<span data-ttu-id="6872c-104">從 C# 7.0 開始，C# 支援參考傳回值 (ref 傳回值)。</span><span class="sxs-lookup"><span data-stu-id="6872c-104">Starting with C# 7.0, C# supports reference return values (ref returns).</span></span> <span data-ttu-id="6872c-105">參考傳回值允許方法將變數參考 (而非值) 傳回給呼叫者。</span><span class="sxs-lookup"><span data-stu-id="6872c-105">A reference return value allows a method to return a reference to a variable, rather than a value, back to a caller.</span></span> <span data-ttu-id="6872c-106">呼叫者接著可以選擇將傳回的變數視為以傳值方式或以傳址方式傳回。</span><span class="sxs-lookup"><span data-stu-id="6872c-106">The caller can then choose to treat the returned variable as if it were returned by value or by reference.</span></span> <span data-ttu-id="6872c-107">呼叫者可建立本身為參考傳回值的新變數，稱為 ref 區域變數。</span><span class="sxs-lookup"><span data-stu-id="6872c-107">The caller can create a new variable that is itself a reference to the returned value, called a ref local.</span></span>

## <a name="what-is-a-reference-return-value"></a><span data-ttu-id="6872c-108">何謂參考傳回值？</span><span class="sxs-lookup"><span data-stu-id="6872c-108">What is a reference return value?</span></span>

<span data-ttu-id="6872c-109">大部分的開發人員都熟悉「以傳址方式」將引數傳遞給已呼叫方法。</span><span class="sxs-lookup"><span data-stu-id="6872c-109">Most developers are familiar with passing an argument to a called method *by reference*.</span></span> <span data-ttu-id="6872c-110">所呼叫方法的引數清單包含以傳址方式傳遞的變數。</span><span class="sxs-lookup"><span data-stu-id="6872c-110">A called method's argument list includes a variable passed by reference.</span></span> <span data-ttu-id="6872c-111">呼叫者會觀察到所呼叫方法對其值進行的任何變更。</span><span class="sxs-lookup"><span data-stu-id="6872c-111">Any changes made to its value by the called method are observed by the caller.</span></span> <span data-ttu-id="6872c-112">「參考傳回值」表示方法會將「參考」 (或別名) 傳回給某個變數。</span><span class="sxs-lookup"><span data-stu-id="6872c-112">A *reference return value* means that a method returns a *reference* (or an alias) to some variable.</span></span> <span data-ttu-id="6872c-113">該變數的範圍必須包含方法。</span><span class="sxs-lookup"><span data-stu-id="6872c-113">That variable's scope must include the method.</span></span> <span data-ttu-id="6872c-114">該變數的存留期必須延伸到傳回方法。</span><span class="sxs-lookup"><span data-stu-id="6872c-114">That variable's lifetime must extend beyond the return of the method.</span></span> <span data-ttu-id="6872c-115">呼叫者對方法的傳回值所進行的修改，是針對方法傳回的變數進行。</span><span class="sxs-lookup"><span data-stu-id="6872c-115">Modifications to the method's return value by the caller are made to the variable that is returned by the method.</span></span>

<span data-ttu-id="6872c-116">宣告方法傳回「參考傳回值」，表示方法會將別名傳回給變數。</span><span class="sxs-lookup"><span data-stu-id="6872c-116">Declaring that a method returns a *reference return value* indicates that the method returns an alias to a variable.</span></span> <span data-ttu-id="6872c-117">此設計通常用於呼叫的程式碼應可透過別名來存取該變數 (包括進行修改) 時。</span><span class="sxs-lookup"><span data-stu-id="6872c-117">The design intent is often that the calling code should have access to that variable through the alias, including to modify it.</span></span> <span data-ttu-id="6872c-118">這樣一來，以傳址方式傳回的方法就不能有傳回型別 `void`。</span><span class="sxs-lookup"><span data-stu-id="6872c-118">It follows that methods returning by reference can't have the return type `void`.</span></span>

- <span data-ttu-id="6872c-125">傳回值不能是常值 `null`。</span><span class="sxs-lookup"><span data-stu-id="6872c-125">The return value cannot be the literal `null`.</span></span> <span data-ttu-id="6872c-126">傳回 `null` 會產生編譯器錯誤 CS8156「無法在此內容中使用運算式，因為其可能不會以傳址方式傳回」。</span><span class="sxs-lookup"><span data-stu-id="6872c-126">Returning `null` generates compiler error CS8156, "An expression cannot be used in this context because it may not be returned by reference."</span></span>

   <span data-ttu-id="6872c-127">具有 ref 傳回的方法可以將別名傳回給目前值為 null （未具現化）值或[可](../nullable-types/index.md)為 null 實數值型別的變數。</span><span class="sxs-lookup"><span data-stu-id="6872c-127">A method with a ref return can return an alias to a variable whose value is currently the null (uninstantiated) value or a [nullable value type](../nullable-types/index.md) for a value type.</span></span>

- <span data-ttu-id="6872c-128">傳回值不能是常數、列舉成員、屬性的以傳值方式傳回值，或是 `class` 或 `struct` 的方法。</span><span class="sxs-lookup"><span data-stu-id="6872c-128">The return value cannot be a constant, an enumeration member, the by-value return value from a property, or a method of a `class` or `struct`.</span></span> <span data-ttu-id="6872c-129">違反此規則會產生編譯器錯誤 CS8156「無法在此內容中使用運算式，因為其可能不會以傳址方式傳回」。</span><span class="sxs-lookup"><span data-stu-id="6872c-129">Violating this rule generates compiler error CS8156, "An expression cannot be used in this context because it may not be returned by reference."</span></span>

<span data-ttu-id="6872c-130">此外，非同步方法上不允許參考傳回值。</span><span class="sxs-lookup"><span data-stu-id="6872c-130">In addition, reference return values are not allowed on async methods.</span></span> <span data-ttu-id="6872c-131">在非同步方法完成執行之前，可能會傳回非同步方法，但其傳回值仍然為未知。</span><span class="sxs-lookup"><span data-stu-id="6872c-131">An asynchronous method may return before it has finished execution, while its return value is still unknown.</span></span>

## <a name="defining-a-ref-return-value"></a><span data-ttu-id="6872c-132">定義 ref 傳回值</span><span class="sxs-lookup"><span data-stu-id="6872c-132">Defining a ref return value</span></span>
