# <a name="nullable-reference-types"></a><span data-ttu-id="e5b10-104">可為 Null 的參考型別</span><span class="sxs-lookup"><span data-stu-id="e5b10-104">Nullable reference types</span></span>

<span data-ttu-id="e5b10-105">C# 8.0 引進**可為 Null 的參考型別**及**不可為 Null 的參考型別**，可讓您針對參考型別變數的屬性撰寫相關重要陳述式：</span><span class="sxs-lookup"><span data-stu-id="e5b10-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="e5b10-106">**參考不應為 Null**。</span><span class="sxs-lookup"><span data-stu-id="e5b10-106">**A reference is not supposed to be null**.</span></span> <span data-ttu-id="e5b10-107">當變數不應為 Null 時，編譯器會實施規則，確保對這些變數取值 (Dereference) 的過程是安全的，而不會事先檢查這些變數是不是 Null：</span><span class="sxs-lookup"><span data-stu-id="e5b10-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it is safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="e5b10-108">變數必須初始化為非 Null 值。</span><span class="sxs-lookup"><span data-stu-id="e5b10-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="e5b10-109">變數永遠不可指派 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="e5b10-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="e5b10-110">**參考可能為 Null**。</span><span class="sxs-lookup"><span data-stu-id="e5b10-110">**A reference may be null**.</span></span> <span data-ttu-id="e5b10-111">當變數可為 Null 時，編譯器會實施不同的規則，確保您已針對 Null 參考正確地進行檢查：</span><span class="sxs-lookup"><span data-stu-id="e5b10-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="e5b10-112">只有在編譯器能保證該值並非為 Null 時，才能對該變數進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="e5b10-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>
  - <span data-ttu-id="e5b10-113">這些變數可使用預設的 `null` 值初始化，也可以在其他程式碼中指派 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="e5b10-113">These variables may be initialized with the default `null` value and may be assigned the value `null` in other code.</span></span>

<span data-ttu-id="e5b10-114">相較於先前版本 C# 中處理參考變數的方式，這項新功能提供極大的好處，因為先前版本中無法從變數宣告判斷設計意圖。</span><span class="sxs-lookup"><span data-stu-id="e5b10-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent couldn't be determined from the variable declaration.</span></span> <span data-ttu-id="e5b10-115">編譯器不針對參考型別的 Null 參考例外狀況提供安全：</span><span class="sxs-lookup"><span data-stu-id="e5b10-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="e5b10-116">**參考可為 Null**。</span><span class="sxs-lookup"><span data-stu-id="e5b10-116">**A reference can be null**.</span></span> <span data-ttu-id="e5b10-117">當參考型別初始化為 null 或稍後為 null 時，不會發出警告。</span><span class="sxs-lookup"><span data-stu-id="e5b10-117">No warning is issued when a reference type is initialized to null, or null is later assigned to it.</span></span>
- <span data-ttu-id="e5b10-118">**參考已假設為並非 Null**。</span><span class="sxs-lookup"><span data-stu-id="e5b10-118">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="e5b10-119">編譯器不會在對參考型別進行取值 (Dereference) 時發出任何警告。</span><span class="sxs-lookup"><span data-stu-id="e5b10-119">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="e5b10-120">(使用可為 Null 參考，編譯器會在您對可能為 Null 的變數進行取值 (Dereference) 時發出警告)。</span><span class="sxs-lookup"><span data-stu-id="e5b10-120">(With nullable references,  the compiler issues warnings whenever you dereference a variable that may be null).</span></span>

<span data-ttu-id="e5b10-121">透過新增的可為 Null 參考型別，您可以更清楚地宣告您的意圖。</span><span class="sxs-lookup"><span data-stu-id="e5b10-121">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="e5b10-122">`null` 值是表示變數並非指向某個值的正確方式。</span><span class="sxs-lookup"><span data-stu-id="e5b10-122">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="e5b10-123">請不要使用這項功能來從您的程式碼中移除所有 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="e5b10-123">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="e5b10-124">相反地，您應該對編譯器及其他閱讀您程式碼之開發人員宣告您的意圖。</span><span class="sxs-lookup"><span data-stu-id="e5b10-124">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="e5b10-125">藉由宣告您的意圖，編譯器會在您撰寫的程式碼與該意圖不一致時通知您。</span><span class="sxs-lookup"><span data-stu-id="e5b10-125">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="e5b10-126">**可為 Null 參考型別**的語法與[可為 Null 實值型別](language-reference/builtin-types/nullable-value-types.md)語法相同：將 `?` 附加至變數的型別。</span><span class="sxs-lookup"><span data-stu-id="e5b10-126">A **nullable reference type** is noted using the same syntax as [nullable value types](language-reference/builtin-types/nullable-value-types.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="e5b10-127">例如，下列變數宣告代表可為 Null 字串變數，`name`：</span><span class="sxs-lookup"><span data-stu-id="e5b10-127">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>

<span data-ttu-id="e5b10-128">任何沒有在型別名稱附加 `?` 的變數都是**不可為 Null 參考型別**。</span><span class="sxs-lookup"><span data-stu-id="e5b10-128">Any variable where the `?` is not appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="e5b10-129">這包含您啟用此功能時現有程式碼中所有的參考型別變數。</span><span class="sxs-lookup"><span data-stu-id="e5b10-129">That includes all reference type variables in existing code when you have enabled this feature.</span></span>


## <a name="nullability-of-types"></a><span data-ttu-id="e5b10-134">型別的可 NULL 性</span><span class="sxs-lookup"><span data-stu-id="e5b10-134">Nullability of types</span></span>

<span data-ttu-id="e5b10-135">任何參考型別都可擁有四種「可 NULL 性」\*\* 中的其中一種，其描述警告產生的時機：</span><span class="sxs-lookup"><span data-stu-id="e5b10-135">Any reference type can have one of four *nullabilities*, which describes when warnings are generated:</span></span>

- <span data-ttu-id="e5b10-136">*不可為空*：無法將 Null 分配給此類型的變數。</span><span class="sxs-lookup"><span data-stu-id="e5b10-136">*Nonnullable*: Null can't be assigned to variables of this type.</span></span> <span data-ttu-id="e5b10-137">此型別的變數不需進行 Null 檢查即可進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="e5b10-137">Variables of this type don't need to be null-checked before dereferencing.</span></span>
- <span data-ttu-id="e5b10-138">*空*：可以為此類型的變數分配 null。</span><span class="sxs-lookup"><span data-stu-id="e5b10-138">*Nullable*: Null can be assigned to variables of this type.</span></span> <span data-ttu-id="e5b10-139">若沒有事先檢查是否為 `null` 即對此型別的變數進行取值 (Dereference)，即會產生警告。</span><span class="sxs-lookup"><span data-stu-id="e5b10-139">Dereferencing variables of this type without first checking for `null` causes a warning.</span></span>

## <a name="nullable-annotation-context"></a><span data-ttu-id="e5b10-182">可為 Null 註釋內容</span><span class="sxs-lookup"><span data-stu-id="e5b10-182">Nullable annotation context</span></span>

- <span data-ttu-id="e5b10-184">您無法在停用的內容中宣告可為 Null 參考。</span><span class="sxs-lookup"><span data-stu-id="e5b10-184">You can't declare nullable references in a disabled context.</span></span>
- <span data-ttu-id="e5b10-185">可以為所有引用變數分配一個 null 值。</span><span class="sxs-lookup"><span data-stu-id="e5b10-185">All reference variables may be assigned a value of null.</span></span>
- <span data-ttu-id="e5b10-186">當對參考型別的變數進行取值 (Dereference) 時，不會產生任何警告。</span><span class="sxs-lookup"><span data-stu-id="e5b10-186">No warnings are generated when a variable of a reference type is dereferenced.</span></span>
- <span data-ttu-id="e5b10-187">Null 容許運算子不可用於停用內容中。</span><span class="sxs-lookup"><span data-stu-id="e5b10-187">The null-forgiving operator may not be used in a disabled context.</span></span>

<span data-ttu-id="e5b10-188">行為與先前版本的 C# 相同。</span><span class="sxs-lookup"><span data-stu-id="e5b10-188">The behavior is the same as previous versions of C#.</span></span>

<span data-ttu-id="e5b10-189">編譯器會在啟用的可為 Null 註釋內容中使用下列規則：</span><span class="sxs-lookup"><span data-stu-id="e5b10-189">The compiler uses the following rules in an enabled nullable annotation context:</span></span>

- <span data-ttu-id="e5b10-190">參考型別的任何變數都是**不可為 Null 參考**。</span><span class="sxs-lookup"><span data-stu-id="e5b10-190">Any variable of a reference type is a **non-nullable reference**.</span></span>
- <span data-ttu-id="e5b10-191">任何不可為 Null 參考都可安全地進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="e5b10-191">Any non-nullable reference may be dereferenced safely.</span></span>
- <span data-ttu-id="e5b10-192">任何可為 Null 參考型別 (在變數宣告中的型別後方標註 `?`) 都可為 Null。</span><span class="sxs-lookup"><span data-stu-id="e5b10-192">Any nullable reference type (noted by `?` after the type in the variable declaration) may be null.</span></span> <span data-ttu-id="e5b10-193">靜態分析會在對其進行取值 (Dereference) 時判斷該值是否已知並非為 Null。</span><span class="sxs-lookup"><span data-stu-id="e5b10-193">Static analysis determines if the value is known to be non-null when it is dereferenced.</span></span> <span data-ttu-id="e5b10-194">若否，則編譯器會警告您。</span><span class="sxs-lookup"><span data-stu-id="e5b10-194">If not, the compiler warns you.</span></span>
- <span data-ttu-id="e5b10-195">您可以使用 Null 容許運算子來宣告可為 Null 參考並非為 Null。</span><span class="sxs-lookup"><span data-stu-id="e5b10-195">You can use the null-forgiving operator to declare that a nullable reference isn't null.</span></span>

<span data-ttu-id="e5b10-196">在啟用的可為 Null 註釋內容中，`?` 字元會附加至宣告**可為 Null 參考型別**的參考型別。</span><span class="sxs-lookup"><span data-stu-id="e5b10-196">In an enabled nullable annotation context, the `?` character appended to a reference type declares a **nullable reference type**.</span></span> <span data-ttu-id="e5b10-197">**可以將放棄運算子**`!`追加到運算式中，以聲明運算式不為空。</span><span class="sxs-lookup"><span data-stu-id="e5b10-197">The **null-forgiving operator** `!` may be appended to an expression to declare that the expression isn't null.</span></span>

## <a name="nullable-warning-context"></a><span data-ttu-id="e5b10-198">可為 Null 警告內容</span><span class="sxs-lookup"><span data-stu-id="e5b10-198">Nullable warning context</span></span>

1. <span data-ttu-id="e5b10-205">變數已確定指派為並非 Null 的值。</span><span class="sxs-lookup"><span data-stu-id="e5b10-205">The variable has been definitely assigned to a non-null value.</span></span>
1. <span data-ttu-id="e5b10-206">變數或運算式已在取值 (Dereference) 之前針對 Null 進行檢查。</span><span class="sxs-lookup"><span data-stu-id="e5b10-206">The variable or expression has been checked against null before de-referencing it.</span></span>

<span data-ttu-id="e5b10-207">啟用空警告上下文時，每當以**可能為空**狀態取消引用變數或運算式時，編譯器都會生成警告。</span><span class="sxs-lookup"><span data-stu-id="e5b10-207">The compiler generates warnings whenever you dereference a variable or expression in a **maybe null** state when the nullable warning context is enabled.</span></span> <span data-ttu-id="e5b10-208">此外，當在啟用的空注釋上下文中將**可能為空**的變數或運算式分配給不可空參考型別時，將生成警告。</span><span class="sxs-lookup"><span data-stu-id="e5b10-208">Furthermore, warnings are generated when a **maybe null** variable or expression is assigned to a nonnullable reference type in an enabled nullable annotation context.</span></span>
