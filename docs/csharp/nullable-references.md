# <a name="nullable-reference-types"></a><span data-ttu-id="fcc2a-104">可為 Null 的參考型別</span><span class="sxs-lookup"><span data-stu-id="fcc2a-104">Nullable reference types</span></span>

<span data-ttu-id="fcc2a-105">C# 8.0 引進**可為 Null 的參考型別**及**不可為 Null 的參考型別**，可讓您針對參考型別變數的屬性撰寫相關重要陳述式：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="fcc2a-106">**參考不應為 Null**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-106">**A reference is not supposed to be null**.</span></span> <span data-ttu-id="fcc2a-107">當變數不應為 Null 時，編譯器會實施規則，確保對這些變數取值 (Dereference) 的過程是安全的，而不會事先檢查這些變數是不是 Null：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it is safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="fcc2a-108">變數必須初始化為非 Null 值。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="fcc2a-109">變數永遠不可指派 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="fcc2a-110">**參考可能為 Null**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-110">**A reference may be null**.</span></span> <span data-ttu-id="fcc2a-111">當變數可為 Null 時，編譯器會實施不同的規則，確保您已針對 Null 參考正確地進行檢查：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="fcc2a-112">只有在編譯器能保證該值並非為 Null 時，才能對該變數進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>

<span data-ttu-id="fcc2a-114">相較於先前版本 C# 中處理參考變數的方式，這項新功能提供極大的好處，因為先前版本中無法從變數宣告判斷設計意圖。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent couldn't be determined from the variable declaration.</span></span> <span data-ttu-id="fcc2a-115">編譯器不針對參考型別的 Null 參考例外狀況提供安全：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="fcc2a-116">**參考可為 Null**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-116">**A reference can be null**.</span></span> <span data-ttu-id="fcc2a-117">不會在參考型別初始化為 Null，或是稍後指派為 Null 時發出任何警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-117">No warnings are issued when a reference type is initialized to null, or later assigned to null.</span></span>
- <span data-ttu-id="fcc2a-118">**參考已假設為並非 Null**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-118">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="fcc2a-119">編譯器不會在對參考型別進行取值 (Dereference) 時發出任何警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-119">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="fcc2a-120">(使用可為 Null 參考，編譯器會在您對可能為 Null 的變數進行取值 (Dereference) 時發出警告)。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-120">(With nullable references,  the compiler issues warnings whenever you dereference a variable that may be null).</span></span>

<span data-ttu-id="fcc2a-121">透過新增的可為 Null 參考型別，您可以更清楚地宣告您的意圖。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-121">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="fcc2a-122">`null` 值是表示變數並非指向某個值的正確方式。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-122">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="fcc2a-123">請不要使用這項功能來從您的程式碼中移除所有 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-123">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="fcc2a-124">相反地，您應該對編譯器及其他閱讀您程式碼之開發人員宣告您的意圖。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-124">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="fcc2a-125">藉由宣告您的意圖，編譯器會在您撰寫的程式碼與該意圖不一致時通知您。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-125">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="fcc2a-126">**可為 Null 參考型別**的語法與[可為 Null 實值型別](programming-guide/nullable-types/index.md)語法相同：將 `?` 附加至變數的型別。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-126">A **nullable reference type** is noted using the same syntax as [nullable value types](programming-guide/nullable-types/index.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="fcc2a-127">例如，下列變數宣告代表可為 Null 字串變數，`name`：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-127">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>

<span data-ttu-id="fcc2a-128">任何沒有在型別名稱附加 `?` 的變數都是**不可為 Null 參考型別**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-128">Any variable where the `?` is not appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="fcc2a-129">這包含您啟用此功能時現有程式碼中所有的參考型別變數。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-129">That includes all reference type variables in existing code when you have enabled this feature.</span></span>

<span data-ttu-id="fcc2a-130">編譯器會使用靜態分析來判斷可為 Null 參考是否已知並非為 Null。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-130">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="fcc2a-131">若您在可為 Null 參考可能為 Null 時對其進行取值 (Dereference)，編譯器會警告您。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-131">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="fcc2a-132">您可以使用[null 容許運算子](language-reference/operators/null-forgiving.md)來覆寫此行為，`!` 遵循變數名稱。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-132">You can override this behavior by using the [null-forgiving operator](language-reference/operators/null-forgiving.md) `!` following a variable name.</span></span> <span data-ttu-id="fcc2a-133">例如，若您知道 `name` 變數並非為 Null，但編譯器卻發出警告，您可以撰寫下列程式碼來覆寫編譯器的分析：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-133">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>

```csharp
name!.Length;
```

## <a name="nullability-of-types"></a><span data-ttu-id="fcc2a-134">型別的可 NULL 性</span><span class="sxs-lookup"><span data-stu-id="fcc2a-134">Nullability of types</span></span>

<span data-ttu-id="fcc2a-135">任何參考型別都可擁有四種「可 NULL 性」中的其中一種，其描述警告產生的時機：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-135">Any reference type can have one of four *nullabilities*, which describes when warnings are generated:</span></span>

- <span data-ttu-id="fcc2a-136">*不可為 null*： Null 無法指派給這個類型的變數。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-136">*Nonnullable*: Null can't be assigned to variables of this type.</span></span> <span data-ttu-id="fcc2a-137">此型別的變數不需進行 Null 檢查即可進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-137">Variables of this type don't need to be null-checked before dereferencing.</span></span>
- <span data-ttu-id="fcc2a-138">*Nullable*： null 可以指派給這個類型的變數。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-138">*Nullable*: Null can be assigned to variables of this type.</span></span> <span data-ttu-id="fcc2a-139">若沒有事先檢查是否為 `null` 即對此型別的變數進行取值 (Dereference)，即會產生警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-139">Dereferencing variables of this type without first checking for `null` causes a warning.</span></span>

## <a name="nullable-annotation-context"></a><span data-ttu-id="fcc2a-181">可為 Null 註釋內容</span><span class="sxs-lookup"><span data-stu-id="fcc2a-181">Nullable annotation context</span></span>

- <span data-ttu-id="fcc2a-183">您無法在停用的內容中宣告可為 Null 參考。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-183">You can't declare nullable references in a disabled context.</span></span>
- <span data-ttu-id="fcc2a-184">所有參考變數都可指派為 Null。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-184">All reference variables may be assigned to null.</span></span>
- <span data-ttu-id="fcc2a-185">當對參考型別的變數進行取值 (Dereference) 時，不會產生任何警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-185">No warnings are generated when a variable of a reference type is dereferenced.</span></span>
- <span data-ttu-id="fcc2a-186">Null 容許運算子不可用於停用內容中。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-186">The null-forgiving operator may not be used in a disabled context.</span></span>

<span data-ttu-id="fcc2a-187">行為與先前版本的 C# 相同。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-187">The behavior is the same as previous versions of C#.</span></span>

<span data-ttu-id="fcc2a-188">編譯器會在啟用的可為 Null 註釋內容中使用下列規則：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-188">The compiler uses the following rules in an enabled nullable annotation context:</span></span>

- <span data-ttu-id="fcc2a-189">參考型別的任何變數都是**不可為 Null 參考**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-189">Any variable of a reference type is a **non-nullable reference**.</span></span>
- <span data-ttu-id="fcc2a-190">任何不可為 Null 參考都可安全地進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-190">Any non-nullable reference may be dereferenced safely.</span></span>
- <span data-ttu-id="fcc2a-191">任何可為 Null 參考型別 (在變數宣告中的型別後方標註 `?`) 都可為 Null。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-191">Any nullable reference type (noted by `?` after the type in the variable declaration) may be null.</span></span> <span data-ttu-id="fcc2a-192">靜態分析會在對其進行取值 (Dereference) 時判斷該值是否已知並非為 Null。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-192">Static analysis determines if the value is known to be non-null when it is dereferenced.</span></span> <span data-ttu-id="fcc2a-193">若否，則編譯器會警告您。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-193">If not, the compiler warns you.</span></span>
- <span data-ttu-id="fcc2a-194">您可以使用 Null 容許運算子來宣告可為 Null 參考並非為 Null。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-194">You can use the null-forgiving operator to declare that a nullable reference isn't null.</span></span>

<span data-ttu-id="fcc2a-195">在啟用的可為 Null 註釋內容中，`?` 字元會附加至宣告**可為 Null 參考型別**的參考型別。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-195">In an enabled nullable annotation context, the `?` character appended to a reference type declares a **nullable reference type**.</span></span> <span data-ttu-id="fcc2a-196">**Null 容許運算子**`!` 可以附加至運算式，以宣告運算式不是 null。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-196">The **null-forgiving operator** `!` may be appended to an expression to declare that the expression isn't null.</span></span>

## <a name="nullable-warning-context"></a><span data-ttu-id="fcc2a-197">可為 Null 警告內容</span><span class="sxs-lookup"><span data-stu-id="fcc2a-197">Nullable warning context</span></span>

<span data-ttu-id="fcc2a-198">可為 Null 警告內容與可為 Null 註釋內容不同。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-198">The nullable warning context is distinct from the nullable annotation context.</span></span> <span data-ttu-id="fcc2a-199">即使停用新的註釋，仍可啟用警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-199">Warnings can be enabled even when the new annotations are disabled.</span></span> <span data-ttu-id="fcc2a-200">編譯器會使用靜態分析來判斷任何參考的 **Null 狀態**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-200">The compiler uses static flow analysis to determine the **null state** of any reference.</span></span> <span data-ttu-id="fcc2a-201">當並未「停用」*可為 Null 警告內容*時，Null 狀態不是**並非為 Null**，就是**可能為 Null**。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-201">The null state is either **not null** or **maybe null** when the *nullable warning context* isn't **disabled**.</span></span> <span data-ttu-id="fcc2a-202">若您在編譯器已判斷其**可能為 Null** 時對參考進行取值 (Dereference)，則編譯器會警告您。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-202">If you dereference a reference when the compiler has determined it's **maybe null**, the compiler warns you.</span></span> <span data-ttu-id="fcc2a-203">除非編譯器可判斷下列兩個狀況的其中一種狀況，否則參考的狀態就會是**可能為 Null**：</span><span class="sxs-lookup"><span data-stu-id="fcc2a-203">The state of a reference is **maybe null** unless the compiler can determine one of two conditions:</span></span>

1. <span data-ttu-id="fcc2a-204">變數已確定指派為並非 Null 的值。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-204">The variable has been definitely assigned to a non-null value.</span></span>
1. <span data-ttu-id="fcc2a-205">變數或運算式已在取值 (Dereference) 之前針對 Null 進行檢查。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-205">The variable or expression has been checked against null before de-referencing it.</span></span>

<span data-ttu-id="fcc2a-206">當可為 null 警告內容已啟用時，每當您以**可能為 null**的狀態來取值變數或運算式時，編譯器就會產生警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-206">The compiler generates warnings whenever you dereference a variable or expression in a **maybe null** state when the nullable warning context is enabled.</span></span> <span data-ttu-id="fcc2a-207">此外，當**可能的 null**變數或運算式指派給啟用的可為 null 注釋內容中的不可為 null 參考型別時，就會產生警告。</span><span class="sxs-lookup"><span data-stu-id="fcc2a-207">Furthermore, warnings are generated when a **maybe null** variable or expression is assigned to a nonnullable reference type in an enabled nullable annotation context.</span></span>
