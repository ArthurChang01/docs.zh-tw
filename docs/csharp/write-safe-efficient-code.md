## <a name="conclusions"></a><span data-ttu-id="6878a-258">結論</span><span class="sxs-lookup"><span data-stu-id="6878a-258">Conclusions</span></span>

<span data-ttu-id="6878a-259">使用實值型別可將配置作業的次數降至最低：</span><span class="sxs-lookup"><span data-stu-id="6878a-259">Using value types minimizes the number of allocation operations:</span></span>

- <span data-ttu-id="6878a-262">實值型別傳回值的儲存體是所配置堆疊。</span><span class="sxs-lookup"><span data-stu-id="6878a-262">Storage for value type return values is stack allocated.</span></span>

<span data-ttu-id="6878a-263">在相同情況下，與參考型別的對比是：</span><span class="sxs-lookup"><span data-stu-id="6878a-263">Contrast that with reference types in those same situations:</span></span>

<span data-ttu-id="6878a-270">最小化配置也包含了取捨。</span><span class="sxs-lookup"><span data-stu-id="6878a-270">Minimizing allocations comes with tradeoffs.</span></span> <span data-ttu-id="6878a-271">您會在 `struct` 的大小大於參考的大小時複製更多記憶體。</span><span class="sxs-lookup"><span data-stu-id="6878a-271">You copy more memory when the size of the `struct` is larger than the size of a reference.</span></span> <span data-ttu-id="6878a-272">參考通常是 64 位元或 32 位元，取決於目標電腦的 CPU。</span><span class="sxs-lookup"><span data-stu-id="6878a-272">A reference is typically 64 bits or 32 bits, and depends on the target machine CPU.</span></span>
