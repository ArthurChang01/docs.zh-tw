## <a name="c-language"></a><span data-ttu-id="2f6e2-111">C# 語言</span><span class="sxs-lookup"><span data-stu-id="2f6e2-111">C# Language</span></span>

 <span data-ttu-id="2f6e2-112">C# 語法的表達能力相當高，同時也很簡單且易於了解。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-112">C# syntax is highly expressive, yet it is also simple and easy to learn.</span></span> <span data-ttu-id="2f6e2-113">熟悉 C、C++ 或 Java 的任何人都能立即辨識出 C# 的大括號語法。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-113">The curly-brace syntax of C# will be instantly recognizable to anyone familiar with C, C++ or Java.</span></span> <span data-ttu-id="2f6e2-114">了解上述任何一種語言的開發人員一般都能在極短時間內開始用 C# 有效率地工作。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-114">Developers who know any of these languages are typically able to begin to work productively in C# within a very short time.</span></span> <span data-ttu-id="2f6e2-115">C# 語法可大幅簡化 C++ 的複雜性，並提供強大的功能，例如可為 Null 的實值型別、列舉、委派、lambda 運算式，還有 Java 中所沒有的直接記憶體存取。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-115">C# syntax simplifies many of the complexities of C++ and provides powerful features such as nullable value types, enumerations, delegates, lambda expressions and direct memory access, which are not found in Java.</span></span> <span data-ttu-id="2f6e2-116">C# 支援泛型方法和型別，提供更高的型別安全和效能。C# 也支援迭代器，讓集合類別的實作者定義自訂的反覆項目表現方式，使得用戶端程式碼可以容易地使用。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-116">C# supports generic methods and types, which provide increased type safety and performance, and iterators, which enable implementers of collection classes to define custom iteration behaviors that are simple to use by client code.</span></span> [!INCLUDE[vbteclinqext](~/includes/vbteclinqext-md.md)]<span data-ttu-id="2f6e2-117"> 運算式可讓強型別查詢成為優先使用的語言建構。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-117"> expressions make the strongly-typed query a first-class language construct.</span></span>  
  
 <span data-ttu-id="2f6e2-118">C# 為物件導向語言，支援封裝、繼承和多型的概念。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-118">As an object-oriented language, C# supports the concepts of encapsulation, inheritance, and polymorphism.</span></span> <span data-ttu-id="2f6e2-119">包括應用程式的進入點 `Main` 方法在內，所有變數和方法都會封裝在類別定義內。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-119">All variables and methods, including the `Main` method, the application's entry point, are encapsulated within class definitions.</span></span> <span data-ttu-id="2f6e2-120">類別可直接繼承自一個父類別，但可以實作數目不拘的介面。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-120">A class may inherit directly from one parent class, but it may implement any number of interfaces.</span></span> <span data-ttu-id="2f6e2-121">覆寫父類別中虛擬方法的方法需要利用 `override` 關鍵字，避免意外重複定義。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-121">Methods that override virtual methods in a parent class require the `override` keyword as a way to avoid accidental redefinition.</span></span> <span data-ttu-id="2f6e2-122">在 C# 中，結構就像輕量型的類別。它是堆疊配置型別，可以實作介面，但不支援繼承。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-122">In C#, a struct is like a lightweight class; it is a stack-allocated type that can implement interfaces but does not support inheritance.</span></span>

## <a name="net-framework-platform-architecture"></a><span data-ttu-id="2f6e2-139">.NET Framework 平台架構</span><span class="sxs-lookup"><span data-stu-id="2f6e2-139">.NET Framework Platform Architecture</span></span>

 <span data-ttu-id="2f6e2-140">C# 程式在 [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] 上執行，其為 Windows 不可或缺的元件，包括稱為通用語言執行平台 (CLR) 的虛擬執行系統和整合的一組類別庫。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-140">C# programs run on the [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)], an integral component of Windows that includes a virtual execution system called the common language runtime (CLR) and a unified set of class libraries.</span></span> <span data-ttu-id="2f6e2-141">CLR 是由 Microsoft 通用語言基礎結構 (CLI) 的商業實作，此國際標準是建立各種語言和程式庫都能一起順暢執行和開發環境的基礎。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-141">The CLR is the commercial implementation by Microsoft of the common language infrastructure (CLI), an international standard that is the basis for creating execution and development environments in which languages and libraries work together seamlessly.</span></span>  
  
 <span data-ttu-id="2f6e2-142">以 C# 撰寫的原始程式碼會編譯成符合 CLI 規格的中繼語言 (IL) 。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-142">Source code written in C# is compiled into an intermediate language (IL) that conforms to the CLI specification.</span></span> <span data-ttu-id="2f6e2-143">IL 程式碼和像是點陣圖和字串的資源，會以稱為組件的可執行檔儲存在磁碟上，副檔名通常為 .exe 或 .dll。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-143">The IL code and resources, such as bitmaps and strings, are stored on disk in an executable file called an assembly, typically with an extension of .exe or .dll.</span></span> <span data-ttu-id="2f6e2-144">組件包含的資訊清單提供有關組件的型別、版本、文化特性及安全性需求資訊。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-144">An assembly contains a manifest that provides information about the assembly's types, version, culture, and security requirements.</span></span>  
  
 <span data-ttu-id="2f6e2-145">C# 程式執行時，組件會載入至 CLR，根據資訊清單中的資訊執行各種不同動作。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-145">When the C# program is executed, the assembly is loaded into the CLR, which might take various actions based on the information in the manifest.</span></span> <span data-ttu-id="2f6e2-146">然後，如果符合安全性需求，CLR 就會執行 Just-In-Time (JIT) 編譯以將 IL 程式碼轉換成原生機器指令。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-146">Then, if the security requirements are met, the CLR performs just in time (JIT) compilation to convert the IL code to native machine instructions.</span></span> <span data-ttu-id="2f6e2-147">CLR 也提供有關自動記憶體回收、例外狀況處理和資源管理的其他服務。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-147">The CLR also provides other services related to automatic garbage collection, exception handling, and resource management.</span></span> <span data-ttu-id="2f6e2-148">由 CLR 執行的程式碼有時稱為「Managed 程式碼 」，相對於會以特定系統為目標編譯成原生機器語言的「Unmanaged 程式碼 」。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-148">Code that is executed by the CLR is sometimes referred to as "managed code," in contrast to "unmanaged code" which is compiled into native machine language that targets a specific system.</span></span> <span data-ttu-id="2f6e2-149">下圖說明 C# 原始程式碼檔、.NET Framework 類別庫、組件及 CLR 的編譯時期和執行階段關聯性。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-149">The following diagram illustrates the compile-time and run-time relationships of C# source code files, the .NET Framework class libraries, assemblies, and the CLR.</span></span>  
  
 <span data-ttu-id="2f6e2-150">![從 C&#35; 原始程式碼到機器執行](../../csharp/getting-started/media/netarchitecture.png "NETarchitecture")</span><span class="sxs-lookup"><span data-stu-id="2f6e2-150">![From C&#35; source code to machine execution](../../csharp/getting-started/media/netarchitecture.png "NETarchitecture")</span></span>  
  
 <span data-ttu-id="2f6e2-151">語言互通性是 [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] 的一項重要功能。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-151">Language interoperability is a key feature of the [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)].</span></span> <span data-ttu-id="2f6e2-152">由於 C# 編譯器所產生的 IL 程式碼符合通用型別規格 (CTS)，所以從 C# 產生的 IL 程式碼可以和從 Visual Basic、Visual C++ 的 .NET 版本所產生的程式碼互動，或和 20 種以上的任何其他 CTS 相容語言互動。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-152">Because the IL code produced by the C# compiler conforms to the Common Type Specification (CTS), IL code generated from C# can interact with code that was generated from the .NET versions of Visual Basic, Visual C++, or any of more than 20 other CTS-compliant languages.</span></span> <span data-ttu-id="2f6e2-153">單一組件可包含以不同的 .NET 語言撰寫的多個模組，而且型別可以彼此參考，如同它們都以相同的語言撰寫一般。</span><span class="sxs-lookup"><span data-stu-id="2f6e2-153">A single assembly may contain multiple modules written in different .NET languages, and the types can reference each other just as if they were written in the same language.</span></span>
