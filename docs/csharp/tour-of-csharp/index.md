<span data-ttu-id="3455e-109">C# 是物件導向的語言，但 C# 更進一步支援「元件導向」\*\*\*\*\*\* 程式設計。</span><span class="sxs-lookup"><span data-stu-id="3455e-109">C# is an object-oriented language, but C# further includes support for ***component-oriented*** programming.</span></span> <span data-ttu-id="3455e-110">現代軟體設計逐漸依賴功能性上獨立與屬於自我描述套件的軟體元件。</span><span class="sxs-lookup"><span data-stu-id="3455e-110">Contemporary software design increasingly relies on software components in the form of self-contained and self-describing packages of functionality.</span></span> <span data-ttu-id="3455e-111">這類元件的關鍵在於，它們會呈現具有屬性、方法和事件的程式設計模型。</span><span class="sxs-lookup"><span data-stu-id="3455e-111">Key to such components is that they present a programming model with properties, methods, and events.</span></span> <span data-ttu-id="3455e-112">它們具有可提供元件相關宣告式資訊的屬性。</span><span class="sxs-lookup"><span data-stu-id="3455e-112">They have attributes that provide declarative information about the component.</span></span> <span data-ttu-id="3455e-113">他們會併入自己的檔。</span><span class="sxs-lookup"><span data-stu-id="3455e-113">They incorporate their own documentation.</span></span> <span data-ttu-id="3455e-114">C # 提供語言結構來直接支援這些概念，讓 c # 成為自然語言，用來建立和使用軟體元件。</span><span class="sxs-lookup"><span data-stu-id="3455e-114">C# provides language constructs to support directly these concepts, making C# a natural language in which to create and use software components.</span></span>

<span data-ttu-id="3455e-115">有數個 c # 功能有助於構建強大和持久的應用程式。</span><span class="sxs-lookup"><span data-stu-id="3455e-115">Several C# features aid in the construction of robust and durable applications.</span></span> <span data-ttu-id="3455e-116">***垃圾收集***會自動回收因為無法使用的物件所佔用的記憶體。</span><span class="sxs-lookup"><span data-stu-id="3455e-116">***Garbage collection*** automatically reclaims memory occupied by unreachable unused objects.</span></span> <span data-ttu-id="3455e-117">***例外狀況處理***提供結構化且可延伸的方法來偵測及復原錯誤。</span><span class="sxs-lookup"><span data-stu-id="3455e-117">***Exception handling*** provides a structured and extensible approach to error detection and recovery.</span></span> <span data-ttu-id="3455e-118">語言的***型別安全***設計使得無法從未初始化的變數讀取、將陣列索引超出其界限，或執行未檢查的型別轉換。</span><span class="sxs-lookup"><span data-stu-id="3455e-118">The ***type-safe*** design of the language makes it impossible to read from uninitialized variables, to index arrays beyond their bounds, or to perform unchecked type casts.</span></span>

<span data-ttu-id="3455e-119">C# 有***統一的型別系統***。</span><span class="sxs-lookup"><span data-stu-id="3455e-119">C# has a ***unified type system***.</span></span> <span data-ttu-id="3455e-120">所有的 C# 型別 (包括 `int` 和 `double` 等基本型別) 都繼承自單一的 `object` 根型別。</span><span class="sxs-lookup"><span data-stu-id="3455e-120">All C# types, including primitive types such as `int` and `double`, inherit from a single root `object` type.</span></span> <span data-ttu-id="3455e-121">因此，所有型別都擁有相同的一組常見作業，且任何型別的值都能以相同方式儲存、傳送及操作。</span><span class="sxs-lookup"><span data-stu-id="3455e-121">Thus, all types share a set of common operations, and values of any type can be stored, transported, and operated upon in a consistent manner.</span></span> <span data-ttu-id="3455e-122">此外，C# 支援使用者定義的參考型別和數值型別，因此能動態配置物件，以及內嵌儲存輕量結構。</span><span class="sxs-lookup"><span data-stu-id="3455e-122">Furthermore, C# supports both user-defined reference types and value types, allowing dynamic allocation of objects as well as in-line storage of lightweight structures.</span></span>

<span data-ttu-id="3455e-123">為了確保 c # 程式和程式庫能夠以相容的方式在一段時間內進化，在 c # 設計中的***版本控制***方面有許多強調。</span><span class="sxs-lookup"><span data-stu-id="3455e-123">To ensure that C# programs and libraries can evolve over time in a compatible manner, much emphasis has been placed on ***versioning*** in C#'s design.</span></span> <span data-ttu-id="3455e-124">許多程式設計語言都不太注意此問題。</span><span class="sxs-lookup"><span data-stu-id="3455e-124">Many programming languages pay little attention to this issue.</span></span> <span data-ttu-id="3455e-125">因此，在引進較新版本的相依程式庫時，以這些其他語言撰寫的程式會有更多的頻率。</span><span class="sxs-lookup"><span data-stu-id="3455e-125">As a result, programs written in those other languages break more often than necessary when newer versions of dependent libraries are introduced.</span></span> <span data-ttu-id="3455e-126">以版本設定考慮直接影響的 c # 設計層麵包括個別`virtual`的和`override`修飾詞、方法多載解析的規則，以及明確介面成員宣告的支援。</span><span class="sxs-lookup"><span data-stu-id="3455e-126">Aspects of C#'s design that were directly influenced by versioning considerations include the separate `virtual` and `override` modifiers, the rules for method overload resolution, and support for explicit interface member declarations.</span></span>

<span data-ttu-id="3455e-127">在較新的版本中，c # 採用其他程式設計範例。</span><span class="sxs-lookup"><span data-stu-id="3455e-127">In more recent versions, C# has embraced other programming paradigms.</span></span> <span data-ttu-id="3455e-128">C # 已包含支援函式程式設計技術（例如 lambda 運算式）的功能。</span><span class="sxs-lookup"><span data-stu-id="3455e-128">C# has included features that support functional programming techniques like lambda expressions.</span></span> <span data-ttu-id="3455e-129">其他新功能支援分隔資料和演算法，例如模式比對。</span><span class="sxs-lookup"><span data-stu-id="3455e-129">Other new features support separating data and algorithms, like pattern matching.</span></span>
