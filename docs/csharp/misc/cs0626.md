# <a name="compiler-warning-level-1-cs0626"></a><span data-ttu-id="7a989-102">編譯器警告 (層級 1) CS0626</span><span class="sxs-lookup"><span data-stu-id="7a989-102">Compiler Warning (level 1) CS0626</span></span>
<span data-ttu-id="7a989-103">方法、運算子或存取子 'method' 已標記為外部，但其上沒有屬性。</span><span class="sxs-lookup"><span data-stu-id="7a989-103">Method, operator, or accessor 'method' is marked external and has no attributes on it.</span></span> <span data-ttu-id="7a989-104">請考慮加入 DllImport 屬性來指定外部實作</span><span class="sxs-lookup"><span data-stu-id="7a989-104">Consider adding a DllImport attribute to specify the external implementation</span></span>  
  
 <span data-ttu-id="7a989-105">方法標記為`extern`也會標示屬性，例如， [DllImport](xref:System.Runtime.InteropServices.DllImportAttribute)屬性。</span><span class="sxs-lookup"><span data-stu-id="7a989-105">A method marked `extern` should also be marked with an attribute, for example, the [DllImport](xref:System.Runtime.InteropServices.DllImportAttribute) attribute.</span></span>
  
 <span data-ttu-id="7a989-106">此屬性會指定實作方法的位置。</span><span class="sxs-lookup"><span data-stu-id="7a989-106">The attribute specifies where the method is implemented.</span></span> <span data-ttu-id="7a989-107">在執行階段，程式將會需要這項資訊。</span><span class="sxs-lookup"><span data-stu-id="7a989-107">At run time, the program will need this information.</span></span>  
  
 <span data-ttu-id="7a989-108">下列範例會產生 CS0626：</span><span class="sxs-lookup"><span data-stu-id="7a989-108">The following sample generates CS0626:</span></span>  
  
```csharp
// CS0626.cs

public class MyClass  
{  
   static extern public void M(); // CS0626  
   // try the following line  
   // [DllImport("mydll.dll")] static extern public void M();  
  
   public static void Main()  
   {  
   }  
}  
```
