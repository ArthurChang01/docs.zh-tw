<span data-ttu-id="4d97a-103">無法指派 'expression' 給範圍變數</span><span class="sxs-lookup"><span data-stu-id="4d97a-103">Cannot assign 'expression' to a range variable.</span></span>  
  
 <span data-ttu-id="4d97a-104">編譯器必須能夠推斷範圍變數的類型，而不論其是在 `from` 子句或 `let` 子句中所引入。</span><span class="sxs-lookup"><span data-stu-id="4d97a-104">The compiler must be able to infer the type of a range variable, whether it is introduced in a `from` clause or a `let` clause.</span></span> <span data-ttu-id="4d97a-105">不可為 Null，因為 null 不是類型，也不可以透過不安全類型的運算式進行指派。</span><span class="sxs-lookup"><span data-stu-id="4d97a-105">It cannot be null because null is not a type, and it cannot be assigned with an expression of an unsafe type.</span></span>

## <a name="example"></a><span data-ttu-id="4d97a-109">範例</span><span class="sxs-lookup"><span data-stu-id="4d97a-109">Example</span></span>

```csharp  
using System.Linq;  
class Test  
{  
    static void Main()  
    {  
  
        var x = from i in Enumerable.Range(1, 100)  
                let k = null // CS1932  
                // Try the following line instead.  
                let k = (string) null  
                select i;  
    }  
}  
```
