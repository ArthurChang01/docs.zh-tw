# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="77608-102">編譯器警告 (層級 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="77608-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="77608-103">已指派私用欄位 'field'，但從未使用過其值</span><span class="sxs-lookup"><span data-stu-id="77608-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="77608-104">此警告可能發生在下列若干案例中，其中編譯器可確認永遠不會參考某個變數：</span><span class="sxs-lookup"><span data-stu-id="77608-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
- <span data-ttu-id="77608-105">私用欄位已指派常數值，但絕不會接著讀取。</span><span class="sxs-lookup"><span data-stu-id="77608-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="77608-106">不必要的指派可能會影響效能。</span><span class="sxs-lookup"><span data-stu-id="77608-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="77608-107">請考慮移除欄位。</span><span class="sxs-lookup"><span data-stu-id="77608-107">Consider removing the field.</span></span>  
  
- <span data-ttu-id="77608-108">只在初始設定式中，才為私用或內部靜態欄位指派常數值。</span><span class="sxs-lookup"><span data-stu-id="77608-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="77608-109">請考慮將欄位變更為常數。</span><span class="sxs-lookup"><span data-stu-id="77608-109">Consider changing the field to a const.</span></span>  
  
- <span data-ttu-id="77608-110">私用或內部欄位已指派常數值，而且只用於 #ifdef 指示詞排除的區塊。</span><span class="sxs-lookup"><span data-stu-id="77608-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="77608-111">請考慮將欄位放到 #ifdef 區塊內。</span><span class="sxs-lookup"><span data-stu-id="77608-111">Consider putting the field inside the #ifdef block.</span></span>  
  
- <span data-ttu-id="77608-112">私用或內部欄位的多個位置中已指派常數值，但卻無法存取。</span><span class="sxs-lookup"><span data-stu-id="77608-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="77608-113">如果您不需要欄位，請考慮將它移除。</span><span class="sxs-lookup"><span data-stu-id="77608-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="77608-114">否則，請以適當的方式來使用欄位。</span><span class="sxs-lookup"><span data-stu-id="77608-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="77608-115">若為其他情況或不接受建議之因應措施的情況下，請使用 #pragma 0414。</span><span class="sxs-lookup"><span data-stu-id="77608-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="77608-116">下列範例顯示會產生 CS0414 的其中一種情況：</span><span class="sxs-lookup"><span data-stu-id="77608-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp  
// CS0414

{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="77608-117">**注意** ：如果變數 `i` 宣告為 `protected or public`，則由於編譯器無法確定是否衍生的類別會使用它，或其他用戶端程式碼是否會具現化相關類別並參考相關變數，因此不會產生任何錯誤</span><span class="sxs-lookup"><span data-stu-id="77608-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
