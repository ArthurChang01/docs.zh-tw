# <a name="compiler-error-cs1930"></a><span data-ttu-id="7c779-102">編譯器錯誤 CS1930</span><span class="sxs-lookup"><span data-stu-id="7c779-102">Compiler Error CS1930</span></span>
<span data-ttu-id="7c779-103">已經宣告範圍變數 'name'</span><span class="sxs-lookup"><span data-stu-id="7c779-103">The range variable 'name' has already been declared</span></span>  
  
 <span data-ttu-id="7c779-104">除非查詢運算式終止，否則查詢運算式中的範圍變數位於範圍中。</span><span class="sxs-lookup"><span data-stu-id="7c779-104">The range variable in a query expression is in scope until the query expression terminates.</span></span> <span data-ttu-id="7c779-105">因此，它必須有唯一的識別項。</span><span class="sxs-lookup"><span data-stu-id="7c779-105">It must therefore have a unique identifier.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="7c779-106">更正這個錯誤</span><span class="sxs-lookup"><span data-stu-id="7c779-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="7c779-107">請為查詢運算式中引進的每個範圍變數提供唯一名稱。</span><span class="sxs-lookup"><span data-stu-id="7c779-107">Give a unique name to each range variable that is introduced in a query expression.</span></span>  
  
## <a name="example"></a><span data-ttu-id="7c779-108">範例</span><span class="sxs-lookup"><span data-stu-id="7c779-108">Example</span></span>  
 <span data-ttu-id="7c779-109">下列範例會產生 CS1930，因為識別項 `num` 是用於 `from` 子句中的範圍變數，以及是用於 `let` 子句引進的範圍變數。</span><span class="sxs-lookup"><span data-stu-id="7c779-109">The following example generates CS1930 because the identifier `num` is used for the range variable in the `from` clause and for the range variable introduced by the `let` clause.</span></span>  
  
```csharp  
// cs1930.cs  
using System.Linq;  
class Program  
{  
    static void Main()  
    {  
        int[] nums = { 0, 1, 2, 3, 4, 5 };  
        var query = from num in nums  
                    let num = 3 // CS1930  
                    select num;   
    }  
}  
```
