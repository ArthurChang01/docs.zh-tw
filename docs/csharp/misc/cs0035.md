# <a name="compiler-error-cs0035"></a><span data-ttu-id="ac639-102">編譯器錯誤 CS0035</span><span class="sxs-lookup"><span data-stu-id="ac639-102">Compiler Error CS0035</span></span>
<span data-ttu-id="ac639-103">運算子 'operator' 用在類型 'type' 的運算元上時其意義會模稜兩可</span><span class="sxs-lookup"><span data-stu-id="ac639-103">Operator 'operator' is ambiguous on an operand of type 'type'</span></span>  
  
 <span data-ttu-id="ac639-104">編譯器有多個可用的轉換，而且不知道套用運算子之前要選擇哪個轉換。</span><span class="sxs-lookup"><span data-stu-id="ac639-104">The compiler has more than one available conversion and does not know which to choose before applying the operator.</span></span> <span data-ttu-id="ac639-105">如需詳細資訊，請參閱[樣板化的使用者定義轉換](https://docs.microsoft.com/cpp/misc/templated-user-defined-conversions)和[轉換運算子](../../csharp/programming-guide/statements-expressions-operators/conversion-operators.md)。</span><span class="sxs-lookup"><span data-stu-id="ac639-105">For more information, see [Templated User Defined Conversions](https://docs.microsoft.com/cpp/misc/templated-user-defined-conversions) and [Conversion Operators](../../csharp/programming-guide/statements-expressions-operators/conversion-operators.md).</span></span>  
  
 <span data-ttu-id="ac639-106">下列範例會產生 CS0035：</span><span class="sxs-lookup"><span data-stu-id="ac639-106">The following sample generates CS0035:</span></span>  
  
```  
// CS0035.cs  
class MyClass  
{  
   private int i;  
  
   public MyClass(int i)  
   {  
      this.i = i;  
   }  
  
   public static implicit operator double(MyClass x)  
   {  
      return (double) x.i;  
   }  
  
   public static implicit operator decimal(MyClass x)  
   {  
      return (decimal) x.i;  
   }  
}  
  
class MyClass2  
{  
   static void Main()  
   {  
      MyClass x = new MyClass(7);  
      object o = - x;   // CS0035  
      // try a cast:  
      // object o = - (double)x;  
   }  
}  
```
