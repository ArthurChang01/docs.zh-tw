<span data-ttu-id="6f664-103">命名空間別名限定詞 '::' 一定會解析為類型或命名空間，所以不能用在這裡。</span><span class="sxs-lookup"><span data-stu-id="6f664-103">The namespace alias qualifier '::' always resolves to a type or namespace so is illegal here.</span></span> <span data-ttu-id="6f664-104">請考慮用 '.' 替代。</span><span class="sxs-lookup"><span data-stu-id="6f664-104">Consider using '.' instead.</span></span>  
  
 <span data-ttu-id="6f664-105">如果您在未預期的位置使用了剖析器解譯為類型的項目，就會發生這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="6f664-105">This error occurs if you used something which the parser interpreted as a type in an unexpected place.</span></span> <span data-ttu-id="6f664-106">只有在使用成員存取 (**.**) 運算子的成員存取運算式中，類型或命名空間名稱才有效。</span><span class="sxs-lookup"><span data-stu-id="6f664-106">A type or namespace name is valid only in a member access expression, using the member access (**.**) operator.</span></span> <span data-ttu-id="6f664-107">如果您在其他內容中使用全域範圍運算子 (::)，則可能會發生這種情況。</span><span class="sxs-lookup"><span data-stu-id="6f664-107">This could occur if you used the global scope operator (::) in another context.</span></span>  
  
## <a name="example"></a><span data-ttu-id="6f664-108">範例</span><span class="sxs-lookup"><span data-stu-id="6f664-108">Example</span></span>  
 <span data-ttu-id="6f664-109">下列範例會產生 CS0687：</span><span class="sxs-lookup"><span data-stu-id="6f664-109">The following sample generates CS0687:</span></span>  
  
```csharp  
// CS0687.cs  
  
using M = Test;  
using System;  
  
public class Test
{  
    public static int x = 77;  
  
    public static void Main()
    {  
        Console.WriteLine(M::x); // CS0687  
        // To resolve use the following line instead:  
        // Console.WriteLine(M.x);  
    }  
}  
```
