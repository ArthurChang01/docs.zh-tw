 <span data-ttu-id="c692a-104">編譯器不允許隱藏名稱，這通常會導致您的程式碼出現邏輯錯誤。</span><span class="sxs-lookup"><span data-stu-id="c692a-104">The compiler does not allow hiding names, which commonly leads to logic errors in your code.</span></span>  
  
## <a name="example"></a><span data-ttu-id="c692a-105">範例</span><span class="sxs-lookup"><span data-stu-id="c692a-105">Example</span></span>  
 <span data-ttu-id="c692a-106">下列範例會產生 CS0135：</span><span class="sxs-lookup"><span data-stu-id="c692a-106">The following sample generates CS0135:</span></span>  
  
```csharp
// CS0135.cs  
public class MyClass2  
{  
   public static int i = 0;  
  
   public static void Main()  
   {  
      {  
         int i = 4;  
         i++;  
      }  
      i = 0;   // CS0135  
   }  
}  
```

 <span data-ttu-id="c692a-108">每次發生指定的識別項為運算式或宣告子中的簡單名稱，區域變數宣告空格 (§ 3.3) 包圍該出現，其他所有出現的運算式中的簡單名稱相同的識別碼或宣告子必須參考相同的實體。</span><span class="sxs-lookup"><span data-stu-id="c692a-108">For each occurrence of a given identifier as a simple-name in an expression or declarator, within the local variable declaration space (§3.3) immediately enclosing that occurrence, every other occurrence of the same identifier as a simple-name in an expression or declarator must refer to the same entity.</span></span> <span data-ttu-id="c692a-109">這項規則可確保某個名稱在指定區塊、switch 區塊、for 陳述式、foreach 陳述式、using 陳述式或匿名函式中一律同義。</span><span class="sxs-lookup"><span data-stu-id="c692a-109">This rule ensures that the meaning of a name is always the same within a given block, switch block, for-, foreach- or using-statement, or anonymous function.</span></span>
