# <a name="compiler-error-cs0457"></a><span data-ttu-id="b8125-102">編譯器錯誤 CS0457</span><span class="sxs-lookup"><span data-stu-id="b8125-102">Compiler Error CS0457</span></span>
<span data-ttu-id="b8125-103">從 'type name 1' 轉換為 'type name 2' 時，發現模稜兩可的使用者定義轉換 'Conversion method name 1' 和 'Conversion method name 2'</span><span class="sxs-lookup"><span data-stu-id="b8125-103">Ambiguous user defined conversions 'Conversion method name 1' and 'Conversion method name 2' when converting from 'type name 1' to 'type name 2'</span></span>  
  
 <span data-ttu-id="b8125-104">兩種轉換方法都適用，而且編譯器無法決定要使用哪一個。</span><span class="sxs-lookup"><span data-stu-id="b8125-104">Two conversion methods are applicable, and the compiler is unable to decide which one to use.</span></span>  
  
 <span data-ttu-id="b8125-105">下列任何一種情況都可能會造成這個錯誤：</span><span class="sxs-lookup"><span data-stu-id="b8125-105">One scenario that can cause this error is as follows:</span></span>  
  
-   <span data-ttu-id="b8125-106">您想要將類別 A 轉換為類別 B，但類別 A 與 B 無關。</span><span class="sxs-lookup"><span data-stu-id="b8125-106">You want to convert class A to class B where A and B are unrelated.</span></span>  
  
-   <span data-ttu-id="b8125-107">A 衍生自 A0，而且沒有方法可從 A0 轉換為 B。</span><span class="sxs-lookup"><span data-stu-id="b8125-107">A is derived from A0, and there is a method that converts from A0 to B.</span></span>  
  
-   <span data-ttu-id="b8125-108">B 具有子類別 B1，而且沒有方法可從 A 轉換為 B1。</span><span class="sxs-lookup"><span data-stu-id="b8125-108">B has a subclass B1 and there is a method that converts from A to B1.</span></span>  
  
 <span data-ttu-id="b8125-109">編譯器將這兩種轉換方法視為相等，因為第一個轉換會提供最佳目的類型，而第二個轉換會提供最佳來源類型。</span><span class="sxs-lookup"><span data-stu-id="b8125-109">The compiler will weight the two conversion methods equally, because the first conversion provides the best destination type, and the second conversion provides the best source type.</span></span> <span data-ttu-id="b8125-110">由於編譯器無法選擇，因此會產生這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="b8125-110">Since the compiler will be unable to choose, this error will be generated.</span></span> <span data-ttu-id="b8125-111">若要解決錯誤，請撰寫將 A 轉換為 B 的新明確方法。</span><span class="sxs-lookup"><span data-stu-id="b8125-111">To resolve, write a new explicit method converting A to B.</span></span>  
  
 <span data-ttu-id="b8125-112">如果有兩個方法可將 A 轉換為 B，也可能會造成這個錯誤。若要修正錯誤，請透過明確轉換指定要使用的轉換。</span><span class="sxs-lookup"><span data-stu-id="b8125-112">Another scenario that causes this error is if there are two methods that convert A to B. To fix, specify which conversion to use through an explicit cast.</span></span>  
  
## <a name="example"></a><span data-ttu-id="b8125-113">範例</span><span class="sxs-lookup"><span data-stu-id="b8125-113">Example</span></span>  
 <span data-ttu-id="b8125-114">下列範例會產生 CS0457。</span><span class="sxs-lookup"><span data-stu-id="b8125-114">The following sample generates CS0457.</span></span>  
  
```csharp  
// CS0457.cs  
using System;  
public class A { }  
  
public class G0 {  }  
public class G1<R> : G0 {  }  
  
public class H0 {  
   public static implicit operator G0(H0 h) {  
      return new G0();  
   }  
}  
public class H1<R> : H0 {  
   public static implicit operator G1<R>(H1<R> h) {  
      return new G1<R>();  
   }  
}  
  
public class Test   
{  
   public static void F0(G0 g) {  }  
   public static void Main()   
   {  
      H1<A> h1a = new H1<A>();  
      F0(h1a);   // CS0457  
   }  
}  
```
