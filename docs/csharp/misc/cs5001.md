# <a name="compiler-error-cs5001"></a>編譯器錯誤 CS5001

程式 'program' 未包含適合進入點的靜態 'Main' 方法  
  
如果在產生可執行檔`Main`的程式碼中找不到具有正確簽章的靜態方法，就會發生這個錯誤。 如果所定義進入點函式 `Main`的大小寫 (例如小寫 `main`) 錯誤，也會發生這個錯誤。 如需適用`Main`于方法之規則的詳細資訊，請參閱[Main （）和命令列引數](../programming-guide/main-and-command-args/index.md)。

如果方法有修飾詞，請確定[ C#選取的語言版本](../language-reference/configure-language-version.md)是7.1 或更高版本。 `async` `Main`

只有`Main`在編譯可執行檔時（也就是指定[-target： exe](/dotnet/csharp/language-reference/compiler-options/target-exe-compiler-option)或[-target： winexe](/dotnet/csharp/language-reference/compiler-options/target-winexe-compiler-option)編譯器選項）時，才需要此方法。 下列 Visual Studio 專案類型預設會指定其中一個選項：

- 主控台應用程式
- ASP.NET Core 應用程式
- WPF 應用程式
- Windows Form 應用程式

## <a name="example"></a>範例  
 下列範例會產生 CS5001：  
  
```csharp  
// CS5001.cs  
// CS5001 expected when compiled with -target:exe or -target:winexe 
public class a  
{  
   // Uncomment the following line to resolve.  
   // static void Main() {}  
}  
```
