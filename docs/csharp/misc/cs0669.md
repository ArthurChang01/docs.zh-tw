# <a name="compiler-error-cs0669"></a><span data-ttu-id="a333e-102">編譯器錯誤 CS0669</span><span class="sxs-lookup"><span data-stu-id="a333e-102">Compiler Error CS0669</span></span>

<span data-ttu-id="a333e-103">具有 ComImport 屬性的類別不能有使用者定義的建構函式。</span><span class="sxs-lookup"><span data-stu-id="a333e-103">A class with the ComImport attribute cannot have a user-defined constructor.</span></span>

<span data-ttu-id="a333e-104">Common Language Runtime 中的 COM Interop 層提供 [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) 類別的建構函式。</span><span class="sxs-lookup"><span data-stu-id="a333e-104">The COM interop layer in the common language runtime supplies the constructor for [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) classes.</span></span> <span data-ttu-id="a333e-105">因此，COM 物件可用來作為執行階段中的 Managed 物件。</span><span class="sxs-lookup"><span data-stu-id="a333e-105">Consequently, a COM object can be used as a managed object in the runtime.</span></span>

<span data-ttu-id="a333e-106">下列範例會產生 CS0669：</span><span class="sxs-lookup"><span data-stu-id="a333e-106">The following sample generates CS0669:</span></span>

```csharp
// CS0669.cs
using System.Runtime.InteropServices;
[ComImport, Guid("00000000-0000-0000-0000-000000000001")]
class TestClass
{
   TestClass()   // CS0669, delete constructor to resolve
   {
   }

   public static void Main()
   {
   }
}
```
