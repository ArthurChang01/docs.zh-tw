 <span data-ttu-id="f05c6-105">已定義查詢方法的多個實作定義，編譯器無法區別哪一個最適用於查詢時，就會產生這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="f05c6-105">This error is generated when multiple implementations of a query method are defined and the compiler cannot disambiguate which one is best to use for the query.</span></span> <span data-ttu-id="f05c6-106">在下列範例中，兩個版本的 `Select` 都具有相同的簽章，因為它們都接受一個 `int` 作為輸入參數，而且以 `int` 作為傳回值。</span><span class="sxs-lookup"><span data-stu-id="f05c6-106">In the following example, both versions of `Select` have the same signature, because they both accept one `int` as an input parameter and have `int` as a return value.</span></span>

## <a name="example"></a><span data-ttu-id="f05c6-109">範例</span><span class="sxs-lookup"><span data-stu-id="f05c6-109">Example</span></span>  
 <span data-ttu-id="f05c6-110">下列程式碼會產生 CS1940：</span><span class="sxs-lookup"><span data-stu-id="f05c6-110">The following code generates CS1940:</span></span>  
  
```  
// cs1940.cs  
using System; //must include explicitly for types defined in 3.5  
class Test  
{  
    public delegate int Dele(int x);  
    int num = 0;  
    public int Select(Func<int, int> d)  
    {  
        return d(this.num);  
    }  
    public int Select(Dele d) // CS1940  
    {  
        return d(this.num) + 1;  
    }  
    public static void Main()  
    {  
        var q = from x in new Test()  
        select x;  
    }  
}  
```
