<span data-ttu-id="73534-121">`LambdaExpression` 類型包含將運算式樹狀架構轉換成可執行程式碼時所使用的 `Compile` 和 `CompileToMethod` 成員。</span><span class="sxs-lookup"><span data-stu-id="73534-121">The `LambdaExpression` type contains `Compile` and `CompileToMethod` members that you would use to convert an expression tree to executable code.</span></span> <span data-ttu-id="73534-122">`Compile` 方法會建立委派。</span><span class="sxs-lookup"><span data-stu-id="73534-122">The `Compile` method creates a delegate.</span></span> <span data-ttu-id="73534-123">`CompileToMethod` 方法會以表示運算式樹狀架構之編譯輸出的 IL 來更新 `MethodBuilder` 物件。</span><span class="sxs-lookup"><span data-stu-id="73534-123">The `CompileToMethod` method updates a `MethodBuilder` object with the IL that represents the compiled output of the expression tree.</span></span> <span data-ttu-id="73534-124">請注意，`CompileToMethod` 僅適用於完整桌面架構，而不適用於 .NET Core。</span><span class="sxs-lookup"><span data-stu-id="73534-124">Note that `CompileToMethod` is only available in the full desktop framework, not in the .NET Core.</span></span>

<span data-ttu-id="73534-125">或者，您也可以提供 `DebugInfoGenerator`，以接收所產生之委派物件的符號偵錯資訊。</span><span class="sxs-lookup"><span data-stu-id="73534-125">Optionally, you can also provide a `DebugInfoGenerator` that will receive the symbol debugging information for the generated delegate object.</span></span> <span data-ttu-id="73534-126">這可讓您將運算式樹狀架構轉換成委派物件，並具有所產生之委派的完整偵錯資訊。</span><span class="sxs-lookup"><span data-stu-id="73534-126">This enables you to convert the expression tree into a delegate object, and have full debugging information about the generated delegate.</span></span>

<span data-ttu-id="73534-128">請注意，委派類型是以運算式類型為基礎。</span><span class="sxs-lookup"><span data-stu-id="73534-128">Notice that the delegate type is based on the expression type.</span></span> <span data-ttu-id="73534-129">如果您想要以強型別方式來使用委派物件，您必須了解傳回型別和引數清單。</span><span class="sxs-lookup"><span data-stu-id="73534-129">You must know the return type and the argument list if you want to use the delegate object in a strongly typed manner.</span></span> <span data-ttu-id="73534-130">`LambdaExpression.Compile()` 方法會傳回 `Delegate` 類型。</span><span class="sxs-lookup"><span data-stu-id="73534-130">The `LambdaExpression.Compile()` method returns the `Delegate` type.</span></span> <span data-ttu-id="73534-131">您必須將它轉換成正確的委派類型，才能讓任何編譯時期工具檢查引數清單或傳回型別。</span><span class="sxs-lookup"><span data-stu-id="73534-131">You will have to cast it to the correct delegate type to have any compile-time tools check the argument list or return type.</span></span>

## <a name="execution-and-lifetimes"></a><span data-ttu-id="73534-132">執行和存留期</span><span class="sxs-lookup"><span data-stu-id="73534-132">Execution and Lifetimes</span></span>

<span data-ttu-id="73534-133">您可以藉由叫用在呼叫 `LambdaExpression.Compile()` 時所建立的委派，來執行程式碼。</span><span class="sxs-lookup"><span data-stu-id="73534-133">You execute the code by invoking the delegate created when you called `LambdaExpression.Compile()`.</span></span> <span data-ttu-id="73534-134">如上所示，`add.Compile()` 會傳回委派。</span><span class="sxs-lookup"><span data-stu-id="73534-134">You can see this above where `add.Compile()` returns a delegate.</span></span> <span data-ttu-id="73534-135">叫用該委派 (藉由呼叫 `func()`) 即可執行程式碼。</span><span class="sxs-lookup"><span data-stu-id="73534-135">Invoking that delegate, by calling `func()` executes the code.</span></span>

<span data-ttu-id="73534-136">該委派代表運算式樹狀架構中的程式碼。</span><span class="sxs-lookup"><span data-stu-id="73534-136">That delegate represents the code in the expression tree.</span></span> <span data-ttu-id="73534-137">您可以保留該委派的控制代碼，稍後再叫用它。</span><span class="sxs-lookup"><span data-stu-id="73534-137">You can retain the handle to that delegate and invoke it later.</span></span> <span data-ttu-id="73534-138">您不需要在每次想要執行運算式樹狀架構所表示的程式碼時編譯運算式樹狀架構</span><span class="sxs-lookup"><span data-stu-id="73534-138">You don't need to compile the expression tree each time you want to execute the code it represents.</span></span> <span data-ttu-id="73534-139">(請記住，運算式樹狀架構為不可變，稍後編譯相同的運算式樹狀架構將會建立執行相同程式碼的委派)。</span><span class="sxs-lookup"><span data-stu-id="73534-139">(Remember that expression trees are immutable, and compiling the same expression tree later will create a delegate that executes the same code.)</span></span>

<span data-ttu-id="73534-140">建議您不要嘗試建立任何更複雜的快取機制來提升效能，方法是避免不必要的編譯呼叫。</span><span class="sxs-lookup"><span data-stu-id="73534-140">I will caution you against trying to create any more sophisticated caching mechanisms to increase performance by avoiding unnecessary compile calls.</span></span> <span data-ttu-id="73534-141">比較兩個任意運算式樹狀架構以判斷其是否代表相同的演算法，執行起來也很耗時。</span><span class="sxs-lookup"><span data-stu-id="73534-141">Comparing two arbitrary expression trees to determine if they represent the same algorithm will also be time consuming to execute.</span></span> <span data-ttu-id="73534-142">您可能會發現避免任何額外的 `LambdaExpression.Compile()` 呼叫所省下的計算時間，超過執行程式碼以判斷兩個不同的運算式樹狀架構是否產生相同的可執行程式碼所花費的時間。</span><span class="sxs-lookup"><span data-stu-id="73534-142">You'll likely find that the compute time you save avoiding any extra calls to `LambdaExpression.Compile()` will be more than consumed by the time executing code that determines of two different expression trees result in the same executable code.</span></span>

## <a name="caveats"></a><span data-ttu-id="73534-143">警告</span><span class="sxs-lookup"><span data-stu-id="73534-143">Caveats</span></span>

<span data-ttu-id="73534-144">您可以使用運算式樹狀架構執行的其中一個最簡單的作業，就是將 Lambda 運算式編譯成委派並叫用該委派。</span><span class="sxs-lookup"><span data-stu-id="73534-144">Compiling a lambda expression to a delegate and invoking that delegate is one of the simplest operations you can perform with an expression tree.</span></span> <span data-ttu-id="73534-145">不過，即使這是簡單的作業，還是必須注意幾點。</span><span class="sxs-lookup"><span data-stu-id="73534-145">However, even with this simple operation, there are caveats you must be aware of.</span></span> 

<span data-ttu-id="73534-146">Lambda 運算式會透過運算式中參考的任何區域變數建立結束型別。</span><span class="sxs-lookup"><span data-stu-id="73534-146">Lambda Expressions create closures over any local variables that are referenced in the expression.</span></span> <span data-ttu-id="73534-147">您必須確保屬於該委派的任何變數都可用於呼叫 `Compile` 的位置，以及在執行所產生的委派時使用。</span><span class="sxs-lookup"><span data-stu-id="73534-147">You must guarantee that any variables that would be part of the delegate are usable at the location where you call `Compile`, and when you execute the resulting delegate.</span></span>

<span data-ttu-id="73534-148">一般而言，編譯器會確保這點。</span><span class="sxs-lookup"><span data-stu-id="73534-148">In general, the compiler will ensure that this is true.</span></span> <span data-ttu-id="73534-149">不過，如果您的運算式存取實作 `IDisposable` 的變數，您的程式碼可能會處置運算式樹狀架構仍持有的物件。</span><span class="sxs-lookup"><span data-stu-id="73534-149">However, if your expression accesses a variable that implements `IDisposable`, it's possible that your code might dispose of the object while it is still held by the expression tree.</span></span>

<span data-ttu-id="73534-150">例如，因為 `int` 不會實作 `IDisposable`，所以下列程式碼會正常運作：</span><span class="sxs-lookup"><span data-stu-id="73534-150">For example, this code works fine, because `int` does not implement `IDisposable`:</span></span>

```csharp
private static Func<int, int> CreateBoundFunc()
{
    var constant = 5; // constant is captured by the expression tree
    Expression<Func<int, int>> expression = (b) => constant + b;
    var rVal = expression.Compile();
    return rVal;
}
```

<span data-ttu-id="73534-151">委派擷取了區域變數 `constant` 的參考。</span><span class="sxs-lookup"><span data-stu-id="73534-151">The delegate has captured a reference to the local variable `constant`.</span></span>
<span data-ttu-id="73534-152">您可以在稍後於 `CreateBoundFunc` 傳回的函式執行時，隨時存取該變數。</span><span class="sxs-lookup"><span data-stu-id="73534-152">That variable is accessed at any time later, when the function returned by `CreateBoundFunc` executes.</span></span>

<span data-ttu-id="73534-153">不過，請考慮實作 `IDisposable` 的這個類別 (而不是經過人為操作的類別)：</span><span class="sxs-lookup"><span data-stu-id="73534-153">However, consider this (rather contrived) class that implements `IDisposable`:</span></span>

```csharp
public class Resource : IDisposable
{
    private bool isDisposed = false;
    public int Argument
    {
        get
        {
            if (!isDisposed)
                return 5;
            else throw new ObjectDisposedException("Resource");
        }
    }

    public void Dispose()
    {
        isDisposed = true;
    }
}
```

<span data-ttu-id="73534-154">如果您將它用於以下所示的運算式中，當您執行 `Resource.Argument` 屬性所參考的程式碼時，您會得到 `ObjectDisposedException`：</span><span class="sxs-lookup"><span data-stu-id="73534-154">If you use it in an expression as shown below, you'll get an `ObjectDisposedException` when you execute the code referenced by the `Resource.Argument` property:</span></span>

```csharp
private static Func<int, int> CreateBoundResource()
{
    using (var constant = new Resource()) // constant is captured by the expression tree
    {
        Expression<Func<int, int>> expression = (b) => constant.Argument + b;
        var rVal = expression.Compile();
        return rVal;
    }
}
```

<span data-ttu-id="73534-155">此方法所傳回的委派已透過 `constant` 物件關閉，該物件已經過處置</span><span class="sxs-lookup"><span data-stu-id="73534-155">The delegate returned from this method has closed over the `constant` object, which has been disposed of.</span></span> <span data-ttu-id="73534-156">(因為已在 `using` 陳述式中宣告，所以已經過處置)。</span><span class="sxs-lookup"><span data-stu-id="73534-156">(It's been disposed, because it was declared in a `using` statement.)</span></span> 

<span data-ttu-id="73534-157">現在，當您執行此方法所傳回的委派時，執行當下會擲回 `ObjectDisposedException`。</span><span class="sxs-lookup"><span data-stu-id="73534-157">Now, when you execute the delegate returned from this method, you'll have a `ObjectDisposedException` thrown at the point of execution.</span></span>

<span data-ttu-id="73534-158">以執行階段錯誤表示編譯時期建構看起來很奇怪，但這是使用運算式樹狀架構時會面臨到的情況。</span><span class="sxs-lookup"><span data-stu-id="73534-158">It does seem strange to have a runtime error representing a compile-time construct, but that's the world we enter when we work with expression trees.</span></span>

<span data-ttu-id="73534-159">此問題有許多變化組合，因此很難提供一般指引予以避免。</span><span class="sxs-lookup"><span data-stu-id="73534-159">There are a lot of permutations of this problem, so it's hard to offer general guidance to avoid it.</span></span> <span data-ttu-id="73534-160">定義運算式時，請小心存取區域變數；建立可由公用 API 傳回的運算式樹狀架構時，請小心存取目前物件 (以 `this` 表示) 中的狀態。</span><span class="sxs-lookup"><span data-stu-id="73534-160">Be careful about accessing local variables when defining expressions, and be careful about accessing state in the current object (represented by `this`) when creating an expression tree that can be returned by a public API.</span></span>

<span data-ttu-id="73534-161">運算式中的程式碼可能會參考其他組件中的方法或屬性。</span><span class="sxs-lookup"><span data-stu-id="73534-161">The code in your expression may reference methods or properties in other assemblies.</span></span> <span data-ttu-id="73534-162">定義運算式、編譯運算式及叫用所產生的委派時，都必須存取該組件。</span><span class="sxs-lookup"><span data-stu-id="73534-162">That assembly must be accessible when the expression is defined, and when it is compiled, and when the resulting delegate is invoked.</span></span> <span data-ttu-id="73534-163">如果沒有該組件，則會發生 `ReferencedAssemblyNotFoundException`。</span><span class="sxs-lookup"><span data-stu-id="73534-163">You'll be met with a `ReferencedAssemblyNotFoundException` in cases where it is not present.</span></span>
